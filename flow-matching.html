Flow Matching 流匹配

Lipman 2022 (ICLR 2023) 簡化版的 diffusion：直接學「速度場 (velocity field)」而非 score。訓練穩、收斂快、採樣步數少——AlphaFlow、CellFlow、MOFlow 等 2024 新模型都採用。

Lipman 2022 (ICLR 2023) — a simplified diffusion: learn the velocity field directly instead of the score. More stable training, faster convergence, fewer sampling steps. Adopted by AlphaFlow, CellFlow, MOFlow and other 2024 models.

核心概念

從 Diffusion 到 Flow Matching

Diffusion 的痛點：需要學 score function ∇log p_t(x)，訓練需要對 noisy data 計算 score 並做 score matching loss——間接、複雜，且採樣需要解 SDE 或 ODE 上百步。

Flow Matching 的洞見（Lipman 2022, ICLR 2023）：放棄 score，直接學 velocity field v_t(x)，定義一個 deterministic 的 ODE：

dx/dt = v_t(x)，   x(0) ~ p_0 (e.g. N(0, I))，   x(1) ~ p_1 (data)

從噪音分布 p₀ 出發、沿 v_t 流動到資料分布 p₁。訓練目標就是讓 ODE 終點分布等於資料分布。

關鍵簡化：Conditional Flow Matching (CFM)。直接建構「給定 x₀ 與 x₁ 的 conditional path」——最常用是 直線插值：

x_t = (1 − t) x₀ + t x₁，   v_t(x_t | x₀, x₁) = x₁ − x₀

訓練目標就是 regression：神經網路 v_θ(x_t, t) 預測 (x₁ − x₀)。沒有 score、沒有 SDE、沒有複雜的 noise schedule！

Diffusion's pain point: learn the score ∇log p_t(x), which requires score matching on noisy data — indirect, complex, and sampling needs hundreds of SDE/ODE steps.

Flow Matching insight (Lipman 2022, ICLR 2023): drop the score; learn the velocity field directly v_t(x), defining a deterministic ODE:

dx/dt = v_t(x),   x(0) ~ p_0 (e.g. N(0, I)),   x(1) ~ p_1 (data)

Start at noise p₀, flow along v_t to data p₁. Training matches ODE endpoint to data distribution.

Key simplification: Conditional Flow Matching (CFM). Construct an explicit conditional path given (x₀, x₁) — usually a straight line:

x_t = (1 − t) x₀ + t x₁,   v_t(x_t | x₀, x₁) = x₁ − x₀

Training is regression: a network v_θ(x_t, t) predicts (x₁ − x₀). No score, no SDE, no noise schedule!

⚡

訓練超穩

純 regression loss，無需 score 或 ELBO；通常 5–10x diffusion 訓練速度。

Pure regression loss — no score, no ELBO; typically 5–10× faster training than diffusion.

🎯

少步採樣

直線路徑近似時 1–4 步即可生成；diffusion 通常需 50+ 步。

Straight-line paths sample in 1–4 steps; diffusion needs 50+.

🔄

OT 自然整合

配對 (x₀, x₁) 時用 OT 找最佳對應 → OT-CFM，路徑更直、採樣更快。

OT-pair (x₀, x₁) for optimal coupling → OT-CFM, straighter paths and faster sampling.

演算法

Conditional Flow Matching 與 Rectified Flow

① CFM 訓練步驟（Lipman 2022）：

  1. 取資料樣本 x₁ ~ p_data，獨立噪音 x₀ ~ N(0, I)。
  2. 取 t ~ U(0, 1)，計算插值 x_t = (1 − t) x₀ + t x₁。
  3. 條件速度為 v* = x₁ − x₀。
  4. Loss: ‖v_θ(x_t, t) − v*‖²。標準 MSE！

② 採樣：x₀ ~ N(0, I)，用 ODE solver（Euler 或 Heun）解 dx/dt = v_θ(x, t) 從 t = 0 到 t = 1。

③ Rectified Flow (Liu 2022)：CFM 訓練後路徑可能彎曲；rectified flow 用「重訓練」拉直：

  • 用訓練好的 v_θ 採樣得到 (x₀ⁱ, x₁ⁱ) 配對。
  • 對這些配對 重新 訓練 v_θ'——目標是直線。
  • 反覆 reflow 2–3 次即可幾乎完全直線化 → 1-step generation。

④ OT Flow Matching (Tong 2023)：不要 random 配對 (x₀, x₁)，而用 batch 內的 OT 配對 → 路徑天然較直，無需 reflow。

⑤ Stochastic Interpolants (Albergo 2023)：更一般的框架——允許 deterministic + stochastic interpolation，可加入 brownian motion。把 diffusion 與 flow matching 統一在一個框架下。

① CFM training (Lipman 2022):

  1. Sample x₁ ~ p_data; independent noise x₀ ~ N(0, I).
  2. Sample t ~ U(0, 1); interpolate x_t = (1 − t) x₀ + t x₁.
  3. Conditional velocity v* = x₁ − x₀.
  4. Loss: ‖v_θ(x_t, t) − v*‖². Standard MSE!

② Sampling: x₀ ~ N(0, I); solve dx/dt = v_θ(x, t) from t = 0 to 1 (Euler or Heun).

③ Rectified Flow (Liu 2022): after CFM, paths may bend. Rectified flow straightens by retraining:

  • Sample (x₀ⁱ, x₁ⁱ) pairs via trained v_θ.
  • Retrain v_θ' on these pairs — target is a straight line.
  • Reflow 2–3 times to near-straight trajectory → 1-step generation.

④ OT Flow Matching (Tong 2023): instead of random (x₀, x₁) pairing, use batch-OT pairing → paths are naturally straighter, no reflow needed.

⑤ Stochastic Interpolants (Albergo 2023): more general framework — allows deterministic + stochastic interpolation; unifies diffusion and flow matching.

為什麼直線路徑這麼重要？ ODE 軌跡越直、Euler 步進誤差越小、需要的步數越少。Diffusion 對應的 path 通常極為扭曲（高曲率），所以 sampling 慢；rectified flow 的核心貢獻就是把這些路徑「拉直」。
Why are straight paths so important? The straighter the ODE trajectory, the smaller the Euler step error, and the fewer steps needed. Diffusion paths are usually highly curved (slow sampling); rectified flow's key contribution is to straighten them.

生物應用

Flow Matching 在生物資訊

AlphaFlow

Jing et al. (2024, ICML) 把 AlphaFold 改成 flow-matching framework——不再給「最可能結構」，而是給「結構 ensemble」（蛋白質構象動態）。可生成 NMR/MD 風格的多重構象 sample。對 IDR (intrinsically disordered region) 蛋白尤其有用。

Jing et al. (2024, ICML) reframe AlphaFold as flow matching — instead of "the most likely structure", produces a structure ensemble (conformational dynamics). Generates NMR/MD-style multi-conformation samples. Especially useful for IDR proteins.

FrameFlow

Yim et al. (2023) 用 SE(3) flow matching 設計蛋白質 backbone（α-carbon trace）——比 RFdiffusion 快 5x 且品質相當。已被 Baker lab 整合進 de novo protein design 流程。

Yim et al. (2023) use SE(3) flow matching for protein backbone (α-carbon trace) design — 5× faster than RFdiffusion with comparable quality. Integrated into the Baker lab's de novo design pipeline.

CellFlow / scCFM

2024 年數個工作把 flow matching 用於單細胞動態——例如從 unperturbed 到 perturbed 細胞分布的 transport。OT-CFM 自然處理「沒有配對細胞」的問題（cells can't be re-measured）。

Several 2024 works apply flow matching to single-cell dynamics — transport from unperturbed to perturbed distributions. OT-CFM naturally handles the "no paired cells" problem (cells can't be re-measured).

MOFlow

分子生成的 flow matching 模型——學 SMILES / molecular graph 的 flow，可條件式生成具有特定性質（如 logP, QED）的小分子。比 diffusion-based DiGress 更快、樣本多樣性更高。

Flow matching for molecular generation — learns SMILES / molecular graph flows; conditional generation of molecules with target properties (logP, QED). Faster than diffusion-based DiGress with higher diversity.

Protein-MPNN-flow

把 ProteinMPNN 的 amino-acid sequence design 改造成 flow matching——用 simplex flow（單純形上的流）生成 sequence。可結合 FrameFlow 做 sequence + structure co-design。

ProteinMPNN-style sequence design as flow matching — simplex flows generate sequences. Combined with FrameFlow for joint sequence-structure design.

ESM-Flow

把 ESM2 protein language model embedding 當 latent space，在這個 latent 中跑 flow matching——可生成「具有特定功能標籤」的蛋白質序列（如「能結合 ATP」）。

Use ESM2 protein language model embeddings as latent space; flow matching in this latent generates proteins with specific functional labels (e.g. "ATP-binding").

深入解析

Flow Matching 進階主題

① 數學基礎：marginal vs conditional velocity

邊際 velocity v_t(x) 是難算的——它是「在 t 時刻、所有可能 (x₀, x₁) 路徑加權平均」。但 conditional velocity v_t(x | x₀, x₁) 可以 closed form 寫出（直線就是 x₁ − x₀）。Lipman 2022 的核心定理：用 conditional v 做 regression，極小化 loss 也會逼近 marginal v——這就是 CFM 為何 work 的關鍵。

② 與 Diffusion 的精確關係

Diffusion 的 probability flow ODE 是 dx/dt = f(x, t) − ½ g(t)² ∇log p_t(x)——一個特殊的 velocity field。Flow matching 把這推廣：v_t 可以是「任意」滿足 continuity equation 的場，不受 score-derived 結構限制。Diffusion 是 FM 的特殊情形（特定 conditional path）。

③ OT 配對與直線化的關係

Random pairing 的 expected path 等於 OT plan 平均；但 individual trajectory 仍可能彎曲（因為交叉）。OT pairing 把這些路徑「事先解纏」，每個 trajectory 接近直線。Tong et al. (2023) 在 ICML 證明這降低 Wasserstein 距離梯度的方差，訓練更穩。

④ Manifold-aware flow matching

對結構受限資料（如 SO(3) 旋轉、單純形 simplex、雙曲空間），需要 Riemannian flow matching (Chen 2023, NeurIPS)——把 conditional path 定義為 geodesic 而非歐式直線。已用於 FrameFlow (SE(3))、MOFlow (categorical simplex)。

⑤ Guided flow matching

類似 classifier-free guidance：在訓練時隨機 drop condition；採樣時 v_θ(x, t, c) − ω(v_θ(x, t, c) − v_θ(x, t, ∅)) 引導生成。實證：FM 的 guidance 比 diffusion 穩定，不容易 oversaturate。

⑥ 訓練實務

  • t 採樣分布：U(0,1) 是標準；logit-normal 對某些任務（如 stable diffusion 3）更好。
  • 網路 conditioning on t：用 sinusoidal embedding + AdaLN（同 diffusion）；不要只把 t concat 進去。
  • EMA 是必要：FM 也吃 EMA model weights 漲分。
  • 採樣 solver：1–4 步用 Euler；要 SOTA 品質用 Heun (RK2) 或 adaptive Dopri5。

⑦ 與 GAN / VAE 的比較

GAN：訓練不穩、mode collapse、樣本品質好。VAE：穩、但 latent 與資料 distribution 不完全匹配。Diffusion：穩、品質好、但慢。Flow matching：穩、品質好、且快——所以是現代 generative model 的事實主流。

① Math: marginal vs conditional velocity

The marginal velocity v_t(x) is intractable — it averages over all (x₀, x₁) paths weighted by mass at x. But conditional velocity v_t(x | x₀, x₁) has a closed form (straight line: x₁ − x₀). Lipman 2022's theorem: regressing toward conditional v also minimizes loss on marginal v — that's why CFM works.

② Exact relation to diffusion

Diffusion's probability flow ODE is dx/dt = f(x, t) − ½ g(t)² ∇log p_t(x) — a particular velocity field. Flow matching generalizes: v_t can be any field satisfying the continuity equation, unconstrained by score-derived structure. Diffusion is a special case of FM (specific conditional path).

③ OT pairing and straightening

Random pairing's expected path equals OT plan on average, but individual trajectories may bend (crossings). OT pairing pre-untangles them, making each trajectory near-straight. Tong et al. (2023, ICML) prove this reduces W-distance gradient variance, making training more stable.

④ Manifold-aware flow matching

For constrained data (SO(3) rotations, simplex, hyperbolic), Riemannian flow matching (Chen 2023, NeurIPS) uses geodesic conditional paths instead of Euclidean lines. Used in FrameFlow (SE(3)), MOFlow (categorical simplex).

⑤ Guided flow matching

Analogous to classifier-free guidance: drop the condition during training; at sampling, use v_θ(x, t, c) − ω(v_θ(x, t, c) − v_θ(x, t, ∅)). Empirically: FM guidance is more stable than diffusion guidance, less prone to oversaturation.

⑥ Training tips

  • t sampling: U(0,1) standard; logit-normal better for some tasks (e.g. Stable Diffusion 3).
  • Conditioning on t: sinusoidal embedding + AdaLN (same as diffusion); don't just concat t.
  • EMA matters: FM also benefits from EMA weights.
  • Sampling solver: Euler for 1–4 steps; Heun (RK2) or adaptive Dopri5 for SOTA quality.

⑦ Compared to GAN / VAE

GAN: unstable, mode collapse, good samples. VAE: stable, latent-data mismatch. Diffusion: stable, good, but slow. Flow matching: stable, good, and fast — hence today's de-facto generative modeling backbone.

潛在改良應用

還能改良哪些任務？

🎯 1. scRNA-seq 軌跡內插

單細胞 time-course 採樣稀疏；想預測「t = 0.5」（在 t = 0 與 t = 1 之間）的細胞分布。提案：用 OT-CFM 從 t = 0 流到 t = 1，沿途任意取點。比 Waddington-OT 更靈活，能生成 novel 中間 cell state。

🎯 2. 藥物擾動 path 學習

不只「最終擾動效應」，而是「擾動隨時間如何作用」。Flow matching 從未受擾動分布流到受擾動分布，整條 path 都可解讀為動態效應。提案：CellFlow-perturbation 整合 Perturb-seq + time-course。

🎯 3. 分子生成與優化

Conditional flow matching 從先驗到目標分子分布；conditioning on desired properties → 直接生成具備特定 docking score / ADMET 屬性的小分子。提案：整合 AutoDock / ADMET predictor 作 guidance。

🎯 4. 跨切片 spatial alignment

Spatial transcriptomics 多切片需要對齊（PASTE 用 GW）。提案：用 flow matching 學「切片 1 → 切片 2」的 vector field，可生成中間切片，做 3D 連續重建。

🎯 5. Cell-cycle / circadian phase generation

已知細胞週期是 circular manifold。提案：Riemannian flow matching on S¹ 學細胞週期相位的條件生成——給定 phase 預測 gene expression。

🎯 6. 跨物種 single-cell translation

Flow matching from human cells to mouse cells（或反向）；用 cell-type label 做 conditioning。提案：跨物種 in silico experiment（「如果這個藥在人身上對 T cell 的反應在鼠身上會像什麼？」）。

🎯 7. RNA secondary structure 生成

RNA 結構是離散字串（base-pair pattern）的 simplex。提案：discrete flow matching 生成 RNA 結構，可條件式於序列（reverse-folding）或功能標籤。

🎯 8. Sequence-to-structure co-design

FrameFlow（結構）+ ProteinMPNN-flow（序列）聯合訓練——同時優化序列與結構，比兩階段更一致。提案：joint flow matching 蛋白質設計，目標是 binder design / enzyme design。

🎯 9. 多體學跨模態 generation

從 scRNA 流到 scATAC（或反向）——用 paired data 學 flow，可在 unpaired data 上生成「對應 modality」。提案：cross-modal CFM 整合 multi-omics。

🎯 1. scRNA-seq trajectory interpolation

Single-cell time-courses are sparse; predict cells at "t = 0.5" between observed t = 0 and 1. Proposal: OT-CFM flowing t = 0 to t = 1 — sample anywhere along the path. More flexible than Waddington-OT, generates novel intermediate cell states.

🎯 2. Drug perturbation path learning

Not just "final effect" but "how the perturbation acts over time". Flow matching from unperturbed to perturbed distribution — the entire path interprets as dynamic effect. Proposal: CellFlow-perturbation joining Perturb-seq + time-course.

🎯 3. Molecule generation and optimization

Conditional FM from prior to target molecular distribution; conditioning on desired properties → directly generate molecules with target docking score / ADMET. Proposal: integrate AutoDock / ADMET predictors as guidance.

🎯 4. Cross-slice spatial alignment

Spatial transcriptomics multi-slice alignment (PASTE uses GW). Proposal: learn a vector field "slice 1 → slice 2" via flow matching — generate intermediate slices for continuous 3D reconstruction.

🎯 5. Cell-cycle / circadian phase generation

Cell cycle is a circular manifold. Proposal: Riemannian FM on S¹ learns conditional phase generation — given phase, predict gene expression.

🎯 6. Cross-species single-cell translation

Flow human → mouse cells (or reverse); condition on cell-type label. Proposal: cross-species in silico experiments ("how would this drug's T-cell response in human translate to mouse?").

🎯 7. RNA secondary structure generation

RNA structure is a discrete string (base-pair pattern) on a simplex. Proposal: discrete flow matching generates RNA structures, conditioned on sequence (reverse-folding) or function label.

🎯 8. Sequence-to-structure co-design

FrameFlow (structure) + ProteinMPNN-flow (sequence) joint training — optimize sequence and structure simultaneously, more consistent than two-stage. Proposal: joint FM protein design for binders / enzymes.

🎯 9. Multi-omics cross-modal generation

Flow scRNA → scATAC (or reverse) on paired data; deploy on unpaired data to generate "corresponding modality". Proposal: cross-modal CFM for multi-omics integration.

實作範例

# pip install torchcfm
import torch
from torchcfm.conditional_flow_matching import (
    ConditionalFlowMatcher,
    ExactOptimalTransportConditionalFlowMatcher)
from torchcfm.models import MLP

FM = ExactOptimalTransportConditionalFlowMatcher(sigma=0.0)
model = MLP(dim=2, time_varying=True).cuda()
opt = torch.optim.Adam(model.parameters(), lr=1e-3)

for step in range(20_000):
    x1 = sample_data(256).cuda()                # real data
    x0 = torch.randn_like(x1)                  # gaussian
    t, xt, ut = FM.sample_location_and_conditional_flow(x0, x1)
    v_pred = model(torch.cat([xt, t[:, None]], -1))
    loss = ((v_pred - ut)**2).mean()
    opt.zero_grad(); loss.backward(); opt.step()

# Sampling: Euler integration
x = torch.randn(1000, 2).cuda()
for i in range(50):
    t = torch.full((1000, 1), i/50).cuda()
    x = x + model(torch.cat([x, t], -1)) / 50
# Pure-PyTorch CFM — for understanding
import torch, torch.nn as nn

class VelocityNet(nn.Module):
    def __init__(self, d=2):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(d+1, 128), nn.SiLU(),
            nn.Linear(128, 128), nn.SiLU(),
            nn.Linear(128, d))
    def forward(self, x, t):
        return self.net(torch.cat([x, t], -1))

model = VelocityNet()
opt = torch.optim.Adam(model.parameters(), lr=1e-3)
for step in range(20_000):
    x1 = sample_data(256)
    x0 = torch.randn_like(x1)
    t  = torch.rand(256, 1)
    xt = (1-t)*x0 + t*x1
    v_star = x1 - x0
    loss = ((model(xt, t) - v_star)**2).mean()
    opt.zero_grad(); loss.backward(); opt.step()
import jax, jax.numpy as jnp
import equinox as eqx
import optax

def cfm_loss(model, x0, x1, t):
    xt = (1-t)*x0 + t*x1
    v_pred = jax.vmap(model)(xt, t)
    return jnp.mean((v_pred - (x1 - x0))**2)

@jax.jit
def step(model, opt_state, x0, x1, t):
    loss, grads = jax.value_and_grad(cfm_loss)(model, x0, x1, t)
    updates, opt_state = opt.update(grads, opt_state)
    model = eqx.apply_updates(model, updates)
    return model, opt_state, loss
