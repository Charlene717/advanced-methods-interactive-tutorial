Hamiltonian MC / NUTS

物理啟發的 MCMC：把 posterior 視為位能、引入「動量」變數，用 Hamiltonian 動力學沿等能曲線移動——比 Random Walk Metropolis 快幾個數量級。NUTS (Hoffman & Gelman 2014) 自動調整 step size 與 trajectory 長度，是 Stan / PyMC / NumPyro 的核心引擎。

A physics-inspired MCMC: treat the posterior as a potential, introduce momentum, follow Hamiltonian dynamics along constant-energy contours — orders of magnitude faster than Random Walk Metropolis. NUTS (Hoffman & Gelman 2014) auto-tunes step size and trajectory length, and is the engine of Stan / PyMC / NumPyro.

核心概念

從 Random Walk 到 Hamiltonian 動力學

Random Walk Metropolis (RWM) 的問題：每步隨機 perturb θ；高維時大多數方向都「走錯」，需要極小 step size、極多步才能 mix。對 d 維 posterior，autocorrelation time ∝ d——維度詛咒。

HMC 的物理直覺 (Neal 2011)：把要採樣的 posterior 看作位能：

U(θ) = − log p(θ | D)

引入輔助動量變數 p ~ N(0, M)（M 稱為 mass matrix）。定義 Hamiltonian：

H(θ, p) = U(θ) + ½ pT M−1 p

聯合分布 exp(−H(θ, p)) = p(θ|D) · N(p|0, M)，其中 θ 邊際就是目標 posterior。

核心想法：用物理動力學「跑」一段時間（leapfrog integrator 解 Hamilton 方程），然後 Metropolis accept/reject——軌跡可以走得很遠（幾十、上百步），卻仍然停留在高機率區域，因為 H 沿動力學保持（近似）守恆。

為何快？HMC 沿 posterior 的等高線「滑行」——避免 RWM 的「卡在局部 ridge」現象。autocorrelation time ∝ d1/4（Neal 2011），對高維非常友善。

The Random Walk Metropolis (RWM) problem: each step randomly perturbs θ; in high dimensions most directions are "wrong", so step size shrinks and mixing slows. For a d-dim posterior, autocorrelation time ∝ d — curse of dimensionality.

HMC physics intuition (Neal 2011): treat the posterior as a potential:

U(θ) = − log p(θ | D)

Introduce auxiliary momentum p ~ N(0, M) (M = mass matrix). Define the Hamiltonian:

H(θ, p) = U(θ) + ½ pT M−1 p

The joint distribution exp(−H(θ, p)) = p(θ|D) · N(p|0, M); marginalising p leaves the target posterior.

The trick: simulate Hamiltonian dynamics for a while (leapfrog integrator), then Metropolis accept/reject — trajectories can travel far (tens to hundreds of steps) while staying in high-probability regions because H is (nearly) conserved along the dynamics.

Why it's fast: HMC "skates" along posterior contours, avoiding RWM's "stuck on a ridge". Autocorrelation time ∝ d1/4 (Neal 2011) — much friendlier to high dimensions.

演算法

Leapfrog、HMC step 與 NUTS

① Leapfrog Integrator：Hamilton 方程 dθ/dt = ∂H/∂p = M−1p，dp/dt = −∂H/∂θ = −∇U。Leapfrog 是 symplectic 數值積分（保持相空間體積、可逆），step size ε：

  • p ← p − (ε/2) ∇U(θ)
  • θ ← θ + ε M−1 p
  • p ← p − (ε/2) ∇U(θ)

跑 L 步（總時間 T = ε·L）。symplectic 性質讓 H 漂移極小，accept rate 高。

② HMC step (Neal 2011)：給定當前 θ：

  1. 抽 p ~ N(0, M)。
  2. 跑 L 步 leapfrog 從 (θ, p) 得 (θ', p')。
  3. Metropolis accept：min(1, exp(H(θ, p) − H(θ', p')))。
  4. 接受時 θ ← θ'，否則 θ 不變。

關鍵超參：ε（step size）、L（trajectory length）、M（mass matrix）。調得好 → 90%+ accept、長距離 mixing；調不好 → divergent transitions / 卡住。

③ NUTS (Hoffman & Gelman 2014, JMLR)：自動決定 L——往前/往後雙向 doubling 樹，直到偵測到 U-turn（軌跡開始彎回）。

  • 從 (θ, p) 雙向 leapfrog，建構 trajectory tree（每次 doubling 樹深度）。
  • 偵測「U-turn」：(θ' − θ) · p' < 0 表示軌跡開始反折，停止擴張。
  • 從整棵樹按 Metropolis 比例隨機選一點作新 θ。
  • 還搭配 dual averaging 自動調 step size ε，使 target acceptance ≈ 0.8。

NUTS 讓 HMC 零調參、零選擇 L——這是 Stan / PyMC / NumPyro 成功的關鍵。

④ Riemannian HMC (Girolami & Calderhead 2011, JRSS-B)：用 Fisher 資訊矩陣作位置依賴 mass matrix M(θ)，讓 HMC 在「彎曲」的 posterior（如 hierarchical model）上更穩。代價是每步 O(d³) Cholesky 分解。

⑤ 處理離散變數：HMC 天然只適用連續變數。離散變數用 marginalize（如 latent class 模型）、Gibbs-within-HMC、或 continuous relaxation（Gumbel-softmax）。

① Leapfrog Integrator: Hamilton's equations dθ/dt = ∂H/∂p = M−1p, dp/dt = −∇U. Leapfrog is a symplectic integrator (volume-preserving, reversible) with step size ε:

  • p ← p − (ε/2) ∇U(θ)
  • θ ← θ + ε M−1 p
  • p ← p − (ε/2) ∇U(θ)

Run L steps (total time T = εL). Symplecticity keeps drift in H minimal — high acceptance rate.

② HMC step (Neal 2011): given current θ:

  1. Draw p ~ N(0, M).
  2. Run L leapfrog steps from (θ, p) to (θ', p').
  3. Metropolis accept: min(1, exp(H(θ, p) − H(θ', p'))).
  4. If accepted, θ ← θ'; else keep θ.

Hyperparams: ε (step size), L (trajectory length), M (mass matrix). Tuned well → 90%+ acceptance, long-range mixing; tuned poorly → divergences / stuck.

③ NUTS (Hoffman & Gelman 2014, JMLR): auto-chooses L — doubles the trajectory bidirectionally until a U-turn is detected.

  • From (θ, p), leapfrog both directions, building a tree (each doubling depth).
  • U-turn detection: (θ' − θ) · p' < 0 means the trajectory is bending back; stop.
  • Randomly choose a new θ from the tree weighted by Metropolis criterion.
  • Dual averaging auto-tunes ε to target ~0.8 acceptance.

NUTS makes HMC zero-tuning, zero-L-choice — the key to Stan / PyMC / NumPyro's success.

④ Riemannian HMC (Girolami & Calderhead 2011, JRSS-B): use position-dependent mass M(θ) = Fisher information — much more robust on funnel/hierarchical posteriors. Cost: O(d³) Cholesky per step.

⑤ Discrete variables: HMC handles continuous variables natively. For discrete: marginalise (latent class models), Gibbs-within-HMC, or continuous relaxation (Gumbel-softmax).

Divergent transitions 是什麼？當 ε 太大或 posterior 太彎曲時，leapfrog 數值積分爆炸——H 突然劇增。NUTS 標記這些 trajectory 為「divergent」，且 sampling 結果不可信。對策：減小 ε（或讓 NUTS 自動），改用 non-centered reparameterization（對 hierarchical model 必備），或用 Riemannian HMC。
What are divergent transitions? When ε is too large or posterior too curved, leapfrog integration explodes — H spikes. NUTS marks these "divergent"; their samples are unreliable. Fixes: reduce ε (or let NUTS adapt), use non-centered reparameterization (a must for hierarchical models), or Riemannian HMC.

生物應用

HMC / NUTS 在生物統計與生物資訊

Stan / PyMC / NumPyro 通用 Bayesian

Stan (Carpenter 2017, JSS)、PyMC、NumPyro 都以 NUTS 為預設 sampler。用於 single-cell normalization、batch correction、GWAS、survival analysis、Bayesian variant calling 等。scVI 雖以 VI 為主但有 HMC 版本做精確推斷對比。

Stan (Carpenter 2017, JSS), PyMC, NumPyro all default to NUTS. Used in single-cell normalization, batch correction, GWAS, survival analysis, Bayesian variant calling. scVI is primarily VI but has HMC variants for exact-inference comparisons.

BayesSpace

Zhao et al. (2021, Nat Biotech) 的 BayesSpace 用 MCMC 在 spatial transcriptomics 上做 sub-spot resolution 與 domain segmentation。Bayesian hidden Markov random field 用 Gibbs + Metropolis，整合 HMC 於連續參數估計。

Zhao et al. (2021, Nat Biotech) BayesSpace performs sub-spot resolution and domain segmentation on spatial transcriptomics. Bayesian hidden MRF uses Gibbs + Metropolis with HMC for continuous parameters.

BEAST2

BEAST2 (Bouckaert 2019, PLOS Comp Biol) 是分子系統發育的標準 Bayesian 軟體——HMC 對連續參數（分子鐘速率、tree height）、Metropolis-Hastings 對離散 tree topology。用於 SARS-CoV-2 演化、古 DNA、species 分歧時間估計。

BEAST2 (Bouckaert 2019, PLOS Comp Biol) is the standard Bayesian phylogenetics tool — HMC for continuous params (clock rate, tree height), MH for discrete tree topology. Used in SARS-CoV-2 evolution, ancient DNA, species divergence dating.

LeafTrack 系譜推斷

用 HMC 對 cell lineage tree（如 CellTagging、Cas9 scarring）估計 lineage 參數與 cell-state transition——比 maximum likelihood 更穩健，能給出不確定性區間。

HMC for cell lineage tree inference (CellTagging, Cas9 scarring) estimates lineage parameters and cell-state transitions — more robust than MLE, with uncertainty intervals.

BUGS-style 多體學

傳統 BUGS / JAGS 用 Gibbs；現代生物統計多體學模型（如 MOFA Bayesian extension、Bayesian multi-omics factorization）改用 Stan + NUTS——梯度型 sampler 對 hierarchical 模型穩定性大幅提升。

Classic BUGS/JAGS use Gibbs; modern Bayesian multi-omics models (Bayesian MOFA, Bayesian factorization) switch to Stan + NUTS — gradient-based samplers handle hierarchical models far better.

Bayesian DCA / SCM

Direct coupling analysis、structural causal models 的 Bayesian 版本用 NUTS 推斷 latent coupling 與 causal effect 的後驗——對小樣本不確定性更可靠。

Bayesian variants of Direct Coupling Analysis and Structural Causal Models use NUTS for posterior over latent couplings and causal effects — more reliable uncertainty under small samples.

Bayesian variant calling

DeepVariant 之外的 Bayesian 路線（如 Octopus, Strelka2 的 trio mode）用 MCMC 推斷 genotype likelihoods 的後驗——對 mosaic、somatic、low-VAF variant 特別有用。HMC 比 RWM 在多 QC metrics 共同推斷時 mix 更快。

Beyond DeepVariant, Bayesian variant callers (Octopus, Strelka2 trio mode) use MCMC for genotype likelihood posteriors — especially valuable for mosaic, somatic, low-VAF variants. HMC mixes faster than RWM on joint QC inference.

Bayesian MR

MR-Egger、Bayesian MR (Berzuini 2018, Genetic Epi)、MR-BMA 等用 HMC/NUTS 推斷因果效應的後驗，內建 horizontal pleiotropy 校正，比 frequentist IVW 更穩。

MR-Egger, Bayesian MR (Berzuini 2018, Genetic Epi), MR-BMA use HMC/NUTS for causal-effect posteriors with built-in horizontal pleiotropy correction — more robust than frequentist IVW.

深入解析

HMC / NUTS 進階主題

① Symplectic integrators 為何重要

Leapfrog 是 second-order symplectic integrator——保持相空間體積、可逆。能量誤差 O(ε²) 隨積分時間有界（不會線性累積）。用 RK4 等非 symplectic integrator 會讓 H 漂移、accept rate 崩潰。Higher-order symplectic（如 4th-order Yoshida）在某些情況更佳但每步開銷更大。

② Mass matrix 調校的關鍵

M = posterior covariance 的反矩陣是「最佳」mass matrix——讓所有方向 mix 速度相當。NUTS 的 dense mass matrix adaptation（warm-up 期估計）對 highly correlated posterior 至關重要。實務：先用 diagonal mass，必要時切到 dense。

③ Non-centered reparameterization

Hierarchical model（如 random effects）有「neal's funnel」幾何——σ² 接近 0 時 θ_i 強相關於 σ²，HMC 很容易 divergent。解法：把 θ_i = μ + σ · z_i（z_i ~ N(0,1)）取代直接 θ_i ~ N(μ, σ²)。這把 funnel 拉直成 isotropic Gaussian。Stan 文件第一條建議。

④ NUTS 的內部運作

NUTS 的核心是 doubling 樹：從 (θ, p) 向前/向後（隨機選）leapfrog 跑 2j 步、檢查 U-turn 與 divergence、達上限或停止條件就停。然後從整棵樹按 Metropolis 比例隨機選一點。為何不偏 bias？因為 U-turn 條件 + 樹採樣方式設計成 detailed balance preserving（Hoffman & Gelman 2014 證明）。

⑤ R-hat 與 ESS 診斷

  • R-hat (Gelman-Rubin)：多 chain 的方差比，R̂ < 1.01 為收斂。
  • ESS (effective sample size)：考慮 autocorrelation 後的「有效」獨立樣本數，> 400 per parameter 是常用門檻。
  • Divergence count：NUTS 出現 divergent transitions → posterior 不可信，必須處理（重 parameterize / 改先驗 / 減 ε）。
  • Tree depth saturation：NUTS 達到 max tree depth（預設 10）→ 軌跡可能不夠長，posterior 多模或極彎曲。

⑥ HMC 與 Langevin 動力學的關係

當 L = 1（只走一步）時 HMC 退化為 Metropolis-Adjusted Langevin Algorithm (MALA)。HMC 是 MALA 的「長軌跡」版——更好 mix 但每步更貴。SGLD、SG-HMC（stochastic gradient HMC, Chen 2014）是大資料版，用 minibatch 梯度估計取代 full ∇U。

⑦ 與 VI 的選擇

VI 快但只給點估計；HMC 慢但給精確後驗。實務經驗：探索期用 VI（pymc 的 ADVI、NumPyro 的 SVI）快速估計；最終 paper 用 NUTS 跑「完整版」。重要決策模型（藥物效應、臨床試驗）必須用 HMC。

① Why symplectic integrators matter

Leapfrog is a second-order symplectic integrator — volume-preserving, reversible. Energy error O(ε²) stays bounded over time (no linear drift). Non-symplectic integrators (RK4) cause H drift and collapsing acceptance rates. Higher-order symplectic (Yoshida 4th-order) can be better, at higher per-step cost.

② Mass matrix tuning is critical

M = (posterior covariance)−1 is the optimal mass matrix — equalises mixing across directions. NUTS's dense mass matrix adaptation during warm-up is essential for highly correlated posteriors. Practice: start with diagonal; upgrade to dense if needed.

③ Non-centered reparameterization

Hierarchical models (random effects) exhibit "Neal's funnel" — when σ → 0, θ_i is strongly tied to σ; HMC diverges easily. Fix: use θ_i = μ + σ · z_i with z_i ~ N(0,1) instead of θ_i ~ N(μ, σ²). Straightens the funnel into an isotropic Gaussian. Top Stan recommendation.

④ NUTS internals

NUTS's core: a doubling tree — leapfrog forward/backward (random direction) for 2j steps, check U-turn and divergence, stop on either or max depth. Then sample from the tree weighted by Metropolis criterion. Why unbiased? Because the U-turn criterion + tree sampling scheme are designed to preserve detailed balance (Hoffman & Gelman 2014).

⑤ R-hat and ESS diagnostics

  • R-hat (Gelman-Rubin): multi-chain variance ratio; R̂ < 1.01 = converged.
  • ESS (effective sample size): autocorrelation-adjusted "independent" samples; > 400 per param is a common bar.
  • Divergence count: NUTS divergences → posterior unreliable; must fix (reparameterize / change prior / smaller ε).
  • Tree depth saturation: hits max depth (default 10) → trajectories may be too short; posterior multimodal or very curved.

⑥ Relation to Langevin dynamics

When L = 1 HMC degenerates to Metropolis-Adjusted Langevin (MALA). HMC is the "long trajectory" version — better mixing, more expensive per step. SGLD, SG-HMC (Chen 2014) are big-data variants using minibatch gradients instead of full ∇U.

⑦ vs Variational Inference

VI is fast but point-estimate; HMC is slow but exact. Workflow: exploration with VI (pymc's ADVI, NumPyro SVI) for fast estimates; final paper uses NUTS for the full posterior. Mission-critical (drug effect, clinical trials) → always HMC.

潛在改良應用

還能改良哪些任務？

🎯 1. 單細胞 batch effect 不確定性

Harmony、scVI 給點估計 batch corrections；下游分析（如 DE）忽略 batch correction 的不確定性。提案：用 NUTS 推斷 batch correction 參數的完整後驗，propagate 到 DE p-value 校正。

🎯 2. 罕病 trio variant calling

Trio analysis（patient + 父 + 母）有複雜的遺傳模式（de novo, mosaic, compound het）。提案：Bayesian trio caller 用 NUTS 推斷 variant origin 後驗，給臨床醫師完整 likelihood 與不確定性，而非單一 yes/no。

🎯 3. Cell lineage tree 不確定性

CellTagging / Cas9 scarring 的 lineage tree 通常以 ML 估計，無不確定性。提案：Bayesian lineage tree，HMC 對 continuous（branch length, mutation rate）+ MH 對 tree topology，給出 tree posterior。

🎯 4. 多體學 joint posterior

多體學整合（如 MOFA、Seurat WNN）多是點估計或 VI。提案：Bayesian multi-omics with NUTS，給出 latent factor 的完整後驗，特別是 missingness 與 zero-inflation 下的不確定性。

🎯 5. Drug-target interaction Bayesian model

DTI prediction 目前主流 deep learning 給點估計。提案：Bayesian neural network + HMC 推斷 DTI posterior，給出 calibrated uncertainty——對藥物 repurposing 與 active learning 至關重要。

🎯 6. Spatial transcriptomics Bayesian deconvolution

Spot 是多 cell-type 混合；cell2location、CARD 給點估計。提案：NUTS 對 deconvolution 模型，給每個 spot 的 cell-type fraction posterior，能標出 spot 中是否有 rare cell type。

🎯 7. CRISPR screening 的 Bayesian effect size

MAGeCK / BAGEL 給 effect size 點估計；對 essentiality 邊界基因不可靠。提案：Bayesian screen analysis with NUTS，給每個 gene 的 essentiality posterior，特別有助 synthetic lethality 識別。

🎯 8. PK/PD Bayesian modeling

藥物 PK/PD 通常用 NONMEM (frequentist NLME)。提案：Stan/PyMC NUTS 做 hierarchical PK/PD，加入 between-subject variability、measurement error、prior knowledge——對 first-in-human dosing 有更穩健的不確定性傳遞。

🎯 1. Batch-correction uncertainty in single-cell

Harmony, scVI give point batch corrections; downstream DE ignores correction uncertainty. Proposal: NUTS over batch-correction parameters with full posterior; propagate into DE p-value calibration.

🎯 2. Rare-disease trio variant calling

Trio analysis (proband + father + mother) has complex inheritance patterns (de novo, mosaic, compound het). Proposal: Bayesian trio caller with NUTS — variant-origin posterior, full likelihood + uncertainty for clinicians, not a yes/no.

🎯 3. Cell lineage tree uncertainty

CellTagging / Cas9 scarring lineage trees are usually ML estimates, no uncertainty. Proposal: Bayesian lineage tree — HMC for continuous params (branch length, mutation rate) + MH for topology — full tree posterior.

🎯 4. Joint multi-omics posterior

Multi-omics integration (MOFA, Seurat WNN) usually point estimate or VI. Proposal: Bayesian multi-omics with NUTS — full latent factor posterior, especially under missingness and zero-inflation.

🎯 5. Bayesian drug-target interaction

DTI prediction is dominated by deep learning point estimates. Proposal: Bayesian NN + HMC for DTI posterior — calibrated uncertainty matters for drug repurposing and active learning.

🎯 6. Bayesian spatial deconvolution

Spots are cell-type mixtures; cell2location, CARD give point estimates. Proposal: NUTS-based deconvolution — per-spot cell-type fraction posteriors, flagging rare cell-type presence.

🎯 7. Bayesian CRISPR screen effect sizes

MAGeCK, BAGEL give point effect sizes; borderline essentials are unreliable. Proposal: Bayesian screen analysis with NUTS — per-gene essentiality posterior, especially useful for synthetic-lethal pairs.

🎯 8. Bayesian PK/PD modelling

Drug PK/PD is usually NONMEM (frequentist NLME). Proposal: Stan/PyMC NUTS for hierarchical PK/PD — between-subject variability, measurement error, prior knowledge — more robust uncertainty for first-in-human dosing.

# Minimal HMC from scratch (didactic)
import numpy as np

def leapfrog(theta, p, grad_U, eps, L):
    p = p - eps/2 * grad_U(theta)
    for _ in range(L-1):
        theta = theta + eps * p
        p     = p - eps * grad_U(theta)
    theta = theta + eps * p
    p     = p - eps/2 * grad_U(theta)
    return theta, -p   # negate p for symmetry

def hmc_step(theta, U, grad_U, eps=0.1, L=20):
    p0 = np.random.randn(*theta.shape)
    theta_new, p_new = leapfrog(theta, p0, grad_U, eps, L)
    H_old = U(theta)       + 0.5*(p0**2).sum()
    H_new = U(theta_new)   + 0.5*(p_new**2).sum()
    if np.log(np.random.rand()) < H_old - H_new:
        return theta_new, True
    return theta, False
