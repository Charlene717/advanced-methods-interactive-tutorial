張量分解 (Tensor Decomposition)

SVD 把矩陣分解為「行向量 × 列向量」加總；張量分解則把 3D、4D、5D 資料同時拆解出多個 mode 的「共變模式」。在「病人 × 細胞 × 基因 × 時間」這類多 mode 生物資料中，它是揭示因子結構的核心工具。

SVD decomposes a matrix into row × column factors; tensor decomposition generalises to 3D, 4D, 5D data, exposing shared modes. For multi-way biological data like "patient × cell × gene × time," it is the core tool for factor structure.

核心概念

從 SVD 到多階張量

2D 矩陣 M 的 SVD：M = U Σ Vᵀ ≈ Σ_r σ_r u_r v_rᵀ——把矩陣寫為「秩 1」外積的加總。

對 3-way 張量 T ∈ ℝ^(I×J×K)，最常見的兩種分解：

① CP / PARAFAC (Carroll & Chang 1970; Harshman 1970)：

T ≈ Σr=1R ar ⊗ br ⊗ cr

每個 component 是 3 個向量的外積。R 就是 tensor rank。優點：可解釋（每個 r 是獨立的「factor」）、在弱條件下 unique。

② Tucker (Tucker 1966)：

T = G ×1 U ×2 V ×3 W

G 是「核心張量 (core)」、U/V/W 是各 mode 的因子矩陣。比 CP 更彈性（可有 cross-mode 互動），但不 unique，須額外約束。

其他變體：Tensor Train (Oseledets 2011)、Hierarchical Tucker、t-SVD (Kilmer 2011) 都針對特定結構或計算效率。

SVD of 2D matrix: M = U Σ Vᵀ ≈ Σ_r σ_r u_r v_rᵀ — write the matrix as a sum of rank-1 outer products.

For a 3-way tensor T ∈ ℝ^(I×J×K), two main decompositions:

① CP / PARAFAC (Carroll & Chang 1970; Harshman 1970):

T ≈ Σr=1R ar ⊗ br ⊗ cr

Each component is the outer product of 3 vectors. R is the tensor rank. Pros: interpretable (each r is an independent factor); unique under mild conditions.

② Tucker (Tucker 1966):

T = G ×1 U ×2 V ×3 W

G is the "core tensor," U/V/W are mode-wise factor matrices. More flexible than CP (allows cross-mode interactions) but not unique without extra constraints.

Other variants: Tensor Train (Oseledets 2011), Hierarchical Tucker, t-SVD (Kilmer 2011), each for specific structures or efficiency.

📐

多 mode 結構

生物資料天生多 mode：個體、細胞、基因、時間、空間、treatment。張量是自然容器。

Biology is inherently multi-way: individual × cell × gene × time × space × treatment. Tensors are the natural container.

✨

CP 的唯一性

Kruskal (1977)：在 mild 條件下 CP 分解 unique（直到 scaling 與 permutation）——可直接「解釋」為獨立 factor。

Kruskal (1977): under mild conditions CP is unique up to scaling and permutation — factors can be directly interpreted.

⚡

壓縮與計算

Tucker(R₁,R₂,R₃) 把 N³ 個元素壓到 R₁R₂R₃ + R₁N + R₂N + R₃N，大幅省空間。

Tucker(R₁,R₂,R₃) compresses N³ elements to R₁R₂R₃ + R₁N + R₂N + R₃N — huge savings.

演算法

ALS、HOSVD 及其變體

① CP-ALS (Alternating Least Squares)

主流 CP 解法。固定 b、c，把 CP 化簡為 a 的線性 LS 問題；交替更新 a → b → c 直到收斂。每步是封閉形式 (matricized tensor times Khatri-Rao product, MTTKRP)：

A ← T(1) (C ⊙ B) (CᵀC ∗ BᵀB)†

(其中 ⊙ 是 Khatri-Rao product、∗ 是 Hadamard product)。容易實作但可能陷入 local optima、收斂慢。

② HOSVD / HOOI（Tucker 解法）

Higher-Order SVD (De Lathauwer 2000)：將張量沿每個 mode unfold 為矩陣 → 各取前 R_k 個 left singular vector → 得到 U, V, W。然後 G = T ×₁ Uᵀ ×₂ Vᵀ ×₃ Wᵀ。HOSVD 給良好初始化但非最佳；Higher-Order Orthogonal Iteration (HOOI) 進一步迭代到最優。

③ Non-negative / Sparse 變體

對 RNA-seq counts 等非負資料用 NCP (non-negative CP)——強制 a, b, c ≥ 0；解釋為「parts-based decomposition」（NMF 在張量的推廣）。Sparse-CP 加 L1 → 自動 feature selection。

④ Online / Streaming 分解

當資料分批到達（例如 single-cell perturbation experiment 隨時間累積）：Online Tensor Decomposition (Mardani 2015)、TensorSketch 在 streaming 下更新分解。

⑤ 統計推論版本

Bayesian Tensor Factorization (Rai 2014)：加入 prior 並用 MCMC / VI 給出 uncertainty。對「個人 × 細胞」這類有 missing data 的場景特別有用——能 imputation + uncertainty 一次完成。

① CP-ALS (Alternating Least Squares)

The workhorse for CP. Fix b, c; CP reduces to a linear LS problem for a; alternate a → b → c until convergence. Closed-form update (MTTKRP):

A ← T(1) (C ⊙ B) (CᵀC ∗ BᵀB)†

(⊙ Khatri-Rao, ∗ Hadamard). Easy but may stick at local minima and converges slowly.

② HOSVD / HOOI (Tucker)

Higher-Order SVD (De Lathauwer 2000): unfold along each mode → take top R_k left singular vectors → U, V, W. Then G = T ×₁ Uᵀ ×₂ Vᵀ ×₃ Wᵀ. HOSVD gives a good init but isn't optimal; Higher-Order Orthogonal Iteration (HOOI) refines.

③ Non-negative / Sparse variants

For non-negative data (RNA-seq counts) use NCP with a, b, c ≥ 0 — interpretable as parts-based decomposition (NMF generalised to tensors). Sparse-CP adds L1 → automatic feature selection.

④ Online / streaming decomposition

When data arrives in batches (e.g. accumulating single-cell perturbation experiments): Online Tensor Decomposition (Mardani 2015), TensorSketch.

⑤ Probabilistic versions

Bayesian Tensor Factorization (Rai 2014): priors + MCMC / VI yield uncertainty. Especially useful for "individual × cell" matrices with missing entries — imputation + uncertainty in one shot.

關鍵差別：CP unique 但 rigid（component 必須是 rank-1 外積）；Tucker 彈性但 non-unique（可加旋轉自由度）。生物資料常用 CP 求「可解釋的 factor」，Tucker 求「壓縮 / interaction」。
Key distinction: CP is unique but rigid (rank-1 outer products); Tucker is flexible but non-unique (extra rotational freedom). Bio data uses CP for interpretable factors, Tucker for compression / interactions.

生物應用

張量分解在生物資訊的應用

Tensor-cell2cell

Armingol et al. (2022, Nature Communications)：建構「病人 × 細胞型別 pair × LR pair」3-way 張量，用 NCP 找出「跨病人共享的 cell-cell communication factors」。發現了單病人分析看不到的 universal LR 模組與 disease-specific 模組。

Armingol et al. (2022, Nat Comm): build a "patient × cell-type pair × LR pair" tensor, apply NCP to discover shared cell–cell communication factors across patients. Reveals universal LR modules and disease-specific modules invisible to single-patient analysis.

LIGER (iNMF)

Welch et al. (2019, Cell)：integrative NMF（NMF 在多個矩陣的推廣，與 CP 等價）對 cross-species/cross-modal scRNA-seq 整合——學「共享 factor」+「dataset-specific factor」。

Welch et al. (2019, Cell): integrative NMF (the multi-matrix NMF, equivalent to CP) for cross-species / cross-modal scRNA-seq integration — learns shared factors + dataset-specific factors.

MOFA+

Argelaguet et al. (2020, Genome Biology)：Bayesian group factor analysis 對「多 view × 多 group」的多體學整合。本質上是 Tucker 變體 + sparsity prior，廣泛用於 CITE-seq、multi-tissue 整合。

Argelaguet et al. (2020, Genome Biol): Bayesian group factor analysis for "multi-view × multi-group" multi-omics integration. Essentially a Tucker variant with sparsity priors — widely used for CITE-seq and multi-tissue integration.

病人 × 細胞 × 時間

Hore et al. (2016) 用 Tucker 對「病人 × 細胞 × 基因」表現張量分解——發現某些病人 share 共同免疫反應模組，另一些有 patient-specific 模組。為 immune profiling 開了門。

Hore et al. (2016) decomposed "patient × cell × gene" expression tensors with Tucker — found shared immune-response modules across patients and patient-specific ones. Opened the door to immune profiling.

MultiBLUP

把「SNP × tissue × phenotype」當張量分解，找跨 tissue 共享 SNP-phenotype 關聯。比單獨 tissue GWAS 統計力更強。

Decompose "SNP × tissue × phenotype" tensors to find cross-tissue shared SNP-phenotype associations. More powerful than per-tissue GWAS.

pTAD（多病人 Hi-C）

Cresswell & Dozmorov (2020) 用 3D 張量（病人 × bin × bin）分解識別 conserved 與 disease-specific TADs。比單樣本 TAD calling 抗噪。

Cresswell & Dozmorov (2020) decompose a "patient × bin × bin" tensor to identify conserved and disease-specific TADs — more noise-robust than per-sample TAD calling.

深入解析

挑張量 rank、unique 條件與計算技巧

① 張量 rank 的詭異性

2D matrix rank 是「max independent rows = max independent cols」；對張量這兩個未必相等。更糟：3-way 張量的 rank 是 NP-hard 問題（Håstad 1990）！實務上用 core consistency diagnostic (CORCONDIA) 或 elbow method 估計合理 R。

② Kruskal 唯一性條件

CP 分解 unique（直到 scaling + permutation）當且僅當 k(A) + k(B) + k(C) ≥ 2R + 2，其中 k(·) 是 Kruskal rank（任 k 行線性獨立的最大 k）。生物資料通常滿足此條件，所以 CP factor 可直接解釋。

③ Tucker 的旋轉模糊性

Tucker 分解 T = G ×₁ U ×₂ V ×₃ W = G' ×₁ (UQ⁻¹) ×₂ V ×₃ W 對任意 invertible Q 都成立。所以 Tucker 「factor」沒有唯一意義，需用 orthogonality 或 sparsity 約束選定 canonical form。

④ Sketching / Randomized 加速

Battaglino et al. (2018) 提出 randomized CP-ALS：每步只用隨機 sketch 解 LS，把單次迭代從 O(N²R) 降到 O(NR log N)。對 10⁸ 元素張量 (1000 cells × 1000 genes × 100 time) 數小時 → 數分鐘。

⑤ Coupled / Joint Tensor Factorization

當多個張量「共享一個 mode」（例如 RNA tensor 與 ATAC tensor 共享 cell mode）：Coupled-CP 強制 cell factor matrix 相同，跨資料學共享 factor。是 multi-omics 張量整合的核心。

⑥ Tensor Train 與深度學習

Oseledets (2011) 的 Tensor Train (TT) 把 d-way 張量寫為 d 個 3-way tensor 的「序列」收縮——適合 d ≥ 5 的高階張量。在 deep learning 中用於壓縮 weight tensor (TT-Net, Novikov 2015)。

⑦ Bures-Tensor 與 quantum information

張量分解與 quantum many-body physics 的 Matrix Product State / Tensor Network 完全是同一數學——Penrose graphical notation、tensor contraction。生物學家用 TT 時，本質上是在做 quantum-style state representation。

⑧ Differentiable Tensor Decomposition

把 CP-ALS 替換成 SGD on |T − Σ a_r ⊗ b_r ⊗ c_r|²，使整個分解可微 → 直接嵌入 deep learning pipeline。tensorly-torch、JAX 版本都支援。可學「tensor factor 為下游任務最佳」而非「最小重建誤差」。

① The quirks of tensor rank

For matrices, rank = max independent rows = max independent cols. For tensors these are unequal. Worse: tensor rank is NP-hard (Håstad 1990). In practice use core consistency diagnostic (CORCONDIA) or the elbow method to pick R.

② Kruskal uniqueness

CP is unique (up to scaling + permutation) iff k(A) + k(B) + k(C) ≥ 2R + 2, where k(·) is the Kruskal rank. Biological data usually meets this — CP factors are directly interpretable.

③ Tucker rotational ambiguity

Tucker T = G ×₁ U ×₂ V ×₃ W = G' ×₁ (UQ⁻¹) ×₂ V ×₃ W for any invertible Q. So Tucker factors have no canonical meaning unless additional constraints (orthogonality, sparsity) pick a unique form.

④ Sketching / randomized speedup

Battaglino et al. (2018): randomized CP-ALS uses sketches per LS step, reducing one iteration from O(N²R) to O(NR log N). For 10⁸-element tensors (1000 cells × 1000 genes × 100 time): hours → minutes.

⑤ Coupled / joint tensor factorization

When multiple tensors share a mode (RNA and ATAC tensors share the cell mode): Coupled-CP forces the cell factor matrix to be shared, learning common factors across data. Core to multi-omics tensor integration.

⑥ Tensor Train and deep learning

Oseledets (2011) Tensor Train (TT) decomposes a d-way tensor into a sequence of 3-way contractions — ideal for d ≥ 5. Used in deep learning to compress weight tensors (TT-Net, Novikov 2015).

⑦ Tensor networks meet quantum many-body physics

Tensor decomposition is mathematically identical to quantum Matrix Product States / Tensor Networks — same Penrose notation, same contractions. Biologists using TT are effectively doing quantum-style state representations.

⑧ Differentiable tensor decomposition

Replace CP-ALS with SGD on |T − Σ a_r ⊗ b_r ⊗ c_r|² → fully differentiable, embeds in deep learning. Supported by tensorly-torch, JAX variants. Learn factors optimal for a downstream task, not just minimum reconstruction.

潛在改良應用

張量分解的潛在新應用

🎯 1. 連續時間 × 個人 × 細胞型別 × 基因 (5D)

單細胞時序資料 + 多病人：建構「病人 × 細胞型別 × 基因 × time bin × treatment」5-way 張量，CP / Tucker 分解學「跨多個 mode 共享的 dynamics factor」。預期能分離出 disease progression、treatment response、circadian 三類 dynamics。

🎯 2. 藥物 × 細胞 × 時間 × 基因（drug screen）

把 LINCS L1000 + 動態 scRNA-seq 整合為 4-way 張量。CP factor 可作為「drug mode of action」的數位指紋——比 Connectivity Map 的 ranked list 更語意化。

🎯 3. 癌症亞型 × 治療 × 時間張量

對 TCGA + 臨床追蹤建構「病人亞型 × treatment × follow-up time × phenotype」張量，找出「對特定治療有反應的亞型 × 時間」聯合 factor。預期能 stratify trial 設計。

🎯 4. Spatial × time × gene tensor

多時間點 spatial transcriptomics：(spatial_x, spatial_y, gene, time) 4D 張量。CP factor 同時揭示「空間模式 × 時間動態」——例如 wound healing 的空間擴散波。

🎯 5. Multi-modal Perturb-seq 張量

「perturbation × cell × gene × modality (RNA/ATAC/protein)」4D 張量。Coupled-CP 學「跨模態保守的擾動 factor」——抓出真正的 mechanism、過濾 modality-specific noise。

🎯 6. Cross-species developmental atlas

「species × cell type × developmental stage × gene」張量。Tucker 分解的 core G 直接量化「跨物種發育保守性」。預期可找出 evo-devo 的關鍵 master regulator。

🎯 7. Single-cell Hi-C 張量分析

「cell × bin × bin」張量，CP 找出 cell-state-specific chromatin organization factor——比 per-cell TAD calling 更穩。

🎯 8. EHR + omics 整合

「病人 × 訪視時間 × clinical feature × omics feature」高階張量。Bayesian CP 處理 missing data，imputation + factor 分析一次完成。對長期慢性病管理有潛力。

🎯 9. Tensor-based GRN

「TF × target × tissue × condition」4D tensor，CP 學 condition-specific regulation factor。比每個 tissue 單獨建 GRN 更高效且利用 cross-tissue info。

🎯 10. Differentiable tensor decomposition + scLLM

把 scGPT/Geneformer 的 latent reshape 為「cell × gene × layer」張量，做 differentiable CP——學「跨層共享的語意 factor」。對 scLLM 可解釋性與壓縮都有貢獻。

🎯 1. Continuous time × individual × cell type × gene (5D)

Single-cell time series + multiple patients: build a "patient × cell type × gene × time bin × treatment" 5-way tensor, CP / Tucker learns shared dynamics factors. Expected to separate disease progression, treatment response, and circadian dynamics.

🎯 2. Drug × cell × time × gene (drug screen)

Integrate LINCS L1000 + dynamic scRNA-seq into a 4-way tensor. CP factors form "drug mode of action" digital fingerprints — more semantic than Connectivity Map ranked lists.

🎯 3. Cancer subtype × treatment × time tensor

TCGA + clinical follow-up: "subtype × treatment × follow-up time × phenotype" tensor finds joint factors "subtype × time responsive to specific treatment." Could stratify clinical trials.

🎯 4. Spatial × time × gene tensor

Multi-timepoint spatial transcriptomics: (spatial_x, spatial_y, gene, time) 4D tensor. CP factors capture spatial pattern × time dynamics — e.g. wound healing diffusion waves.

🎯 5. Multi-modal Perturb-seq tensors

"perturbation × cell × gene × modality (RNA/ATAC/protein)" 4D tensor. Coupled-CP learns modality-conserved perturbation factors — true mechanism, filtering modality-specific noise.

🎯 6. Cross-species developmental atlas

"species × cell type × developmental stage × gene" tensor. The Tucker core G directly quantifies cross-species developmental conservation, surfacing evo-devo master regulators.

🎯 7. Single-cell Hi-C tensors

"cell × bin × bin" tensor with CP factors finds cell-state-specific chromatin organization, more robust than per-cell TAD calling.

🎯 8. EHR + omics integration

"patient × visit time × clinical feature × omics feature" high-order tensor. Bayesian CP handles missing data — imputation + factor analysis in one step. Promising for chronic disease management.

🎯 9. Tensor-based GRN

"TF × target × tissue × condition" 4D tensor, CP learns condition-specific regulation. More efficient than per-tissue GRNs and leverages cross-tissue information.

🎯 10. Differentiable tensor decomposition + scLLM

Reshape scGPT/Geneformer latents as "cell × gene × layer" tensors with differentiable CP — learn cross-layer semantic factors. Improves both interpretability and compression of scLLMs.

程式碼

實作範例

import tensorly as tl
from tensorly.decomposition import parafac, tucker, non_negative_parafac
import numpy as np

# T shape: (patient, cell_type, gene)
T = tl.tensor(np.random.rand(20, 15, 3000))

# CP decomposition with rank=10
weights, factors = parafac(T, rank=10, n_iter_max=100)
patient_F, cell_F, gene_F = factors   # each (mode_dim, R)

# Non-negative CP for counts / abundance
weights, factors_nn = non_negative_parafac(T, rank=10)

# Tucker with multilinear ranks (5, 4, 20)
core, factors_t = tucker(T, rank=[5, 4, 20])
import cell2cell as c2c
# Tensor: (sample, lr_pair, sender_celltype, receiver_celltype)
tensor = c2c.tensor.InteractionTensor(
    rnaseq_matrices={s: adata_dict[s] for s in samples},
    metadata=meta_dict,
    interaction_columns=("ligand", "receptor"),
    communication_score="expression_mean")
tensor.compute_tensor_factorization(rank=8, init="random")
# Output: per-sample factor loadings, per-LR factor loadings
c2c.plotting.factor_plot.plot_factors(tensor.factors)
# install.packages("rTensor")
library(rTensor)

T <- as.tensor(array(rnorm(20*15*3000), c(20,15,3000)))

# CP decomposition
cp_fit <- cp(T, num_components = 10, max_iter = 100)
cp_fit$U                          # list of factor matrices
cp_fit$norm_percent               # variance explained

# Tucker decomposition
tk_fit <- tucker(T, ranks = c(5,4,20))
tk_fit$Z@data                     # core tensor
