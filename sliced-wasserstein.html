切片 Wasserstein / 快速 OT

Sinkhorn OT 一次比較 N 個樣本要 O(N² log N)。1D Wasserstein 卻只要 sort：O(N log N)。Sliced-Wasserstein 把高維分布投影到隨機 1D 方向，平均多個 1D W——快、可微、樣本複雜度低，已成單細胞、影像、生成模型的標準距離。

Sinkhorn OT costs O(N² log N) to compare N samples. 1D Wasserstein needs only sorting: O(N log N). Sliced-Wasserstein projects high-dim distributions onto random 1D directions and averages — fast, differentiable, sample-efficient. Now standard in single-cell, imaging and generative modelling.

核心概念

從 OT 到 Sliced-Wasserstein

OT 的痛點：標準 p-Wasserstein 距離 W_p(μ, ν) 在 d 維需解線性規劃，N 樣本下複雜度 O(N³ log N)；entropic regularised Sinkhorn (Cuturi 2013) 把成本降到 O(N² / ε²)，但仍是 quadratic in N，且涉及非平凡 hyperparam ε。

1D 的奇蹟：當 d = 1，兩個 1D 分布的 W_p 有 closed-form：

W_p^p(μ, ν) = ∫_0^1 | F_μ^{-1}(t) − F_ν^{-1}(t) |^p dt

對 N 個樣本就是 sort + 對齊兩個 sorted 陣列：複雜度 O(N log N)。沒有 LP、沒有 hyperparameter。

Sliced-Wasserstein (Bonneel et al. 2015, JMIV; Rabin et al. 2011)：把高維問題化為 L 個 1D 問題的平均：

SW_p^p(μ, ν) = ∫_{S^{d-1}} W_p^p(θ_# μ, θ_# ν) dσ(θ) ≈ (1/L) Σ_{l=1}^L W_p^p(P_{θ_l} μ, P_{θ_l} ν)

其中 θ ∈ S^{d-1} 是隨機單位方向，P_θ μ 是 μ 沿 θ 的 1D 邊際分布。理論：SW 仍是 metric (Bonnotte 2013) 且與 W 拓樸等價（在緊緻空間上）。

OT pain point: the standard p-Wasserstein W_p(μ, ν) in d-D is a linear program with O(N³ log N) complexity for N samples. Entropic Sinkhorn (Cuturi 2013) drops this to O(N² / ε²) — still quadratic, and adds a non-trivial hyperparameter ε.

The 1D miracle: for d = 1, the W_p between two 1D distributions has a closed form:

W_p^p(μ, ν) = ∫_0^1 | F_μ^{-1}(t) − F_ν^{-1}(t) |^p dt

For N samples this is just sort + pair the two sorted arrays: complexity O(N log N). No LP, no hyperparameter.

Sliced-Wasserstein (Bonneel et al. 2015, JMIV; Rabin et al. 2011): reduces the high-dim problem to an average of L 1D problems:

SW_p^p(μ, ν) = ∫_{S^{d-1}} W_p^p(θ_# μ, θ_# ν) dσ(θ) ≈ (1/L) Σ_{l=1}^L W_p^p(P_{θ_l} μ, P_{θ_l} ν)

θ ∈ S^{d-1} is a random unit direction, P_θ μ the 1D marginal of μ along θ. Theory: SW is still a metric (Bonnotte 2013) and topologically equivalent to W on compact spaces.

⚡

近線性複雜度

每個切片 O(N log N) · L 個切片 → O(L N log N)，比 Sinkhorn 的 O(N²) 快數量級。

Per slice O(N log N) · L slices → O(L N log N), orders of magnitude faster than Sinkhorn's O(N²).

🎯

可微分

sort 雖不可微，但「sorted index 對齊」可微 → SW 是 generator / autoencoder 損失的好選擇。

Sorting itself isn't differentiable, but "aligning indices of sorted arrays" is — SW is ideal for generator / autoencoder losses.

📊

樣本複雜度

SW 在維度上樣本複雜度 O(N^{-1/2})（與 d 無關），而 W 是 O(N^{-1/d}) — 雙重勝利。

SW has dimension-free O(N^{-1/2}) sample complexity, vs W's O(N^{-1/d}) — a double win.

演算法

SW 計算、Max-SW、Generalised-SW

① SW 計算 — 三步驟（Bonneel 2015）：

  1. 取 L 個隨機單位方向 θ_l ~ Unif(S^{d-1})（簡單做法：抽 d 維高斯後 normalise）。
  2. 對每個 θ_l：計算投影 a_i = ⟨x_i, θ_l⟩、b_j = ⟨y_j, θ_l⟩（兩組 N 個 1D 值）。
  3. 分別 sort 兩組，配對成 (a_(i), b_(i))，計算 W_p^p = (1/N) Σ_i |a_(i) − b_(i)|^p。

最終 SW^p ≈ (1/L) Σ_l W_p^p。當 L → ∞ 收斂到真 SW；通常 L = 50–500 已足夠穩。

② Max-Sliced Wasserstein (Deshpande et al. 2019, CVPR)：

Max-SW(μ, ν) = max_{θ ∈ S^{d-1}} W_p(P_θ μ, P_θ ν)

不平均所有方向，而找「最判別性」的方向。高維下 SW 的 L 個隨機方向多半 informative 低（curse of dimensionality）；Max-SW 集中在最重要方向，sample efficiency 更好。實作：對 θ 做 projected gradient ascent。

③ Distributional Sliced-Wasserstein (Nguyen et al. 2021, ICLR)：學一個方向分布 q(θ) 而非單一最大方向——介於 SW 和 Max-SW 之間，更穩健。可用於 GAN-style 訓練。

④ Generalised SW (Kolouri et al. 2019, NeurIPS)：把 linear projection P_θ 換成 非線性 projection g_θ：μ → ℝ。常見：神經網路、polynomial、Radon transform 廣義版。對「捲曲」manifold 友善。

⑤ Sliced Gromov-Wasserstein (Vayer et al. 2019, NeurIPS)：Gromov-Wasserstein (GW) 對 metric space 之間做比較，但成本是 O(N⁴)；Sliced-GW 把兩個 metric space 各投影到 1D 後計算 GW、平均。應用：跨物種 / 跨模態 single-cell 對齊。

⑥ Sinkhorn-Sliced 混合：Sliced-W loss 可作為 Sinkhorn 的 warm-up，或在 batch 級別交替使用，兼顧速度與精確 coupling。GeomLoss (Feydy 2019) 同時提供。

① SW computation — three steps (Bonneel 2015):

  1. Draw L unit directions θ_l ~ Unif(S^{d-1}) (sample d-D Gaussian, normalise).
  2. For each θ_l: project a_i = ⟨x_i, θ_l⟩, b_j = ⟨y_j, θ_l⟩ (two arrays of N values).
  3. Sort both, pair (a_(i), b_(i)), compute W_p^p = (1/N) Σ_i |a_(i) − b_(i)|^p.

SW^p ≈ (1/L) Σ_l W_p^p; L = 50–500 typically suffices.

② Max-Sliced Wasserstein (Deshpande et al. 2019, CVPR):

Max-SW(μ, ν) = max_{θ ∈ S^{d-1}} W_p(P_θ μ, P_θ ν)

Instead of averaging, find the most discriminative direction. In high dim most random directions carry low information (curse of dimensionality); Max-SW concentrates effort and is sample-efficient. Implementation: projected gradient ascent on θ.

③ Distributional Sliced-Wasserstein (Nguyen et al. 2021, ICLR): learn a distribution q(θ) over directions instead of a single max — between SW and Max-SW, more robust. Used in GAN-style training.

④ Generalised SW (Kolouri et al. 2019, NeurIPS): replace linear P_θ with a non-linear projection g_θ: μ → ℝ (NNs, polynomials, generalised Radon). Helpful for curved manifolds.

⑤ Sliced Gromov-Wasserstein (Vayer et al. 2019, NeurIPS): GW compares two metric spaces but costs O(N⁴); Sliced-GW projects each to 1D, runs GW per slice, averages. Used for cross-species / cross-modality single-cell alignment.

⑥ Sinkhorn-Sliced hybrid: use Sliced-W as warm-up to Sinkhorn, or alternate per batch for speed + exact coupling. Provided by GeomLoss (Feydy 2019).

為什麼 sort 就夠？ 對 1D 兩個分布的 W_p，最佳 coupling 就是「分位數對齊」——把兩邊都 sort 後對應位置直接配。這是 1D OT 的特有結構（不需 LP）；高維沒有這個免費午餐。
Why is sorting enough? For two 1D distributions, the optimal W_p coupling is "quantile alignment" — sort both sides and pair by position. This is a 1D-only structural shortcut (no LP needed); high-dim lacks this free lunch.

⚡ 何時用 Sliced-Wasserstein？

情境1:
大樣本 (N > 10⁴): Sinkhorn O(N²) 已不可行；SW O(L N log N) 仍可行。
情境2:
需要梯度: 訓練 generator / VAE / autoencoder 時，SW 損失比 GAN 損失穩定、比 Sinkhorn 快。
情境3:
跨 batch / 跨 donor 比較: 比較 single-cell population distance；SW 是好 metric。
不適合:
需要明確 transport plan / coupling（例 batch alignment 要直接配對） → 用 Sinkhorn 或 unbalanced OT；極高維且高度稀疏（直接 random projection 失效）→ Generalised SW。
Case 1:
Large N (>10⁴): Sinkhorn O(N²) is impractical; SW O(L N log N) still works.
Case 2:
Gradient required: training generators / VAEs / AEs — SW loss is more stable than GAN, faster than Sinkhorn.
Case 3:
Cross-batch / cross-donor comparison: single-cell population distance — SW is a good metric.
Not for:
Need an explicit transport plan / coupling (e.g. batch alignment requires direct pairing) → use Sinkhorn or unbalanced OT; extremely high-dim and very sparse (linear projections lose info) → Generalised SW.
生物應用

SW 在單細胞、空間、生成模型

Sliced-Wasserstein AutoEncoder

Kolouri et al. (2018, ICLR) SWAE 用 SW loss 取代 WAE 的 GAN/MMD critic — 訓練穩、無 adversarial、隱空間分布貼合 prior。生物應用：scVI 的變體、藥物分子 VAE 損失改良（更穩定的 latent geometry）。

Kolouri et al. (2018, ICLR) SWAE replaces the WAE GAN/MMD critic with an SW loss — stable, adversarial-free, latent matches the prior. Used in scVI variants and molecule-VAE losses with better latent geometry.

SW-PD 跨 donor population 距離

single-cell 多 donor 比較需要 population-level distance，但 N 可達 10⁵+。Sliced-Wasserstein Population Distance (Cao, Yan 2022; 衍生工作 2023–24) 用 SW 計算 donor pairwise distance，可用於做 hierarchical clustering 或 association test 與 disease 表型相關性。

Comparing multiple donors at the single-cell population level needs a distance with N up to 10⁵+. Sliced-Wasserstein Population Distance (Cao, Yan 2022; follow-ups 2023–24) computes pairwise donor distances via SW for hierarchical clustering and disease-association tests.

空間轉錄體 slice alignment

PASTE (Zeira et al. 2022, Nat Methods) 用 OT 對齊連續切片；後續 PASTE2、SPACEL 採用 Sliced-GW 提速、處理大樣本切片。對大規模 Visium / Slide-seqV2 資料尤其關鍵。

PASTE (Zeira et al. 2022, Nat Methods) aligns consecutive spatial slices via OT; later PASTE2 and SPACEL use Sliced-GW for speed and scalability. Especially critical for large Visium / Slide-seqV2 datasets.

Batch comparison 與 effect quantification

不同 batch 的 single-cell 嵌入分佈可用 SW 比較；SW 距離可作為 batch effect 強度 metric，比 kBET / iLISI 更原則化、可微（可做正則項）。已見於 scIB benchmark 變體。

Embeddings from different batches can be compared via SW; SW distance is a principled, differentiable batch-effect metric — more rigorous than kBET / iLISI and usable as a regulariser. Appears in scIB benchmark variants.

Sliced Gromov-Wasserstein 跨物種

Vayer et al. (2019, NeurIPS) Sliced-GW 把兩個無法直接共享 feature 的 metric space 對齊 — 用於人–鼠 single-cell 對齊（不同 gene set）、跨模態（RNA ↔ ATAC）embedding 對齊。比 full GW 快 100×。

Vayer et al. (2019, NeurIPS) Sliced-GW aligns two metric spaces lacking shared features — used for human–mouse single-cell alignment (different gene sets) and cross-modal (RNA ↔ ATAC) embedding alignment. 100× faster than full GW.

Cross-modal embedding 對齊

CITE-seq 同一細胞的 RNA + protein latent 對齊；ATAC + RNA multiome；spatial + scRNA-seq 整合。SW 比 Sinkhorn 在 100k-cell 級別資料快 30–100×，且更易加入 GPU 訓練。

CITE-seq RNA + protein latents, ATAC + RNA multiome, spatial + scRNA-seq integration — SW is 30–100× faster than Sinkhorn on 100k-cell datasets and integrates easily into GPU training.

生成模型損失

Wu et al. (2019) SW-GAN 用 sliced-W 替換 WGAN critic — 無 adversarial 不穩。後續用於 protein structure 生成、分子 VAE、cell embedding GAN。Diffusion 變體中也有人用 SW 替代 ELBO 對齊樣本分布。

Wu et al. (2019) SW-GAN replaces the WGAN critic with sliced-W — no adversarial instability. Later used for protein structure generation, molecule VAEs, cell-embedding GANs. Some diffusion variants use SW instead of ELBO to align sample distributions.

scVI variants

scVI / totalVI / multiVI 是 VAE-based single-cell 模型；近年變體（如 scvi-tools 內部實驗、社群 PR）用 SW prior matching 替代 KL，對 latent geometry 控制更好、batch correction 更乾淨。

scVI / totalVI / multiVI are VAE-based single-cell models; recent variants (scvi-tools experiments, community PRs) replace KL with SW prior matching for better latent geometry and cleaner batch correction.

深入解析

進階主題

① 方向選取的維度詛咒

SW 對高維 d 不友善——隨機方向幾乎正交於資料 manifold，多數方向只看到「mean shift」沒有結構資訊。Nadjahi et al. (2020, NeurIPS) 對 SW 樣本複雜度做詳細分析；Max-SW、Distributional-SW、Generalised-SW 都是對此問題的回應。實務：在 PCA 後的低維 latent 計算 SW 更有效。

② SW 與 W 的關係

SW ≤ W（容易證），但反向沒有；Bonnotte (2013) 證明在緊緻 d 維域上兩者拓樸等價，但 SW 的「rate」比 W 弱。實務：SW 是 W 的好 surrogate，但要量化 transport cost 仍需 W / Sinkhorn。

③ Sample complexity

W_p 的 empirical estimator 收斂率為 O(N^{-1/d})（curse of dim），SW 為 O(N^{-1/2})（與 d 無關）。對 single-cell N = 10⁵ 量級且 d = 50 維 latent，SW 估計穩定，W 估計極為 noisy。

④ Differentiable sorting trick

Sliced-W 訓練時需對 sort 操作微分。實作技巧：sort 不直接可微，但其「permutation」相對輸入連續變化（measure-zero except at ties），實務上對輸入梯度可用——PyTorch torch.sort() 即提供。Cuturi et al. (2019) 提出 soft-sort 平滑近似，數值更穩。

⑤ Convex Sliced-W

Bonneel & Coeurjolly (2019) Spot the Spot 用 max-SW 找最判別方向；對 GAN-style 訓練更穩。後續 distributional sliced (Nguyen 2021) 介於 mean 與 max 之間。

⑥ Sliced for non-Euclidean / Hyperbolic

近年延伸到 hyperbolic SW (Bonet 2023)、circular SW、graph SW。對 cell ontology / phylogeny 等非歐式資料給出快速距離。

⑦ 何時改用 Sinkhorn？

當需要明確 transport plan（如 batch correction 要把 cell A 對應到 cell B 整合）、或計算 OT-derived quantities (Wasserstein gradient flow, OT barycentre)，仍以 Sinkhorn / unbalanced Sinkhorn 為主。SW 主要當「相似度」用。

① Curse of dimensionality on directions

SW degrades in very high d — random directions are nearly orthogonal to the data manifold; most carry only mean-shift info. Nadjahi et al. (2020, NeurIPS) analyse SW sample complexity; Max-SW, Distributional-SW, Generalised-SW respond to this. Practical fix: compute SW in a PCA-reduced latent.

② SW vs W

SW ≤ W (easy); the reverse fails. Bonnotte (2013) shows topological equivalence on compact d-D domains, but SW's "rate" is weaker. SW is a good W surrogate, but for transport cost itself you still want W / Sinkhorn.

③ Sample complexity

Empirical W_p converges at O(N^{-1/d}) — curse of dim; SW at O(N^{-1/2}) — dim-free. For single-cell with N ~ 10⁵ and d = 50 latent, SW is stable, W is extremely noisy.

④ Differentiable sorting

Training SW needs gradients through sort. Sort itself isn't differentiable, but the permutation is locally constant in input (except at ties), so gradient w.r.t. input values is well-defined a.e. — PyTorch torch.sort() supports it. Cuturi et al. (2019) propose smooth soft-sort for added stability.

⑤ Convex Sliced-W

Bonneel & Coeurjolly (2019) "Spot the Spot" use Max-SW for the most discriminative direction — stable for GAN-style training. Distributional-SW (Nguyen 2021) interpolates between mean and max.

⑥ Non-Euclidean / hyperbolic Sliced-W

Recent: hyperbolic SW (Bonet 2023), circular SW, graph SW. Provides fast distances on cell ontologies / phylogenies and other non-Euclidean structures.

⑦ When to use Sinkhorn instead?

When you need an explicit transport plan (e.g. batch correction that pairs cell A to cell B), or OT-derived quantities (Wasserstein gradient flow, OT barycentres), stay with Sinkhorn / unbalanced Sinkhorn. SW is mainly used as a "similarity".

import torch

def sliced_w2(X, Y, L=128):
    """Sliced 2-Wasserstein for X (N,d), Y (M,d). Differentiable."""
    d = X.shape[1]
    theta = torch.randn(d, L, device=X.device); theta = theta / theta.norm(dim=0, keepdim=True)
    A = X @ theta                            # (N, L)
    B = Y @ theta                            # (M, L)
    # Sort per direction independently; pad if N != M (here assume equal)
    A_s, _ = A.sort(dim=0)
    B_s, _ = B.sort(dim=0)
    return ((A_s - B_s)**2).mean()       # average over N AND L
