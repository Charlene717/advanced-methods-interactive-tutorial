專家混合 (Mixture of Experts, MoE)

為什麼 GPT-4、Mixtral、Gemini 都用 MoE？因為「N 個 expert 網路 + gating」可以把參數量加到天文數字，但每個 token 只啟用 top-K 個 expert——計算量幾乎不變。Jacobs 1991 提出、Shazeer 2017 真正 scale，現已是大模型主流架構。

Why do GPT-4, Mixtral, and Gemini use MoE? Because "N experts + gating" lets you scale parameters astronomically while only top-K experts activate per token — compute stays nearly flat. Jacobs 1991 introduced it; Shazeer 2017 scaled it; today it's the mainstream architecture for large models.

核心概念

MoE 的基本想法

標準 dense MLP 的問題：所有 input 都通過同一組 weight matrix。但實際上不同 input（不同 token、不同 cell type、不同 tissue）需要不同的處理 — 把所有 specialty 塞進一組 weights 是浪費。

MoE 的設計 (Jacobs et al. 1991, Neural Comp; Jordan & Jacobs 1994)：用 N 個小型「expert」網路 E₁, …, E_N + 一個「gating」網路 g。輸入 x 的輸出是 gating 加權的 expert 組合：

y = Σ_{i=1}^N g_i(x) · E_i(x)，   g(x) ∈ Δ^{N-1}（單純形）

Dense MoE 的問題：每個 expert 都要計算，等於把 dense network 變大 N 倍——沒有真正節省 compute。

Sparse MoE 的關鍵洞見 (Shazeer et al. 2017, ICLR)：讓 gating 是 top-K sparse——每個 input 只啟用 K（通常 1 或 2）個 expert：

g(x) = TopK(softmax(W_g x))

於是參數量 ∝ N（很大），但 FLOPs ∝ K（很小）。Mixtral 8×7B 總共 47B params，但 inference 時只用 13B（top-2 of 8 experts）。

Sparsity 帶來的核心優勢：

  • 更大模型容量：同等 inference cost 下放入更多 knowledge。
  • 專業分工：不同 expert 自然學到不同 specialty（語言、領域、modality）。
  • 條件式計算 (Conditional Computation)：困難 token 用更多 expert（部分變體），簡單 token 走 fast path。

The dense MLP problem: every input passes through the same weight matrix. But different inputs (different tokens, cell types, tissues) need different processing — packing all specialties into one weight set is wasteful.

MoE design (Jacobs et al. 1991, Neural Comp; Jordan & Jacobs 1994): N small "expert" networks E₁, …, E_N + a "gating" network g. Output is a gating-weighted expert mixture:

y = Σ_{i=1}^N g_i(x) · E_i(x),   g(x) ∈ Δ^{N-1}

Dense MoE problem: every expert is computed — equivalent to making a dense network N× larger, no compute saving.

Sparse MoE key insight (Shazeer et al. 2017, ICLR): make gating top-K sparse — each input activates only K (often 1 or 2) experts:

g(x) = TopK(softmax(W_g x))

So parameters ∝ N (huge) but FLOPs ∝ K (small). Mixtral 8×7B has 47B total params, but inference uses only 13B (top-2 of 8).

What sparsity buys you:

  • Larger capacity at the same inference cost.
  • Specialization: experts naturally learn different specialties (language, domain, modality).
  • Conditional computation: harder inputs use more experts (in some variants), easy ones take the fast path.

演算法

從 Jacobs 1991 到 Mixtral / Switch / Soft MoE

① Original MoE (Jacobs 1991, Jordan & Jacobs 1994)：Dense，全部 expert 計算後 gating 加權。用 EM-style training：E-step 計算 responsibilities g_i(x_t)，M-step 更新 expert。理論優美但 N 增大時 compute 線性增加。

② Sparsely-gated MoE (Shazeer 2017, ICLR)：真正的突破。Top-K gating + 加 Gaussian noise 防止 collapse + load balancing loss：

L_aux = α · CV(load_i)² + α · CV(importance_i)²

load_i = expert i 處理的 token 數；importance_i = expert i 收到的 gating mass。CV = coefficient of variation。最小化 → 強迫各 expert 均勻使用。Google 用此架構訓 137B params 翻譯模型。

③ Switch Transformer (Fedus 2021, JMLR)：Shazeer 後續，把 K = 1（單 expert per token）。優勢：通訊量減半、實作極簡。配 capacity factor C 限制每 expert max tokens，避免 imbalance。Fedus 訓 1.6T params Switch-XL，當時最大公開 LM。

④ GLaM / ST-MoE / Mixtral：更大規模工程化：

  • GLaM (Du 2022, Google)：1.2T params, top-2 routing, 64 experts。energy 比 GPT-3 少 1/3、品質更好。
  • ST-MoE (Zoph 2022)：解決 training instability — router z-loss、轉換 dense layer 部分為 MoE。
  • Mixtral 8×7B / 8×22B (Jiang 2024)：第一個開源 SOTA MoE LLM——8 expert per layer, top-2 routing。47B params total, 13B active。

⑤ Expert Choice routing (Zhou 2022)：反過來——讓「每個 expert 選擇 top-K tokens」而非「每個 token 選 top-K experts」。優勢：load balance 自動達成，無需 aux loss。

⑥ Soft MoE (Puigcerver 2024, ICLR)：完全可微 — 用 "slot" 機制，每個 expert 處理「所有 token 的 weighted average」slot。無 discrete top-K、無 load balance 問題，但喪失稀疏 FLOPs 優勢。對視覺等任務表現超好。

⑦ Hash routing / Random routing (Roller 2021)：用 hash 函數固定 token → expert 映射，跳過 gating learning。簡單、可預測，但效果略遜於 learned gating。

① Original MoE (Jacobs 1991, Jordan & Jacobs 1994): Dense — all experts computed, gating-weighted. EM-style training: E-step computes responsibilities g_i(x_t), M-step updates experts. Elegant but compute scales linearly with N.

② Sparsely-gated MoE (Shazeer 2017, ICLR): The real breakthrough. Top-K gating + Gaussian noise on gating + load balancing loss:

L_aux = α · CV(load_i)² + α · CV(importance_i)²

load_i = tokens routed to expert i; importance_i = gating mass received. CV = coefficient of variation. Minimizing forces uniform expert use. Google trained 137B-parameter translation models with this.

③ Switch Transformer (Fedus 2021, JMLR): Shazeer-followup with K = 1 (single expert per token). Halves communication, simplifies code. Uses a capacity factor C capping max tokens per expert to avoid imbalance. Fedus trained 1.6T-param Switch-XL — largest public LM at the time.

④ GLaM / ST-MoE / Mixtral: Engineering at scale:

  • GLaM (Du 2022, Google): 1.2T params, top-2 routing, 64 experts. 1/3 less energy than GPT-3, higher quality.
  • ST-MoE (Zoph 2022): Addresses training instability — router z-loss, converting some dense layers to MoE.
  • Mixtral 8×7B / 8×22B (Jiang 2024): First open SOTA MoE LLM — 8 experts per layer, top-2 routing. 47B total params, 13B active.

⑤ Expert Choice routing (Zhou 2022): Reversed — each expert picks its top-K tokens (not the other way). Load balance is automatic, no aux loss needed.

⑥ Soft MoE (Puigcerver 2024, ICLR): Fully differentiable — "slot" mechanism where each expert processes a weighted average of all tokens. No discrete top-K, no load-balance problems, but loses sparse-FLOP advantage. Excellent for vision.

⑦ Hash / Random routing (Roller 2021): Fix token → expert via a hash, skipping gating learning. Simple, deterministic, slightly worse than learned gating.

Load balancing 是 MoE 訓練的核心痛點：若不做 balancing，gating 會 collapse 到「只用 1–2 個 expert」，浪費其他 expert 的容量並造成 GPU 利用率不均。Shazeer 2017 的 CV-based loss 是經典；Expert Choice 從架構面解決；Soft MoE 完全避免這問題（但失去稀疏優勢）。實務上 Mixtral / Switch 仍使用 aux loss + capacity factor 雙重保險。
Load balancing is MoE training's central pain: without it, gating collapses to "use only 1–2 experts", wasting other experts' capacity and creating uneven GPU load. Shazeer 2017's CV loss is classic; Expert Choice solves it architecturally; Soft MoE bypasses it (but loses sparsity). Mixtral / Switch use both aux loss + capacity factor in practice.

🌳 什麼時候用 MoE？

情境1:
巨型語言 / 多模態模型——同等 inference cost 下要最大 capacity：Mixtral, GLaM, GPT-4-MoE。
情境2:
跨領域 / 跨模態異質資料——不同 tissue × cell type × disease：每 expert 處理一類。scGPT-MoE, ESM-MoE。
情境3:
計算受限但記憶體相對充裕——can store 大模型但 FLOPs 預算固定。
不適合:
(a) 小 dataset（expert 太多易 underfit）；(b) inference 要極低延遲（routing overhead）；(c) memory 受限環境。
Case 1:
Giant LLM / multimodal models — maximize capacity at same inference cost: Mixtral, GLaM, GPT-4-MoE.
Case 2:
Heterogeneous cross-domain data — tissues × cell types × diseases: one expert per class. scGPT-MoE, ESM-MoE.
Case 3:
Memory-rich, compute-bound — can store huge models but fixed FLOPs budget.
Not for:
(a) Small datasets (too many experts → underfit); (b) ultra-low latency inference (routing overhead); (c) memory-tight envs.
生物應用

MoE 在生物與醫學

scGPT-MoE / 細胞型別專家

scGPT 基礎模型在巨型 cell atlas (>33M cells) 上訓練。scGPT-MoE 變體（2024 工作）讓不同 expert 各自專注於不同 cell type / tissue——免疫細胞 expert、腦 expert、肝 expert 等，整體 capacity 大幅提升但 inference cost 不變。

scGPT pretrained on >33M cells. The scGPT-MoE variant (2024 work) routes different experts to different cell types / tissues — immune expert, brain expert, liver expert — boosting capacity without raising inference cost.

DRP-MoE 藥物響應預測

藥物響應預測 (DRP) 因 cell line × drug × tissue 異質性大而困難。DRP-MoE 用 cell-type-specific experts，每個 expert 學一個 lineage 的響應模式，比 single dense model 在罕見 lineage 上準確度高 15–20%。

Drug response prediction is hard due to cell-line × drug × tissue heterogeneity. DRP-MoE uses cell-type-specific experts — one per lineage — gaining 15–20% accuracy on rare lineages.

ESM2-MoE / 蛋白語言模型

ESM2 (Lin 2023, Science) 是 protein 標準。MoE 變體（2024）讓不同 expert 處理不同 protein family / domain class（α-helical / β-sheet / IDR / membrane）——尤其改進稀有 family 表現。

ESM2 (Lin 2023, Science) is the protein-LM standard. MoE variants (2024) route experts to different families / domain classes (α / β / IDR / membrane) — especially boosting rare families.

多組織 GTEx 表現預測

GTEx 53 個組織的 gene expression prediction 需要 tissue-aware model。MoE 讓 53 個 expert（或 cluster 後較少組）各專一組織，比 multi-task dense model 在 cross-tissue 任務上更穩。

Predicting gene expression across GTEx's 53 tissues requires tissue-aware modeling. MoE assigns experts to tissues (or tissue clusters), beating dense multi-task models on cross-tissue tasks.

多疾病 phenotype 預測

EHR-based phenotyping 同時面對心血管、神經、代謝、癌症等領域。Med-MoE（Singhal 2024 風格）用 medical-domain expert，提升 rare disease classification 效能。

EHR phenotyping spans cardiovascular, neurology, metabolic, oncology domains. Med-MoE (Singhal 2024-style) uses medical-domain experts, boosting rare-disease classification.

Pan-cancer 分子分型

TCGA 33 種癌型差異極大。Pan-cancer MoE 設計專一 expert 處理 each cancer type 或 TME 亞型，整合 mutation + expression + methylation 預測 prognosis，比 generic model 提升 c-index 0.05–0.08。

TCGA's 33 cancer types are diverse. Pan-cancer MoE dedicates experts to each cancer type / TME subtype, integrating mutation + expression + methylation for prognosis — c-index gain 0.05–0.08 over generic models.

多模態 omics 整合

CITE-seq (RNA + protein), Multiome (RNA + ATAC), spatial 等多模態整合，每個 modality 有獨特特徵。Modality-specific experts 比 shared encoder 更靈活，配合 cross-modal gating 達成 modality fusion。

CITE-seq (RNA + protein), Multiome (RNA + ATAC), spatial integration — each modality has unique features. Modality-specific experts with cross-modal gating handle fusion more flexibly than shared encoders.

Federated MoE 跨醫院模型

不同醫院資料 distribution shift 大。Federated MoE（2024）每醫院訓自家 expert + 共享 gating，protecting privacy 同時聚合知識。比 FedAvg + dense 在 site-heterogeneous data 上提升 2–5% AUC。

Distribution shifts across hospitals are severe. Federated MoE (2024) trains site-specific experts + shared gating — preserving privacy while aggregating knowledge. Beats FedAvg + dense by 2–5% AUC on site-heterogeneous data.

Genomic foundation model MoE

DNABERT, Enformer 等 genome 模型用 MoE 變體，讓不同 expert 處理不同 genomic region (promoter / enhancer / intergenic) 或不同 species。長序列 + 多區域類型剛好適合 MoE 架構。

Genome models like DNABERT, Enformer adopt MoE variants — experts for different region types (promoter / enhancer / intergenic) or species. Long sequences + many region types are MoE's sweet spot.

深入解析

MoE 進階主題

① Top-K = 1 vs Top-K = 2 的權衡

Switch (K=1) 通訊量最低、實作最簡；但 routing decision irreversible — 若選錯 expert 完全沒救。Top-2 (Shazeer, Mixtral) 留 backup expert，loss landscape 更平滑、收斂更穩，但通訊雙倍。實證上 K=2 普遍品質更好；K=1 在極限規模上有 throughput 優勢。

② Auxiliary load-balancing loss 的數學

Shazeer 2017 的兩個 CV：

L_aux = α · ⟨f, P⟩ × N，  f_i = fraction routed to i,  P_i = avg gate prob to i

當 f, P 都均勻時 ⟨f, P⟩ = 1/N，loss = α；某個 expert 被偏好時兩者都偏大，loss 上升。α 通常 0.01–0.1。ST-MoE 的 router z-loss：對 router logit 大小做 L2 約束，預防數值爆炸與 training divergence。

③ Capacity factor C

Switch / GShard 引入：每 expert 的容量 = (tokens / num_experts) × C。超過上限的 token 被 drop（zero-padded）。C = 1.0 完全均勻分配；C = 1.25 給一些 slack；C = 2.0 大幅 slack 但浪費 memory。實務 C = 1.0–1.5。

④ Expert collapse 與 fix

沒 load balancing 時，gate 會學到「永遠選 expert A」——其他 expert 永遠收不到梯度、永遠不學東西。除了 aux loss，亦可：(a) Random initialization variance（experts 初始差異要夠大）；(b) Noise on gating logits（Shazeer 訓練時加 Gaussian）；(c) Expert Choice（從架構解決）；(d) Soft MoE（沒 discrete routing → 無 collapse）。

⑤ Routing stability 與 fine-tuning

Pretrained MoE 在 fine-tuning 時 routing 可能 drift——突然全部 token 走同一 expert。解法：(a) freeze routing during fine-tuning；(b) BTX (Branch-Train-MiX, Sukhbaatar 2024)：dense 模型 fine-tune 多個專家後再 mix；(c) lower learning rate for router。

⑥ Scaling laws of MoE

Clark et al. (2022, ICML) 與 ST-MoE 發現：MoE scaling law 與 dense 不同。Dense: loss ~ N^(-α)（N=params）；MoE: loss ~ N_active^(-α') × (N_total/N_active)^(-β)，其中 N_active 是 active params。實證 α' < α 但有額外項——MoE 在「同等 train compute」下 always win，但 inference memory 更貴。

⑦ MoE 與 modular / lifelong learning 的關係

MoE 天然是「模組化神經網路」。Veniat & Denoyer (2018) 等 lifelong learning 工作把 expert 視為「可加可拔」模組——新 task 來訓新 expert 不忘舊知識。生物上的應用：每加入新 cohort / new disease，增 expert 而不重訓 base。

⑧ Soft MoE 與 Slot attention 的關係

Puigcerver et al. (2024, ICLR) 的 Soft MoE 與 Slot Attention (Locatello 2020) 結構相似：用 learnable "slots" 對 input set 做 attention pooling，每 slot 給一個 expert。差別：Soft MoE 對 inputs 做 weighted average；Slot Attention 競爭性地 assign inputs to slots。Soft MoE 完全可微，特別適合 ViT 等視覺任務。

① Top-K = 1 vs 2 trade-off

Switch (K=1) is lowest-comm and simplest; but routing is irreversible — wrong pick is fatal. Top-2 (Shazeer, Mixtral) keeps a backup, smoother loss, more stable convergence, but doubles communication. Empirically K=2 wins in quality; K=1 wins in throughput at extreme scale.

② Auxiliary load-balancing loss math

Shazeer 2017's two CVs:

L_aux = α · ⟨f, P⟩ × N,  f_i = fraction routed to i,  P_i = avg gate prob to i

Uniform f, P → ⟨f, P⟩ = 1/N → loss = α; skewed → loss rises. α typically 0.01–0.1. ST-MoE router z-loss: L2 on router logits prevents numerical blowup and training divergence.

③ Capacity factor C

Switch / GShard introduce: expert capacity = (tokens / num_experts) × C. Excess tokens are dropped (zero-padded). C = 1.0 strict; C = 1.25 slack; C = 2.0 huge slack but memory waste. Typical C = 1.0–1.5.

④ Expert collapse and fixes

Without load balancing, gating learns "always pick A" — others never receive gradients. Beyond aux loss: (a) variance in init (experts start differently); (b) noise on gating logits (Shazeer's Gaussian); (c) Expert Choice (architectural); (d) Soft MoE (no discrete routing → no collapse).

⑤ Routing stability and fine-tuning

Pretrained MoE may drift during fine-tuning — all tokens collapse to one expert. Fixes: (a) freeze routing; (b) BTX (Branch-Train-MiX, Sukhbaatar 2024): fine-tune dense experts then mix; (c) lower LR on the router.

⑥ MoE scaling laws

Clark et al. (2022, ICML) and ST-MoE: MoE laws differ from dense. Dense: loss ~ N^(-α); MoE: loss ~ N_active^(-α') × (N_total/N_active)^(-β). Empirically α' < α with an extra term — MoE always wins at iso-train-compute, but inference memory is costlier.

⑦ MoE and modular / lifelong learning

MoE is naturally a "modular network". Veniat & Denoyer (2018) and others treat experts as plug-and-play modules — add new experts for new tasks without forgetting. Biological angle: add expert per cohort / new disease without retraining base.

⑧ Soft MoE and Slot Attention

Puigcerver et al. (2024, ICLR) Soft MoE resembles Slot Attention (Locatello 2020): learnable "slots" attention-pool inputs, one expert per slot. Difference: Soft MoE weighted-averages inputs; Slot Attention competitively assigns. Soft MoE is fully differentiable — great for ViT-style vision tasks.

潛在改良應用

還能改良哪些任務？

🎯 1. Cell-type-aware foundation model

scGPT / Geneformer 等 cell foundation model 用 dense Transformer。提案：替換 FFN 為 MoE，每 expert 自然 specialize 一個 cell-type lineage——比 dense 更高效於罕見 cell type，且每 expert 可解釋為「lineage program」。

🎯 2. Tissue-specific drug response MoE

藥物在不同組織響應差異大。提案：tissue × cancer × cell-line 三層 expert hierarchy，gating 根據 multi-omics features 自動 route，提升 rare lineage 與 first-in-class drug 預測。

🎯 3. 跨物種 ortholog-aware MoE

不同物種共享 conserved core + species-specific 差異。提案：shared expert（core biology）+ species-specific experts（divergent regulation），gating 根據 species + ortholog group。利於 cross-species translation 與 evolutionary biology。

🎯 4. 多模態 omics MoE 整合

RNA + ATAC + methylation + spatial + protein 各模態結構差異大。提案：modality-specific experts + cross-modal gating；gating 根據 missing-modality pattern 自動調整 routing。比現行 concat / cross-attention 更靈活。

🎯 5. Continual learning of cell atlases

新 cell atlas（如 HCA 持續擴增）持續加入。提案：add-expert 機制——每加入新 cohort 訓新 expert，old experts freeze。配合 routing fine-tune 達到零遺忘 continual learning。

🎯 6. Federated multi-hospital diagnostic MoE

每家醫院資料 distribution 不同，傳統 FL 模型 underperform。提案：每家醫院 local expert + 共享 lightweight gating（只送 features）。配合 differential privacy 進一步 protect。

🎯 7. Genomic regulatory region experts

Enformer 等用 dense Transformer 處理全 genome。提案：每 chromatin context (promoter / enhancer / silencer / intergenic) 訓 expert；gating 根據 epigenetic features (H3K27ac, ATAC) 自動 route。提升 distal regulation 預測。

🎯 8. Protein design with structural experts

RFdiffusion / FrameFlow 等 protein design model 用 dense network。提案：fold-class experts（α-helical bundle / Rossmann fold / β-barrel / IDR）；gating 根據 target topology 自動 route。對 multi-fold 設計尤其有用。

🎯 9. Microbiome MoE for ecological context

不同棲位（gut / skin / soil / marine）微生物群落結構差異大。提案：biome-specific experts + 共享 taxonomic embedding，提升 cross-biome 預測（如「這個物種在 gut 與 soil 的 function 差異」）。

🎯 10. MoE for clinical trial subgroup analysis

同一藥物在不同 patient subgroup 效果差異大。提案：subgroup-aware MoE（age / genotype / comorbidity-conditioned routing），自動識別 responder vs non-responder subgroup，輔助 precision oncology / immunology 試驗設計。

🎯 1. Cell-type-aware foundation model

scGPT / Geneformer use dense Transformers. Proposal: replace FFN with MoE — experts naturally specialize per cell-type lineage, more efficient for rare types, each expert interpretable as a "lineage program".

🎯 2. Tissue-specific drug response MoE

Drug response varies by tissue. Proposal: three-level expert hierarchy (tissue × cancer × cell-line); gating routes by multi-omics features, improving rare-lineage and first-in-class predictions.

🎯 3. Cross-species ortholog-aware MoE

Species share conserved core but diverge in regulation. Proposal: shared expert (core biology) + species-specific experts (divergence); gating uses species + ortholog group. Benefits cross-species translation and evolutionary biology.

🎯 4. Multimodal omics MoE integration

RNA + ATAC + methylation + spatial + protein vary structurally. Proposal: modality-specific experts + cross-modal gating; gating adapts to missing-modality patterns. More flexible than concat / cross-attention.

🎯 5. Continual learning of cell atlases

Atlases (e.g. HCA) keep growing. Proposal: add-expert mechanism — train new expert per new cohort, freeze old; tune routing for zero-forgetting continual learning.

🎯 6. Federated multi-hospital MoE

Cross-hospital distributions differ; FL underperforms. Proposal: local expert per hospital + shared lightweight gating (features only); add DP for further privacy.

🎯 7. Genomic regulatory-region experts

Enformer uses dense Transformer for the whole genome. Proposal: experts per chromatin context (promoter / enhancer / silencer / intergenic); gating routes by epigenetic features (H3K27ac, ATAC). Boosts distal regulation prediction.

🎯 8. Protein design with structural experts

RFdiffusion / FrameFlow use dense networks. Proposal: fold-class experts (α-helical / Rossmann / β-barrel / IDR); gating routes by target topology. Especially useful for multi-fold design.

🎯 9. Microbiome MoE for ecological context

Niches (gut / skin / soil / marine) have distinct community structure. Proposal: biome-specific experts + shared taxonomic embedding for cross-biome inference (e.g. "function of this species in gut vs soil").

🎯 10. MoE for clinical trial subgroup analysis

The same drug differs across subgroups. Proposal: subgroup-aware MoE (age / genotype / comorbidity-conditioned routing) auto-identifies responder vs non-responder, aiding precision oncology / immunology trials.

# Minimal top-K sparse MoE layer in PyTorch
import torch, torch.nn as nn, torch.nn.functional as F

class Expert(nn.Module):
    def __init__(self, d, h):
        super().__init__()
        self.net = nn.Sequential(nn.Linear(d, h), nn.GELU(), nn.Linear(h, d))
    def forward(self, x): return self.net(x)

class SparseMoE(nn.Module):
    def __init__(self, d=256, h=1024, n_experts=8, k=2):
        super().__init__()
        self.k = k
        self.gate = nn.Linear(d, n_experts)
        self.experts = nn.ModuleList([Expert(d, h) for _ in range(n_experts)])

    def forward(self, x):                       # x: (B, T, d)
        B, T, d = x.shape
        x_flat = x.view(-1, d)                  # (B*T, d)
        logits = self.gate(x_flat)              # (B*T, n_experts)
        topk_val, topk_idx = logits.topk(self.k, dim=-1)
        topk_w = F.softmax(topk_val, dim=-1)    # renormalize
        out = torch.zeros_like(x_flat)
        for i, expert in enumerate(self.experts):
            mask = (topk_idx == i).any(dim=-1)   # tokens routed here
            if mask.sum() == 0: continue
            tok = x_flat[mask]
            w   = topk_w[mask] * (topk_idx[mask] == i).float()
            w   = w.sum(dim=-1, keepdim=True)
            out[mask] += w * expert(tok)
        # Auxiliary load-balance loss (Shazeer 2017)
        probs = F.softmax(logits, dim=-1)
        f = (topk_idx.flatten()[:, None] == torch.arange(8, device=x.device))\
            .float().mean(0)              # fraction routed
        P = probs.mean(0)                     # avg gate prob
        L_aux = (f * P).sum() * 8             # N=8 experts
        return out.view(B, T, d), L_aux
