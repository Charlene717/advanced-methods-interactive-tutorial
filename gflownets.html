GFlowNets 生成性流網路

Bengio 2021 提出的新範式：像 RL 但不只追求最高 reward，而是「按 reward 比例採樣」——產出多樣化的高品質候選。專為「分子設計、序列設計、組合優化」這種 mode-diverse 任務而生。

A new paradigm by Bengio 2021: like RL, but instead of maximising reward it samples proportionally to reward — producing diverse high-quality candidates. Built for mode-diverse tasks: molecular design, sequence design, combinatorial optimisation.

核心概念

從 RL 到 GFlowNet 的範式轉變

RL 的問題：標準 RL（PPO, DQN）的目標是 找出 argmax reward 的軌跡——對「找最佳解」的任務（如下棋）很完美，但對「找一籃多樣的好解」的任務（如藥物設計）會失敗——RL 會 collapse 到單一 mode，缺乏多樣性。

GFlowNet 的目標 (Bengio et al. 2021, NeurIPS)：學一個 sampling policy π，使得

P(終態 = x) ∝ R(x)

也就是「reward 高的 x 被取樣機率高，但其他不錯的 x 也按比例被覆蓋」。直觀地說，GFlowNet 把 reward 視為密度函數，學會從這個密度採樣。

「Flow」的由來：把生成過程視為 DAG（有向無環圖）——從初始狀態 s₀ 走若干步動作到達終態 x。在每個邊上定義一個 flow F(s→s')，類似水流。要求：

  • 守恆：每個內部節點的「入流總和 = 出流總和」（Kirchhoff law）。
  • 邊界：到達終態 x 的總流入 = R(x)。

當 flow 滿足這兩個條件，從 s₀ 按 normalized flow 行走的軌跡會自動使 P(x) ∝ R(x)。

與 MCMC 比較：MCMC 也採樣 ∝ R(x)，但需要 burn-in 與長 chain；GFlowNet 是 amortized——學一次 policy 後可一次採樣，無 chain mixing 問題。

The RL problem: standard RL (PPO, DQN) targets argmax-reward trajectories — perfect for "find the best solution" (chess), but failure mode for "find a basket of diverse good solutions" (drug design) — RL collapses to a single mode, missing diversity.

GFlowNet objective (Bengio et al. 2021, NeurIPS): learn a sampling policy π such that

P(terminal = x) ∝ R(x)

"High-reward x is sampled often, but other good x are covered proportionally." Effectively, GFlowNet treats reward as a density and learns to sample from it.

The "flow" metaphor: view generation as a DAG — from initial state s₀, take actions to reach terminal x. Define flow F(s→s') on each edge, like water. Constraints:

  • Conservation: every internal node satisfies "in-flow = out-flow" (Kirchhoff).
  • Boundary: total flow into terminal x equals R(x).

If flow satisfies both, walking from s₀ along normalised flow makes P(x) ∝ R(x) automatically.

vs MCMC: MCMC also samples ∝ R(x), but needs burn-in + long chains. GFlowNet is amortised — learn a policy once, sample in one pass, no mixing problem.

P(x) ∝ R(x) automatically covers multiple modes — no RL-style collapse.

演算法

Flow 守恆、Trajectory Balance 與 Subtrajectory

① Flow Matching loss (Bengio 2021, NeurIPS) — 原版：對每個內部狀態 s，要求

Σs'→s F(s'→s) = Σs→s'' F(s→s'')

loss = Σ_s (log Σ in − log Σ out)²。直接、但對長軌跡梯度很差。

② Detailed Balance (Bengio 2023, ICLR)：學 forward policy P_F(s'|s) 與 backward policy P_B(s|s')，加上 state flow F(s)。要求每條邊：

F(s) · P_F(s'|s) = F(s') · P_B(s|s')

梯度比 FM loss 穩，仍逐邊 update。

③ Trajectory Balance (Malkin et al. 2022, NeurIPS) — 主流：整條 trajectory τ = (s₀ → s₁ → ... → x)：

Z · Π_t P_F(s_{t+1}|s_t) = R(x) · Π_t P_B(s_t|s_{t+1})

loss 取 log 後做 squared error。Z 是 partition function（一個可學標量）。梯度大幅改善，現為實務首選。

④ Subtrajectory Balance (Madan 2023)：不要求整條軌跡完整，子片段 s_i → s_j 也可作 loss term。樣本利用率提升，對長軌跡尤其有用（如分子有 30+ atoms）。

⑤ FL loss / Forward-Looking (Pan 2023)：把 R(s) 也訓練在中間狀態（用 forward-looking estimator）——加速收斂，特別適合 sparse reward。

⑥ Continuous GFlowNet (Lahlou 2023, ICML)：把離散 DAG 換成連續狀態空間，用 SDE 描述 flow，可應用於 protein 折疊、控制問題等。

① Flow Matching loss (Bengio 2021, NeurIPS) — original: for every internal state s,

Σs'→s F(s'→s) = Σs→s'' F(s→s'')

loss = Σ_s (log Σ in − log Σ out)². Direct, but poor gradients on long trajectories.

② Detailed Balance (Bengio 2023, ICLR): learn forward P_F(s'|s), backward P_B(s|s'), and state flow F(s). Edge-wise:

F(s) · P_F(s'|s) = F(s') · P_B(s|s')

Smoother gradients than FM, still edge-level updates.

③ Trajectory Balance (Malkin et al. 2022, NeurIPS) — workhorse: whole trajectory τ = (s₀ → s₁ → ... → x):

Z · Π_t P_F(s_{t+1}|s_t) = R(x) · Π_t P_B(s_t|s_{t+1})

Loss is squared log error. Z is a learnable partition function. Much better gradient signal; default in practice.

④ Subtrajectory Balance (Madan 2023): allow loss terms on sub-segments s_i → s_j. Better sample efficiency, especially for long trajectories (e.g. 30+ atom molecules).

⑤ FL loss / Forward-Looking (Pan 2023): learn R(s) at intermediate states via a forward-looking estimator — speeds convergence on sparse rewards.

⑥ Continuous GFlowNet (Lahlou 2023, ICML): replace discrete DAG with continuous state space, flow via SDE — applies to protein folding, control problems, etc.

實務選擇：新手入門用 Trajectory Balance（穩定、好調）；長軌跡或 sparse reward 切到 Subtrajectory Balance + FL；連續狀態（如 protein 構象、控制）用 Continuous GFlowNet。
Choice in practice: default to Trajectory Balance (stable, tunable). Long trajectories or sparse reward → Subtrajectory Balance + FL. Continuous states (protein conformations, control) → Continuous GFlowNet.

生物應用

GFlowNets 在生物與藥物設計

GFlowNet-Mol

Bengio et al. (2021, NeurIPS) 原始論文應用：用 GFlowNet 逐步建構分子（每步添加 atom/bond），reward 由 docking score 或 QSAR 給出。比 RL-based 方法（REINVENT）生成更多樣化的候選，特別是發掘 binding pocket 的多個 scaffold。

The original Bengio et al. (2021, NeurIPS) application: build molecules atom-by-atom; reward = docking or QSAR score. Generates far more diverse candidates than RL-based REINVENT, especially across distinct scaffolds for a binding pocket.

蛋白質序列設計

Jain et al. (2022, NeurIPS) 用 GFlowNet 設計 antimicrobial peptide：reward = 預測 activity + diversity。比 ProteinMPNN-RL 更能避免 mode collapse，找到「結構不同但都有效」的 sequence family。

Jain et al. (2022, NeurIPS) used GFlowNets for antimicrobial peptide design: reward = predicted activity + diversity. Avoids ProteinMPNN-RL's mode collapse, finds structurally distinct yet effective families.

反應序列規劃

SynFlow / RetroFlow (2023) 用 GFlowNet 做合成路線規劃——從目標分子逆推至 commercially available building blocks，每步是一個反應規則。多樣化的路徑讓化學家有多重選擇。

SynFlow / RetroFlow (2023) plan synthesis routes with GFlowNets — back-track from target to commercial building blocks via reaction templates. Diverse routes give chemists genuine alternatives.

多目標藥物優化

藥物設計通常要平衡 docking、ADMET、合成可行性、新穎性等。Jain (2023) 用 multi-objective GFlowNet (MOGFN)：定義 reward 為多目標 weighted sum 或 Pareto-set proxy，一次採樣得到 Pareto-front 上的多個候選。

Drug design balances docking, ADMET, synthesisability, novelty. Jain (2023) introduced multi-objective GFlowNet (MOGFN): reward as weighted sum or Pareto proxy — sampling yields the entire Pareto front in one go.

對接構象多樣化

傳統 docking 給「最佳」pose；DiverseDock 用 GFlowNet 給 reward ∝ binding affinity，採樣 多個構象 ensemble，反映真實 binding mode 多樣性（cryptic site、induced fit）。

Classic docking returns one "best" pose. DiverseDock uses GFlowNets with reward ∝ binding affinity to sample conformation ensembles reflecting genuine binding diversity (cryptic, induced fit).

主動學習 + 多樣性

Active learning 通常選「最不確定」樣本——但這常 collapse 到相似樣本。Jain (2023) 用 GFlowNet 選 batch：reward = uncertainty × diversity。多樣化、互補性 的 batch 加速模型收斂。

Active learning typically picks "most uncertain" samples — often clusters similar items. Jain (2023) uses GFlowNets to pick batches with reward = uncertainty × diversity. Diverse, complementary batches accelerate model convergence.

基因擾動多樣化

CRISPR combinatorial screens：選哪些 gene set 一起 knock-out？常規方法選「預測效應最大」的組合，缺乏多樣性。提案：用 GFlowNet 採樣多樣化高效應組合，揭示不同 pathway 的 synergistic 效應。

CRISPR combinatorial screens — which gene set to knock out? Standard methods pick top-predicted combos, often redundant. Proposal: GFlowNet samples diverse high-effect combos, revealing synergies across pathways.

RNA aptamer 設計

SELEX 實驗成本高；用 GFlowNet 從 sequence space 採樣 diverse high-affinity aptamer 候選。已在 fluorescent aptamer（如 Spinach 家族）設計中應用。

SELEX is expensive; GFlowNets sample diverse high-affinity aptamer candidates from sequence space. Applied in fluorescent aptamer design (Spinach family).

深入解析

GFlowNet 進階主題

① 與 MaxEnt RL / Soft Q-learning 的關係

Soft Q-learning (Haarnoja 2017) 也採樣 ∝ exp(R/T)。當 T → 0 退化為標準 RL；T → ∞ 退化為均勻採樣。GFlowNet 可視為 T = 1 的特例，但定義在 DAG 而非 MDP 上，且訓練目標是 flow 守恆而非 Bellman。Malkin (2023) 證明 GFlowNet 與 entropic RL 在某些設定下等價。

② 與 amortised MCMC / VI 的關係

GFlowNet 是 amortised sampler——學一次後可快速採樣。可視為「適用於組合空間的 amortised VI」。在 categorical posterior 上，GFlowNet 比 mean-field VI 更能捕捉多模。

③ Off-policy 訓練

GFlowNet 的 loss 可在任意 trajectory 上算（不需要當前 policy）。實務優勢：可用 prioritised replay、expert demonstration、random exploration 等補充訓練資料。Tutorial 推薦：50% on-policy + 50% off-policy / replay。

④ Conditional GFlowNet

把條件 c（如 target property）feed 進 policy：π(s'|s, c)。可做 conditional generation（給定目標屬性生成分子）、目標驅動的 RL。已用於多目標藥物設計。

⑤ GFlowNet 的數值陷阱

  • Z 的學習：partition function Z 在 log-scale 訓練；初始化必須合理（用 reward 的 log-mean）。
  • Reward scaling：R 太集中（峰太尖）會讓 P_F 學不出來，常用 R^β（β < 1）soften。
  • Backward policy 設計：uniform P_B 是預設；學 P_B 可改善但增加複雜度。
  • 長軌跡：超過 50+ 步時 trajectory balance 也會崩潰，改用 subtrajectory 或 FL loss。

⑥ Tempering 與多樣性控制

實務常用 P(x) ∝ R(x)^β：β = 1 標準 GFlowNet；β < 1 更平坦（多樣性更高）；β > 1 更尖銳（接近 RL）。藥物設計常用 β = 0.5–1.0 平衡探索與利用。

⑦ 與 Diffusion 的最新整合

Sendera (2024) 等工作把 GFlowNet 用於訓練 diffusion model：把 diffusion 的去噪步視為 DAG，用 GFlowNet 學去噪策略，可一次學到 sample-efficient + diversity-aware 的 generation。是 2024–2025 的熱門方向。

① Relation to MaxEnt RL / Soft Q-learning

Soft Q-learning (Haarnoja 2017) also samples ∝ exp(R/T). T → 0 collapses to standard RL; T → ∞ to uniform. GFlowNet is the T = 1 case on a DAG (not an MDP), trained via flow conservation rather than Bellman. Malkin (2023) shows equivalence under certain settings.

② Relation to amortised MCMC / VI

GFlowNet is an amortised sampler — train once, sample fast. Like "amortised VI for combinatorial spaces". For categorical posteriors, GFlowNets capture multimodality better than mean-field VI.

③ Off-policy training

The loss can be evaluated on any trajectory (not just current policy). Practical win: mix prioritised replay, expert demos, random exploration. Recipe: 50% on-policy + 50% off-policy / replay.

④ Conditional GFlowNets

Feed condition c (e.g. target property) into the policy: π(s'|s, c). Enables conditional generation (molecules with target affinity) and goal-driven RL. Used in multi-objective drug design.

⑤ Numerical pitfalls

  • Z learning: partition function on log scale; initialise to log-mean reward.
  • Reward scaling: overly peaked R breaks P_F learning — use R^β with β < 1 to soften.
  • Backward policy: uniform P_B is default; learning P_B helps but adds complexity.
  • Long trajectories: trajectory balance breaks past 50+ steps; switch to subtrajectory or FL loss.

⑥ Tempering and diversity control

Use P(x) ∝ R(x)^β: β = 1 standard; β < 1 flatter (more diverse); β > 1 sharper (RL-like). Drug design commonly uses β = 0.5–1.0.

⑦ Integration with diffusion (2024–25 trend)

Sendera (2024) uses GFlowNets to train diffusion models: treat denoising steps as a DAG, learn the denoising policy via GFlowNet — sample-efficient and diversity-aware in one shot. A hot research direction.

潛在改良應用

還能改良哪些任務？

🎯 1. Combinatorial perturbation 設計

從 Perturb-seq 預測「多基因 KO 組合的最大協同效應」。提案：用 GFlowNet 採樣 top-K 多樣化的 K-gene 組合（reward = 預測 synergy），覆蓋不同 pathway，比 brute-force top-K 更有實驗價值。

🎯 2. Bayesian network 結構發現

從觀察資料推斷 causal DAG 是 NP-hard。提案：GFlowNet over DAG space 採樣多個 plausible DAG（reward = posterior score），呈現模型不確定性，比 single MAP 更有資訊量。Deleu (2022) 已開先例。

🎯 3. 蛋白質結構多 conformer 採樣

AlphaFold 給單一最可能結構；許多蛋白質實際上是 ensemble（IDR、kinase activate-inactive）。提案：Continuous GFlowNet over backbone conformations，reward = energy + experimental constraints，採樣 conformer ensemble。

🎯 4. 多目標 sgRNA 設計

guide RNA 設計需要平衡 on-target activity、off-target avoidance、PAM availability。提案：MO-GFlowNet 採樣 Pareto-front 上的多樣化 guide candidates，加速 CRISPR library 設計。

🎯 5. 抗體 CDR 設計

抗體互補決定區（CDRs）需要 binding affinity、developability、低免疫原性等多目標平衡。提案：GFlowNet 在 CDR sequence space 採樣，比 ESM-IF / RFdiffusion 更能保證候選多樣性。

🎯 6. RNA secondary structure ensemble

RNA 通常有 metastable 構象 ensemble，非單一最低能量結構。提案：GFlowNet 對 RNA structure DAG 採樣，reward = 自由能 + SHAPE 數據相容性，覆蓋多個 functionally distinct 構象。

🎯 7. Cell type 標註的多解釋

一個 cluster 可能對應多種合理的 cell type 解釋。提案：GFlowNet 採樣 top-K 標註方案（reward = marker gene overlap × biological prior），呈現生物學家可選的合理解釋。

🎯 8. ChIP-seq peak 解釋 motif 組合

單個 peak 通常有多個 TF binding motifs 並存。提案：GFlowNet 在 motif combination space 採樣 multiple 解釋（reward = motif score 和），比 single greedy assignment 更能反映 TF 複雜性。

🎯 1. Combinatorial perturbation design

From Perturb-seq, predict "synergistic multi-gene KO combos". Proposal: GFlowNet samples top-K diverse K-gene combos (reward = predicted synergy), covering distinct pathways — more experimentally valuable than brute-force top-K.

🎯 2. Bayesian network structure discovery

Causal DAG inference is NP-hard. Proposal: GFlowNet over DAG space samples plausible DAGs (reward = posterior score), exposing model uncertainty — more informative than a single MAP. Deleu (2022) set the precedent.

🎯 3. Multi-conformer protein sampling

AlphaFold gives one structure; many proteins are ensembles (IDR, kinase active/inactive). Proposal: Continuous GFlowNet over backbone conformations; reward = energy + experimental constraints; sample conformer ensembles.

🎯 4. Multi-objective sgRNA design

Guide RNA design balances on-target activity, off-target avoidance, PAM. Proposal: MO-GFlowNet samples diverse Pareto-optimal guides — accelerates CRISPR library design.

🎯 5. Antibody CDR design

CDRs balance affinity, developability, immunogenicity. Proposal: GFlowNet over CDR sequence space — better candidate diversity than ESM-IF / RFdiffusion.

🎯 6. RNA secondary structure ensembles

RNA often has a metastable ensemble, not one minimum-energy structure. Proposal: GFlowNet over RNA structure DAG; reward = free energy + SHAPE compatibility; sample functionally distinct conformers.

🎯 7. Multi-hypothesis cell type annotation

A cluster may admit several plausible cell-type explanations. Proposal: GFlowNet samples top-K annotations (reward = marker overlap × biological prior) — presenting reviewable alternatives.

🎯 8. ChIP-seq motif combination explanations

A single peak often hosts multiple co-occurring TF motifs. Proposal: GFlowNet over motif combinations samples multiple explanations (reward = summed motif scores) — better than a single greedy assignment.

import torch, torch.nn as nn

class PolicyNet(nn.Module):
    def __init__(self, n_states, n_actions):
        super().__init__()
        self.emb = nn.Embedding(n_states, 64)
        self.head = nn.Linear(64, n_actions)
    def forward(self, s):
        return self.head(self.emb(s))

pf = PolicyNet(N_STATES, N_ACTIONS)
pb = PolicyNet(N_STATES, N_ACTIONS)
logZ = nn.Parameter(torch.zeros(1))
opt = torch.optim.Adam(list(pf.parameters())+list(pb.parameters())+[logZ], lr=1e-3)

for step in range(10_000):
    # sample trajectory s0 -> ... -> x
    s, traj_pf, traj_pb = sample_trajectory(pf, pb)
    R = reward(s[-1])

    # Trajectory Balance loss
    logPF = sum(torch.log_softmax(pf(s_t), -1)[a_t]
                for s_t, a_t in traj_pf)
    logPB = sum(torch.log_softmax(pb(s_t), -1)[a_t]
                for s_t, a_t in traj_pb)
    loss = (logZ + logPF - torch.log(R) - logPB).pow(2)
    opt.zero_grad(); loss.backward(); opt.step()
