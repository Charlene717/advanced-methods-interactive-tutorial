存活分析 × 深度學習 (DeepSurv)

Cox 比例風險（Cox 1972, JRSS-B）是醫學統計的基石；DeepSurv (Katzman 2018, BMC Med Res) 用神經網路替換線性 β·x，捕捉非線性互動。DeepHit (Lee 2018, AAAI) 處理 competing risks，DeepProg (Poirion 2021, Genome Medicine) 整合 multi-omics——存活分析正在被深度學習徹底改寫。

Cox proportional hazards (Cox 1972, JRSS-B) is the cornerstone of medical statistics; DeepSurv (Katzman 2018, BMC Med Res) replaces linear β·x with a neural network, capturing non-linear interactions. DeepHit (Lee 2018, AAAI) handles competing risks, DeepProg (Poirion 2021, Genome Medicine) integrates multi-omics — survival analysis is being rewritten by deep learning.

核心概念

從 Cox 模型到 DeepSurv

存活資料的特殊性：給定 n 個病人，每人有 (xᵢ, Tᵢ, δᵢ)，其中

  • xᵢ ∈ ℝᵖ：特徵（age, gene expression, image features...）
  • Tᵢ：觀察到的時間
  • δᵢ ∈ {0, 1}：1 = 事件（如死亡）發生，0 = censored（病人在研究結束時仍存活，或 lost-to-follow-up）

關鍵：censoring 不能當「未死亡」處理——病人可能在 follow-up 結束後不久死亡。傳統 regression / classification 都無法正確處理。

Cox 模型（Cox 1972）：定義 hazard function h(t|x)（時間 t 瞬間死亡率），假設 proportional hazards：

h(t|x) = h₀(t) · exp(β·x)

h₀(t)（baseline hazard）對所有病人共用、與 x 無關；exp(β·x) 是「相對風險（hazard ratio）」。Cox 的天才之處：用 partial likelihood，β 的估計不需顯式建模 h₀(t)。

Partial likelihood（Cox loss）：

ℓ(β) = −Σ_{i: δᵢ=1} [β·xᵢ − log Σ_{j: Tⱼ≥Tᵢ} exp(β·xⱼ)]

對每個 event time，比較 event 病人的「risk score」與當下所有「at-risk」病人。

DeepSurv 的突破（Katzman 2018, BMC Med Res）：把 β·x 替換為 NN(x; θ)——神經網路的輸出就是 log hazard ratio。loss 一樣是 Cox partial likelihood：

ℓ(θ) = −Σ_{i: δᵢ=1} [NN(xᵢ) − log Σ_{j: Tⱼ≥Tᵢ} exp(NN(xⱼ))]

整個 NN 在 Cox loss 下端到端訓練。可捕捉非線性互動（如「age × gene mutation」的交互風險）。

What makes survival data special: n patients, each with (xᵢ, Tᵢ, δᵢ):

  • xᵢ ∈ ℝᵖ: features (age, gene expression, image features, ...)
  • Tᵢ: observed time
  • δᵢ ∈ {0, 1}: 1 = event (e.g. death) observed, 0 = censored (still alive at study end / lost to follow-up)

Critical: censoring ≠ "did not die" — patients may die shortly after follow-up ends. Standard regression/classification cannot handle this correctly.

Cox model (Cox 1972): define hazard h(t|x) (instantaneous death rate at t), assume proportional hazards:

h(t|x) = h₀(t) · exp(β·x)

h₀(t) (baseline hazard) is shared across patients; exp(β·x) is the "hazard ratio". Cox's genius: partial likelihood estimates β without modeling h₀(t).

Partial likelihood (Cox loss):

ℓ(β) = −Σ_{i: δᵢ=1} [β·xᵢ − log Σ_{j: Tⱼ≥Tᵢ} exp(β·xⱼ)]

For each event time, compare the event patient's risk score against all currently at-risk patients.

DeepSurv breakthrough (Katzman 2018, BMC Med Res): replace β·x with NN(x; θ) — the NN output is the log hazard ratio. Same Cox partial likelihood:

ℓ(θ) = −Σ_{i: δᵢ=1} [NN(xᵢ) − log Σ_{j: Tⱼ≥Tᵢ} exp(NN(xⱼ))]

End-to-end NN training under Cox loss. Captures non-linear interactions (e.g. age × mutation joint risk).

演算法

DeepSurv、DeepHit 與 Cox-Time

① DeepSurv (Katzman 2018, BMC Med Res)：

  1. 輸入 xᵢ，網路 NN(x; θ) 輸出純量 log hazard ratio。
  2. 對每個 event sample i，計算分母 Σ_{j ∈ R(Tᵢ)} exp(NN(xⱼ))，其中 R(Tᵢ) 是「at-risk set」。
  3. Loss = − Σ event δᵢ · [NN(xᵢ) − log denominator]。
  4. 梯度下降到收斂；用 dropout / L2 regularization 防 overfit。

② Cox-Time (Kvamme 2019, JMLR)：放寬 proportional hazards 假設——讓 hazard ratio 隨時間變化。網路輸入 (x, t)，輸出 h(t | x)。loss 變成 case-control sampling 的近似 partial likelihood。對「early vs late events 模式不同」的資料有用。

③ DeepHit (Lee 2018, AAAI)：處理 competing risks（如病人可能死於癌症、心血管、或其他原因）。把時間離散化 t ∈ {1, ..., K}，網路輸出 P(event = k, time = t | x) 的聯合分布。loss 包含 log-likelihood + ranking loss（鼓勵高風險者較早事件）。

④ Survival Forests (Ishwaran 2008, Annals of Applied Statistics)：隨機森林的存活版本——每棵樹用 log-rank split 切割節點，aggregate 出 Nelson-Aalen estimator。對小樣本、tabular 資料常勝過 DeepSurv，且天然處理 missing data。

⑤ Pseudo-observation methods：把 censored 資料轉成「pseudo-observation」（每病人一個近似 survival probability），然後用標準 regression。可搭配任何 ML 模型，但對 censoring 重 (> 50%) 的情況偏誤大。

⑥ Concordance Index (C-index)：評估標準。對所有「可比較的 pair (i, j)」（即 Tᵢ < Tⱼ 且 δᵢ = 1），計算「模型給 i 較高 risk score」的比例。C-index = 0.5 隨機；1.0 完美；臨床有用 ≈ 0.65-0.85。

① DeepSurv (Katzman 2018, BMC Med Res):

  1. NN(x; θ) outputs scalar log hazard ratio.
  2. For each event sample i, compute denominator Σ_{j ∈ R(Tᵢ)} exp(NN(xⱼ)), where R(Tᵢ) is the at-risk set.
  3. Loss = − Σ events δᵢ · [NN(xᵢ) − log denominator].
  4. Gradient descent; dropout / L2 to control overfit.

② Cox-Time (Kvamme 2019, JMLR): relaxes proportional hazards — hazard ratio varies with time. Network takes (x, t), outputs h(t | x). Loss is approximate partial likelihood via case-control sampling. Useful when "early vs late events differ".

③ DeepHit (Lee 2018, AAAI): handles competing risks (patient may die from cancer, CVD, or other). Discretize time t ∈ {1, ..., K}, network outputs joint P(event = k, time = t | x). Loss = log-likelihood + ranking loss (encourage high-risk → earlier events).

④ Survival Forests (Ishwaran 2008, Annals of Applied Statistics): RF for survival — log-rank splits, aggregate to Nelson-Aalen estimator. Often beats DeepSurv on small / tabular datasets and handles missing data natively.

⑤ Pseudo-observation methods: convert censored data into "pseudo-observation" (per-patient approximate survival prob), then standard regression. Compatible with any ML model but biased under heavy censoring (> 50%).

⑥ Concordance Index (C-index): the standard metric. Over all comparable pairs (Tᵢ < Tⱼ, δᵢ = 1), fraction where model assigns higher risk to i. C-index = 0.5 random; 1.0 perfect; clinically useful ≈ 0.65-0.85.

為何 Cox loss 自帶 ranking 性質？Partial likelihood 比較的是「at-risk set 中誰先死」，本質上是 ranking loss。所以 Cox 訓練出的 risk score 自然具有 ordinal 意義——比某些直接學 P(survival) 的方法更貼近臨床需求（醫生關心「哪個病人 risk 較高」而非絕對死亡機率）。
Why does Cox loss inherently rank? Partial likelihood compares "who in the at-risk set dies first" — it is intrinsically a ranking loss. Cox-trained risk scores carry ordinal meaning, more clinically aligned than methods that directly model P(survival) (clinicians care about "which patient has higher risk", not the absolute death probability).

🌳 什麼時候用 DeepSurv？

情境1:
高維特徵 + 中型樣本——基因 expression (p = 20k) + n = 500 病人。Cox 容易 overfit，DeepSurv + dropout 較穩。
情境2:
強非線性交互——age × mutation 等。DeepSurv 自動發現，Cox 需手動 feature engineering。
情境3:
多種事件類型——用 DeepHit 處理 competing risks（癌症 vs 心血管 vs 其他）。
不適合:
n < 100 小樣本——線性 Cox 或 Random Survival Forest 通常更穩；DeepSurv 易 overfit。Censoring 極高 (> 90%)——任何方法都困難。
Case 1:
High-dim features + medium sample — gene expression (p = 20k) with n = 500. Cox overfits; DeepSurv + dropout is stabler.
Case 2:
Strong non-linear interactions — age × mutation, etc. DeepSurv discovers them; Cox needs manual feature engineering.
Case 3:
Multiple event types — use DeepHit for competing risks (cancer vs CVD vs other).
Not for:
n < 100 — linear Cox or Random Survival Forest usually wins; DeepSurv overfits. > 90% censoring — every method struggles.
生物應用

存活分析 + DL 在臨床與生物資訊

TCGA pan-cancer 預後

TCGA 33 種癌症、20k+ 病人是 DeepSurv 最常見的 benchmark。Cheerla & Gevaert (2019, Bioinformatics) 用 multi-modal DeepSurv（基因 expression + 病理影像 + 臨床），多種癌症 C-index 從線性 Cox 的 0.65 提升到 0.72-0.80。

TCGA's 33 cancers / 20k+ patients is DeepSurv's classic benchmark. Cheerla & Gevaert (2019, Bioinformatics) use multi-modal DeepSurv (expression + pathology + clinical), boosting C-index from linear Cox's 0.65 to 0.72-0.80 across cancers.

DeepSurv 肝移植

Katzman et al. (2018, BMC Med Res) 原始論文展示在 UNOS 肝移植資料（n > 50k）上，DeepSurv 比 Cox 的 C-index 提升 ~3%。後續被用於預測移植後 1/5/10 年存活、優化分配演算法。

Katzman et al. (2018, BMC Med Res) original paper shows DeepSurv beats Cox by ~3% C-index on UNOS liver transplant data (n > 50k). Used downstream for 1/5/10-year post-transplant prediction and allocation optimization.

DeepHit 乳癌 competing risks

Lee et al. (2018, AAAI) 在 METABRIC 乳癌 cohort 比較 DeepHit vs Cox vs Random Survival Forest。DeepHit 在 competing risks (cancer death vs other death) 設定下 C-index 顯著較高，且能輸出每種 cause-specific incidence function。

Lee et al. (2018, AAAI) compare DeepHit vs Cox vs RSF on METABRIC breast cancer. DeepHit significantly improves C-index under competing risks (cancer-specific vs other death) and outputs cause-specific incidence functions.

Cox-PASNet — pathway-based

Hao et al. (2019, BMC Med Inform Decis Mak) 把 pathway 知識嵌入 DeepSurv 架構——第一層神經元對應 KEGG pathway，第二層做 pathway-pathway 互動。比 vanilla DeepSurv 更可解釋，且效能可比。

Hao et al. (2019, BMC Med Inform Decis Mak) embed pathway knowledge into DeepSurv — first-layer neurons = KEGG pathways, second-layer = pathway-pathway interactions. More interpretable than vanilla DeepSurv, comparable performance.

DeepProg multi-omics

Poirion et al. (2021, Genome Medicine) 用 ensemble of autoencoder + DeepSurv 整合 RNA + miRNA + DNA methylation 預測 hepatocellular carcinoma 預後——比單 omics DeepSurv 顯著好，且識別出可移植到外部 cohort 的 risk signatures。

Poirion et al. (2021, Genome Medicine) ensemble autoencoder + DeepSurv on RNA + miRNA + DNA methylation for hepatocellular carcinoma — significantly beats single-omics, identifying risk signatures transferable to external cohorts.

ICU mortality from EHR

電子病歷縱貫資料 + transformer + DeepSurv：MIMIC-IV / eICU 上預測 30-day ICU mortality。對 censoring 處理是關鍵——病人 discharge 後不再追蹤算 censored。BEHRT、Med-BERT 等用於 EHR embedding 後接 DeepSurv head。

Longitudinal EHR + transformer + DeepSurv: 30-day ICU mortality on MIMIC-IV / eICU. Handling censoring (discharge → censored) is critical. BEHRT, Med-BERT generate EHR embeddings, with DeepSurv head on top.

藥物反應 duration

免疫治療反應持續時間預測：對 melanoma checkpoint inhibitor 反應者，DeepSurv 預測「response 維持多久」。input = 基因 expression + immune cell composition。預測 high vs low durable response，指導 second-line therapy 時機。

Predict duration of immunotherapy response: for melanoma checkpoint inhibitor responders, DeepSurv predicts "how long the response lasts". Inputs = gene expression + immune cell composition. Guides timing of second-line therapy.

Brain MRI 預後

Glioblastoma 病人 MRI + DeepSurv：CNN 抽取影像 feature → DeepSurv head 預測 risk。Lao et al. (2017, Sci Rep) 在 TCGA-GBM 上 C-index ≈ 0.71，比 RANO 標準臨床評估好。後續結合 multi-parametric MRI、radiomics、組織學影像。

Glioblastoma MRI + DeepSurv: CNN extracts image features → DeepSurv head predicts risk. Lao et al. (2017, Sci Rep) achieve C-index ≈ 0.71 on TCGA-GBM, beating standard RANO assessment. Later works combine multi-parametric MRI, radiomics, histology.

Real-world evidence

臨床試驗外的 real-world cohort（如 Flatiron Health 腫瘤資料庫）用 DeepSurv 建立 risk model，校準到 RCT 樣本——可估計新藥對 underrepresented sub-population 的 effect。FDA 已開始接受此類 DL-based survival evidence。

Outside trials, real-world cohorts (Flatiron Health oncology DB) use DeepSurv to build risk models calibrated to RCT samples — estimating drug effects in underrepresented sub-populations. FDA increasingly accepts DL-based survival evidence.

Single-cell-resolved survival

Tumor microenvironment 的單細胞 composition 與預後關聯——對每位病人的 scRNA-seq 算 cell-type abundance vector，餵 DeepSurv。Newman et al. (2019, Nat Biotechnol) CIBERSORTx 之後，把 single-cell composition 作為 DeepSurv input 是新興方向。

Tumor microenvironment single-cell composition correlates with prognosis — per-patient scRNA-seq → cell-type abundance vector → DeepSurv. Following Newman et al. (2019, Nat Biotechnol) CIBERSORTx, single-cell composition as DeepSurv input is an emerging frontier.

深入解析

存活分析 + DL 的進階主題

① Proportional hazards 假設的檢驗與放寬

PH 假設「hazard ratio 不隨 t 變」——可用 Schoenfeld residuals 檢驗。違反時：時間-依賴 covariate、time-varying coefficients、或直接用 Cox-Time / DeepHit（discrete-time）。臨床上，cancer treatment effect 常隨時間衰減 → PH 違反 → 選用較彈性模型。

② Ties 處理

當多個 event 同時發生（ties），partial likelihood 有 Efron / Breslow / exact 三種近似。Efron 預設、Breslow 偏誤大但簡單、exact 慢但精確。DeepSurv 預設 Breslow（最簡單），但對 ties 多的資料應改用 Efron。

③ Time-dependent C-index

標準 C-index 對長時間 follow-up 偏誤大。Uno's C-index (Uno 2011) 用 IPCW 校正；Antolini's C-index (2005) 對 discrete-time 模型友善；Brier score 給 absolute calibration。臨床論文應同時報告 C-index 與 Brier score。

④ Calibration

Ranking 對 (C-index 高) ≠ 機率校準對。臨床決策（如 5-year survival ≥ 50% → 給治療）需要 calibration。常用 calibration plot：把預測 P 分成 deciles，比較預測 vs 觀察存活率。若不 calibrated 可用 Platt scaling 或 isotonic regression 後處理。

⑤ Causal vs predictive

DeepSurv 學的是「conditional risk」，不是 causal effect。若想評估「治療效果 → 存活」的因果，需用 g-computation, IPTW, target trial emulation 等因果推論方法。「DeepSurv 預測 + causal correction」是新興研究方向。

⑥ 不確定性量化

Standard DeepSurv 給點估計。要 uncertainty：

  • Bootstrap：反覆訓練、取 ensemble。
  • MC dropout：推論時保持 dropout, 多次 sample。
  • Bayesian DeepSurv：變分推論 + Cox loss。
  • Conformal survival prediction (Chen 2024)：給 valid coverage 的 survival interval。

⑦ Federated DeepSurv

多醫院聯合訓練 DeepSurv 但不分享病人資料：FederatedSurvival (Wang 2022) 用 federated averaging on Cox gradient + secure aggregation。對 rare cancer 跨機構研究極有用。

⑧ Interpretability

DeepSurv 是 black box——用 SHAP、attention、integrated gradients 解釋。Cox-PASNet 從架構層面注入可解釋性。對臨床部署（FDA approval）解釋性 ≈ 必要。

① Testing and relaxing PH assumption

PH assumes "hazard ratio constant in t" — check via Schoenfeld residuals. If violated: time-dependent covariates, time-varying coefficients, or Cox-Time / DeepHit. Cancer treatment effects often decay → PH violated → use more flexible models.

② Ties handling

For tied event times, partial likelihood has Efron / Breslow / exact approximations. Efron default; Breslow biased but simple; exact slow but accurate. DeepSurv defaults to Breslow; use Efron for many ties.

③ Time-dependent C-index

Standard C-index biased under long follow-up. Uno's C-index (Uno 2011) uses IPCW; Antolini's C-index (2005) for discrete-time models; Brier score for absolute calibration. Clinical papers should report both C-index and Brier score.

④ Calibration

Good ranking (high C-index) ≠ well-calibrated probabilities. Clinical decisions ("5-year survival ≥ 50% → treat") need calibration. Calibration plot: bin predicted P into deciles, compare predicted vs observed survival. If miscalibrated, post-process with Platt scaling / isotonic regression.

⑤ Causal vs predictive

DeepSurv learns "conditional risk", not causal effect. For "treatment → survival" causal estimation use g-computation, IPTW, target trial emulation. "DeepSurv + causal correction" is an emerging direction.

⑥ Uncertainty quantification

Standard DeepSurv gives point estimates. For uncertainty:

  • Bootstrap: retrain ensemble.
  • MC dropout: keep dropout at inference, multi-sample.
  • Bayesian DeepSurv: VI + Cox loss.
  • Conformal survival prediction (Chen 2024): valid-coverage survival intervals.

⑦ Federated DeepSurv

Multi-hospital training without sharing patient data: FederatedSurvival (Wang 2022) uses federated averaging on Cox gradients + secure aggregation. Crucial for rare-cancer multi-institution research.

⑧ Interpretability

DeepSurv is black-box — explain with SHAP, attention, integrated gradients. Cox-PASNet bakes interpretability into the architecture. For clinical deployment (FDA approval), interpretability is nearly mandatory.

潛在改良應用

還能改良哪些任務？

🎯 1. scLLM-derived patient embedding for survival

用 scGPT/Geneformer 從病人 tumor scRNA-seq 抽 patient embedding → DeepSurv。提案：scLLM-DeepSurv，可跨 cohort transfer（scLLM 已預訓練）。

🎯 2. Spatial transcriptomics + survival

Spatial 圖譜可揭示 TME structure（tumor-immune interaction patterns）。提案：Graph-DeepSurv——把 spatial graph 餵 GNN，輸出 risk。對 immunotherapy response 預測尤其有潛力。

🎯 3. Conformal DeepSurv

給臨床決策需 valid uncertainty。提案：結合 conformal prediction 與 DeepSurv，給每病人一個「5-year survival 機率區間」，提升 trustworthiness。

🎯 4. Causal DeepSurv with target trial emulation

結合 target trial emulation 思路與 DeepSurv flexible representation。提案：causal-DeepSurv 估計 treatment effect on survival，可用 instrumental variable extension。

🎯 5. Multi-task survival

同時學「overall survival」、「disease-free survival」、「response duration」。共享 representation 可在 small cohort 上提升 power。提案：Multi-Task DeepSurv with task balancing。

🎯 6. Time-varying biomarker integration

很多 biomarker 隨時間變化（CRP, blood pressure, tumor size）。提案：Longitudinal DeepSurv with transformer encoder for biomarker trajectory → 整合動態資訊預測 survival。

🎯 7. Federated multi-modal DeepSurv

跨醫院聯合：基因 + 影像 + 臨床。提案：Federated Multi-Modal DeepSurv 用於 rare disease 與 underserved population——資料分散但模型可學。

🎯 1. scLLM patient embeddings for survival

scGPT/Geneformer extract patient embeddings from tumor scRNA-seq → DeepSurv. Proposal: scLLM-DeepSurv, transferable across cohorts (pretrained scLLM).

🎯 2. Spatial transcriptomics + survival

Spatial maps reveal TME structure (tumor-immune interactions). Proposal: Graph-DeepSurv — feed spatial graph into GNN, output risk. Particularly promising for immunotherapy response.

🎯 3. Conformal DeepSurv

Clinical decisions need valid uncertainty. Proposal: conformal + DeepSurv giving per-patient "5-year survival interval" for trustworthy deployment.

🎯 4. Causal DeepSurv with target trial emulation

Combine target trial emulation with DeepSurv's flexible representation. Proposal: causal-DeepSurv estimating treatment effect on survival; IV extensions possible.

🎯 5. Multi-task survival

Jointly learn "overall survival", "disease-free survival", "response duration". Shared representation boosts small-cohort power. Proposal: Multi-Task DeepSurv with task balancing.

🎯 6. Time-varying biomarker integration

Biomarkers change over time (CRP, BP, tumor size). Proposal: Longitudinal DeepSurv with transformer biomarker encoder for dynamic-informed survival prediction.

🎯 7. Federated multi-modal DeepSurv

Cross-hospital: genes + imaging + clinical. Proposal: Federated Multi-Modal DeepSurv for rare disease and underserved populations — data stays distributed, model still learns.
