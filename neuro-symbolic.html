Neuro-Symbolic AI / 知識圖譜（完成 50 章！）

「神經網路擅長感知與插值；符號推理擅長組合與解釋。」Neuro-symbolic 結合兩者，配合 Knowledge Graph 三元組 (subject, predicate, object)，是 LLM 時代克服「幻覺」與「無法解釋」的關鍵。從 TransE (2013) 到 PrimeKG (2023 Nature Sci Data)、TxGNN (2023 Nature Med)，KG + GNN + LLM 已成為精準醫療的核心 stack。

"Neural networks excel at perception and interpolation; symbolic reasoning excels at composition and explanation." Neuro-symbolic combines the two with Knowledge Graph triples (subject, predicate, object) — a key strategy to fight LLM hallucination and opacity. From TransE (2013) to PrimeKG (2023 Nat Sci Data) and TxGNN (2023 Nat Med), KG + GNN + LLM is now the core stack of precision medicine.

核心概念

為什麼要結合神經與符號？

兩個 AI 傳統的分裂：

  • 連接主義 (Neural)：感知強、可從資料學；但「黑盒」、無法解釋、邏輯推理弱、易幻覺、需大量資料。
  • 符號主義 (Symbolic)：邏輯一致、可解釋、樣本效率高；但脆弱、無法處理連續/感知資料、知識需手寫。

過去 50 年兩派輪流主導（symbolic 主導 1960–80 年代專家系統時代，neural 自 2012 ImageNet 後主導）。Neuro-symbolic（Garcez & Lamb 2020）即「兩者整合」——讓神經處理感知 + 學習，符號處理組合 + 推理 + 解釋。

Knowledge Graph (KG) 是符號側的主要載體：

  • 由 三元組 (subject, predicate, object) 組成。例：(BRCA1, mutated_in, breast_cancer)；(aspirin, treats, headache)。
  • 結構化知識：實體 → 節點、關係 → 帶標籤的邊。
  • 支援 SPARQL 查詢、規則推理（如 transitivity）、路徑解釋。

三種典型整合方式：

  1. KG embedding（神經學習符號）：把 KG 三元組 embed 到向量空間（TransE, ComplEx, RotatE）。能補全缺失邊，但失去原本的離散結構。
  2. GNN on KG（神經訊息傳遞）：R-GCN, CompGCN — 在 KG 上跑訊息傳遞，把 relational 結構帶進神經表示。
  3. LLM + RAG + KG（混合架構，現代主流）：LLM 生成查詢 → 從 KG retrieve 相關三元組 → context 注入 prompt → LLM 基於可驗證證據作答。GraphRAG (Microsoft 2024) 是代表。

The two AI traditions:

  • Connectionist (Neural): strong perception, learns from data; but black-box, weak logic, hallucinates, data-hungry.
  • Symbolic: logically consistent, interpretable, sample-efficient; but brittle, no perception, knowledge is hand-coded.

For 50 years they alternated dominance (symbolic ruled 1960s–80s expert systems; neural since ImageNet 2012). Neuro-symbolic (Garcez & Lamb 2020) = the integration — let neural handle perception + learning, symbolic handle composition + reasoning + explanation.

The Knowledge Graph (KG) is the main symbolic substrate:

  • Built from triples (subject, predicate, object). E.g. (BRCA1, mutated_in, breast_cancer); (aspirin, treats, headache).
  • Structured knowledge: entities → nodes, relations → labeled edges.
  • Supports SPARQL queries, rule-based inference (transitivity, etc.), path explanations.

Three typical integration modes:

  1. KG embedding (neural learns the symbolic): embed KG triples in vector space (TransE, ComplEx, RotatE). Completes missing edges but discards discrete structure.
  2. GNN on KG (neural message passing): R-GCN, CompGCN — message passing on KG, importing relational structure into neural reps.
  3. LLM + RAG + KG (modern hybrid stack): LLM generates queries → retrieve relevant KG triples → inject as context → LLM answers from verifiable evidence. GraphRAG (Microsoft 2024) is canonical.

演算法

KG embedding, R-GCN, LTN, DeepProbLog

① TransE (Bordes 2013, NeurIPS)：把每個實體 e、關係 r 嵌成向量。「真」三元組滿足 e_subj + r ≈ e_obj。Loss：margin-based ranking — 把真三元組 score 拉高、corrupt 三元組 score 推低。簡單但無法表達 1-to-N、symmetric 關係。

② ComplEx (Trouillon 2016, ICML)：用 複數向量，score 為 Re(⟨e_subj, r, ē_obj⟩)。複數的非對稱共軛可以自然表達 asymmetric 關係（e.g. parent_of），是 KG embedding 的事實標竿。

③ RotatE (Sun 2019, ICLR)：關係看作「複平面旋轉」：e_obj = e_subj ∘ r，其中 |r_i| = 1。可同時表達 symmetric、antisymmetric、inversion、composition 四類關係。當前最強的單模型 KG embedding。

④ R-GCN (Schlichtkrull 2018, ESWC)：relational GCN — 每種關係類型有自己的線性變換矩陣 W_r，message passing 為 h_v^{(l+1)} = σ(Σ_r Σ_{u ∈ N_r(v)} (1/c_{v,r}) W_r^{(l)} h_u^{(l)})。可做 entity classification 與 link prediction。

⑤ Logical Tensor Networks (LTN, Serafini & Garcez 2016)：把一階邏輯公式用 fuzzy logic 變成可微 loss。例：∀x, ∀y, mutated_in(x, cancer) ∧ codes_for(x, kinase) → druggable(x) — 用 product t-norm 變成 differentiable，可與 neural classifier 聯合訓練。

⑥ DeepProbLog (Manhaeve 2018, NeurIPS)：把神經網路當成 probabilistic logic program 的 atomic predicate。MNIST + 加法可用「神經辨識數字 + 邏輯規則做加法」聯合訓練——遠比純神經 sample efficient。

⑦ LLM + RAG + KG（現代主流）：

  1. 使用者 query。
  2. LLM 解析為實體 / 關係，產生 KG 查詢（SPARQL 或 Cypher）。
  3. 從 KG retrieve top-k 三元組 / 路徑。
  4. 把 KG 證據 + query 一起 prompt 給 LLM 產生帶引用的答案。

GraphRAG (Edge 2024, Microsoft) 進一步用 community detection 與 hierarchical summarization 預處理 KG，提升 retrieve 品質。

① TransE (Bordes 2013, NeurIPS): embed each entity e and relation r as vectors. True triples satisfy e_subj + r ≈ e_obj. Loss: margin-based ranking — push true scores up, corrupt scores down. Simple but can't represent 1-to-N or symmetric relations.

② ComplEx (Trouillon 2016, ICML): use complex-valued vectors; score Re(⟨e_subj, r, ē_obj⟩). Asymmetric conjugates naturally represent asymmetric relations (parent_of). De-facto baseline.

③ RotatE (Sun 2019, ICLR): relations as rotations in the complex plane: e_obj = e_subj ∘ r with |r_i| = 1. Captures symmetric, antisymmetric, inversion, composition — currently the strongest single-model KG embedding.

④ R-GCN (Schlichtkrull 2018, ESWC): relational GCN — each relation type has its own W_r; message passing h_v^{(l+1)} = σ(Σ_r Σ_{u ∈ N_r(v)} (1/c_{v,r}) W_r^{(l)} h_u^{(l)}). Used for entity classification and link prediction.

⑤ Logical Tensor Networks (LTN, Serafini & Garcez 2016): turn first-order logic formulas into differentiable losses via fuzzy logic. E.g. ∀x ∀y, mutated_in(x, cancer) ∧ codes_for(x, kinase) → druggable(x) — product t-norm makes it differentiable; jointly trained with a neural classifier.

⑥ DeepProbLog (Manhaeve 2018, NeurIPS): use NNs as the atomic predicates of a probabilistic logic program. MNIST + addition trains "NN recognizes digits + logic adds" jointly — far more sample-efficient than pure NN.

⑦ LLM + RAG + KG (modern stack):

  1. User query.
  2. LLM parses entities / relations, generates KG query (SPARQL or Cypher).
  3. Retrieve top-k triples / paths from KG.
  4. Provide retrieved KG evidence + query in prompt; LLM returns cited answer.

GraphRAG (Edge 2024, Microsoft) adds community detection + hierarchical summarization to improve retrieval.

關鍵直覺：KG embedding 把離散三元組變連續向量、可微，但失去組合 explainability；LLM + RAG 保留 KG 的離散結構與可解釋 path，但 retrieve 不準時仍會幻覺。實務最強的 stack 通常是：LLM (高層意圖) + RAG (證據檢索) + KG embedding (推測缺失邊) + 規則引擎 (硬性約束)，四者協作。
Key intuition: KG embedding makes discrete triples continuous and differentiable, but loses compositional explainability. LLM + RAG preserves discrete structure and explainable paths, but fails on poor retrieval. Strongest stacks combine four: LLM (high-level intent) + RAG (evidence retrieval) + KG embedding (missing-edge inference) + rule engine (hard constraints).

情境1:
已有大量結構化知識（醫學 ontology, drug-target DB, pathway DB）→ KG-based。比硬塞給 LLM 更可靠。
情境2:
需要解釋性 / 監管要求（藥物審查、臨床決策）→ KG path explanation > black-box 神經網路。

Case 1:
Large existing structured knowledge (medical ontology, drug-target DB, pathway DB) → KG-based is more reliable than dumping into LLM.
Case 2:
Explainability / regulation (drug review, clinical decision) → KG path explanation beats black-box.
Case 3:
Few-shot + strong structure (rare disease, orphan drugs) → DeepProbLog / LTN combining domain rules infers from limited data.
Case 4:
LLM answers must be verifiable (biomedical Q&A) → LLM + RAG + KG (GraphRAG, MedGraphRAG).
Not for:
Pure perception tasks (image / speech) — symbolic layer is redundant. Or brand-new domains without a KG.

Microsoft (Edge et al. 2024) GraphRAG — converts long documents to a community-structured KG before retrieval. Applied as MedGraphRAG on 30M PubMed abstracts, 20%+ improvement on "summary-level" questions like "recent treatments for disease X".

深入解析

進階主題

① 為何 LLM 仍會幻覺？KG 如何修正？

LLM 的生成是「下一 token 機率最大化」——當 prompt 涉及訓練時罕見的事實，模型仍會「自信地造假」。KG 提供 ground-truth 三元組，可在生成過程：(i) 檢索：把相關三元組作為 context；(ii) 驗證：對 LLM 輸出做 fact-check 比對 KG；(iii) 規則約束：硬約束生成不能違背 KG（如「不可推薦該病人禁忌的藥」）。

② KG embedding 的 expressivity 階層

  • TransE：無法表達 symmetric / 1-to-N / composition 三類關係。
  • DistMult：所有關係都 symmetric（dot product 對稱性）。
  • ComplEx：解決 antisymmetric 問題。
  • RotatE：同時涵蓋 symmetric, antisymmetric, inversion, composition 四類。
  • BoxE (Abboud 2020)：用 hyper-box 表達實體與關係，可表達更廣泛的邏輯模式。
  • HypE / hyperbolic KG embedding：把實體嵌入 Poincaré 球，自然處理 hierarchical KG（如 disease ontology）。

③ R-GCN 的計算瓶頸

R-GCN 每種關係一個 W_r — 若 KG 有數百種關係，參數爆炸。basis decomposition（共享 basis matrices）與 block-diagonal decomposition（Schlichtkrull 2018）為解。CompGCN (Vashishth 2020) 進一步用 composition function 把 entity 與 relation 同時嵌入。

④ Logical Tensor Networks 的 differentiable 邏輯

LTN 把布林邏輯換成 [0,1] fuzzy logic：A ∧ B → A · B（product t-norm）、A ∨ B → A + B − A·B、∀x → 對所有 x 取「平均」。所有 quantifier 變 differentiable，可與 neural classifier 聯合反傳。實務：規則作為 soft constraint，violation 是 loss term。

⑤ DeepProbLog 的工作流

程式碼是 ProbLog（probabilistic logic）。某些 atomic predicate 由 NN 提供（e.g. digit(image, X) 由 CNN 輸出機率分布）。Loss 反傳：邏輯推理結果 → ProbLog 的機率 → 對 NN 參數的梯度。關鍵突破：把符號程式變 differentiable 同時保留可解釋性。

⑥ GraphRAG 與傳統 RAG 的差異

傳統 RAG：把文檔切 chunk → embed → vector DB → top-k 取回。對「single fact」query 有效，但對「整本書摘要」這類 summary 需求很差。GraphRAG：用 LLM 先把文檔結構化為 entity-relation KG → 對 KG 做 community detection → 對每個 community 生成 hierarchical summary → query 時根據需要取回不同層級的 summary。

⑦ KG 不完整性（OWA vs CWA）

KG 是 open-world assumption：「沒看到的三元組」不代表「不成立」，只是「未知」。這與 closed-world 機器學習假設衝突。實務：negative sampling 是「假設大部分 corrupt 三元組是 false」的近似；conformal prediction / probabilistic KG（如 PSL）給出帶不確定度的推理。

⑧ Neuro-symbolic 的未來方向

  • Differentiable reasoning：把 SAT / SMT solver 變 differentiable（Wang 2019, SATNet）。
  • LLM 作為 KG 構建器：用 LLM 從非結構化文本自動抽取三元組構建 KG；REBEL (Cabot 2021) 是代表。
  • Neuro-symbolic agent：結合 LLM planner + KG world model + 符號 executor（如 ToolFormer, ReAct 模式）。

① Why does LLM hallucinate? How does KG fix it?

LLM generation = "max next-token probability" — when prompts touch rarely-seen facts, the model invents content confidently. KG provides ground truth: (i) retrieval: inject relevant triples as context; (ii) verification: fact-check LLM output against KG; (iii) rule constraints: hard-disallow generation that violates KG (e.g. "cannot recommend a contraindicated drug").

② KG-embedding expressivity hierarchy

  • TransE: can't represent symmetric / 1-to-N / composition relations.
  • DistMult: all relations are symmetric (dot-product symmetry).
  • ComplEx: fixes asymmetry.
  • RotatE: covers symmetric, antisymmetric, inversion, composition.
  • BoxE (Abboud 2020): hyper-box-based, captures broader logical patterns.
  • HypE / hyperbolic KG embedding: Poincaré ball naturally handles hierarchical KG (disease ontology).

③ R-GCN computational bottleneck

R-GCN allocates one W_r per relation — parameters explode with hundreds of relations. Solutions: basis decomposition (shared bases) and block-diagonal decomposition (Schlichtkrull 2018). CompGCN (Vashishth 2020) uses composition functions to jointly embed entities and relations.

④ Logical Tensor Networks: differentiable logic

LTN swaps boolean for [0,1] fuzzy logic: A ∧ B → A·B (product t-norm); A ∨ B → A + B − A·B; ∀x → mean over x. All quantifiers become differentiable, jointly trainable with neural classifiers. Practically: rules act as soft constraints; violation is a loss term.

⑤ DeepProbLog workflow

Code is ProbLog (probabilistic logic). Some atomic predicates come from NNs (e.g. digit(image, X) from CNN softmax). Loss backprops: logical inference → ProbLog probability → gradient into NN parameters. Key breakthrough: differentiable symbolic programs that retain interpretability.

⑥ GraphRAG vs traditional RAG

Traditional RAG: chunk document → embed → vector DB → top-k. Works for single-fact queries but fails on "summarize the whole book". GraphRAG: LLM first structures the corpus into an entity-relation KG → community detection → hierarchical summary per community → retrieve appropriate summary level per query.

⑦ KG incompleteness (OWA vs CWA)

KG uses open-world assumption: missing triple ≠ false; merely unknown. Clashes with closed-world ML assumption. Practical: negative sampling approximates "most corrupt triples are false"; conformal prediction / probabilistic KG (PSL) give calibrated uncertainty.

⑧ Future of neuro-symbolic

  • Differentiable reasoning: differentiable SAT / SMT solvers (SATNet, Wang 2019).
  • LLM as KG builder: auto-extract triples from text (REBEL, Cabot 2021).
  • Neuro-symbolic agents: LLM planner + KG world model + symbolic executor (ToolFormer, ReAct patterns).
潛在改良應用

還能改良哪些任務？

🎯 1. 個人化 medical RAG agent

結合病人 EMR + PrimeKG + LLM 構建「個人 KG」。提案：每個病人即一個 ego-graph，agent 可問「對這個病人，這個新藥是否安全？」並輸出 KG 路徑解釋（過敏史、共病、藥物代謝基因）。

🎯 2. 多重組學 evidence-aware KG

當前 KG 多為 categorical（gene-disease association）。提案：把 KG 邊權重來自實驗證據（GWAS p-value、Perturb-seq effect size、表型一致性），用 R-GCN + uncertainty 推理時帶誤差棒。

🎯 3. Causal KG：超越 association

目前 biomedical KG 多表達「相關性」。提案：結合 causal discovery（如 Ch 14 的方法）構建「causal KG」，邊代表 do-intervention 效應。配合 LLM 做 counterfactual 推理（「如果敲除這基因會怎樣？」）。

🎯 4. Temporal KG for disease progression

當前 KG 是靜態的；但疾病進展、治療反應都有時間性。提案：temporal KG embedding（如 TNTComplEx）模擬「初期 → 中期 → 末期」的關係演化，用於個人化 trajectory prediction。

🎯 5. 規則學習自動化（Rule mining）

當前 KG 規則需專家手寫。提案：用 AnyBURL / RNNLogic 自動從 KG 學「Horn rule」，發現如「藥 A treats 疾病 X ∧ X has_symptom Y → 藥 A relieves Y」這類規則。

🎯 6. KG-aware 蛋白質設計

結合 RFdiffusion / FrameFlow + KG（功能標籤、結合伴侶）。提案：generative design 條件式於 KG-derived constraints（「設計能 bind 此 target 但不 bind 此 off-target 的蛋白質」）。

🎯 7. 臨床試驗匹配 (clinical trial matching)

當前匹配主要靠關鍵字。提案：把試驗 eligibility criteria 轉成符號規則，病人 EMR 轉成 KG，做 symbolic + neural 匹配。可大幅提升 rare-disease 試驗招募效率。

🎯 8. LLM-fact-checking via KG

給 LLM 一個 medical 回答，自動把 claim 抽取後對 PrimeKG check。提案：FactScore-Bio benchmark + 訓練 LLM 「在 KG 沒有支持時主動說不知道」。

🎯 9. Cross-species KG alignment

人類 KG 與小鼠 KG 共享結構但 entity 不同（orthologs、homologous diseases）。提案：跨物種 KG embedding 對齊——可 transfer 小鼠豐富的 perturbation 資料到人類疾病推理。

🎯 1. Personalized medical RAG agent

Combine EMR + PrimeKG + LLM into a "patient KG". Proposal: each patient is an ego-graph; agent answers "is this drug safe for this patient?" with KG path explanations (allergies, comorbidities, drug-metabolism genes).

🎯 2. Multi-omics evidence-aware KG

Current KGs are categorical (gene-disease association). Proposal: weight KG edges by experimental evidence (GWAS p-value, Perturb-seq effect size, phenotype consistency); R-GCN + uncertainty inference with error bars.

🎯 3. Causal KG: beyond association

Biomedical KGs mostly encode correlation. Proposal: combine with causal discovery (Ch 14) to build "causal KG" where edges represent do-intervention effects; LLM-based counterfactual reasoning ("what if we knock out this gene?").

🎯 4. Temporal KG for disease progression

Current KGs are static; disease progression and treatment response are time-dependent. Proposal: temporal KG embeddings (e.g. TNTComplEx) to model "early → mid → late" relation evolution for personalized trajectory prediction.

🎯 5. Automated rule mining

Current KG rules are hand-written. Proposal: AnyBURL / RNNLogic learn Horn rules from KG — discover patterns like "drug A treats disease X ∧ X has_symptom Y → drug A relieves Y".

🎯 6. KG-aware protein design

Combine RFdiffusion / FrameFlow + KG (functional labels, binding partners). Proposal: generative design conditioned on KG-derived constraints ("design a binder for target X that does NOT bind off-target Y").

🎯 7. Clinical-trial matching

Matching is keyword-based today. Proposal: convert eligibility criteria to symbolic rules + patient EMR to KG; do symbolic + neural matching. Major boost to rare-disease trial recruitment.

🎯 8. LLM fact-checking via KG

Auto-extract claims from LLM medical answers and verify against PrimeKG. Proposal: FactScore-Bio benchmark + train LLM to say "I don't know" when KG lacks support.

🎯 9. Cross-species KG alignment

Human and mouse KGs share structure but differ in entities (orthologs, homologous diseases). Proposal: cross-species KG embedding alignment — transfer mouse-rich perturbation data to human disease reasoning.
