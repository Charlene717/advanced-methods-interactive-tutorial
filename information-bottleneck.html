資訊瓶頸 (Information Bottleneck)

Tishby 在 1999 年提出的「最小充分統計量」原則——找到一個表徵 T，盡可能壓縮輸入 X，但保留與目標 Y 相關的訊息。今天它支配著可解釋表徵學習、scRNA-seq 整合與 scLLM 微調。

Tishby's 1999 "minimal sufficient statistic" principle — find a representation T that compresses X as much as possible while preserving information relevant to target Y. It now underpins interpretable representation learning, scRNA-seq integration, and scLLM fine-tuning.

核心概念

壓縮與保留之間的最佳折衷

Tishby, Pereira & Bialek (1999) 提出資訊瓶頸 (Information Bottleneck, IB) 原則：學一個隨機表徵 T = f(X)，使得

LIB = I(X; T) − β · I(T; Y)

最小化此目標等價於：T 對 X 「忘掉」越多越好（壓縮 I(X;T)↓），但對 Y 「保留」越多越好（預測 I(T;Y)↑）。係數 β 控制兩者權衡——β 小強調壓縮、β 大強調預測。

幾何上，這在 (I(X;T), I(T;Y)) 平面定義一條「資訊瓶頸曲線」(information curve)：所有可行的 (壓縮、預測) 組合的 Pareto front。

Tishby, Pereira & Bialek (1999) proposed the Information Bottleneck (IB) principle: learn a stochastic representation T = f(X) that minimizes

LIB = I(X; T) − β · I(T; Y)

This trades off compression (forget X as much as possible — I(X;T)↓) against prediction (retain Y as much as possible — I(T;Y)↑). β controls the balance — small β favours compression, large β favours prediction.

Geometrically, this traces the information curve in the (I(X;T), I(T;Y)) plane — the Pareto frontier of all feasible (compression, prediction) pairs.

🧬

最小充分統計量

IB 給出「統計學的最佳壓縮」——保留所有對 Y 有用的訊息，丟棄其他。

IB defines the statistically optimal compression — keeps everything useful for Y, discards the rest.

⚖️

β 的調控

β=0 → 常數編碼；β→∞ → 完整 X（無壓縮）。中間有「相變」現象，T 維度跳階增加。

β=0 → constant code; β→∞ → full X (no compression). Phase transitions appear at critical β values where T's effective dimensionality jumps.

🔬

深度學習詮釋

Tishby & Schwartz-Ziv (2017) 主張 SGD 隱含執行 IB——先擬合再壓縮。雖有爭議，但啟發 VIB 等實用方法。

Tishby & Schwartz-Ziv (2017) argued SGD implicitly performs IB — fit-then-compress. Controversial, but inspired VIB and related practical methods.

演算法

從經典到深度的演算法家族

① Classical IB（離散 X, Y）

當 X、Y 為離散變量時，IB 可由類似 Blahut-Arimoto 的迭代解出：交替更新 p(t|x)、p(t)、p(y|t)，類似 EM。Tishby 原始 1999 paper 給出收斂保證。缺點：需要 p(X,Y) 已知或良好估計，無法擴展到高維連續資料。

② Variational Information Bottleneck (VIB)

Alemi et al. (2017, ICLR) 把 IB 變成可微目標：

LVIB = Eq(t|x)[−log p(y|t)] + β · KL(q(t|x) ‖ r(t))

其中 q(t|x) 為編碼器（如 NN 輸出 μ, σ 的 Gaussian），p(y|t) 為解碼器，r(t) 為先驗（通常 N(0,I)）。第一項是 cross-entropy（−I(T;Y) 的上界），第二項是 KL（I(X;T) 的上界）。整個架構長得像 VAE，但條件是 Y 不是 X。

③ IB-VAE / Conditional IB

當 Y 不存在（無監督）時，可把 IB 變成自我預測（X 預測自身的部分）；或加入額外 condition（細胞型別、batch）做 Conditional IB——用於 scRNA-seq batch removal。

④ Recursive / Sloppy / Sparse IB

Recursive IB 把 T 進一步分層；Sloppy IB 處理 ill-conditioned 情境；Sparse IB 把 KL 換成 L1，得到「sparse code」。

① Classical IB (discrete X, Y)

For discrete variables, IB is solved by Blahut-Arimoto-like iteration: alternately update p(t|x), p(t), p(y|t), similar to EM. Tishby's 1999 paper guarantees convergence. Limitation: requires p(X,Y) known or well estimated — doesn't scale to high-dim continuous data.

② Variational Information Bottleneck (VIB)

Alemi et al. (2017, ICLR) made IB differentiable:

LVIB = Eq(t|x)[−log p(y|t)] + β · KL(q(t|x) ‖ r(t))

where q(t|x) is the encoder (e.g. NN outputting Gaussian μ, σ), p(y|t) the decoder, and r(t) a prior (usually N(0,I)). The first term upper-bounds −I(T;Y); the second upper-bounds I(X;T). Architecturally similar to a VAE but conditioned on Y, not X.

③ IB-VAE / Conditional IB

Without explicit Y (unsupervised), replace Y by self-prediction (predicting masked X) or add a side variable (cell type, batch) for Conditional IB — used in scRNA-seq batch removal.

④ Recursive / Sloppy / Sparse IB

Recursive IB stacks IB layers; Sloppy IB handles ill-conditioning; Sparse IB swaps KL for L1, yielding sparse codes.

數學連結：VAE = IB 的特例（取 Y = X、β = 1）。β-VAE (Higgins 2017) 就是 β ≠ 1 的 IB-VAE，β > 1 鼓勵 disentanglement。所以 IB 是 VAE 家族的「母原理」。
Math link: VAE = IB special case (Y = X, β = 1). β-VAE (Higgins 2017) is IB-VAE with β ≠ 1; β > 1 encourages disentanglement. IB is the "mother principle" of the VAE family.

🌳 如何選 β？

情境1:
跨 batch 整合——希望 T 不含 batch info → β 小，壓縮主導。
情境2:
下游分類（如 cell type）——預測效能優先 → β 大。
情境3:
可解釋 / disentanglement——β 略大於 1（β-VAE 配方），但會犧牲重建。
情境4:
隱私保護——β 越小，T 含的個人資訊越少（k-anonymity 連結）。
Case 1:
Cross-batch integration — want T free of batch info → small β, compression dominates.
Case 2:
Downstream classification (e.g. cell type) — prediction priority → large β.
Case 3:
Interpretability / disentanglement — β slightly > 1 (β-VAE recipe), at the cost of reconstruction fidelity.
Case 4:
Privacy — smaller β → less personal info in T (k-anonymity-like guarantee).
生物應用

IB 在單細胞與多體學的應用

scVI + IB batch removal

Lopez et al. (2018, Nature Methods) 的 scVI 是 IB-VAE 的單細胞版本——把 batch 當 condition、cell type 當 Y，β 壓低使 T 不含 batch info，但保留 cell type 訊號。是 scRNA-seq 整合的 gold standard 之一。

Lopez et al. (2018, Nat Methods) scVI is the single-cell version of IB-VAE — batch as a condition, cell type as Y, small β so T loses batch info while retaining cell-type signal. A gold-standard scRNA-seq integration method.

IB-based marker selection

mRMR (minimum-redundancy-maximum-relevance, Peng 2005) 可重新詮釋為 IB 的離散近似——找一組基因 T ⊂ X 使 I(T; cell_type) 最大、I(T; X\T) 最小。

mRMR (Peng 2005) is an IB-flavoured discrete approximation — find genes T ⊂ X maximizing I(T; cell_type) while minimizing I(T; X\T).

Brain decoding (視覺神經科學)

Strouse & Schwab (2017) 用 IB 從 fMRI 解碼受試者看到的視覺刺激類別——T 為 minimal sufficient neural code。比 raw SVM 在 generalization 上更穩。

Strouse & Schwab (2017) used IB to decode visual stimuli from fMRI — T as the minimal sufficient neural code, generalizing better than raw SVM.

DNA motif discovery

Slonim et al. (2005, PNAS) 用 IB 把 promoter sequence 壓縮到「足以預測基因表現」的最小 motif 集合——比 MEME 等做出更精簡且生物上有意義的 motif。

Slonim et al. (2005, PNAS) used IB to compress promoter sequences to the minimal motif set sufficient to predict expression — yielding more parsimonious and biologically meaningful motifs than MEME.

scRNA-seq imputation

DCA (Eraslan 2019)、scVI 與後續 imputation 方法的 latent space 本質上是 IB-style 表徵——壓縮掉 dropout 噪音，保留真實表現訊號。

DCA (Eraslan 2019), scVI, and successors use IB-style latent spaces — compress out dropout noise while preserving true expression signal.

Cross-modal IB

CITE-seq / 10x Multiome 整合：T 編碼 RNA，預測對應 surface protein 表現。β 控制「跨模態映射」的緊密度，太大反而過擬合單細胞噪音。

CITE-seq / 10x Multiome integration: T encodes RNA, predicts surface protein levels. β controls cross-modal mapping tightness; too large over-fits single-cell noise.

深入解析

陷阱、估計與相變

① 互資訊估計是難題

I(X;T) 在高維連續資料估計極不穩定。VIB 用 KL 上界繞過——但這只是上界、不是真值。若想得到「真實」的資訊曲線，需 MINE (Belghazi 2018)、InfoNCE 下界、或 Donsker-Varadhan dual。實務上：把 KL 視為 surrogate 即可，不必執著於 absolute bits。

② 相變 (Phase transitions)

Wu et al. (2019, ICLR) 嚴格證明：在 IB-VAE 中，當 β 跨越某些 critical 值時，T 的「有效維度」會跳階——這是 disentanglement 與「latent collapse」（部分維度突然失活）的根本原因。實務上，當訓練曲線突然不穩，往往是 β 太接近 critical point。

③ Information plane controversy

Tishby & Schwartz-Ziv (2017) 在 MNIST + tanh activation 上看到「擬合 → 壓縮」兩階段。Saxe et al. (2018, ICLR) 用 ReLU 重做後反駁——壓縮並非 SGD 的普遍行為，而依賴 activation 飽和。今日共識：information plane 是有趣的觀察視角，但「SGD = IB」仍未被嚴格證實。

④ Open IB & General IB

對連續 Y，標準 IB 退化（I(T;Y) 可能無界）。Achille & Soatto (2018) 提出 Information Dropout / Bottleneck 改進；Yu & Yang (2023) 提出 Open IB 對「未見類別」也能 well-defined。

⑤ Mutual Information Neural Estimation

結合 IB 與 MINE：用 MINE 估 I(T;Y)、用 KL 估 I(X;T) → 直接優化 IB 不再透過 VAE-style reparameterization。在 high-dim image 與 sequence 任務上更穩。

⑥ IB 與 SDPI / Fano 不等式

IB 的可行區域受 SDPI (Strong Data Processing Inequality) 限制：I(T;Y) ≤ I(X;Y)。所以 IB 不會「無中生有」——預測上限受真實 X–Y 依賴決定。資訊曲線的「右上界」就是 I(X;Y)。

⑦ Conditional IB 的精確形式

對 batch removal 等任務，目標是 min I(X;T) − β·I(T;Y) + γ·I(T;B)，其中 B 為 batch label。實作上把第三項換成 adversarial loss（讓 T 騙過 batch classifier），即 scVI 的核心。

⑧ IB vs Sufficient Statistic

古典統計：sufficient statistic T 滿足 I(T;Y) = I(X;Y)。IB 是其「鬆綁」版本——允許略小於 I(X;Y) 換取壓縮。Achille & Soatto 證明：在 deep learning 中，「invariant」表徵（不受 nuisance 影響）必然在 IB 曲線上的特定點。

① MI estimation is hard

I(X;T) is unstable to estimate in high dims. VIB sidesteps via the KL upper bound — but it's only a bound, not the truth. For "real" info curves use MINE (Belghazi 2018), InfoNCE lower bound, or Donsker-Varadhan dual. In practice, treat KL as a surrogate; don't fixate on absolute bits.

② Phase transitions

Wu et al. (2019, ICLR) proved rigorously that in IB-VAE, T's "effective dimension" jumps at critical β values — the root cause of disentanglement and latent collapse. If training becomes unstable, β is likely near a critical point.

③ Information-plane controversy

Tishby & Schwartz-Ziv (2017) observed a "fit-then-compress" pattern with tanh on MNIST. Saxe et al. (2018, ICLR) refuted this with ReLU — compression isn't universal in SGD; it depends on activation saturation. Today's consensus: information plane is an interesting lens, but "SGD = IB" remains unproven.

④ Open IB & General IB

For continuous Y, standard IB degenerates (I(T;Y) can be unbounded). Achille & Soatto (2018) proposed Information Dropout / Bottleneck; Yu & Yang (2023) proposed Open IB for well-definedness on unseen classes.

⑤ Mutual-Information Neural Estimation

Combine IB with MINE: estimate I(T;Y) via MINE and I(X;T) via KL → optimize IB directly without VAE-style reparam. More stable on high-dim images and sequences.

⑥ IB and SDPI / Fano's inequality

IB's feasible region is bounded by SDPI: I(T;Y) ≤ I(X;Y). IB doesn't create information — the prediction ceiling is the true X–Y dependence. The right-upper bound of the info curve is exactly I(X;Y).

⑦ Conditional IB in detail

For batch removal: min I(X;T) − β·I(T;Y) + γ·I(T;B), B = batch label. In practice, the third term becomes an adversarial loss (T fools a batch classifier) — the core of scVI.

⑧ IB vs sufficient statistic

Classical stats: sufficient T satisfies I(T;Y) = I(X;Y). IB is a relaxation — slightly less than I(X;Y) in exchange for compression. Achille & Soatto showed that "invariant" deep representations (free of nuisance variables) sit on specific IB-curve points.

潛在改良應用

IB 還能改良哪些生物資訊任務？

🎯 1. scGPT / Geneformer 加 IB 正則化

現有 scLLM latent 過度高維（512–1024）、難解釋。加入 IB 正則化（VIB head）可強制 latent 「對下游任務」最小且足夠——預期更可解釋、更 robust 於 batch、計算下游更便宜。

🎯 2. 藥物效應 latent disentanglement

Perturb-seq 訓練時把「擾動類型」當 Y，IB 學出「擾動相關」 vs 「基線變異」分離的 latent。可用於 perturbation transfer learning（在 K562 學的擾動效應，預測 HEK293 上的反應）。

🎯 3. Rare disease 表徵

大型 atlas 加 IB（Y = disease label，β 小）→ 可學「在病人中保持與健康細胞最大相似度」的 latent，凸顯真正的疾病訊號。對 N-of-1 罕病診斷有潛力。

🎯 4. 隱私保護表徵學習

對病人多體學資料學 latent，β 設小到 mutual info I(T; patient_id) 接近 0，但保留 I(T; phenotype)。即 differential privacy 的 IB 版本（Asoodeh 2020）。

🎯 5. Multi-task / Multi-Y IB

同時預測多個 Y（cell type、disease、batch）的 IB——分配不同 β_k 給每個 Y_k。可學「對所有下游任務最佳壓縮」的 universal representation。

🎯 6. IB 用於 Spatial transcriptomics

把 spatial coordinate 當 Y，學「保留空間結構但壓縮細胞 idiosyncrasy」的 latent。預期對 niche 與 trajectory 同時偵測有助。

🎯 7. IB-based GRN 簡化

傳統 GRN 過度連結。對每個 TF–target 邊用 IB 評估「條件 mutual info 是否大於壓縮成本」——保留邊只在「真的傳訊息」時。預期得到更稀疏、可解釋的網路。

🎯 8. Cross-species IB

把「物種 label」當 nuisance、cell type 當 Y → 學「跨物種保守」的 cell-type latent。用於 evolutionary atlas 比對。

🎯 9. 細胞軌跡 + IB

把 pseudotime 當 Y，IB 學「對 pseudotime 必要」的最小基因集——這就是「動態 marker discovery」。比 tradeSeq GAM 更聚焦於「資訊瓶頸」基因。

🎯 10. IB for Foundation-model evaluation

給定 scGPT 與 Geneformer，計算各自 latent 與 cell type 的 I(T;Y) 在固定 I(X;T) 下誰高？這給 foundation-model 比較的客觀指標。

🎯 1. scGPT / Geneformer + IB regularization

Current scLLM latents are oversized (512–1024) and hard to interpret. A VIB head forces the latent to be minimal yet sufficient for downstream tasks — more interpretable, more batch-robust, cheaper downstream.

🎯 2. Drug-effect latent disentanglement

Train Perturb-seq with perturbation type as Y, IB learns a latent that separates perturbation-related signal from baseline variability — useful for perturbation transfer learning (K562 → HEK293).

🎯 3. Rare-disease representations

Atlas + IB (Y = disease label, small β) → latent that maximally resembles healthy cells while preserving disease signal. Promising for N-of-1 rare-disease diagnostics.

🎯 4. Privacy-preserving representations

For patient multi-omics, set β small so I(T; patient_id) ≈ 0 while I(T; phenotype) stays high. IB version of differential privacy (Asoodeh 2020).

🎯 5. Multi-task / multi-Y IB

Predict multiple Y (cell type, disease, batch) with per-Y β_k. Learns a universal representation optimally compressed for all downstream tasks.

🎯 6. IB for spatial transcriptomics

Treat spatial coordinates as Y — learn a latent preserving spatial structure while compressing cell idiosyncrasy. Useful for joint niche + trajectory detection.

🎯 7. IB-based GRN simplification

Classical GRNs are too dense. For each TF–target edge, ask whether conditional MI exceeds compression cost — retain edge only if it really conveys information. Yields sparser, interpretable networks.

🎯 8. Cross-species IB

Species label as nuisance, cell type as Y → cross-species conserved cell-type latents. Useful for evolutionary atlas alignment.

🎯 9. Trajectory + IB

Pseudotime as Y, IB learns the minimal gene set required for the trajectory — "dynamic marker discovery." More focused on bottleneck genes than tradeSeq GAM.

🎯 10. IB for foundation-model evaluation

Given scGPT and Geneformer, at fixed I(X;T), whose I(T;Y) is higher for cell type prediction? Provides an objective metric for foundation-model comparison.

import tensorflow as tf
import tensorflow_probability as tfp
tfd = tfp.distributions

# Encoder: x -> (mu, sigma)
encoder = tf.keras.Sequential([
    tf.keras.layers.Dense(256, activation='relu'),
    tf.keras.layers.Dense(2 * D)])

def vib_loss(x, y, beta=0.01):
    h = encoder(x)
    mu, log_sigma = tf.split(h, 2, axis=-1)
    q = tfd.Normal(mu, tf.exp(log_sigma))
    t = q.sample()
    # Prior r(t) = N(0, I)
    r = tfd.Normal(0., 1.)
    kl = tf.reduce_sum(tfd.kl_divergence(q, r), axis=-1)
    logits = classifier(t)
    ce = tf.nn.sparse_softmax_cross_entropy_with_logits(y, logits)
    return tf.reduce_mean(ce + beta * kl)
import torch
import torch.nn as nn
import torch.nn.functional as F

class VIB(nn.Module):
    def __init__(self, d_in, d_z, n_class):
        super().__init__()
        self.enc = nn.Sequential(nn.Linear(d_in, 256), nn.ReLU(),
                                  nn.Linear(256, 2*d_z))
        self.cls = nn.Linear(d_z, n_class)
        self.d_z = d_z
    def forward(self, x):
        h = self.enc(x)
        mu, logvar = h[:, :self.d_z], h[:, self.d_z:]
        std = (0.5*logvar).exp()
        z = mu + std * torch.randn_like(std)
        return self.cls(z), mu, logvar

def vib_loss(logits, y, mu, logvar, beta=1e-2):
    ce = F.cross_entropy(logits, y)
    kl = -0.5 * (1 + logvar - mu.pow(2) - logvar.exp()).sum(-1).mean()
    return ce + beta * kl
# Classical (discrete) IB via Blahut-Arimoto-like updates
import numpy as np

def classical_ib(p_xy, n_T, beta, n_iter=200):
    nx, ny = p_xy.shape
    p_x  = p_xy.sum(1); p_y_x = p_xy / p_x[:, None]
    p_t_x = np.random.rand(nx, n_T); p_t_x /= p_t_x.sum(1, keepdims=True)
    for _ in range(n_iter):
        p_t   = p_x @ p_t_x
        p_y_t = (p_t_x.T * p_x) @ p_y_x / (p_t[:, None] + 1e-12)
        kl    = (p_y_x[:, None, :] *
                 np.log((p_y_x[:, None, :] + 1e-12) /
                        (p_y_t[None, :, :] + 1e-12))).sum(-1)
        p_t_x = p_t[None, :] * np.exp(-beta * kl)
        p_t_x /= p_t_x.sum(1, keepdims=True)
    return p_t_x, p_t, p_y_t
