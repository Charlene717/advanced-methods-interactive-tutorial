聯邦學習

McMahan et al. (2017, AISTATS) 提出 FedAvg：多個 client（醫院、實驗室）各自在本地訓練，僅上傳 model update 給 server 聚合；資料永不離開機構。對基因體、EHR、影像等敏感醫療資料是法規層級必備。

McMahan et al. (2017, AISTATS) introduced FedAvg: multiple clients (hospitals, labs) train locally and upload only model updates to a server for aggregation. Raw data never leaves the institution — a regulatory must-have for genomics, EHR, and imaging.

核心概念

從 centralised 到 federated

問題：10 家醫院各有 10 萬病人 EHR。集中所有資料訓練 → 違反 HIPAA / GDPR / 台灣個資法。每家獨立訓練 → 樣本少、模型差且各自不同。

Federated 思路（McMahan 2017）：共享「模型參數更新」而非「資料」。

  1. Server 把當前 global model θ_t 廣播給 K 個 client。
  2. 每個 client k 在本地資料 D_k 上跑 E 個 epoch 的 SGD，得到 θ_t^k。
  3. Client 把 Δ_k = θ_t^k − θ_t（或 θ_t^k 本身）上傳。
  4. Server 聚合（FedAvg）：θ_{t+1} = Σ (n_k/n) θ_t^k，其中 n_k 是 client-k 樣本數。

關鍵保證：聚合僅看 model updates、不看 raw data。配合 Differential Privacy + Secure Aggregation → 即使單個 client 與 server 合謀，也難反推個別樣本。

三大挑戰：

  • Non-IID 資料：不同醫院 patient mix 差異大 → 各 client 梯度方向不一致、收斂慢。
  • System heterogeneity：client 算力、網路頻寬差距大 → 平均要等慢者。
  • Privacy 攻擊：gradient inversion attack 可從梯度反推 batch 樣本（Zhu & Han 2019, NeurIPS）。

Problem: 10 hospitals each hold 100k EHR records. Pooling all data → violates HIPAA / GDPR / Taiwan PIPA. Training each in isolation → too few samples, divergent models.

Federated idea (McMahan 2017): share model updates, not data.

  1. The server broadcasts the current global model θ_t to K clients.
  2. Each client k runs E SGD epochs on local data D_k to produce θ_t^k.
  3. The client uploads Δ_k = θ_t^k − θ_t (or θ_t^k itself).
  4. The server aggregates (FedAvg): θ_{t+1} = Σ (n_k/n) θ_t^k where n_k is client-k's sample count.

Key guarantee: aggregation sees only updates, never raw data. Combined with differential privacy + secure aggregation → even a colluding client and server cannot easily invert individual samples.

Three challenges:

  • Non-IID data: hospitals have different patient mixes → client gradients disagree, slow convergence.
  • System heterogeneity: compute / bandwidth varies → straggler problem.
  • Privacy attacks: gradient-inversion attacks can recover batches (Zhu & Han 2019, NeurIPS).

演算法

FedAvg, FedProx, SCAFFOLD & DP-SGD

① FedAvg (McMahan 2017)：基準演算法。每 round：

  • Server broadcast θ_t。
  • 每個 client 跑 E 次 local SGD，batch B：θ_t^k ← θ_t^k − η ∇F_k(θ_t^k; ξ)。
  • Server 聚合：θ_{t+1} = Σ (n_k/n) θ_t^k。

② FedProx (Li 2020, MLSys)：對 non-IID 加 proximal term：local loss = F_k(θ) + (μ/2)‖θ − θ_t‖²，避免 client 跑太遠偏離 global。

③ SCAFFOLD (Karimireddy 2020, ICML)：用 control variate 修正 client drift——server 與 client 各維持一個 c, c_k；local update 變成 θ ← θ − η (∇F_k − c_k + c)。對 non-IID 加速 3–5x。

④ DP-SGD（Abadi 2016, CCS）：在 local SGD 前 (i) 對 per-sample gradient 做 norm clipping 到 C；(ii) 加 N(0, σ²C²I) noise。給出 (ε, δ)-DP 保證 by moments accountant。

⑤ Secure aggregation (Bonawitz 2017, CCS)：用 pairwise additive masks，每 client 上傳 Δ_k + r_k - r_{kk'}，所有 mask 兩兩抵銷只剩 Σ Δ_k。server 看不到個別 Δ_k。

⑥ Personalized FL：共享 backbone、各 client 留自己的 head（pFedMe, Ditto, FedRep）。或 multi-task / meta-learning (Per-FedAvg)。對極 non-IID 場景明顯優於 FedAvg。

① FedAvg (McMahan 2017): the baseline. Each round:

  • Server broadcasts θ_t.
  • Each client runs E local SGD epochs with batch B: θ_t^k ← θ_t^k − η ∇F_k(θ_t^k; ξ).
  • Server aggregates: θ_{t+1} = Σ (n_k/n) θ_t^k.

② FedProx (Li 2020, MLSys): adds a proximal term for non-IID: local loss = F_k(θ) + (μ/2)‖θ − θ_t‖², preventing clients from drifting too far.

③ SCAFFOLD (Karimireddy 2020, ICML): uses control variates to correct client drift — server and clients each carry c, c_k; update becomes θ ← θ − η(∇F_k − c_k + c). 3–5× speed-up on non-IID.

④ DP-SGD (Abadi 2016, CCS): before local SGD (i) clip per-sample gradient to norm C; (ii) add N(0, σ²C²I) noise. Provides (ε, δ)-DP via moments accountant.

⑤ Secure aggregation (Bonawitz 2017, CCS): pairwise additive masks make Δ_k + r_k − r_{kk'} cancel pairwise — server sees only Σ Δ_k, not individuals.

⑥ Personalised FL: shared backbone with per-client heads (pFedMe, Ditto, FedRep), or multi-task / meta-learning (Per-FedAvg). Clearly beats FedAvg on extreme non-IID.

為什麼 local epoch 數 E 是關鍵旋鈕？ E=1 等同於分散式 SGD（同步開銷大）；E 太大則 client 過度擬合本地分布，平均後 global model 振盪甚至發散（"client drift"）。實務 E ∈ [1, 5] 配 SCAFFOLD 或 FedProx 通常最穩。
Why is the number of local epochs E the key knob? E=1 is essentially distributed SGD (high sync cost). Large E overfits local distributions → averaged global model oscillates or diverges ("client drift"). Practitioners typically use E ∈ [1, 5] with SCAFFOLD or FedProx.

深入解析

Federated Learning 進階主題

① Non-IID 量化：Dirichlet partition

學術 benchmark 用 Dirichlet(α) 把資料分給 K client。α → 0 極端 non-IID（每 client 幾乎只有一個 class）；α → ∞ IID。實證：α < 0.1 時 FedAvg 直接發散，需 SCAFFOLD 或 personalised FL。生物資料（醫院間 patient mix）對應 α ≈ 0.3–0.5。

② Convergence analysis

Li et al. (2020, ICLR) 證明 FedAvg 在 convex setting O(1/T) 但係數隨 non-IID Γ 增加。Karimireddy 2020 證明 SCAFFOLD 對 Γ 不敏感。對 non-convex DNN 沒有 tight theory，但 FedProx 的 proximal term 已被多 study 證實穩定。

③ Communication efficiency

每 round 上傳整個 model（千萬參數）vs 數萬樣本——bandwidth 是 bottleneck。技術：(a) gradient sparsification (top-k)、(b) quantisation (1-bit, signSGD)、(c) low-rank update (FedLin)、(d) one-shot FL（只一次通訊，knowledge distillation）。

④ Privacy attacks & defences

主要攻擊：(i) gradient inversion (Zhu & Han 2019, Geiping 2020) 從梯度反推樣本；(ii) membership inference (Shokri 2017)；(iii) model inversion。防禦：DP-SGD（噪聲）+ secure aggregation（加密）+ small batch。實證 ε ≈ 1–4 在多數醫療任務影響 acc < 3%。

⑤ Cross-silo vs cross-device

Cross-silo（醫院、藥廠）：K 小、機器穩定、資料量大。Cross-device（手機）：K 巨大、device 隨機 dropout、資料小、heterogeneous compute。生物資料幾乎全是 cross-silo，演算法更接近 distributed optimisation。

⑥ Federated foundation model

2024 開始討論「federated pretraining」of 醫療 LLM、scLLM。挑戰：模型大（10B+）、通訊上限。技術方向：LoRA-only federation（FedLoRA），只共享 low-rank adapter，可降 1000x 通訊。

⑦ Vertical vs horizontal FL

Horizontal：不同機構有相同 feature、不同 sample（醫院聯合）。Vertical：相同 sample、不同 feature（醫院 + 保險公司 + 基因檢測）；需 entity resolution + SMC，技術更難。生物應用主要 horizontal；vertical 在多 omics + clinical 整合時關鍵。

⑧ Federated evaluation & benchmark

FLAIR (Yu 2023)、FederatedScope、FATE 是主要 benchmark/framework。Bio 專用：MELLODDY benchmark（藥）、FeTS（影像）、NVFlare medical examples。

① Quantifying non-IID: Dirichlet partition

Academic benchmarks split data via Dirichlet(α). α → 0 is extreme non-IID; α → ∞ is IID. Empirically, FedAvg diverges at α < 0.1 and needs SCAFFOLD or personalised FL. Real biomedical data (hospital patient mix) sit at α ≈ 0.3–0.5.

② Convergence analysis

Li et al. (2020, ICLR) proved O(1/T) for FedAvg in convex settings with constants growing in non-IID Γ. Karimireddy 2020 proved SCAFFOLD is Γ-insensitive. For non-convex DNNs no tight theory, but FedProx's proximal term is widely shown to stabilise training.

③ Communication efficiency

Uploading a multi-million-parameter model each round vs a few-thousand samples — bandwidth is the bottleneck. Techniques: (a) gradient sparsification (top-k), (b) quantisation (1-bit, signSGD), (c) low-rank update (FedLin), (d) one-shot FL (single round of distillation).

④ Privacy attacks & defences

Main attacks: (i) gradient inversion (Zhu & Han 2019, Geiping 2020); (ii) membership inference (Shokri 2017); (iii) model inversion. Defences: DP-SGD + secure aggregation + small batch. ε ≈ 1–4 reduces accuracy by < 3% on most medical tasks.

⑤ Cross-silo vs cross-device

Cross-silo (hospitals, pharma): small K, stable machines, large per-client data. Cross-device (phones): huge K, random dropout, small data, heterogeneous compute. Biomedical FL is almost always cross-silo — closer to distributed optimisation.

⑥ Federated foundation models

2024 brought "federated pretraining" for medical LLMs / scLLMs. Challenge: huge models (10B+), bandwidth limits. Direction: LoRA-only federation (FedLoRA) shares small adapters, reducing comms ~1000×.

⑦ Vertical vs horizontal FL

Horizontal: same features, different samples (hospitals). Vertical: same samples, different features (hospital + insurer + genome testing); needs entity resolution + SMC, technically harder. Biology is mostly horizontal; vertical matters for multi-omics + clinical integration.

⑧ Federated evaluation & benchmark

FLAIR (Yu 2023), FederatedScope, FATE are main frameworks. Bio-specific: MELLODDY (pharma), FeTS (imaging), NVFlare medical examples.

import torch, copy
from torch import nn

def fedavg(global_model, clients, rounds=20, E=3, lr=1e-2):
    for r in range(rounds):
        local_states, ns = [], []
        for ds in clients:                          # ds = (loader, n_samples)
            m = copy.deepcopy(global_model)
            opt = torch.optim.SGD(m.parameters(), lr=lr)
            for _ in range(E):
                for x, y in ds[0]:
                    opt.zero_grad()
                    loss = nn.functional.cross_entropy(m(x), y)
                    loss.backward(); opt.step()
