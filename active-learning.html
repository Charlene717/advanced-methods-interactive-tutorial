主動學習 / 貝氏優化

當每次實驗都很貴 — Perturb-seq、藥物 screen、CRISPR library — 用 GP + acquisition function 智慧地選下一個要做的實驗。從 100 次標記學會傳統需要 10,000 次的內容。

When each experiment is expensive — Perturb-seq, drug screens, CRISPR libraries — use GP + acquisition functions to intelligently choose the next experiment. Learn from 100 labels what would normally need 10,000.

核心概念

昂貴函數的最佳化問題

給定一個昂貴函數 f: X → ℝ（每次 query 可能要 $5000 或一週實驗），目標 argmaxx f(x)，同時最小化 query 次數。傳統 grid search、random search 在 d ≥ 5 維就完全失效。

貝氏優化（Bayesian Optimization, BO）的精神：

  1. Surrogate model：用便宜的代理模型（通常是 Gaussian Process）擬合到目前已測過的點。GP 給出每個 x 的「預測均值 μ(x)」+「不確定性 σ(x)」。
  2. Acquisition function：α(x) 決定下一個該 query 哪個 x。結合 exploitation（μ 高的地方）與 exploration（σ 高的地方）。
  3. 迭代：選 x* = argmax α(x)，做實驗得 y*，更新 GP，重複。

主動學習（Active Learning, AL）是同一精神的不同包裝：給有限 label budget，每次選「最有資訊量」的樣本去標記。BO 是針對「找最大值」的特例；AL 更廣（找最大不確定性、最大模型改進等）。

Given expensive f: X → ℝ (each query costs $5000 or a week), find argmaxx f(x) with minimum queries. Grid / random search fail at d ≥ 5.

Bayesian Optimization (BO):

  1. Surrogate model: cheap proxy (typically a Gaussian Process) fit to measured points. GP gives predicted mean μ(x) + uncertainty σ(x).
  2. Acquisition function: α(x) decides next query — balances exploitation (high μ) and exploration (high σ).
  3. Iterate: select x* = argmax α(x), measure y*, update GP, repeat.

Active Learning (AL) is the same idea: with a limited label budget, query the most informative samples. BO is the special case for finding maxima; AL is broader (max uncertainty, max model improvement).

演算法

GP 與 Acquisition Functions

① Gaussian Process Regression

GP 假設 f(x) 在任意有限點集上服從多元高斯分布。給定 kernel k(x, x')（衡量相似度，常用 RBF/Matérn），看過資料 D = {(xi, yi)} 後，對新點 x* 的後驗：

μ(x*) = k* · (K + σ²I)−1 y

σ²(x*) = k(x*,x*) − k* · (K + σ²I)−1 k*T

其中 Kij = k(xi, xj)，k* = [k(x*, x1), ..., k(x*, xn)]。複雜度 O(n³) 因矩陣反轉——大樣本要用 sparse GP、inducing points 等技巧。

② 主要 Acquisition Functions

  • Expected Improvement (EI)：EI(x) = E[max(0, f(x) − f*)] 其中 f* 為目前已知最佳。封閉解：EI(x) = σ(x) · [Z·Φ(Z) + φ(Z)], Z = (μ(x) − f*)/σ(x)。
  • Upper Confidence Bound (UCB)：UCB(x) = μ(x) + β·σ(x)。β 控制 exploration 強度，理論上 β ∝ √log(t) 給 sublinear regret 保證 (Srinivas 2010, ICML)。
  • Thompson Sampling (TS)：從 GP 後驗抽樣一個 f̃，取 argmax f̃。自然的隨機探索，在多 batch 場景表現好。
  • Probability of Improvement (PI)：PI(x) = Φ((μ(x) − f*)/σ(x))。簡單但偏 exploitation。
  • Knowledge Gradient (KG)：考慮「knowing y will help future decisions」。理論上最 sample-efficient，但計算很貴。

③ Batch BO（並行 query）

當實驗可以並行（96-well plate、Perturb-seq batch），需要選 q 個 query 點。難點：選同一批 q 個點不能太相似。

  • q-EI (Wang & Clark 2016)：多點期望改進的 Monte Carlo 估計。
  • Local penalization (Gonzalez 2016)：已選點周圍 penalize acquisition。
  • DPP-based diverse selection：用 determinantal point process 強制多樣性。

④ Multi-objective BO（Pareto front）

實際生物 task 常有多目標：efficacy + toxicity + cost。MOBO 找 Pareto front 而非單點：

  • ParEGO：scalarize 多目標為單目標。
  • EHVI (Expected Hypervolume Improvement)：直接最大化 Pareto front 覆蓋面積。
  • qEHVI：batch 版本（Daulton 2020, NeurIPS）。

① Gaussian Process Regression

GP assumes f(x) is jointly Gaussian over any finite set. With kernel k(x, x') (RBF/Matérn typically) and data D = {(xi, yi)}, posterior for new x*:

μ(x*) = k* · (K + σ²I)−1 y

σ²(x*) = k(x*,x*) − k* · (K + σ²I)−1 k*T

where Kij = k(xi, xj), k* = [k(x*, x1), ..., k(x*, xn)]. O(n³) for inversion — large n needs sparse GP, inducing points.

② Major Acquisition Functions

  • Expected Improvement (EI): EI(x) = E[max(0, f(x) − f*)] with f* current best. Closed form: EI(x) = σ(x) · [Z·Φ(Z) + φ(Z)], Z = (μ(x) − f*)/σ(x).
  • Upper Confidence Bound (UCB): UCB(x) = μ(x) + β·σ(x). β controls exploration; β ∝ √log(t) gives sublinear regret (Srinivas 2010, ICML).
  • Thompson Sampling (TS): sample f̃ from GP posterior, take argmax. Natural stochastic exploration, excels in batch settings.
  • Probability of Improvement (PI): PI(x) = Φ((μ(x) − f*)/σ(x)). Simple but exploitation-biased.
  • Knowledge Gradient (KG): "knowing y will help future decisions". Most sample-efficient theoretically but expensive.

③ Batch BO (parallel queries)

When experiments parallelize (96-well plate, Perturb-seq batch), select q points. Challenge: q points must not be too similar.

  • q-EI (Wang & Clark 2016): Monte Carlo multi-point EI.
  • Local penalization (Gonzalez 2016): penalize acquisition near already-selected points.
  • DPP-based diverse selection: determinantal point process enforces diversity.

④ Multi-objective BO (Pareto front)

Real biology has multi-objectives: efficacy + toxicity + cost. MOBO finds the Pareto front:

  • ParEGO: scalarize to single objective.
  • EHVI: directly maximize Pareto front hypervolume.
  • qEHVI: batch version (Daulton 2020, NeurIPS).

生物應用

BO / AL 在生命科學的應用

LaMBO

用 BO + denoising autoencoder 在離散序列空間做 protein optimization。在 GFP brightness、antibody binding 上以 100× 少實驗達到 wet lab 等同效能。後續 LaMBO-2 (2023) 加入 multi-objective（folding stability + binding）。

BO + denoising autoencoder for protein optimization in discrete sequence space. 100× fewer experiments to match wet-lab efficacy on GFP brightness, antibody binding. LaMBO-2 (2023) adds multi-objective (folding stability + binding).

BoTorch + GPyTorch

Meta/Stanford 出品的 BO 工業框架 — Ax 是其 UX 層。已被 Pfizer、Genentech 用於 reaction optimization、drug screening、bioprocess optimization。配合 PyTorch 後端可擴展至 GP 之外的代理模型。

Meta/Stanford industrial BO framework; Ax is the UX layer. Used by Pfizer, Genentech for reaction optimization, drug screening, bioprocess optimization. PyTorch backend supports non-GP surrogates.

COMETS

在組合 CRISPR screen 中，可能 sgRNA 對數為 ⁾20000²/2 ≈ 2億，wet lab 跑不完。COMETS (Najm 2024, Nat Biotech) 用 BO 主動學習 epistasis，從 10⁴ 組合預測 10⁸ 中真實 synergy。

Combinatorial CRISPR screens have ~200M possible pairs. COMETS (Najm 2024, Nat Biotech) uses BO to actively learn epistasis — predicts synergy across 10⁸ pairs from just 10⁴ measurements.

主動 Perturb-seq 設計

下一個 Perturb-seq 該打哪些基因？Roohani et al. (2024, Nat Biotech) 提出 active learning framework：用現有 perturbation 資料訓模型，acquisition 選「最有資訊量」的 gene perturbation，10× 減少實驗成本。

Which genes to perturb next? Roohani et al. (2024, Nat Biotech): active-learning framework using existing perturbation data; acquisition picks "most informative" gene perturbations — 10× cost reduction.

Adaptive Drug Screening

Ahuja et al. (2024) 在 NCI-60 cancer cell line panel 上用 BO 主動選 drug × cell line 配對 — 預測 cross-cell-line synergy。配合 PRISM 大型 screen，從 1% 標記學會 80% 的 IC50 landscape。

Ahuja et al. (2024) on NCI-60 cancer panel: BO actively selects drug × cell line pairs to predict cross-cell synergy. With PRISM data, learns 80% of IC50 landscape from 1% labeled.

Spatial 主動成像

MERFISH、Visium 全切片成像耗時 + 昂貴。主動式：先掃 10% 區域，BO 選下一個要掃的「資訊密集」區域。對 sparse signal (rare cell types in tissue) 特別有效。

MERFISH/Visium whole-slide imaging is slow + expensive. Active version: scan 10% first, BO picks next "information-dense" region. Especially powerful for sparse signals (rare cell types in tissue).

深入解析

BO 的進階主題與最新動向

① 高維 BO 的挑戰

GP 在 d ≥ 20 維上不靈光（curse of dimensionality）。解決方案：

  • REMBO / LineBO：假設 f 有 low-dim active subspace，BO 在投影空間進行。
  • TuRBO (Eriksson 2019, NeurIPS)：trust region BO，在局部小區域擬合 GP。
  • SAASBO (Eriksson 2021, AISTATS)：用 sparse axis-aligned subspace prior，自動選 important dims。
  • Deep Kernel Learning：NN 學 embedding，GP 在 embedding 上做。對 protein, molecule 等結構化輸入有效。

② 離散與結構化空間

蛋白質序列、SMILES、CRISPR sgRNA 都是離散字串，無法直接做 GP。方案：

  • Latent Space BO：VAE 學 continuous latent，BO 在 latent 進行（LaMBO、ChemBO）。
  • String kernel：直接在序列上定義 kernel（k-mer overlap、edit distance）。
  • Surrogate ≠ GP：用 NN ensemble、Bayesian NN 作為代理。對大型離散空間更實用。

③ 隨成本變化的 BO（Cost-aware BO）

真實實驗中 query 成本未必相等：高分辨率 MERFISH 比低分辨率貴 10×；某些 CRISPR target 比其他更難 clone。Cost-aware BO (Snoek 2012, Lee 2020) 加入成本模型，最大化 expected improvement per unit cost。

④ Multi-fidelity BO

有 cheap proxy（模擬、in silico screen）與 expensive ground truth（wet lab）時。BOCA、qMF-EI 用低保真度資料加速優化，只在關鍵點用 expensive query。對 drug discovery 至關重要——分子 docking 廉價，cell-based assay 昂貴。

⑤ Safe BO

實驗失敗可能很貴（病人試藥）。Safe BO 在 query 前驗證「機率 ≥ 1−δ 安全」(Sui 2015, ICML)。對 clinical trial design 與 robot lab 都重要。

⑥ 與 Reinforcement Learning 的關係

BO ≈ 連續 stateless bandit；RL ≈ stateful bandit。對 sequential experimental design（每次實驗影響下一個的 state），可用 model-based RL 取代純 BO。Bagchi (2024) 展示 RL-based experiment design 在 multi-step Perturb-seq 表現勝 BO。

⑦ GP 之外的代理模型

  • Deep Ensemble：多個 NN 的 std 作為 uncertainty。對大數據與高維輸入更實用。
  • Bayesian NN：Stochastic Weight Averaging Gaussian (SWAG)、Hamiltonian MC 等。
  • Random Forest with quantile regression：樹模型 + 分位數估計做不確定性。

⑧ Active Learning 的不同 query strategies

除 BO 的「找最大值」外，AL 有其他策略：

  • Uncertainty sampling：選最不確定樣本標記。
  • Query by Committee：多個模型最不一致的樣本。
  • Expected Model Change：標記後模型參數變化最大的樣本。
  • Core-set selection：能代表整個資料分布的最小子集。
  • BALD (Houlsby 2011)：Bayesian Active Learning by Disagreement — 在 Bayesian NN 框架下衡量 information gain。

⑨ 與 Conformal 的結合

用 conformal prediction 給 BO 的 surrogate 加 calibrated uncertainty — 替代 GP 的 Gaussian 假設。Stanton et al. (2023) 提出 conformal Bayesian optimization，對 misspecified prior 更穩健。

① High-Dimensional BO Challenges

GPs struggle at d ≥ 20 (curse of dimensionality). Remedies:

  • REMBO / LineBO: assume f has low-dim active subspace; BO in projection.
  • TuRBO (Eriksson 2019, NeurIPS): trust-region BO, local GP fit.
  • SAASBO (Eriksson 2021, AISTATS): sparse axis-aligned subspace prior; auto-select important dims.
  • Deep Kernel Learning: NN learns embedding, GP on embedding. Effective for protein, molecule inputs.

② Discrete and Structured Spaces

Protein sequences, SMILES, CRISPR sgRNAs are discrete — can't run GP directly. Solutions:

  • Latent Space BO: VAE learns continuous latent; BO in latent (LaMBO, ChemBO).
  • String kernels: kernel defined on sequences (k-mer overlap, edit distance).
  • Surrogate ≠ GP: NN ensemble or Bayesian NN — more practical for huge discrete spaces.

③ Cost-aware BO

Real query costs vary: high-res MERFISH costs 10× low-res; some CRISPR targets are harder to clone. Cost-aware BO (Snoek 2012, Lee 2020) adds a cost model, maximizes expected improvement per unit cost.

④ Multi-fidelity BO

Cheap proxy (simulation, in silico screen) + expensive ground truth (wet lab). BOCA, qMF-EI use low-fidelity data to accelerate optimization — critical for drug discovery (cheap docking, expensive cell assays).

⑤ Safe BO

Failed experiments can be costly (patient trials). Safe BO (Sui 2015, ICML) verifies "≥ 1−δ probability safe" before query — important for clinical trial design and robotic labs.

⑥ Connection to Reinforcement Learning

BO ≈ continuous stateless bandit; RL ≈ stateful bandit. For sequential experimental design where each experiment changes state, model-based RL can replace pure BO. Bagchi (2024) shows RL-based experiment design beats BO on multi-step Perturb-seq.

⑦ Beyond GP Surrogates

  • Deep Ensemble: NN ensemble's std as uncertainty. Better for big-data, high-dim inputs.
  • Bayesian NN: SWAG, Hamiltonian MC, etc.
  • RF + quantile regression: tree models with quantile-based uncertainty.

⑧ Active Learning Query Strategies

Beyond BO's max-finding, AL has other strategies:

  • Uncertainty sampling: pick most uncertain samples.
  • Query by Committee: samples where multiple models disagree most.
  • Expected Model Change: samples whose label would shift parameters most.
  • Core-set selection: smallest set representing the full distribution.
  • BALD (Houlsby 2011): Bayesian Active Learning by Disagreement — info gain in BNN.

⑨ Combining with Conformal

Conformal prediction provides calibrated uncertainty to BO surrogates — replaces GP's Gaussian assumption. Stanton et al. (2023) propose conformal Bayesian optimization, robust to misspecified priors.

潛在改良應用

BO/AL 還能改良哪些任務？

🎯 1. 下一個 Perturb-seq 該打哪些基因？

有 ~2 萬個 protein-coding gene，但只能打 ~1000 個。用 BO 從現有 Perturb-seq 資料學 surrogate，selectively pick next batch of targets that maximize information about gene regulatory network structure。提案：active Perturb-seq pipeline，與 scGen 等 prediction model 結合。

🎯 2. 下一個藥物候選與劑量

給定 chemical space + ADMET constraints + efficacy target，用 multi-objective BO 找 Pareto-optimal candidates。提案：integrated drug discovery loop — in silico screen → BO → wet lab → iterate。

🎯 3. 最有資訊量的 Patient Sample

Biobank 有 50 萬樣本但只能 deep-profile 5000 個。BO 主動選「最會改善現有模型」的 sample — 偏重邊界 case、minority populations、phenotype outliers。提升 polygenic risk model 跨人種 calibration。

🎯 4. Rare Cell Type Discovery via Active Outlier Detection

用 BO 引導的 outlier detection — 在 atlas-scale scRNA-seq 中主動探索 "uncertain" 區域。每次發現新 rare cell type 後更新 surrogate，繼續找其他 hidden populations。

🎯 5. Optimal Experimental Design 跨組織 Atlas

給定 tissue × condition × technology 的 3D budget grid，BO 選下一個該做的組合 — 最大化整個 atlas 的 coverage 與 quality。提案：HuBMAP / HCA 的 adaptive sampling scheduler。

🎯 6. CRISPR sgRNA Library Design

對 genome-wide library 用 BO 主動選 sgRNA：考慮 efficacy prediction + off-target risk + sequence diversity。比 random library 少 50% 數量達同等 hit rate。

🎯 7. Protein Engineering 的 Directed Evolution

取代隨機 mutagenesis，用 BO 在序列空間導航。LaMBO 已展示，但可推廣到 antibody affinity maturation、enzyme thermostability optimization 等任務。

🎯 8. Spatial Transcriptomics 主動採樣

稀疏 spatial omics（如 single-cell Hi-C）無法 full coverage。BO 從 sparse scan 預測「最有資訊量的下一個 cell/location」。對 patient biopsy 樣本特別重要（樣本量有限）。

🎯 9. Multi-Omics Modality Selection

給定 RNA、ATAC、protein、methylation 四種模態，每種預算不同。BO 決定「為了某個 cell type characterization，哪些模態最有資訊量」。降低不必要 multi-omics 成本。

🎯 10. Hyperparameter Tuning for scLLMs

scGPT、Geneformer 的 fine-tuning 有大量 hyperparameters（learning rate, batch size, masking rate, vocabulary size）。BO 用 validation performance 作目標函數，幾十次 trial 找到最佳組合。

🎯 1. Which Genes to Perturb Next?

~20k protein-coding genes but only ~1000 perturbed. BO trains a surrogate on existing Perturb-seq data; acquisition picks targets maximizing info about GRN structure. Proposal: active Perturb-seq pipeline coupled with scGen-style predictors.

🎯 2. Next Drug Candidate & Dose

Multi-objective BO over chemical space + ADMET + efficacy → Pareto candidates. Proposal: integrated drug discovery loop — in silico → BO → wet lab → iterate.

🎯 3. Most Informative Patient Sample

Biobanks have 500k samples but only 5k can be deep-profiled. BO actively picks samples that most improve current models — boundary cases, minority populations, phenotype outliers. Improves cross-ethnicity polygenic risk calibration.

🎯 4. Rare Cell Type Discovery via Active Outlier Detection

BO-guided outlier detection in atlas-scale scRNA-seq — actively explores "uncertain" regions. After each novel rare type found, update surrogate to find more hidden populations.

🎯 5. Optimal Experimental Design for Atlases

BO selects next combination on tissue × condition × technology budget grid to maximize atlas coverage and quality. Proposal: HuBMAP / HCA adaptive sampling scheduler.

🎯 6. CRISPR sgRNA Library Design

BO selects sgRNAs for genome-wide libraries balancing efficacy + off-target risk + sequence diversity. 50% smaller libraries with equal hit rate vs random.

🎯 7. Directed Evolution for Protein Engineering

Replace random mutagenesis with BO-driven sequence-space navigation. LaMBO showed promise; extendable to antibody affinity maturation, enzyme thermostability.

🎯 8. Active Spatial Transcriptomics Sampling

Sparse spatial omics (single-cell Hi-C) can't fully cover. BO from sparse scans predicts the next-most-informative cell/location. Critical for limited patient biopsies.

🎯 9. Multi-Omics Modality Selection

For RNA/ATAC/protein/methylation with different costs, BO picks which modalities most inform a given cell-type characterization — cuts unnecessary multi-omics costs.

🎯 10. Hyperparameter Tuning for scLLMs

scGPT, Geneformer have many tuning knobs (LR, batch size, mask rate, vocab). BO uses validation performance as objective — finds best combos in tens of trials.

import torch
from botorch.models import SingleTaskGP
from botorch.fit import fit_gpytorch_mll
from botorch.acquisition import ExpectedImprovement
from gpytorch.mlls import ExactMarginalLogLikelihood

# Initial data
train_X = torch.rand(10, 3)                   # 10 samples, 3D
train_Y = some_expensive_fn(train_X).unsqueeze(-1)

# Fit GP
gp = SingleTaskGP(train_X, train_Y)
mll = ExactMarginalLogLikelihood(gp.likelihood, gp)
fit_gpytorch_mll(mll)

# BO loop
for i in range(30):
    EI = ExpectedImprovement(gp, best_f=train_Y.max())
    from botorch.optim import optimize_acqf
    cand, _ = optimize_acqf(EI, bounds=torch.tensor([[0.]*3,[1.]*3]),
                            q=1, num_restarts=10, raw_samples=256)
    new_Y = some_expensive_fn(cand).unsqueeze(-1)
    train_X = torch.cat([train_X, cand]); train_Y = torch.cat([train_Y, new_Y])
    gp = SingleTaskGP(train_X, train_Y)
    fit_gpytorch_mll(ExactMarginalLogLikelihood(gp.likelihood, gp))

# Minimal GP regression + EI from scratch
import numpy as np
from scipy.stats import norm

def rbf(x1, x2, l=0.2, sigma=1.0):
    sqd = (x1[:,None] - x2[None,:])**2
    return sigma**2 * np.exp(-0.5 * sqd / l**2)

def gp_posterior(X_train, y_train, X_star, noise=1e-4):
    K = rbf(X_train, X_train) + noise * np.eye(len(X_train))
    K_inv = np.linalg.inv(K)
    K_star = rbf(X_train, X_star)
    K_ss = rbf(X_star, X_star)
    mu = K_star.T @ K_inv @ y_train
    cov = K_ss - K_star.T @ K_inv @ K_star
    return mu, np.sqrt(np.maximum(np.diag(cov), 1e-8))

def expected_improvement(mu, sigma, f_best, xi=0.01):
    Z = (mu - f_best - xi) / (sigma + 1e-9)
    return (mu - f_best - xi) * norm.cdf(Z) + sigma * norm.pdf(Z)

# BO loop
X_grid = np.linspace(0, 1, 200)
X_train = np.array([0.1, 0.5, 0.9])
y_train = expensive_fn(X_train)
for i in range(20):
    mu, sigma = gp_posterior(X_train, y_train, X_grid)
    ei = expected_improvement(mu, sigma, y_train.max())
    x_next = X_grid[np.argmax(ei)]
    X_train = np.append(X_train, x_next)
    y_train = np.append(y_train, expensive_fn(np.array([x_next])))

Bayesian Optimization 中 acquisition function 的主要任務是什麼？
