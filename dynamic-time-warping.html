動態時間規整 (Dynamic Time Warping)

比較兩條時間序列、但允許「時間軸局部伸縮」的經典演算法（Sakoe & Chiba 1978, IEEE）。從語音辨識起家，今天已是 cellAlign（單細胞 pseudotime 對齊）、ECG/EEG、microbiome 縱貫資料的標配。Soft-DTW (Cuturi 2017) 讓它可微，可直接訓 NN。

A classical algorithm (Sakoe & Chiba 1978, IEEE) that compares two time series allowing "local time-axis stretching". Originally for speech recognition, now ubiquitous in cellAlign (pseudotime alignment), ECG/EEG, longitudinal microbiome. Soft-DTW (Cuturi 2017) makes it differentiable and end-to-end trainable.

核心概念

為什麼需要 DTW？

歐式距離的局限：兩條時間序列 x[1..N]、y[1..M] 的歐式距離要求 N = M 且「逐點對齊」。但實際上：

  • 同一首歌不同人哼唱——速度不一致
  • 同一通路活化於兩個樣本——時序略有偏移
  • 同一心跳——R 波位置會漂移

歐式距離會誤判這些「形狀相同、時間略偏移」的訊號為「不同」。

DTW 的核心：允許 x 的時間軸 i 與 y 的時間軸 j 做「彈性對應」——以 monotone non-decreasing path (i₁, j₁) → (i₂, j₂) → ... 的方式對應兩條序列，使路徑上點對的距離總和最小。

遞推公式：定義 DP 矩陣 D ∈ ℝ^(N × M)，

D(i, j) = d(x_i, y_j) + min{D(i-1, j), D(i, j-1), D(i-1, j-1)}

其中 d(·, ·) 通常是 |x_i − y_j| 或 (x_i − y_j)²。最終 DTW(x, y) = D(N, M)。Backtracking 還原最佳對齊路徑。時間複雜度 O(NM)。

關鍵限制：路徑必須 (i) 從 (1,1) 到 (N, M)；(ii) 單調不下降（時間不能倒流）；(iii) 連續（不能跳格）。實務常加 Sakoe-Chiba band：|i − j| ≤ w 的對角帶約束，把複雜度降到 O(N·w)。

Euclidean's limitation: Euclidean distance between x[1..N], y[1..M] requires N = M and "pointwise alignment". In practice:

  • Same song hummed by different people — different tempo
  • Same pathway activated in two samples — slight time shift
  • Same heartbeat — R-wave position drifts

Euclidean misclassifies these "same shape, time-shifted" signals as "different".

DTW's core: allow a flexible time-axis correspondence — monotone non-decreasing path (i₁, j₁) → (i₂, j₂) → ... that pairs the two series, minimizing total pairwise distance along the path.

Recursion: DP matrix D ∈ ℝ^(N × M),

D(i, j) = d(x_i, y_j) + min{D(i-1, j), D(i, j-1), D(i-1, j-1)}

where d(·, ·) is |x_i − y_j| or (x_i − y_j)². Final DTW(x, y) = D(N, M). Backtracking recovers the optimal alignment. Time complexity O(NM).

Constraints: path must (i) go from (1,1) to (N, M); (ii) be monotone non-decreasing (time cannot reverse); (iii) be continuous. In practice add Sakoe-Chiba band: |i − j| ≤ w diagonal band, reducing complexity to O(N·w).

演算法

從經典 DTW 到 Soft-DTW 與 FastDTW

① 經典 DTW (Sakoe & Chiba 1978)：O(NM) DP，O(NM) 記憶體（也可 O(min(N,M)) 流式存）。Sakoe-Chiba band w 通常取 10-20% 的序列長度——既加速又防止「病態對齊」（如把整條 x 對到 y 的單點）。

② FastDTW (Salvador & Chan 2007)：多解析度近似——先在低解析度（粗化序列）做 DTW、再在原解析度沿 coarse path 周圍 refine。複雜度降至 O(N)，但只是近似最佳路徑。對「序列極長」有用，對「需要嚴格最佳」則不適合。

③ Soft-DTW (Cuturi & Blondel 2017, ICML)：把 hard min 換成 soft-min（log-sum-exp）：

D_γ(i, j) = d(x_i, y_j) − γ · log(e^(−D(i-1,j)/γ) + e^(−D(i,j-1)/γ) + e^(−D(i-1,j-1)/γ))

當 γ → 0 退化回原 DTW。優點：所有 entry 可微，可作為 loss 訓練 NN（如時序預測模型的「輸出與 ground truth 的軟對齊損失」）。在 PyTorch / JAX 中有完整實作（soft-dtw-jax、pysdtw）。

④ DBA — DTW Barycenter Averaging (Petitjean 2011)：給定一群時間序列，找一個「中心序列」使所有序列到它的 DTW 距離總和最小（類似 k-means 的 centroid）。是 DTW-clustering 的基礎，已用於 ECG cluster、cell trajectory 中心化。iDBA (Forestier 2017) 是其增量更新版。

⑤ Shape DTW (Zhao & Itti 2018)：對 x_i 與 y_j 不只比較單點，而是比較「local shape descriptor」（如 HOG-1D / 子序列）。對形狀相似但 absolute value 略偏的序列更穩健。

⑥ 其他變體：

  • Derivative DTW (Keogh 2001)：對 dx/dt 序列做 DTW，避免「形狀相似但縱軸偏移」。
  • Weighted DTW：對對角步加 weight (1) 比水平/垂直步 (1+w) 大——鼓勵對角線、抑制「橫衝直撞」。
  • Itakura parallelogram：另一種替代 Sakoe-Chiba band 的形狀約束，限制更嚴。

① Classic DTW (Sakoe & Chiba 1978): O(NM) DP, O(NM) memory (or O(min(N,M)) streaming). Sakoe-Chiba band w is typically 10-20% of sequence length — accelerates and prevents pathological alignments.

② FastDTW (Salvador & Chan 2007): multi-resolution approximation — DTW at low resolution, then refine around coarse path at original resolution. Complexity O(N), approximate. Good for very long sequences; not for strict optima.

③ Soft-DTW (Cuturi & Blondel 2017, ICML): replaces hard min with soft-min (log-sum-exp):

D_γ(i, j) = d(x_i, y_j) − γ · log(e^(−D(i-1,j)/γ) + e^(−D(i,j-1)/γ) + e^(−D(i-1,j-1)/γ))

γ → 0 recovers classic DTW. Benefit: differentiable everywhere, usable as NN loss (e.g. soft-aligned forecast loss). Implementations: soft-dtw-jax, pysdtw.

④ DBA — DTW Barycenter Averaging (Petitjean 2011): given a set of series, find a "central series" minimizing total DTW distance (like k-means centroid). Basis of DTW-clustering — used for ECG clustering, cell trajectory centroids. iDBA (Forestier 2017) is incremental.

⑤ Shape DTW (Zhao & Itti 2018): instead of pointwise comparison, compare "local shape descriptors" (HOG-1D / subsequences). More robust when shapes match but absolute values differ.

⑥ Other variants:

  • Derivative DTW (Keogh 2001): DTW on dx/dt — avoids "same-shape, vertical-offset" issue.
  • Weighted DTW: diagonal step weight (1) less than horizontal/vertical (1+w) — encourages diagonal moves.
  • Itakura parallelogram: alternative shape constraint to Sakoe-Chiba band, more restrictive.
DTW vs. Optimal Transport：兩者都是「最小代價對應」，但 DTW 強制「monotone, continuous」（時間不能倒流、不能跳），而 OT 允許任意 mass transport。對單細胞 pseudotime alignment——若兩條軌跡分支結構相同，DTW 比 OT 更合適。若是 cross-population mass redistribution，OT 才對。
DTW vs. Optimal Transport: both find minimum-cost correspondence, but DTW enforces "monotone, continuous" (time cannot reverse or skip), whereas OT allows arbitrary mass transport. For single-cell pseudotime alignment with matching branching, DTW is more appropriate. For cross-population mass redistribution, OT is better.

🌳 什麼時候用 DTW？

情境1:
同類訊號、時間軸略偏——心跳對齊、語音、不同人 ECG。
情境2:
不同條件下的同 trajectory——cellAlign 對齊兩條件 scRNA-seq pseudotime。
情境3:
序列分類與聚類——配 DBA 或 k-medoid + DTW 距離做時間序列分群。
不適合:
序列分支或拓撲不一致；訊號需「絕對時間」（如 schedule, calendar event）；資料極長且嚴格最佳路徑必要。
Case 1:
Same signal type, slight time shift — heartbeat, speech, multi-subject ECG.
Case 2:
Same trajectory under different conditions — cellAlign aligning two-condition scRNA-seq pseudotime.
Case 3:
Sequence classification / clustering — with DBA or k-medoid + DTW distance.
Not for:
Branching topology mismatch; absolute time required (schedule, calendar); very long sequences needing strict optimal path.
生物應用

DTW 在生物資訊與醫學的應用

cellAlign

Alpert et al. (2018, Nature Methods) 用 DTW 對齊不同條件下的 scRNA-seq pseudotime——例如比較 WT 與 KO 的細胞分化軌跡。對齊後可逐 pseudotime 點做差異基因分析，發現「軌跡何時開始分歧」。已成單細胞跨條件比較的標配。

Alpert et al. (2018, Nature Methods) align scRNA-seq pseudotimes across conditions with DTW — e.g. WT vs KO differentiation trajectories. After alignment, per-pseudotime DE analysis identifies "when trajectories diverge". Now standard for cross-condition single-cell comparison.

ECG / 心律分析

心跳之間有自然變異（HRV），DTW 對齊不同心跳的 P-QRS-T 波——比歐式距離更穩健地檢測 arrhythmia。在 PhysioNet challenge 多次被用作 baseline 或特徵工程基礎。也用於 ICU 連續心電監測的患者特徵聚類。

Heartbeats vary naturally (HRV); DTW aligns P-QRS-T waves robustly — better than Euclidean for arrhythmia detection. Featured in many PhysioNet challenges as baseline / feature engineering. Also used in ICU continuous-monitoring patient clustering.

Spike train alignment

神經元 spike train 比較常用 Victor-Purpura metric 或 van Rossum distance，但 DTW 也可——對於相同 stimulus 下不同 trial 的 spike sequence，DTW 對齊揭示「神經元編碼的時序不變特徵」。已用於 hippocampus replay 分析。

Spike train comparison often uses Victor-Purpura or van Rossum distance, but DTW works too — aligning spikes from different trials of the same stimulus reveals "temporally invariant neural coding". Used in hippocampus replay analysis.

Mass spec retention-time alignment

LC-MS 不同批次的 retention time 會漂移——DTW（或其變體 COW: correlation optimized warping）對齊 peak 位置，是 proteomics / metabolomics 預處理的核心步驟。對 plasma metabolite 大型 cohort 研究尤其關鍵。

LC-MS retention times drift between batches — DTW (or COW: correlation optimized warping) aligns peak positions, a key proteomics / metabolomics preprocessing step. Crucial for plasma-metabolite large-cohort studies.

時序 RNA-seq 比較

兩個基因（或兩個條件下的同基因）的時序 expression profile 用 DTW 比較——可發現「跟隨關係」（gene A 先表達、gene B 延後表達 → DTW path 上偏對角）。用於 gene regulation 推論與 chrono-therapy 研究。

DTW-comparing time-course expression of two genes (or one gene under two conditions) reveals "follower" relationships (gene A first, gene B delayed → off-diagonal DTW path). Used in gene regulation inference and chrono-therapy research.

晝夜節律基因震盪

不同組織的 circadian gene 震盪相位不同（肝臟比心臟快 4 小時等）。DTW 對齊揭示組織-組織相位差，用於「中樞時鐘 vs 周邊時鐘」研究（Hughes et al. 2009 Genome Research 的後續工作）。

Circadian genes oscillate with tissue-specific phase shifts (liver 4h ahead of heart, etc.). DTW alignment reveals tissue-tissue phase differences for "central vs peripheral clock" research (Hughes et al. 2009 Genome Research follow-ups).

Microbiome 縱貫資料

飲食干預研究中，不同病人對相同飲食的微生物群響應時間不同——DTW 對齊 microbiome 時序，揭示「快響應者 vs 慢響應者」。被用於 IBS、IBD 介入研究與 personalized nutrition。

In dietary intervention studies, patients respond on different timescales — DTW aligns microbiome time-series to identify "fast vs slow responders". Used in IBS, IBD intervention studies and personalized nutrition.

DTW + Sliced-OT integration

2023-24 年數個工作把 DTW 與 sliced-Wasserstein 結合——對「時序 + 分布」的混合資料（如多細胞 single-cell 的 pseudotime alignment 同時要對齊細胞分布）。提供 cross-condition single-cell 動態比較的新工具。

Several 2023-24 works combine DTW with sliced-Wasserstein for "time + distribution" data (e.g. simultaneously aligning pseudotime and cell distributions). Provides new tools for cross-condition single-cell dynamics.

深入解析

DTW 進階主題

① DTW 為何不是「距離」

DTW 不滿足三角不等式——DTW(x, z) 可能 > DTW(x, y) + DTW(y, z)。所以嚴格說它是 dissimilarity 而非 metric。實務後果：用 DTW 距離做 k-means 不保證收斂（需用 k-medoid 或 DBA）。

② Soft-DTW 的微分可導

Cuturi & Blondel (2017) 證明 soft-min 版的 DP 對所有 entry 連續可微，梯度可用「reverse-mode DP」高效計算（O(NM)）。這讓 DTW 進入 NN 的 loss zoo——時序預測網路可直接用「soft-DTW(prediction, ground_truth)」做訓練。

③ DTW 與「subsequence」變體

UCR Suite (Rakthanmanon 2012, KDD) 提供 subsequence DTW — 在長序列中找與短 query 最相似的子段，配合 LB_Keogh lower bound 可在 trillion-point 資料庫中查詢。對基因組 motif scanning 也有應用。

④ Sakoe-Chiba band 的選擇

太緊（w → 0）退化為歐式距離；太鬆（w → N）失去意義（任何兩序列幾乎都「相似」）。實證 w = 10-20% 序列長度通常最佳；Itakura 平行四邊形是另一種選擇。對 scRNA-seq pseudotime alignment，cellAlign 建議 w = 20%。

⑤ DTW clustering

k-medoid + DTW：簡單但慢（每對序列都要算 DTW）；可用 LB_Keogh 預篩。
DBA + k-means：找 centroid 序列、把序列分到最近 centroid。Petitjean (2011, Pattern Recognition) 原版及 iDBA (Forestier 2017) 是主流選擇。
Spectral clustering：以 DTW 距離建 similarity matrix、做 spectral embedding。對 time-series database 的探索性分析有用。

⑥ 多變量 DTW（multivariate DTW）

對 d 維時間序列 x[i] ∈ ℝ^d，d(x_i, y_j) 改用 Euclidean / Mahalanobis 距離即可。但 multivariate DTW 對「不同維度有不同 dynamics」時容易失敗——可考慮 dimension-wise DTW + 聚合，或 SOM-DTW (Sakurai 2007)。

⑦ GPU / parallel DTW

DP 的 anti-diagonal 可平行化——cuDTW 在 GPU 上對大規模序列 corpus 加速 100x+。tslearn 的 CUDA 版本可實時處理百萬序列的 pairwise DTW。

⑧ 與 Edit distance 的關係

DTW ≈ continuous-valued 的 edit distance：editing operations 是 insert/delete/replace；DTW 的 horizontal/vertical/diagonal step 對應 insert/delete/match。對 categorical sequences 用 edit distance；對 continuous 用 DTW；對混合資料看具體 metric。

① DTW is not a metric

DTW violates the triangle inequality — DTW(x, z) may exceed DTW(x, y) + DTW(y, z). Strictly a dissimilarity, not a metric. Consequence: k-means with DTW doesn't guarantee convergence (use k-medoid or DBA).

② Soft-DTW differentiability

Cuturi & Blondel (2017) prove the soft-min DP is everywhere differentiable; gradients computable by reverse-mode DP in O(NM). Brings DTW into the NN loss zoo — time-series forecasting nets train end-to-end with "soft-DTW(prediction, ground_truth)".

③ DTW subsequence variants

UCR Suite (Rakthanmanon 2012, KDD) provides subsequence DTW — finds query best-match in long series with LB_Keogh lower bound, scaling to trillion-point databases. Applied to genome motif scanning too.

④ Choosing Sakoe-Chiba band w

Too tight (w → 0) degenerates to Euclidean; too loose (w → N) makes everything "similar". Empirically w = 10-20% of length is best; Itakura parallelogram is an alternative. For scRNA-seq pseudotime alignment, cellAlign suggests w = 20%.

⑤ DTW clustering

k-medoid + DTW: simple but slow (all-pairs); LB_Keogh pruning helps.
DBA + k-means: find centroid series, assign by nearest centroid. Petitjean (2011, Pattern Recognition) original, iDBA (Forestier 2017) — both standard.
Spectral clustering: similarity matrix from DTW + spectral embedding. Useful for time-series database exploration.

⑥ Multivariate DTW

For d-dim x[i] ∈ ℝ^d, use Euclidean / Mahalanobis in d(x_i, y_j). But multivariate DTW struggles when dimensions have different dynamics — consider dimension-wise DTW + aggregate, or SOM-DTW (Sakurai 2007).

⑦ GPU / parallel DTW

DP anti-diagonals parallelize — cuDTW gives 100x+ speedup on large corpora. tslearn's CUDA variant handles million-series pairwise DTW in real time.

⑧ Relation to edit distance

DTW ≈ continuous-valued edit distance: edits are insert/delete/replace; DTW's horizontal/vertical/diagonal steps correspond to insert/delete/match. Use edit distance for categorical, DTW for continuous, ad-hoc metrics for mixed.

潛在改良應用

還能改良哪些任務？

🎯 1. cellAlign + branching trajectory

目前 cellAlign 假設兩 trajectory 是線性。提案：branching-DTW——允許在 branch 點分叉，對齊不同 branching topology 的 trajectories。可用於 hematopoiesis 的 cross-condition 比較。

🎯 2. Spatial DTW for spatial trajectories

對 spatial transcriptomics 的「沿空間軸的細胞變化」做 DTW 對齊——同一 tissue 不同 sample 的空間 gradient 對齊。提案：SpatialDTW with 2D 路徑限制。

🎯 3. Multi-omics longitudinal DTW

同一病人同時記錄 RNA / metabolome / microbiome 時序——多 modality 變化速度不同，需要 modality-specific DTW + 聯合對齊。提案：Multi-Modal DTW for personalized longitudinal medicine。

🎯 4. Soft-DTW VAE for time-series generation

用 Soft-DTW 當 VAE decoder loss，可生成「時序合理」的訊號（不只 pointwise 匹配）。提案：Soft-DTW VAE for ECG generation、單細胞 trajectory simulation。

🎯 5. DTW + Conformal prediction

對齊後做時序預測時，用 conformal prediction 給「未來軌跡」的 uncertainty band。提案：Conformal Soft-DTW Forecasting for ICU monitoring。

🎯 6. Causal DTW

找 X(t) leading Y(t) 的「時序因果方向」——DTW path 偏 X 之上 → Y 領先；偏 X 之下 → X 領先。提案：causal-DTW 用於 dyadic interaction (microbiome co-evolution、神經元 cross-correlation)。

🎯 7. DTW on graphs (Graph-DTW)

對「圖結構隨時間演化」（如 dynamic protein-protein interaction）做圖序列的 DTW——把 graph kernel 當作 d(x_i, y_j)。提案：Graph-DTW for dynamic biological networks。

🎯 1. cellAlign with branching

cellAlign assumes linear trajectories. Proposal: branching-DTW — allow forks at branch points, align trajectories with different branching topology. Useful for hematopoiesis cross-condition comparison.

🎯 2. Spatial DTW for spatial trajectories

DTW on "cell change along spatial axis" in spatial transcriptomics — align spatial gradients across samples. Proposal: SpatialDTW with 2D path constraints.

🎯 3. Multi-omics longitudinal DTW

Same patient with RNA + metabolome + microbiome time-series — different modalities, different paces; need per-modality DTW + joint alignment. Proposal: Multi-Modal DTW for personalized longitudinal medicine.

🎯 4. Soft-DTW VAE for time-series generation

Use Soft-DTW as VAE decoder loss to generate "temporally plausible" signals (not just pointwise). Proposal: Soft-DTW VAE for ECG generation, single-cell trajectory simulation.

🎯 5. DTW + Conformal prediction

After alignment, use conformal prediction for "future trajectory" uncertainty bands. Proposal: Conformal Soft-DTW Forecasting for ICU monitoring.

🎯 6. Causal DTW

Find leading/lagging direction in X(t) ↔ Y(t) — path above diagonal → Y leads; below → X leads. Proposal: causal-DTW for dyadic interactions (microbiome co-evolution, neuronal cross-correlation).

🎯 7. Graph-DTW

DTW on "graph-evolving over time" (dynamic PPI) — use graph kernel as d(x_i, y_j). Proposal: Graph-DTW for dynamic biological networks.

from tslearn.metrics import dtw, dtw_path
import numpy as np

t = np.linspace(0, 4*np.pi, 200)
x = np.sin(t)
y = np.sin(1.3*t + 0.5)             # shifted & faster

d = dtw(x, y)                       # DTW distance
path, d2 = dtw_path(x, y)           # optimal alignment path

# With Sakoe-Chiba band
from tslearn.metrics import dtw_path_from_metric
path, _ = dtw_path_from_metric(x, y, metric="euclidean",
    sakoe_chiba_radius=20)         # band width

# Clustering with DBA
from tslearn.clustering import TimeSeriesKMeans

import numpy as np

def dtw_simple(x, y, band=None):
    N, M = len(x), len(y)
    D = np.full((N+1, M+1), np.inf)
    D[0, 0] = 0
    for i in range(1, N+1):
        j0 = max(1, i-band) if band else 1
        j1 = min(M, i+band)+1 if band else M+1
        for j in range(j0, j1):
            cost = abs(x[i-1] - y[j-1])
            D[i, j] = cost + min(D[i-1, j], D[i, j-1], D[i-1, j-1])
    return D[N, M], D

# Backtracking
def trace_path(D):
    i, j = D.shape[0]-1, D.shape[1]-1
    path = [(i, j)]
    while i > 1 or j > 1:
        candidates = [(i-1, j-1), (i-1, j), (i, j-1)]
        candidates = [(a,b) for a,b in candidates if a>0 and b>0]
        i, j = min(candidates, key=lambda p: D[p])
        path.append((i, j))
    return path[::-1]
