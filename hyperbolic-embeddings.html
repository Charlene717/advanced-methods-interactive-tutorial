雙曲嵌入 (Hyperbolic Embeddings)

為什麼把細胞型別、蛋白質家族塞進雙曲空間？因為負曲率讓「半徑指數成長的體積」自然容納樹狀階層。Nickel & Kiela 2017 後爆紅，已成為單細胞 ontology、跨物種對齊、TCR repertoire 等任務的新利器。

Why embed cell types and protein families in hyperbolic space? Negative curvature gives volume that grows exponentially with radius — perfectly suited to tree-like hierarchies. Since Nickel & Kiela 2017, hyperbolic embeddings have become a new tool for single-cell ontology, cross-species alignment, and TCR repertoires.

核心概念

為何需要負曲率空間？

歐式空間的困境：樹（branching factor b, 深度 d）的節點數為 b^d，但歐式 ℝ^k 中半徑 r 的球體積只增長 r^k——若想在不太扭曲距離的前提下嵌入一棵樹，歐式維度必須隨深度線性增長（Sarkar 2011: 任何有界扭曲嵌入需要 Ω(d) 維）。

雙曲空間的優勢：常曲率 −c 的雙曲空間 ℍ_c 中，半徑 r 的球體積為 sinh(cr)^(d−1) ≈ e^((d−1)cr)——體積隨 r 指數成長。剛好對應樹狀結構，故可在 2 維 就把任意樹幾乎無失真嵌入（Sarkar 2011）。

直觀比喻：歐式空間像平地——往外走 r 步看到的東西大約是 O(r²)。雙曲空間像「永遠在發散的喇叭」——往外走 r 步看到的東西指數增加，就像走進一棵不斷分枝的樹。

The Euclidean problem: A tree (branching factor b, depth d) has b^d nodes, but a ball of radius r in ℝ^k has volume ~r^k. To embed a tree with bounded distortion, Euclidean dimension must scale linearly with depth (Sarkar 2011: any low-distortion embedding requires Ω(d) dimensions).

The hyperbolic advantage: In hyperbolic space ℍ_c with constant curvature −c, a ball of radius r has volume sinh(cr)^(d−1) ≈ e^((d−1)cr) — volume grows exponentially in r. This matches tree structure exactly, so any tree can be embedded almost isometrically in just 2 dimensions (Sarkar 2011).

Intuition: Euclidean space is flat — walking r steps outward exposes O(r²) area. Hyperbolic space is "an ever-flaring trumpet" — walking r steps exposes exponentially more, like stepping into a tree that keeps branching.

演算法

Poincaré disk / Lorentz 模型與優化

① Poincaré disk 模型：定義在開單位球 𝔻 = {x ∈ ℝ^d : ‖x‖ < 1}，距離為

d(x,y) = acosh(1 + 2 ‖x − y‖² / ((1 − ‖x‖²)(1 − ‖y‖²)))

越靠邊界距離越「拉長」——一棵樹的根放在原點、葉子放在邊界附近，葉子之間距離指數成長，剛好符合樹的層次。

② Lorentz / hyperboloid 模型：把雙曲空間視為 ℝ^(d+1) 中的單側雙曲面 ⟨x,x⟩_L = −1，其中 ⟨x,y⟩_L = −x₀y₀ + Σx_iy_i。距離為 d(x,y) = acosh(−⟨x,y⟩_L)。數值更穩定，因為避免了 Poincaré disk 在邊界附近的奇異性。

③ Riemannian SGD：標準 SGD 不能用——歐式更新會把點推出單位球或雙曲面。Nickel & Kiela (2017) 提出在 Poincaré disk 上的 Riemannian SGD：

θ ← proj_𝔻(θ − η · (1 − ‖θ‖²)²/4 · ∇L)

(1 − ‖θ‖²)² 是黎曼度量的倒數，proj_𝔻 把超出邊界的點拉回。Sala et al. (2018) 改用 Lorentz + 高精度 (multi-precision floats) 進一步降低嵌入誤差。

① Poincaré disk model: defined on the open unit ball 𝔻 = {x ∈ ℝ^d : ‖x‖ < 1}, with distance

d(x,y) = acosh(1 + 2 ‖x − y‖² / ((1 − ‖x‖²)(1 − ‖y‖²)))

Distance "stretches" near the boundary — root at origin, leaves near the edge, leaf-to-leaf distance grows exponentially, matching tree hierarchy.

② Lorentz / hyperboloid model: view hyperbolic space as a one-sheeted hyperboloid in ℝ^(d+1) with ⟨x,x⟩_L = −1, where ⟨x,y⟩_L = −x₀y₀ + Σx_iy_i. Distance: d(x,y) = acosh(−⟨x,y⟩_L). More numerically stable — avoids the boundary singularity of Poincaré disk.

③ Riemannian SGD: Standard SGD doesn't work — Euclidean updates push points outside the unit ball. Nickel & Kiela (2017) introduced Riemannian SGD on Poincaré disk:

θ ← proj_𝔻(θ − η · (1 − ‖θ‖²)²/4 · ∇L)

(1 − ‖θ‖²)² is the inverse Riemannian metric; proj_𝔻 clips out-of-disk points. Sala et al. (2018) switched to Lorentz + high-precision floats for lower distortion.

數學性質：在曲率 −1 的雙曲空間，深度為 d 的二元樹可在 2 維中以扭曲 1 + O(1/d) 嵌入（Sarkar 2011）。歐式空間在同樣扭曲下需要 Ω(d) 維。對單細胞 atlas 動輒上百個 cell type 的 ontology——雙曲幾乎是「強烈推薦」。
Theory: In curvature −1 hyperbolic space, a depth-d binary tree embeds with distortion 1 + O(1/d) in just 2 dimensions (Sarkar 2011). Euclidean needs Ω(d) for the same distortion. For cell-type ontologies with hundreds of classes, hyperbolic is strongly preferred.

🌳 什麼時候用雙曲？

情境1:
明確階層資料——cell ontology、phylogeny、Pfam protein family、ICD-10 疾病碼。
情境2:
大量葉節點、淺層中間——TCR repertoire、語意分類等。歐式維度成本太高。
情境3:
跨物種對齊——若兩物種共享 ontology 結構，雙曲嵌入比歐式更容易對齊。
不適合:
無明顯階層、流形是「circle / torus / sphere」這類正曲率時——應該用對應的幾何（spherical embedding、torus embedding）。
Case 1:
Clear hierarchies — cell ontology, phylogeny, Pfam, ICD-10.
Case 2:
Many leaves, shallow middle — TCR repertoires, semantic taxonomies. Euclidean is dimension-expensive.
Case 3:
Cross-species alignment — if two species share an ontology, hyperbolic aligns easier than Euclidean.
Not for:
No hierarchy, or manifold is circle/torus/sphere (positive curvature) — use the matching geometry instead.
生物應用

雙曲嵌入在單細胞與生物資訊

HypMor / Cell ontology

Klimovskaia et al. (2020, Nature Communications) 把 scRNA-seq 細胞型別嵌入 Poincaré disk——根節點為「all cells」、不同階層的 cluster 自然分佈在不同半徑。比 UMAP 更好地保留階層關係。

Klimovskaia et al. (2020, Nature Communications) embed scRNA-seq cell types in a Poincaré disk — root = "all cells", clusters distribute by hierarchy level. Preserves hierarchy better than UMAP.

HypAlign 跨物種對齊

2023 年數個工作（如 Klimovskaia 後續、Skinnider lab）用雙曲嵌入做人–鼠細胞型別對齊：兩物種共享的 cell-type ontology 結構在雙曲空間自然對齊，比歐式 + CCA 更穩。

Several 2023 works (Klimovskaia follow-up, Skinnider lab) align human–mouse cell types in hyperbolic space — shared ontology structure aligns naturally, more robust than Euclidean + CCA.

Pfam / 蛋白質家族階層

Nickel & Kiela (2017, NeurIPS) 原始論文就有 protein family taxonomy 範例——Pfam 的 family / clan / subfamily 三層階層幾乎無扭曲嵌入。後續被用於 protein function prediction。

The original Nickel & Kiela (2017, NeurIPS) paper used Pfam family/clan/subfamily as a benchmark — nearly distortion-free. Later extended to protein function prediction.

Single-cell phylogeny

CellTagging / lineage barcode 加上雙曲嵌入可重建細胞家譜——譜系樹本質是樹，雙曲幾何是最自然選擇。Macaulay et al. (2024) 應用於 zebrafish 發育。

CellTagging / lineage barcodes + hyperbolic embedding reconstruct cell lineage trees — phylogenies are intrinsically trees, hyperbolic is the natural choice. Macaulay et al. (2024) applied this to zebrafish development.

ATAC-seq 染色質階層

TAD (topological associated domains) 與 sub-TAD 構成樹狀階層；雙曲嵌入可用於對齊不同細胞型別的染色質結構。

TADs and sub-TADs form a hierarchical tree — hyperbolic embeddings help align chromatin architectures across cell types.

TCR repertoire

TCR V/D/J 段組合構成隱性樹狀階層；雙曲嵌入比 Levenshtein clustering 更能保留 clonotype 結構，已用於 COVID-19 / 癌症免疫資料。

TCR V/D/J segments form an implicit hierarchical tree — hyperbolic embeddings preserve clonotype structure better than Levenshtein clustering. Applied to COVID-19 and cancer immunology data.

深入解析

雙曲嵌入的進階主題

① Hyperbolic Neural Networks

Ganea et al. (2018, NeurIPS) 把神經網路的線性層、活化函數推廣到雙曲空間：使用 Möbius 加法、scalar mul，並用 exp/log map 在切空間做運算。HyperboloidNN、HGCN (Hyperbolic GCN, Chami 2019) 是後續代表作。

② 與 Lorentz model 的選擇

Poincaré disk 直覺但在邊界 (‖x‖→1) 數值不穩；Lorentz / hyperboloid 數值穩但需投影。實務建議：訓練用 Lorentz、視覺化用 Poincaré（可雙向轉換）。

③ 混合幾何（product space）

Gu et al. (2019) 提出 ℍ × 𝕊 × ℝ product space embedding——資料可能有「樹 + 環 + 平面」多種結構同時存在（如細胞型別樹 + 細胞週期環）。每個因子分別嵌入後 concatenate。

④ 評估指標

  • distortion: Σ |d_embed(i,j) − d_tree(i,j)| / d_tree(i,j)
  • mAP: 對每個節點，其鄰居在嵌入空間中也應該是最近的 → mean average precision
  • hierarchical clustering 一致性: 嵌入空間做 HDBSCAN / k-means 後與 ground truth ontology 比較

⑤ 與 UMAP / t-SNE 的關係

UMAP 假設資料在 manifold 上局部歐式；對階層資料其實是「強拉直」。HyperMap、Hyperbolic UMAP（Klimovskaia 2020, Sala 2024）用雙曲做投影，能在低維保留樹狀關係——對細胞 ontology 視覺化比標準 UMAP 更忠實。

⑥ 數值陷阱

雙曲距離 acosh 在 argument 接近 1 時數值梯度爆炸（因 acosh'(1) = ∞）；常用 clamp(argument, ≥ 1 + ε) 防止。Sala et al. (2018) 用 multi-precision 直接避免。geoopt 套件已內建這些 trick。

① Hyperbolic Neural Networks

Ganea et al. (2018, NeurIPS) generalize linear layers and activations to hyperbolic space via Möbius operations and tangent-space exp/log maps. HyperboloidNN, HGCN (Chami 2019) are notable follow-ups.

② Poincaré disk vs Lorentz

Poincaré is intuitive but numerically unstable near the boundary; Lorentz is stable but needs projection. Rule of thumb: train in Lorentz, visualize in Poincaré (they are inter-convertible).

③ Product-space geometry

Gu et al. (2019) propose ℍ × 𝕊 × ℝ product embeddings — data may have tree + cycle + flat structure simultaneously (e.g. cell-type tree + cell-cycle ring). Each factor embeds separately, then concatenate.

④ Evaluation metrics

  • Distortion: Σ |d_embed − d_tree| / d_tree
  • mAP: for each node, are its neighbors nearest in embedding? → mean average precision
  • Hierarchical clustering consistency: HDBSCAN/k-means on embedding vs ground-truth ontology

⑤ Relation to UMAP / t-SNE

UMAP assumes local Euclidean manifold; on hierarchical data it forces flatness. HyperMap, Hyperbolic UMAP (Klimovskaia 2020, Sala 2024) project into hyperbolic space, preserving tree structure in low dimensions — more faithful than standard UMAP for cell ontology visualization.

⑥ Numerical pitfalls

The acosh gradient blows up as argument → 1 (since acosh'(1) = ∞). Use clamp(argument, ≥ 1 + ε). Sala et al. (2018) use multi-precision to bypass this. The geoopt library bundles these tricks.

潛在改良應用

還能改良哪些任務？

🎯 1. 把 cell ontology 當先驗約束嵌入

目前 scRNA-seq 嵌入（PCA、scVI、UMAP）大多 ignore 已知 cell ontology。提案：把 Cell Ontology (CL) 樹當作 prior，學一個 hyperbolic latent 使「ontology 相近的細胞嵌入相近」，可同時做 cluster + 階層 annotation。

🎯 2. 跨物種 lineage 對齊

用雙曲嵌入做 Tabula Sapiens × Tabula Muris 對齊——共享 ontology 結構（如「免疫細胞 → T cell → CD4+」階層）在雙曲空間可對齊更穩。提案：HypHarmony 或類似的雙曲版 Harmony。

🎯 3. 多解析度細胞圖譜

單一嵌入難以同時看「粗類別」與「細亞型」。雙曲嵌入天然有「半徑 = 解析度」的詮釋——靠原點看到粗結構，靠邊界看到細結構。提案：互動式雙曲細胞圖譜瀏覽器。

🎯 4. TCR/BCR clonotype 動態

免疫 repertoire 隨時間演化——某 clonotype 在某時刻擴增為 clonal family。雙曲嵌入可同時表示家系結構與擴增動態（時間 = 半徑變化）。提案：雙曲時序 TCR 模型，用於疫苗反應 / 癌症免疫追蹤。

🎯 5. Single-cell phylogeny + spatial 整合

結合 lineage barcoding 與空間轉錄體（如 spatial CellTagging）：細胞既有時間階層（譜系）又有空間鄰居關係。雙曲 × 歐式 product space 可同時保留兩者。提案：HypSpatial framework。

🎯 6. Hyperbolic embedding 評估 scLLM 階層理解

scGPT/Geneformer 的 latent 是否「真的」學到細胞型別階層？將其 latent 投影到 hyperbolic 空間，看 ontology 是否在嵌入中保留。提案：scLLM 階層解釋性 benchmark。

🎯 7. 染色質階層的 dynamic 模型

TAD-subTAD 階層在發育過程中動態重組。雙曲嵌入可緊湊表示這種「半徑變化」的階層動態。提案：Hyperbolic Hi-C dynamics，比現有 graph-based 方法更節省維度。

🎯 1. Use cell ontology as a prior

Current scRNA-seq embeddings (PCA, scVI, UMAP) usually ignore known cell ontology. Proposal: learn a hyperbolic latent where the Cell Ontology (CL) tree acts as a prior — ontology-close cells embed close — enabling joint clustering + hierarchical annotation.

🎯 2. Cross-species lineage alignment

Hyperbolic alignment for Tabula Sapiens × Tabula Muris — shared ontology (immune → T cell → CD4+) aligns more stably in hyperbolic space. Proposal: HypHarmony — a hyperbolic version of Harmony.

🎯 3. Multi-resolution cell atlas

A single embedding rarely shows coarse + fine simultaneously. Hyperbolic embedding's "radius = resolution" interpretation handles this naturally — near origin you see coarse, near boundary you see fine. Proposal: interactive hyperbolic atlas browser.

🎯 4. TCR/BCR clonotype dynamics

Immune repertoires evolve over time — clonal families expand at specific moments. Hyperbolic embedding can represent both family structure and expansion dynamics (time = radial change). Proposal: hyperbolic temporal TCR model for vaccine response / cancer immune tracking.

🎯 5. Phylogeny + spatial integration

Combine lineage barcoding with spatial transcriptomics: cells have temporal hierarchy (lineage) and spatial neighbors. Hyperbolic × Euclidean product space preserves both. Proposal: HypSpatial framework.

🎯 6. Evaluate scLLM hierarchy understanding

Do scGPT/Geneformer latents truly learn cell-type hierarchy? Project their latents to hyperbolic space and check ontology preservation. Proposal: scLLM hierarchy interpretability benchmark.

🎯 7. Dynamic chromatin hierarchy

TAD-subTAD hierarchies reorganize during development. Hyperbolic embedding compactly represents radial-dynamic hierarchy. Proposal: Hyperbolic Hi-C dynamics, more dimension-efficient than current graph methods.

實作範例

# pip install geoopt torch
import torch
import geoopt
from geoopt import PoincareBall, ManifoldParameter
from geoopt.optim import RiemannianAdam

ball = PoincareBall(c=1.0)                    # curvature = 1
emb = ManifoldParameter(
    ball.random(n_nodes, 2, std=0.01),     # 2D Poincaré
    manifold=ball)

opt = RiemannianAdam([emb], lr=1e-2)

for step in range(5000):
    # margin-based contrastive loss on tree edges
    pos = ball.dist(emb[u], emb[v])           # tree edges
    neg = ball.dist(emb[u], emb[w])           # random negatives
    loss = torch.nn.functional.relu(pos - neg + 1.0).mean()
    opt.zero_grad(); loss.backward(); opt.step()
import numpy as np

def poincare_dist(x, y, eps=1e-7):
    """x, y : (..., d) inside open unit ball"""
    nx2 = (x*x).sum(-1)
    ny2 = (y*y).sum(-1)
    nxy2 = ((x - y)**2).sum(-1)
    arg = 1 + 2 * nxy2 / ((1 - nx2)*(1 - ny2) + eps)
    return np.arccosh(np.clip(arg, 1+eps, None))

# Lorentz model — more stable
def lorentz_dist(x, y):
    inn = -x[..., 0]*y[..., 0] + (x[..., 1:]*y[..., 1:]).sum(-1)
    return np.arccosh(np.clip(-inn, 1+1e-7, None))
# install.packages("hydra")
library(hydra)

# Hyperbolic Distance Recovery and Approximation
# Input: pairwise distance matrix D (e.g. from a tree)
emb <- hydra(D, dim = 2, curvature = 1)
# emb$r, emb$directional — polar coords in Poincaré disk

# Convert to Cartesian on disk
xy <- emb$r * emb$directional
