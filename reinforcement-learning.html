強化學習於生物應用 (Reinforcement Learning for Bio)

當「答案」要靠互動產生——藥物分子生成、蛋白質設計、CRISPR 靶點選擇、自適應臨床試驗——強化學習給出統一框架：以 reward 引導策略，平衡探索與利用。從 REINVENT 到 RLHF，RL 已是 AI 驅動實驗循環的核心。

When the "answer" must emerge through interaction — molecule generation, protein design, CRISPR target selection, adaptive clinical trials — reinforcement learning provides a unifying framework: a reward-guided policy balancing exploration and exploitation. From REINVENT to RLHF, RL is at the heart of AI-driven experimental loops.

核心概念

MDP、價值、策略

Markov Decision Process (MDP) 由五元組 (S, A, P, R, γ) 定義：

  • S：狀態空間（例：當前分子片段、實驗條件）
  • A：動作空間（例：加上一個 atom、選一個 CRISPR guide）
  • P(s'|s,a)：轉移機率（環境動態）
  • R(s,a)：reward 函數（例：QED + 合成可行性）
  • γ ∈ [0,1)：折扣因子（長遠 vs 短期）

目標：學一個策略 π(a|s) 使期望折扣回報 J(π) = E[Σₜ γᵗ R(sₜ, aₜ)] 最大。

三大流派：

  • 價值法 (Value-based)：學 Q(s,a) ≈ 期望回報；π = argmax_a Q。代表：Q-learning、DQN (Mnih 2015, Nature)
  • 策略法 (Policy-based)：直接參數化 π_θ(a|s)，用 ∇J(θ) 更新。代表：REINFORCE、PPO (Schulman 2017)
  • Actor-Critic：兩者結合——actor (π) + critic (V or Q)。代表：A2C、SAC (Haarnoja 2018)、DDPG

探索 vs 利用 (Exploration vs Exploitation)：RL 的核心張力——是繼續做已知好的（exploit），還是嘗試未知（explore）？常見做法包括 ε-greedy、entropy bonus（SAC 的最大熵思想）、UCB（上信賴界）、Thompson sampling、curiosity-driven exploration。

A Markov Decision Process (MDP) is the tuple (S, A, P, R, γ):

  • S: state space (e.g. current molecular fragment, experimental condition)
  • A: action space (e.g. attach an atom, choose a CRISPR guide)
  • P(s'|s,a): transition probability (environment dynamics)
  • R(s,a): reward function (e.g. QED + synthesizability)
  • γ ∈ [0,1): discount factor (long-term vs near-term)

Goal: learn policy π(a|s) maximising J(π) = E[Σₜ γᵗ R(sₜ, aₜ)].

Three families:

  • Value-based: learn Q(s,a) ≈ expected return; π = argmax_a Q. Examples: Q-learning, DQN (Mnih 2015, Nature)
  • Policy-based: parameterise π_θ(a|s) directly; update via ∇J(θ). Examples: REINFORCE, PPO (Schulman 2017)
  • Actor-Critic: combines both — actor (π) + critic (V or Q). Examples: A2C, SAC (Haarnoja 2018), DDPG

Exploration vs Exploitation: the core RL tension — keep choosing the known-good action (exploit) or try something new (explore)? Standard tricks: ε-greedy, entropy bonus (SAC's max-entropy idea), UCB, Thompson sampling, curiosity-driven exploration.

演算法

REINFORCE、PPO、SAC、AlphaZero

① REINFORCE (Williams 1992) — policy gradient 的最直接形式：

∇_θ J(θ) = E_π [ Σₜ ∇_θ log π_θ(aₜ|sₜ) · Gₜ ]

其中 Gₜ = Σ_{k≥t} γ^(k−t) R_k。直接用 Monte Carlo 估計 return，gradient 高 variance，但 unbiased。加上 baseline b(s) 可降低 variance：用 advantage Aₜ = Gₜ − b(sₜ) 取代 Gₜ。

② PPO (Proximal Policy Optimization, Schulman 2017) — 目前最廣用的 policy-gradient：

L^CLIP(θ) = E [ min(rₜ(θ) Âₜ, clip(rₜ(θ), 1−ε, 1+ε) Âₜ) ]

rₜ(θ) = π_θ(aₜ|sₜ) / π_θ_old(aₜ|sₜ) 為機率比；clip 限制更新幅度避免分布劇烈漂移。穩定、實作簡單，是 RLHF（如 ChatGPT、AlphaFold-Multimer 微調）首選。

③ SAC (Soft Actor-Critic, Haarnoja 2018) — 最大熵 RL：

π* = argmax_π E [ Σₜ R(sₜ, aₜ) + α H(π(·|sₜ)) ]

同時最大化 reward 與策略熵 H；α 為熵權重（自動調整）。連續動作空間表現最佳，特別適合分子 / 蛋白質連續設計變數。

④ AlphaZero / MuZero (Silver 2017, 2020) — model-based + MCTS：學一個 world model（MuZero 不需真實規則），用 Monte Carlo Tree Search 規劃，再用 policy/value network 引導 MCTS。AlphaTensor (Fawzi 2022, Nature) 將此框架用於矩陣乘法演算法發現；類似思想已被探索於合成路徑規劃。

⑤ RLHF (Reinforcement Learning from Human Feedback) — Ouyang 2022 (InstructGPT)：

  1. SFT：在示範資料上監督微調基底模型
  2. Reward model：用人類偏好對 (chosen, rejected) 訓練 reward r_φ
  3. PPO：以 r_φ 為 reward 微調策略，加上 KL penalty 防漂離 SFT

已從 LLM 蔓延到生物：人類專家可標註「哪個 protein 設計看起來合理」、「哪個分子值得合成」，再做 RLHF。Anthropic 的 RLAIF / Constitutional AI 變體也適用。

① REINFORCE (Williams 1992) — vanilla policy gradient:

∇_θ J(θ) = E_π [ Σₜ ∇_θ log π_θ(aₜ|sₜ) · Gₜ ]

Gₜ = Σ_{k≥t} γ^(k−t) R_k. Monte Carlo return is unbiased but high-variance. A baseline b(s) gives advantage Aₜ = Gₜ − b(sₜ), reducing variance.

② PPO (Schulman 2017) — today's most-used policy gradient:

L^CLIP(θ) = E [ min(rₜ(θ) Âₜ, clip(rₜ(θ), 1−ε, 1+ε) Âₜ) ]

rₜ(θ) = π_θ / π_θ_old; clipping prevents large policy shifts. Stable, easy to implement; first choice for RLHF (ChatGPT, AlphaFold-Multimer fine-tuning).

③ SAC (Haarnoja 2018) — max-entropy RL:

π* = argmax_π E [ Σₜ R(sₜ, aₜ) + α H(π(·|sₜ)) ]

Maximises both reward and policy entropy H; α auto-tuned. Best for continuous action spaces — ideal for continuous design variables in molecules / proteins.

④ AlphaZero / MuZero (Silver 2017, 2020) — model-based + MCTS: learn a world model (MuZero needs no real rules), plan with Monte Carlo Tree Search, guided by policy/value networks. AlphaTensor (Fawzi 2022, Nature) applied this to matrix-multiplication algorithm discovery; similar ideas appear in synthesis-route planning.

⑤ RLHF (Reinforcement Learning from Human Feedback) — Ouyang 2022 (InstructGPT):

  1. SFT — supervised fine-tune base model on demonstrations
  2. Reward model — train r_φ on (chosen, rejected) preference pairs
  3. PPO — fine-tune policy using r_φ as reward, with KL penalty vs SFT

Spreading from LLMs to biology: human experts can label "which protein design looks plausible" or "which molecule is worth synthesising", then run RLHF. Anthropic's RLAIF / Constitutional AI variants apply too.

選哪個？ 離散動作（分子片段加減、CRISPR guide 選擇）→ PPO 或 DQN；連續動作（蛋白質扭轉角、實驗劑量）→ SAC；可建模 + 規劃（合成路徑、實驗排程）→ MCTS / MuZero；有人類偏好資料 → RLHF / DPO（Rafailov 2023）。
Which one? Discrete actions (add/remove fragments, pick CRISPR guides) → PPO or DQN; continuous (protein torsion angles, dosage) → SAC; planning-friendly (synthesis routes, experiment scheduling) → MCTS / MuZero; human preferences available → RLHF / DPO (Rafailov 2023).

生物應用

RL 在藥物、蛋白質、CRISPR、臨床

REINVENT — RL 生成分子

Olivecrona et al. (2017, J. Cheminf.) 提出 RNN-prior + REINFORCE 微調，以 QED、predicted activity 為 reward。REINVENT 2 / 3 / 4 (Blaschke 2020; Loeffler 2024, J. Cheminf.) 加入 scaffold constraint、強化學習策略，成為製藥業 de novo design 主力工具。

Olivecrona et al. (2017, J. Cheminf.) — RNN prior + REINFORCE with QED / predicted activity as reward. REINVENT 2/3/4 (Blaschke 2020; Loeffler 2024, J. Cheminf.) added scaffold constraints and richer RL strategies; now a workhorse for industry de novo design.

DeepCDR / drug response

Liu et al. (2020, Bioinformatics) DeepCDR 結合 cell-line omics 與藥物結構預測 IC50；後續 RL 變體 (e.g. MOReL, BO-RL) 把 drug response 模型當 oracle，主動選下一個要測的 (cell, drug) 組合。

Liu et al. (2020, Bioinformatics) — DeepCDR predicts IC50 from cell-line omics + drug structure. Later RL variants use the model as an oracle to actively choose the next (cell, drug) pair to test.

蛋白質設計 (ProteinSGM-RL)

Wang et al. (2022) ProteinSGM 用 score-based 模型生成蛋白質結構；後續加入 RL 微調，把「可摺疊性 (pLDDT)、目標親和力」當 reward。同類工作：RFdiffusion (Watson 2023, Nature) 加 RL guidance、ProteinMPNN-RL。

Wang et al. (2022) ProteinSGM — score-based protein structure generation; later extended with RL fine-tuning using foldability (pLDDT) and target affinity as reward. Related: RFdiffusion (Watson 2023, Nature) with RL guidance, ProteinMPNN-RL.

CRISPR guide 選擇

每個 gene 有數十至數百個可能 guide RNA；要選 cut efficiency 高、off-target 低者。RL agent 把每次 guide 選擇視為動作，experimental readout 為 reward，逐步學偏好。Wang et al. (2023, Cell Reports Methods) 將此用於高通量 CRISPR 篩選優化。

Each gene has tens–hundreds of candidate guide RNAs; must pick high-efficiency, low-off-target ones. RL treats each guide pick as an action, experimental readout as reward. Wang et al. (2023, Cell Reports Methods) used this for high-throughput CRISPR screen optimisation.

自適應臨床試驗

傳統 RCT 固定分配；adaptive trial 依累積結果動態調整 arm 比例。RL（特別是 contextual bandits + Thompson sampling）可在 ethical/regulatory 限制下持續探索。Yauney & Shah (2018) 把化療劑量選擇建模為 RL；I-SPY 系列乳癌試驗已實際採用 Bayesian adaptive 策略。

Classical RCTs fix allocation; adaptive trials adjust arm ratios using accumulating data. RL (especially contextual bandits + Thompson sampling) explores within ethical/regulatory limits. Yauney & Shah (2018) modelled chemo dosing as RL; the I-SPY breast-cancer trial series uses Bayesian adaptive allocation.

順序 CRISPR perturbation

Perturb-seq 一次扰動一基因；要選下一個最有 information gain 的擾動，可用 RL + active learning。King-Smith et al. (2024) 框架把每次擾動視為 RL 動作，以「下游表型 entropy」為 reward。

Perturb-seq screens one gene at a time; the next "most informative" perturbation can be chosen via RL + active learning. King-Smith et al. (2024) modelled each perturbation as an RL action with "downstream phenotype entropy" reward.

AlphaFold-Multimer RL 微調

Evans et al. (2022) AlphaFold-Multimer 推理時可加 RL 風格的 refinement，以 interface score 為 reward 微調預測；後續多體蛋白質設計 (Watson 2023, RFdiffusion) 採類似思路 — 用結構 oracle 當 reward 引導生成模型。

Evans et al. (2022) AlphaFold-Multimer can be refined at inference with RL-style updates using interface score as reward; later multimer design works (Watson 2023, RFdiffusion) use the structural oracle similarly to guide generation.

RLHF / RLAIF for bio-LLM

ProtST、ESM-3、scGPT 等生物大模型可用 RLHF 微調：人類專家比較「合理 vs 不合理」 protein / cell embedding。Anthropic-style RLAIF（用 LLM 評分代替人類）也適用，可大幅降低標註成本。

Bio LLMs like ProtST, ESM-3, scGPT can be RLHF-tuned: experts compare "plausible vs implausible" protein / cell embeddings. Anthropic-style RLAIF (LLM-as-judge) cuts annotation cost dramatically.

深入解析

進階主題：reward hacking、offline RL、安全

① Reward hacking 與 Goodhart 失靈

當 reward 是「真實目標的代理」(QED 代理 oral bioavailability、pLDDT 代理 foldability)，agent 會學會「鑽 reward 漏洞」生成 QED 高但毫無意義的分子。對策：multi-objective reward、constraint penalty、KL regularisation 對齊先驗策略（RLHF 的核心 trick）。Skalse et al. (2022, NeurIPS) 給出 reward hacking 的形式化定義。

② Offline RL — 從歷史資料學策略

生物實驗無法 online 互動；只有過去的 (s, a, r, s') log。Offline RL（CQL Kumar 2020、IQL Kostrikov 2021）解決「分布外動作高估」問題。應用：歷史 EHR 學最佳劑量；歷史化學文獻學合成策略。

③ Inverse RL / Imitation Learning

從專家示範反推 reward（Ng & Russell 2000）或直接克隆策略（Behavioral Cloning）。當你「不知道怎麼設計 reward 但有專家樣本」時的選擇。GAIL (Ho 2016) 用 GAN 思想做 imitation。

④ Multi-objective RL

藥物設計同時要 potency、selectivity、ADMET、合成可行 — 多 reward。常見：scalarisation (加權和、Chebyshev)、Pareto front methods、constrained MDP。REINVENT 4 支援多目標 reward 組合。

⑤ Sim-to-real / 模擬器忠實度

分子 RL 常依賴 docking、MD、QM 模擬器當 oracle；模擬與實驗的 gap 可能讓 RL 學到「擊敗模擬器」而非「擊敗實驗」。對策：domain randomisation、頻繁加入濕實驗驗證、unc​ertainty-aware reward。

⑥ 安全與不可逆動作

臨床 RL 不像遊戲——錯誤決策可能傷害病人。Safe RL（Achiam 2017 CPO、Garcia 2015 review）強制 constraint 滿足；contextual bandit + conservative exploration 是實際首選。Anthropic 的 Constitutional AI 思想也適用：用顯式規則限制 policy。

⑦ DPO — RLHF 的替代

Rafailov et al. (2023, NeurIPS) Direct Preference Optimization：直接從偏好對導出 closed-form loss，跳過 reward model + PPO，訓練更穩、計算更省。2024–25 年生物 LLM 微調很快採用 DPO / IPO / KTO。

① Reward hacking and Goodhart failure

When reward is a proxy for the true goal (QED for oral bioavailability, pLDDT for foldability), the agent learns to exploit the proxy — high QED but nonsense molecules. Fixes: multi-objective reward, constraint penalties, KL regularisation against a prior policy (RLHF's core trick). Skalse et al. (2022, NeurIPS) formalise reward hacking.

② Offline RL — learn from logged data

Biological experiments rarely allow online interaction; you only have past (s, a, r, s') logs. Offline RL (CQL — Kumar 2020; IQL — Kostrikov 2021) addresses over-estimation on out-of-distribution actions. Applications: EHR dose-optimisation, synthesis strategy from literature.

③ Inverse RL / Imitation Learning

Recover reward from expert demos (Ng & Russell 2000) or just clone the policy (Behavioral Cloning). Use when reward is hard to design but expert traces exist. GAIL (Ho 2016) uses GAN-style imitation.

④ Multi-objective RL

Drug design needs potency + selectivity + ADMET + synthesizability — multiple rewards. Standard tools: scalarisation (weighted, Chebyshev), Pareto-front methods, constrained MDPs. REINVENT 4 supports multi-objective combinations.

⑤ Sim-to-real / simulator fidelity

Molecular RL often uses docking / MD / QM as oracles; the sim–experiment gap can let RL beat the simulator without beating the bench. Counter-measures: domain randomisation, frequent wet-lab validation, uncertainty-aware reward.

⑥ Safety and irreversible actions

Clinical RL isn't a game — wrong decisions can hurt patients. Safe RL (Achiam 2017 CPO, Garcia 2015 review) enforces constraints; contextual bandit + conservative exploration is the practical default. Anthropic-style Constitutional AI also fits: hard rules constrain the policy.

⑦ DPO — the RLHF alternative

Rafailov et al. (2023, NeurIPS) — Direct Preference Optimization derives a closed-form loss directly from preference pairs, skipping reward model + PPO. More stable, cheaper. 2024–25 bio-LLM fine-tuning has rapidly adopted DPO / IPO / KTO.

class DrugDesignEnv(gym.Env):

        done = len(self.state) >= 30

# Minimal REINFORCE with baseline (pure PyTorch)
import torch, torch.nn as nn
from torch.distributions import Categorical

policy = nn.Sequential(nn.Linear(obs_dim, 64), nn.ReLU(), nn.Linear(64, n_act))
value  = nn.Sequential(nn.Linear(obs_dim, 64), nn.ReLU(), nn.Linear(64, 1))
opt = torch.optim.Adam(list(policy.parameters()) + list(value.parameters()), lr=1e-3)

for epi in range(2000):
    s, log_probs, rewards, vals = env.reset(), [], [], []
    while True:
        logits = policy(torch.tensor(s, dtype=torch.float32))
        dist = Categorical(logits=logits); a = dist.sample()
        log_probs.append(dist.log_prob(a))
        vals.append(value(torch.tensor(s, dtype=torch.float32)))
        s, r, done, *_ = env.step(a.item()); rewards.append(r)
        if done: break
    R, returns = 0, []
    for r in rewards[::-1]:
        R = r + 0.99*R; returns.insert(0, R)
    returns = torch.tensor(returns, dtype=torch.float32)
    adv = returns - torch.cat(vals).squeeze()
    loss = -(torch.stack(log_probs) * adv.detach()).mean() + 0.5*(adv**2).mean()
    opt.zero_grad(); loss.backward(); opt.step()
# Sketch of REINVENT-style RL fine-tune of a SMILES RNN
import torch
from reinvent_models import ReinventModel  # pseudo-import

prior = ReinventModel.load("prior.ckpt"); prior.eval()    # frozen prior
agent = ReinventModel.load("prior.ckpt")              # to be fine-tuned
opt = torch.optim.Adam(agent.parameters(), lr=5e-4)

def score(smiles):
    return qed(smiles) + 0.5*active_predictor(smiles)

for step in range(500):
    smis, agent_lp = agent.sample(batch=64)
    with torch.no_grad(): prior_lp = prior.log_prob(smis)
    R = torch.tensor([score(s) for s in smis])
    # Augmented likelihood: KL-regularised reward
    augmented = prior_lp + 60*R
    loss = ((augmented - agent_lp)**2).mean()
    opt.zero_grad(); loss.backward(); opt.step()
