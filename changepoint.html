變點偵測 (Change-point Detection)

時間序列裡突然「均值跳一階」、「方差爆掉」或「整個分布變了」的位置在哪？從基因組 CNV breakpoint、ChIP-seq 區域邊界、藥物治療 onset，到 ICU 病程斷點——變點偵測是把連續觀測切成不同 regime 的核心工具。

Where does a time series suddenly shift in mean, variance, or distribution? From CNV breakpoints to ChIP-seq domain boundaries, drug-onset detection, and ICU disease-stage shifts — change-point detection is the core tool for slicing continuous data into distinct regimes.

核心概念

什麼是變點？

定義：給定時間序列 y₁, y₂, …, y_T，變點 τ 是滿足「y_{1:τ} 的分布 ≠ y_{τ+1:T} 的分布」的位置。最常見的差異類型：

  • 均值變點 (mean shift): μ₁ → μ₂，方差不變——例如基因組 segment 的 log2 ratio 從 0 跳到 +0.5。
  • 方差變點 (variance shift): σ₁² → σ₂²，均值不變——例如治療後生理訊號雜訊驟增。
  • 分布變點 (distribution shift): 整個分布族都換——非參數方法（如 E-Divisive）擅長處理。
  • 趨勢變點 (trend break): 斜率變化——疾病進程從緩慢到急劇惡化。

Offline vs Online：

  • Offline (batch): 拿到完整序列後找全部變點——典型如 CNV 分析、回溯研究。
  • Online (streaming): 邊接收邊偵測，盡早 alarm——典型如 ICU 監護、wearable 即時警報。

Single vs Multiple：單變點是經典統計（Page 1954, CUSUM），多變點則是 NP-hard 在無剪枝下，需要 binary segmentation、PELT、dynamic programming 等技巧。

Definition: Given a series y₁, y₂, …, y_T, a change-point τ is a location where the distribution of y_{1:τ} differs from y_{τ+1:T}. Common types:

  • Mean shift: μ₁ → μ₂ with constant variance — e.g. genomic log2-ratio jumping from 0 to +0.5.
  • Variance shift: σ₁² → σ₂² with constant mean — e.g. physiological signal noise jumping post-treatment.
  • Distribution shift: the whole distribution family changes — non-parametric methods (E-Divisive) handle this.
  • Trend break: slope changes — slow → steep disease progression.

Offline vs Online:

  • Offline (batch): Find all change-points after observing the full series — CNV analysis, retrospective studies.
  • Online (streaming): Detect as data arrive, with low delay — ICU monitoring, wearable alarms.

Single vs Multiple: Single change-point is classic (Page 1954, CUSUM). Multiple change-points are NP-hard without pruning — needs binary segmentation, PELT, or dynamic programming.

演算法

主流變點偵測演算法

① CUSUM (Page 1954, Biometrika)：累積和檢定。對均值 shift，計算 S_t = max(0, S_{t-1} + (y_t − μ₀ − k))，當 S_t 超過 threshold h 時宣告變點。Online 友善，但只擅長 single change-point。

② Binary Segmentation (Scott & Knott 1974)：先找全局單一最佳變點 τ*，再對左右兩段遞迴找下一個變點，直到無顯著改善。複雜度 O(T log T)，但貪婪——可能漏掉成對近距變點。

③ PELT (Killick 2012, JASA)：Pruned Exact Linear Time。用 dynamic programming 找全局最佳分段（penalty + cost）：

F(s) = min_{τ < s} { F(τ) + C(y_{τ+1:s}) + β }

關鍵剪枝定理：若 F(τ) + C(y_{τ+1:s}) ≥ F(s) 則 τ 永遠不會是後續最佳起點 → 可剪掉。複雜度幾乎線性（與變點數成比例），是 offline 主流。

④ BOCPD (Adams & MacKay 2007)：Bayesian Online Change-point Detection。每步更新「run length r_t」的後驗——r_t 表「自上次變點以來的時間」。Hazard function H(r) 決定變點機率，後驗以遞迴更新：

P(r_t | y_{1:t}) ∝ Σ_{r_{t-1}} P(r_t | r_{t-1}) P(y_t | r_{t-1}, y_{1:t-1}) P(r_{t-1} | y_{1:t-1})

輸出是每時刻變點的機率分布，而非 hard alarm。可整合任意 conjugate likelihood（Gaussian、Poisson、Bernoulli）。

⑤ E-Divisive (Matteson & James 2014, JASA)：非參數，不假設任何分布。用能量距離 (energy distance) 衡量「左半段 vs 右半段」差異，遞迴找最大分隔點。對非高斯、heavy-tailed、多模態資料尤其穩健。

⑥ NEUROC / 神經網路法：近年用 transformer/CNN 從 raw 訊號直接 regress 變點位置（如 TIRE, KL-CPD, NEUROC）。優勢：可學複雜模式（如同時 mean + variance shift）；缺點：需要訓練資料。

① CUSUM (Page 1954, Biometrika): Cumulative sum test. For mean shift, S_t = max(0, S_{t-1} + (y_t − μ₀ − k)); alarm when S_t exceeds h. Online-friendly but only suited to single change-point.

② Binary Segmentation (Scott & Knott 1974): Find the global single best τ*, then recurse on left/right segments. Complexity O(T log T), but greedy — may miss pairs of close change-points.

③ PELT (Killick 2012, JASA): Pruned Exact Linear Time. Dynamic programming for optimal segmentation (penalty + cost):

F(s) = min_{τ < s} { F(τ) + C(y_{τ+1:s}) + β }

Key pruning rule: if F(τ) + C(y_{τ+1:s}) ≥ F(s), then τ can never be the future best origin → pruned. Near-linear complexity (with change-point count); the offline workhorse.

④ BOCPD (Adams & MacKay 2007): Bayesian Online Change-point Detection. Update posterior over "run length r_t" — time since last change-point. Hazard H(r) controls change-point probability, with recursive update:

P(r_t | y_{1:t}) ∝ Σ_{r_{t-1}} P(r_t | r_{t-1}) P(y_t | r_{t-1}, y_{1:t-1}) P(r_{t-1} | y_{1:t-1})

Output is a distribution over change-points at each step, not a hard alarm. Works with any conjugate likelihood (Gaussian, Poisson, Bernoulli).

⑤ E-Divisive (Matteson & James 2014, JASA): Non-parametric, distribution-free. Uses energy distance between left and right halves, recursively finding maximum separation. Robust to non-Gaussian, heavy-tailed, multi-modal data.

⑥ NEUROC / Neural approaches: Recent transformer/CNN models regress change-points from raw signals (TIRE, KL-CPD, NEUROC). Pros: learns complex patterns (joint mean + variance shifts). Cons: needs training data.

實務選擇指南：已知變點數量少 + Gaussian → PELT；streaming + 要機率不確定性 → BOCPD；分布未知 / 多模態 → E-Divisive；有大量標註資料 + 複雜 pattern → 神經網路法。切記： penalty β 是 PELT 最敏感的超參——太小會 over-segment、太大會錯過。常用 BIC: β = 2 log T。
Choice guide: Few changes + Gaussian → PELT; streaming + uncertainty needed → BOCPD; unknown distribution / multimodal → E-Divisive; large labeled data + complex patterns → neural methods. Note: the PELT penalty β is the most sensitive hyperparameter — too small over-segments, too large misses changes. Standard BIC: β = 2 log T.

🌳 什麼時候用變點偵測？

情境1:
基因組 segmentation——array-CGH / WGS 找 CNV breakpoints；用 PELT 或 CBS。
情境2:
ICU / wearable 即時警報——BOCPD 提供機率不確定性，避免 hard threshold 抖動。
情境3:
分布未知 / 非高斯——E-Divisive 不需任何分布假設。
不適合:
趨勢平緩的漸變（非「跳變」）——用 trend filtering、structural break test 更合適。
Case 1:
Genome segmentation — array-CGH / WGS CNV breakpoints; use PELT or CBS.
Case 2:
ICU / wearable alarms — BOCPD's posterior probabilities avoid hard-threshold flicker.
Case 3:
Unknown / non-Gaussian distributions — E-Divisive is distribution-free.
Not for:
Smooth gradual drift (not a "jump") — trend filtering or structural break tests fit better.
生物應用

變點偵測在生物與醫學

CNV breakpoint

Olshen et al. (2004, Biostatistics) 的 CBS (Circular Binary Segmentation) 是 array-CGH / WGS 拷貝數變異 (CNV) 偵測的金標準——把基因組 log2 ratio 切成 segment，每 segment 對應一個 copy state。已嵌入 DNAcopy R 套件、Sequenza、PennCNV。

Olshen et al. (2004, Biostatistics) CBS (Circular Binary Segmentation) is the gold standard for array-CGH / WGS copy-number variant detection — splits genome log2-ratio into segments, each a copy state. Built into DNAcopy R, Sequenza, PennCNV.

ChIP-seq broad domain

H3K27me3、H3K9me3 等 broad domain 沒有 narrow peak 結構，需要變點偵測切「啟動子–沉默區」邊界。SICER、RSEG、ChIP-seqcompare 都內建變點偵測模組。

Broad domains like H3K27me3 lack narrow-peak structure — change-point detection slices "promoter–silenced region" boundaries. SICER, RSEG, ChIP-seqcompare embed change-point modules.

DNase-seq peak boundary

DNase hypersensitive site 邊界是 chromatin accessibility 從「閉合 → 開放」的變點。F-Seq、Hotspot 等工具用變點+核密度估計定 boundary，比固定 window 更準。

DNase hypersensitive site boundaries are accessibility "closed → open" change-points. F-Seq, Hotspot combine change-point + kernel density for boundary calling — more accurate than fixed windows.

疾病進程斷點

阿茲海默症、帕金森氏症的 cognitive decline 不是連續的——常有「stable → 急遽惡化」的轉折點。Bayesian change-point models (Younes 2014) 從 longitudinal MMSE / lab values 自動找這些轉折，輔助早期介入。

Alzheimer's / Parkinson's cognitive decline isn't smooth — there's often a "stable → rapid" turning point. Bayesian change-point models (Younes 2014) detect these from longitudinal MMSE / labs, aiding early intervention.

ECG arrhythmia onset

心房顫動 (AFib) 啟動瞬間是 RR interval 變異度的變點。BOCPD 配 Gaussian-Inverse-Gamma likelihood 可實時偵測，已被 Apple Watch / KardiaMobile 類消費醫材採用變種。

Atrial fibrillation onset is a change-point in RR-interval variability. BOCPD with Gaussian-Inverse-Gamma likelihood enables real-time detection — adopted in variants by Apple Watch / KardiaMobile consumer devices.

microbiome longitudinal

抗生素治療後腸道菌群「dysbiosis → recovery」是分布變點。Mukherjee et al. (2018, Nat Methods) 用 multivariate BOCPD 找菌群組成突變點，預測 C. difficile 感染復發。

Post-antibiotic gut microbiome "dysbiosis → recovery" is a distribution change-point. Mukherjee et al. (2018, Nat Methods) use multivariate BOCPD on community composition to predict C. difficile recurrence.

藥物 onset detection

CGM (連續血糖) 中胰島素注射的 onset、HRV 中 β-blocker 起效時間——用 BOCPD 自動標記反應起點，可量化 PK/PD 個體差異。

CGM insulin onset, HRV β-blocker onset — BOCPD auto-labels response start, quantifying PK/PD individual variability.

細胞狀態轉變

scRNA pseudotime 沿 trajectory 上的 branching point 本質是分布變點。Saelens et al. (2019, Nat Biotechnol) 的 dynverse benchmark 顯示變點偵測法（如 dyno-cpoint）在某些 dataset 上比 Monocle、Slingshot 更穩。

Branching points along scRNA pseudotime are distribution change-points. Saelens et al. (2019, Nat Biotechnol) dynverse benchmark shows change-point methods (e.g. dyno-cpoint) can beat Monocle / Slingshot on some datasets.

Switching dynamics scRNA

scVelo dynamics 中 gene "induction → repression" 的 switching time 是內部變點。最近 (2024) DeepCpD-Velo 把變點偵測整合進 RNA velocity 框架，提升 fate decision 時間預測。

In scVelo dynamics, "induction → repression" switching time is an internal change-point. DeepCpD-Velo (2024) integrates change-point detection into RNA velocity for better fate-decision timing.

深入解析

變點偵測的進階主題

① Penalty 選擇：BIC、AIC、SIC、改良型

PELT 的 β 直接影響變點數量。常用：BIC β = 2 log T（保守）、AIC β = 2（激進）、SIC β = log T（介於兩者）。Killick et al. (2012) 推薦 BIC；Haynes et al. (2017) 提出 CROPS 一次掃描多個 β，得到完整 Pareto front。

② Hazard function 與先驗

BOCPD 的 hazard H(r) 編碼「我認為變點多久發生一次」。常數 hazard = 幾何分布的 run length。對 ECG 等高頻訊號 H 可能要設小（< 0.001）；對 longitudinal microbiome 可能設大（>0.1）。實務：對 hazard 加 hyperprior 並 marginal-likelihood 學它。

③ Multivariate change-point

多變量序列同時找變點的方法：(a) 分別跑 univariate 再 aggregate；(b) Cho & Fryzlewicz (2015) 的 Sparsified Binary Segmentation；(c) Wang & Samworth (2018) 的 inspect 演算法用 sparse projection；(d) Multivariate BOCPD 用 multivariate-normal Wishart conjugate。微生物群、scRNA latent 等都屬於這類問題。

④ 與 HMM / state-space 的關係

HMM (Hidden Markov Model) 是「軟版本」變點偵測——它允許狀態之間反覆切換。當 HMM transition 機率極小（self-transition ≈ 1），HMM 退化成 change-point。對「regimes 確實罕見切換」的問題，change-point 更高效；對「快速 switching」的問題（如 ion channel state），HMM 更合適。

⑤ False discovery 控制

多重檢定問題嚴重——T 個時刻每個都可能是變點。可用 Bonferroni（太保守）、BH FDR、或 Frick et al. (2014) 的 SMUCE (Simultaneous Multiscale Change-point Estimator) 給出 confidence band，控制 family-wise error。

⑥ Latent / functional / graph change-points

不只是 scalar 序列：(a) Functional CPD——序列每點是一條曲線（如連續 spectrum），用 Hilbert-space distance；(b) Graph CPD——序列每點是一張圖（如 dynamic brain connectome），用 graph kernel / Laplacian spectrum；(c) Tensor CPD——序列每點是高維 tensor，用 low-rank approximation 之差。

⑦ Online 中的 latency 與 false alarm trade-off

Online CPD 永遠在「delay」與「false alarm」間權衡。CUSUM 的 ARL₀ (in-control average run length) 與 ARL₁ (out-of-control average run length) 對應這個 trade-off：threshold h 高 → 少 false alarm 但偵測慢，h 低反之。Lorden (1971) 的最佳性結果：CUSUM 在 worst-case detection delay 意義下對 known shift 是最佳的。

⑧ 神經網路方法的興起

(a) KL-CPD (Chang 2019, ICLR)——用 deep kernel 比較窗口內外的 KL divergence；(b) TIRE (De Ryck 2021)——time-invariant representation embedding；(c) NEUROC——end-to-end transformer，但需要充分標註資料。優勢在於可處理「mean + variance + 高階矩」同時改變的複雜情境。

① Penalty choice: BIC, AIC, SIC, refinements

PELT's β directly controls change-point count. Common: BIC β = 2 log T (conservative), AIC β = 2 (aggressive), SIC β = log T (intermediate). Killick et al. (2012) recommend BIC; Haynes et al. (2017) propose CROPS to scan β values for a full Pareto front.

② Hazard function and prior

BOCPD's hazard H(r) encodes "how often changes happen". Constant hazard = geometric run-length prior. For ECG-like high-frequency signals H may be tiny (< 0.001); for longitudinal microbiome larger (> 0.1). Practice: place a hyperprior on hazard and learn it by marginal likelihood.

③ Multivariate change-point

Methods for joint change-points across multivariate series: (a) per-variable univariate aggregation; (b) Cho & Fryzlewicz (2015) Sparsified Binary Segmentation; (c) Wang & Samworth (2018) inspect with sparse projection; (d) Multivariate BOCPD with Normal-Wishart conjugate. Useful for microbiome composition, scRNA latent series.

④ Relation to HMM / state-space

HMM is a "soft" change-point detector — states may switch back and forth. When self-transition ≈ 1, HMM degenerates to change-point. For rare regime switches, change-point is more efficient; for fast switching (e.g. ion-channel states), HMM is better.

⑤ False discovery control

Multiple-testing is severe — every time point is a candidate. Use Bonferroni (too conservative), BH FDR, or Frick et al. (2014) SMUCE (Simultaneous Multiscale Change-point Estimator) providing confidence bands with family-wise error control.

⑥ Latent / functional / graph change-points

Beyond scalar series: (a) Functional CPD — each time point is a curve (e.g. spectrum); use Hilbert-space distance; (b) Graph CPD — each point is a graph (dynamic brain connectome); use graph kernels / Laplacian spectra; (c) Tensor CPD — high-dim tensors; use low-rank approximation differences.

⑦ Latency vs false alarm in online CPD

Online CPD always trades delay against false alarms. CUSUM's ARL₀ (in-control) and ARL₁ (out-of-control) correspond to this trade-off: high threshold h → few false alarms but slow detection. Lorden (1971) proves CUSUM is optimal in worst-case detection delay for known shifts.

⑧ Rise of neural methods

(a) KL-CPD (Chang 2019, ICLR) — deep kernel comparing in/out-window KL; (b) TIRE (De Ryck 2021) — time-invariant representation embedding; (c) NEUROC — end-to-end transformer, needing labeled data. Strengths: handle joint mean + variance + higher-moment changes.

潛在改良應用

還能改良哪些任務？

🎯 1. scRNA-seq trajectory branching auto-detect

目前 Monocle、Slingshot 需手動指定 branch 結構或用 graph-based 啟發式。提案：把 pseudotime 上的 cell 視為時間序列，multivariate BOCPD 自動定 branching point 與 lineage transition。

🎯 2. WGS 結構變異 (SV) breakpoint refinement

SV caller (Manta, Delly) 給出粗略位置但 base-pair 精度不足。提案：在候選區用 ultra-local PELT + read depth + split-read 訊號，精修到 base-pair 級別。

🎯 3. 連續藥物代謝 PK/PD 個體差異

固定劑量下，不同患者的「drug onset / peak / elimination」時間點差異大。提案：對個體血藥濃度時間序列做 BOCPD，自動標 onset / peak / clearance，量化個體 PK/PD 異質性。

🎯 4. ICU 多模態變點融合預警

ICU 監護有 HR、BP、SpO₂、temperature、lab values 等多模態。提案：multivariate BOCPD 把這些訊號融合，比單通道警報降低 false alarm（已知 ICU alarm fatigue 是大問題）。

🎯 5. scRNA + ATAC + spatial 跨模態變點對齊

同一細胞群在不同 modality 中變化發生的時間點可能不同（如 chromatin 開放 → 表現上升有 lag）。提案：跨模態 lag-aware change-point detection 揭示分子事件因果順序。

🎯 6. Long-COVID 病程斷點刻畫

Long-COVID 患者症狀軌跡複雜——可能有「acute → recovery → relapse」多個 phase。提案：對 wearable + EHR longitudinal data 做 multimodal BOCPD，自動 stratify 病程亞型。

🎯 7. AlphaFold ensemble 動態解析

AlphaFlow 等模型生成的 conformational ensemble 沿 time/sample 軸是時間序列。提案：變點偵測找「conformer cluster 切換」點，揭示 allostery / induced fit 機制。

🎯 8. 神經元 spike train 模式變化偵測

大腦 ensemble recording 中神經元 firing pattern 可能因 task switch / arousal 突變。提案：graph CPD 於動態 functional connectivity，追蹤 cognitive state transitions。

🎯 9. CRISPR screen 響應動力學

時序 CRISPR-LOF 篩選看 gene KO 後 phenotype 的時間動態。提案：對每個 perturbation 的 readout 時序做 CPD，把 gene 分類為 "early effect" vs "late effect"——比 endpoint 分析資訊更多。

🎯 1. scRNA-seq trajectory branching auto-detection

Monocle / Slingshot need manual branch specification or graph heuristics. Proposal: view cells along pseudotime as a series; multivariate BOCPD auto-locates branching points and lineage transitions.

🎯 2. WGS structural variant breakpoint refinement

SV callers (Manta, Delly) give coarse positions but lack bp precision. Proposal: ultra-local PELT in candidate regions combining read depth + split-read signal for bp-level refinement.

🎯 3. PK/PD individual variability detection

At fixed dose, "onset / peak / elimination" timings differ greatly across patients. Proposal: BOCPD on individual concentration series auto-labels these and quantifies PK/PD heterogeneity.

🎯 4. ICU multimodal change-point fusion alarm

ICU monitoring spans HR, BP, SpO₂, temperature, labs. Proposal: multivariate BOCPD fuses these — fewer false alarms than single-channel (alarm fatigue is a known ICU problem).

🎯 5. Cross-modal scRNA + ATAC + spatial change-point alignment

The same cell population may show timing differences across modalities (chromatin opens → expression rises with lag). Proposal: lag-aware cross-modal CPD reveals molecular event causal ordering.

🎯 6. Long-COVID disease-stage characterization

Long-COVID trajectories show complex acute → recovery → relapse phases. Proposal: multimodal BOCPD on wearable + EHR longitudinal data auto-stratifies disease-course subtypes.

🎯 7. AlphaFold ensemble dynamics

AlphaFlow's conformer ensembles along time/sample axes are series. Proposal: CPD finds "conformer cluster switches", revealing allostery / induced-fit mechanisms.

🎯 8. Neuronal spike-train pattern change detection

Ensemble recordings may show sudden firing-pattern shifts at task switches / arousal. Proposal: graph CPD on dynamic functional connectivity tracks cognitive state transitions.

🎯 9. CRISPR screen response dynamics

Time-resolved CRISPR-LOF screens show post-KO phenotype dynamics. Proposal: CPD on each perturbation's readout classifies genes as "early effect" vs "late effect" — more informative than endpoint analysis.
