Modern Hopfield Networks 現代霍普菲爾德網路

Hopfield (1982) 的聯想式記憶網路在 2020 年迎來復興：Ramsauer et al. (2021, ICLR) 證明指數能量函數的新版 Hopfield 與 Transformer attention 數學上等價，且具有指數級記憶容量。一夜之間，Hopfield 不再只是歷史，而是 attention 的「能量視角」。

Hopfield's (1982) associative memory network was resurrected in 2020: Ramsauer et al. (2021, ICLR) proved that the new exponential-energy Hopfield is mathematically equivalent to Transformer attention, with exponential memory capacity. Overnight, Hopfield went from history to "the energy view of attention".

核心概念

從經典 Hopfield 到 Modern Hopfield

經典 Hopfield (Hopfield 1982, PNAS)：給定一組記憶 {x¹, ..., x^N}（二元 ±1 向量），權重矩陣 W = Σ_i x^i (x^i)^T（Hebbian）。能量函數為

E(ξ) = −½ ξ^T W ξ

輸入「噪音樣本」ξ_0，反覆做 ξ ← sign(W ξ)，能量單調下降，最終收斂到最近的記憶。容量上限約為 0.14 N——只能儲存少於 14% 神經元數的 pattern。

Modern Hopfield (Krotov & Hopfield 2016, NeurIPS; Demircigil 2017)：把能量改為「polynomial 或 exponential interaction function」，記憶容量隨之提升。Demircigil et al. (2017) 用指數能量達成 指數級容量 ~2^(d/2)。

Continuous Modern Hopfield (Ramsauer et al. 2021, ICLR)：把記憶 X = [x¹ ... x^N] 收進矩陣，連續查詢向量 ξ，新能量函數

E(ξ) = −lse(β, X^T ξ) + ½ ξ^T ξ + const

其中 lse = log-sum-exp。更新規則為：

ξ_new = X · softmax(β · X^T · ξ)

這正是 Transformer 的 attention！ X 是 keys/values、ξ 是 query、β 是溫度倒數（在 Transformer 中是 1/√d_k）。一次更新就近似收斂到最近記憶。

Classical Hopfield (Hopfield 1982, PNAS): given memories {x¹, ..., x^N} (binary ±1), weight matrix W = Σ_i x^i (x^i)^T (Hebbian rule). Energy:

E(ξ) = −½ ξ^T W ξ

From a noisy input ξ_0, iterate ξ ← sign(W ξ); energy decreases monotonically, converging to the nearest memory. Capacity is only ~0.14 N — fewer than 14% of neuron count can be stored.

Modern Hopfield (Krotov & Hopfield 2016, NeurIPS; Demircigil 2017): replace the quadratic energy with polynomial / exponential interactions to increase capacity. Demircigil et al. (2017) used exponential energy to achieve exponential capacity ~2^(d/2).

Continuous Modern Hopfield (Ramsauer et al. 2021, ICLR): stack memories into X = [x¹ ... x^N], use continuous query ξ, with new energy

E(ξ) = −lse(β, X^T ξ) + ½ ξ^T ξ + const

where lse = log-sum-exp. Update rule:

ξ_new = X · softmax(β · X^T · ξ)

This is exactly the Transformer attention! X plays the role of keys/values, ξ is the query, and β is the inverse temperature (= 1/√d_k in Transformers). A single update converges close to the nearest memory.

💾

指數容量

經典 Hopfield 容量 ≈ 0.14 N；Modern 版可儲存 2^(d/2) 個 pattern，遠超神經元數。

Classical capacity ≈ 0.14 N; Modern achieves 2^(d/2) patterns — far beyond neuron count.

🔗

等於 Attention

更新規則 ξ ← X·softmax(βX^Tξ) 正是 Transformer 的 self-attention 公式。

Update rule ξ ← X·softmax(βX^Tξ) is literally Transformer self-attention.

⚡

一步收斂

β 大時，一次 update 就完成檢索；不像經典 Hopfield 需要多輪迭代。

For large β, retrieval converges in a single update — no multi-round iteration like classical Hopfield.

演算法

Continuous Modern Hopfield 與 Hopfield Layers

① 能量函數的設計（Ramsauer 2021）：新能量寫成

E(ξ) = −β⁻¹ log(Σ_i exp(β x^i · ξ)) + ½‖ξ‖² + const

第一項 lse 對「最相似的記憶」取軟最大；第二項是 self-energy 防止 ξ 無限增大。梯度為 ∇E = ξ − X·softmax(β X^T ξ)，令其為零就得到 fixed-point ξ = X·softmax(β X^T ξ)。

② 三種 Hopfield 操作模式（hflayers）：

  • Hopfield（標準）: Q from input, K/V from learnable memory matrix → 同 self-attention。
  • HopfieldPooling: Q 為 learnable 單向量，所有 K/V 從一組輸入聚合 → set 上的 attention pooling。
  • HopfieldLayer: Q from data, K/V = learnable lookup table → 大型 stored-pattern memory（似 retrieval-augmented）。

③ β 的角色：β 小 → softmax 平滑、回傳 multiple memories 的混合（meta-stable state）；β 大 → softmax 銳化、單一記憶（pure memory state）。Ramsauer 證明：β > β_c 時，每個 memory 各有 basin of attraction，誤差指數小。

④ 與 Transformer 對應：Transformer attention(Q, K, V) = softmax(QK^T/√d)V。若 V = X、K = X、Q = ξ、β = 1/√d，則完全等價。Ramsauer 還推出「兩層 Hopfield = standard attention block」的對應。

⑤ 訓練：把 Hopfield 當作 differentiable layer 直接接到 NN，標準 backprop 即可。記憶 X 可以是 learnable parameter，也可以是 batch-dependent input（如 DeepRC：把整個 repertoire 當 keys）。

① Energy design (Ramsauer 2021):

E(ξ) = −β⁻¹ log(Σ_i exp(β x^i · ξ)) + ½‖ξ‖² + const

The first term (lse) soft-maxes over similarity to the most similar memory; the second is a self-energy preventing ξ from blowing up. Gradient ∇E = ξ − X·softmax(β X^T ξ); setting it to zero gives fixed point ξ = X·softmax(β X^T ξ).

② Three Hopfield modes (hflayers):

  • Hopfield (standard): Q from input, K/V from learnable memory matrix → same as self-attention.
  • HopfieldPooling: Q is a single learnable vector; K/V aggregate inputs → attention pooling over sets.
  • HopfieldLayer: Q from data, K/V = learnable lookup table → large stored-pattern memory (akin to retrieval-augmented).

③ Role of β: small β → smooth softmax → mixture of memories (meta-stable state); large β → sharp softmax → a single memory (pure state). Ramsauer proves: for β > β_c, each memory has its basin of attraction with exponentially small retrieval error.

④ Correspondence to Transformer: attention(Q, K, V) = softmax(QK^T/√d)V. With V = X, K = X, Q = ξ, β = 1/√d, they are identical. Ramsauer also derived "two Hopfield updates = one standard attention block".

⑤ Training: drop Hopfield in as a differentiable layer; standard backprop. Memory X can be a learnable parameter or batch-dependent input (e.g. DeepRC: the entire repertoire as keys).

歷史轉折：Hopfield 在 2024 年因「對人工神經網路的奠基貢獻」與 Geoffrey Hinton 共獲諾貝爾物理獎。Modern Hopfield 的復興表明：經典模型在新的能量函數下，仍可以與最先進的深度學習對齊。
Historical turn: Hopfield received the 2024 Nobel Prize in Physics with Geoffrey Hinton for "foundational contributions to artificial neural networks". The Modern Hopfield revival shows: classical models under a new energy function can still align with state-of-the-art deep learning.

🌳 何時用 Hopfield Layer？

情境1:
Set / Bag 輸入聚合——TCR 倉庫、藥物分子集合、影像 patch 集合：用 HopfieldPooling 把 set 聚合成單向量。
情境2:
記憶式檢索——把訓練集當 stored patterns，查詢時取 attention-weighted 平均（如 retrieval-augmented LLM、藥物-靶點檢索）。
情境3:
類比 attention 但需要能量視角——關注 attractor 收斂、basin 結構或聯想記憶分析。
不適合:
純語言序列建模——標準 Transformer 已足夠；Hopfield 不會帶來顯著收益。
Case 1:
Set / bag aggregation — TCR repertoires, molecule sets, image patch sets: HopfieldPooling collapses a set into one vector.
Case 2:
Memory-based retrieval — treat training set as stored patterns; query via attention-weighted recall (retrieval-augmented LLM, drug-target retrieval).
Case 3:
Attention with an energy view — when attractor convergence, basin structure, or associative-memory analysis matters.
Not for:
Pure language sequence modeling — standard Transformer suffices; Hopfield offers no clear gain.
生物應用

Hopfield Layers 在免疫與分子生物資訊

DeepRC — 免疫倉庫分類

Widrich et al. (2020, NeurIPS) 用 Modern Hopfield (HopfieldPooling) 把 patient 的 TCR/BCR 倉庫（10^5 − 10^6 序列）聚合成單向量，用於疾病分類（CMV、HIV、CRC）。Hopfield attention 自動挑出 disease-associated clonotypes。

Widrich et al. (2020, NeurIPS) used HopfieldPooling to aggregate a patient's TCR/BCR repertoire (10⁵–10⁶ sequences) into one vector for disease classification (CMV, HIV, CRC). Hopfield attention automatically selects disease-associated clonotypes.

HopfieldRepertoire

Schäfl et al. (2024) 改良 DeepRC，把 TCR repertoire 變成 stored memories，新樣本查詢時做 attention retrieval。可解釋哪些 TCR 對診斷貢獻最大，並支援少樣本 transfer。

Schäfl et al. (2024) extended DeepRC by treating repertoires as stored memories; new queries retrieve via attention. Interpretable which TCRs drive diagnosis, and supports few-shot transfer.

藥物-靶點結合預測

將已知 ligand-target pairs 當作 Hopfield 記憶，新查詢藥物透過 attention 檢索相似配體，預測結合強度。Schmidhuber 團隊的 Hopfield-DTI 與後續 MolBind 都採此架構。

Store known ligand-target pairs as Hopfield memories; new drug queries retrieve similar binders via attention to predict binding. Schmidhuber group's Hopfield-DTI and MolBind follow this design.

Conformal-DeepRC 倉庫預測

把 DeepRC 與 Conformal Prediction 結合：對患者 TCR repertoire 做 disease prediction 時同時給出可信度區間。對臨床轉譯很關鍵（FDA-style uncertainty）。

Combines DeepRC with Conformal Prediction: alongside disease prediction from a TCR repertoire, output a calibrated uncertainty interval. Crucial for clinical translation (FDA-style uncertainty).

Mol-Hopfield 分子相似度檢索

把化合物庫的 ECFP / 圖嵌入當記憶，新分子查詢做 attention-weighted similarity；用於 hit expansion 與 scaffold hopping。比傳統 Tanimoto NN 更善於捕捉「軟相似度」。

Store compound library ECFP / graph embeddings as memory; query via attention-weighted similarity for hit expansion and scaffold hopping. Captures "soft similarity" better than Tanimoto NN.

蛋白功能標註 — Multiple-Instance Learning

每個蛋白由多個 domain / motif 組成（bag of instances）。HopfieldPooling 自然處理 MIL：把 domain embeddings 聚合成蛋白級表示，預測 GO term 或 enzyme function。

Proteins consist of domains / motifs (bag of instances). HopfieldPooling naturally handles MIL: aggregate domain embeddings into a protein-level vector for GO term or enzyme prediction.

深入解析

Modern Hopfield 進階主題

① 容量爆炸：從多項式到指數

Krotov & Hopfield (2016) 用 polynomial interaction F(s) = s^n 把容量提升到 O(d^(n−1))；Demircigil (2017) 用 exp(s) 進一步達到 2^(d/2)。Ramsauer (2021) 的 lse + ½‖ξ‖² 形式繼承了指數容量，並使能量在連續空間平滑可導。

② 與 Attention 的兩種對應

「軟」對應：一次 Hopfield update = 一次 attention forward。「硬」對應：Ramsauer 證明連續多次 Hopfield update 與「跨層 attention 累積」等價，這暗示 Transformer 各層其實在做漸進記憶檢索。

③ Meta-stable state 與 mixture retrieval

當 β 取中等值，softmax 不會收斂到單一記憶，而是「鄰近記憶的加權混合」。這對應神經科學的 schema/prototype 概念，也是 retrieval-augmented generation 的數學基礎——查詢得到多個相關記憶的混合。

④ Hopfield Layer 變體

  • HopfieldPooling: Q 為單向量、output 為單向量 — set aggregation。
  • HopfieldLayer: learnable memory（外部 K/V） — 大型 lookup table。
  • Multi-head Hopfield: 多組獨立 (Q, K, V) 平行 — 同 multi-head attention。

⑤ 與 Energy-Based Model 的關係

Modern Hopfield 是 EBM 的特殊型：能量為 lse + self-term；採樣的 Langevin 動力學 = continuous Hopfield update + noise。本章下一節「EBM & DCA」會看到更通用的 EBM 框架。

⑥ 訓練實務

  • β 初始化：1/√d_k（同 Transformer）；過大會 saturate softmax 造成梯度消失。
  • 記憶數量 N：容量上限 ~2^(d/2)，但實務中 N 過大會增加 attention 計算成本（O(N²)）。
  • Normalization：對 K, V 做 LayerNorm 通常穩定訓練；對 ξ 不要 normalize（破壞 attractor 結構）。
  • Dropout：對 attention score 做 dropout 可避免「記住 noise」。

⑦ 與 retrieval-augmented LLM 的關係

RAG / kNN-LM 在大型知識庫上做檢索 + attention——這本質就是大尺度的 HopfieldLayer。把外部知識庫當 stored patterns，查詢時取最相似 top-k 並 attention-weighted aggregation。Hopfield 提供了能量解讀。

① Capacity explosion: polynomial to exponential

Krotov & Hopfield (2016) used polynomial interactions F(s) = s^n to lift capacity to O(d^(n−1)); Demircigil (2017) reached 2^(d/2) via exp(s). Ramsauer (2021)'s lse + ½‖ξ‖² inherits the exponential capacity while remaining smooth and differentiable in continuous space.

② Two correspondences to attention

"Soft" correspondence: one Hopfield update = one attention forward. "Hard" correspondence: Ramsauer proved iterated Hopfield ≡ stacking attention across layers — suggesting Transformer layers implicitly perform progressive memory retrieval.

③ Meta-stable states and mixture retrieval

At moderate β, softmax does not converge to a single memory but to a "weighted mixture of nearby memories". This matches schema/prototype concepts in neuroscience and grounds retrieval-augmented generation — queries retrieve a mixture of related memories.

④ Hopfield Layer variants

  • HopfieldPooling: Q = single vector, output = single vector → set aggregation.
  • HopfieldLayer: learnable memory (external K/V) → large lookup table.
  • Multi-head Hopfield: parallel (Q, K, V) groups, mirroring multi-head attention.

⑤ Relation to Energy-Based Models

Modern Hopfield is a special EBM: energy is lse + self-term; Langevin sampling = continuous Hopfield update + noise. The next chapter on EBM & DCA covers the broader framework.

⑥ Training tips

  • β init: 1/√d_k (same as Transformer); too large saturates softmax → vanishing gradient.
  • Number of memories N: capacity ~2^(d/2), but large N inflates O(N²) attention cost.
  • Normalization: LayerNorm on K, V usually stabilizes training; don't normalize ξ (breaks attractor structure).
  • Dropout: attention dropout helps avoid memorizing noise.

⑦ Connection to retrieval-augmented LLMs

RAG / kNN-LM perform retrieval + attention on large knowledge bases — essentially a large-scale HopfieldLayer. External knowledge = stored patterns; query retrieves top-k via attention-weighted aggregation. Hopfield supplies the energy interpretation.

潛在改良應用

還能改良哪些任務？

🎯 1. 整合 patient-level 多模態倉庫

每位患者有 TCR、BCR、scRNA、scATAC 各自為「set」。提案：多模態 HopfieldPooling，每個 modality 分別聚合後跨模態 attention，可用於免疫表型分類與臨床預後。

🎯 2. 細胞型別 prototype memory

把 scLLM (scGPT / Geneformer) 學到的細胞 embedding 平均成 cell-type prototype，存進 HopfieldLayer；新細胞做 attention retrieval 即可標註。提案：scHopfield zero-shot annotation。

🎯 3. 蛋白家族 prototype-based annotation

Pfam family 的 consensus embedding 當記憶；query 蛋白序列做 attention 即可分到 family / clan。提案：HopfieldHMMer，可解釋 + 可信度區間。

🎯 4. 影像病理 patch aggregation

Whole-slide image (WSI) 由數萬 patch 組成，現有 MIL 用 Attention-MIL。提案：HopfieldPooling MIL 訓練更穩、解釋性更佳（哪些 patch 是 attractor）。

🎯 5. CRISPR perturbation library 檢索

Perturb-seq 已知 perturbation → expression profile 映射存為記憶。提案：新 perturbation 直接 attention 檢索最相似 perturbation 預測效應，無需從頭訓練。

🎯 6. RAG-style 生物資訊知識增強

把 GO / Reactome / KEGG pathway annotation 全部嵌入 HopfieldLayer，作為 LLM 的外部記憶。提案：BioRAG = 生物知識基底 + Hopfield retrieval。

🎯 7. Conformal-Hopfield uncertainty

Hopfield retrieval 天然有 attention weight 可作為信心度，但缺乏統計保證。提案：結合 conformal prediction 對 Hopfield top-k 給出嚴格 coverage。

🎯 1. Patient-level multimodal repertoires

Each patient has TCR, BCR, scRNA, scATAC sets. Proposal: multimodal HopfieldPooling — each modality pooled separately, then cross-modal attention — for immune phenotyping and clinical prognosis.

🎯 2. Cell-type prototype memory

Average scLLM (scGPT / Geneformer) embeddings into cell-type prototypes; store in a HopfieldLayer; new cells retrieve via attention. Proposal: scHopfield zero-shot annotation.

🎯 3. Protein-family prototype annotation

Pfam family consensus embeddings as memory; query proteins attention into family/clan. Proposal: HopfieldHMMer — interpretable, with calibrated uncertainty.

🎯 4. Pathology WSI patch aggregation

Whole-slide images aggregate via Attention-MIL today. Proposal: HopfieldPooling MIL — more stable training, clearer interpretability (which patches are attractors).

🎯 5. CRISPR perturbation library retrieval

Perturb-seq stored as perturbation → expression mapping. Proposal: new perturbations retrieve nearest known effects via attention, no retraining needed.

🎯 6. RAG-style bioinformatics knowledge augmentation

Embed GO / Reactome / KEGG annotations into a HopfieldLayer as LLM external memory. Proposal: BioRAG = biological knowledge base + Hopfield retrieval.

🎯 7. Conformal-Hopfield uncertainty

Attention weights serve as a confidence proxy but lack statistical guarantees. Proposal: combine conformal prediction with Hopfield top-k for rigorous coverage.

程式碼

實作範例

# pip install hflayers
import torch
from hflayers import Hopfield, HopfieldPooling, HopfieldLayer

# Standard Hopfield: input → attention over learnable memory
hop = Hopfield(input_size=128, hidden_size=64,
               num_heads=4, scaling=0.125)   # β = 1/√64
x = torch.randn(16, 32, 128)               # (B, seq, dim)
y = hop(x)                                       # same shape

# HopfieldPooling: aggregate set → single vector
pool = HopfieldPooling(input_size=128, hidden_size=64,
                       num_heads=4, output_size=128)
bag = torch.randn(16, 200, 128)             # 200 instances
out = pool(bag)                                  # (16, 128)

# HopfieldLayer: external learnable memory of N patterns
layer = HopfieldLayer(input_size=128, hidden_size=64,
                      quantity=5000)            # 5000 stored patterns
q = torch.randn(16, 128)
r = layer(q.unsqueeze(1)).squeeze(1)         # attention over memory
# Pure PyTorch — Modern Hopfield update
import torch, torch.nn as nn, torch.nn.functional as F

class ModernHopfield(nn.Module):
    def __init__(self, d, n_mem, beta=None):
        super().__init__()
        self.X = nn.Parameter(torch.randn(n_mem, d) * 0.02)
        self.beta = beta if beta is not None else 1/d**0.5
    def forward(self, xi, n_steps=1):
        # xi : (B, d) query; iterate update
        for _ in range(n_steps):
            sim = xi @ self.X.t()             # (B, n_mem)
            w = F.softmax(self.beta * sim, dim=-1)
            xi = w @ self.X                   # (B, d)
        return xi

mh = ModernHopfield(d=128, n_mem=500, beta=0.5)
xi = torch.randn(16, 128)
retrieved = mh(xi, n_steps=3)             # 3 iterations
# DeepRC-style repertoire classifier sketch
import torch.nn as nn
from hflayers import HopfieldPooling

class RepertoireClassifier(nn.Module):
    def __init__(self, seq_dim=256, n_classes=2):
        super().__init__()
        self.seq_encoder = nn.Sequential(           # per-TCR encoder
            nn.Conv1d(21, 128, 5, padding=2), nn.ReLU(),
            nn.AdaptiveMaxPool1d(1))
        self.pool = HopfieldPooling(
            input_size=128, hidden_size=64,
            num_heads=8, output_size=128)
        self.cls = nn.Linear(128, n_classes)

    def forward(self, tcr_set):
        # tcr_set : (B, N_clonotypes, 21, L_aa)
        B, N, A, L = tcr_set.shape
        h = self.seq_encoder(tcr_set.view(B*N, A, L)).squeeze(-1)
        h = h.view(B, N, 128)
        v = self.pool(h)                             # Hopfield over repertoire
        return self.cls(v)
