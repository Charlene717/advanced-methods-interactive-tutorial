變分推論與 VAE

scVI、totalVI、MultiVI、PeakVI 等單細胞工具的數學骨幹——把貝氏推論變成可微優化。

The mathematical backbone of scVI, totalVI, MultiVI, PeakVI — turning Bayesian inference into differentiable optimization.

核心概念

從 EM 到 VAE

單細胞測量值 x（每個基因的 count）的生成模型：細胞有個未知的「隱變數」z（低維表徵），條件分布 p(x|z) 是 NB / ZINB / Poisson。我們想求後驗 p(z|x) 但難算 → 用 qφ(z|x)（神經網路 encoder）近似。

ELBO（Evidence Lower BOund）：

log p(x) ≥ Eq(z|x)[log p(x|z)] − KL[q(z|x) ‖ p(z)]

右邊是「重構誤差」+「先驗正則化」。最大化 ELBO ⇔ 同時學 encoder φ 與 decoder θ。

重參數技巧 (reparameterization trick)：z = μ + σ ⊙ ε, ε ~ N(0, I)。讓抽樣可微，可用反向傳播訓練。

Generative model for single-cell counts x: each cell has a latent z (low-dim rep); p(x|z) is NB / ZINB / Poisson. The posterior p(z|x) is intractable → approximate with qφ(z|x) (a neural net encoder).

ELBO (Evidence Lower BOund):

log p(x) ≥ Eq(z|x)[log p(x|z)] − KL[q(z|x) ‖ p(z)]

= reconstruction + prior regularization. Maximizing ELBO simultaneously trains encoder φ and decoder θ.

Reparameterization trick: z = μ + σ ⊙ ε, ε ~ N(0, I) — makes sampling differentiable for backprop.

scVI 家族

統一的機率框架

scVI (Lopez 2018)

為 scRNA-seq 設計：encoder 輸出 z 與 library size，decoder 輸出 NB 或 ZINB 參數。原生支援批次效應移除（把 batch 當條件變數）。

For scRNA-seq: encoder outputs z + library size; decoder outputs NB / ZINB parameters. Native batch correction (batch as conditional input).

totalVI (Gayoso 2021)

擴展到 CITE-seq——同時模擬 RNA (NB) 與蛋白質 (NB + foreground/background mixture)。

Extends to CITE-seq — jointly models RNA (NB) and protein (NB + foreground/background mixture).

MultiVI / PeakVI

處理 scATAC（PeakVI 為 Bernoulli 模型）與多模態整合（MultiVI 對齊 paired 或 unpaired RNA + ATAC）。

scATAC (PeakVI = Bernoulli model) and multi-modal integration (MultiVI for paired or unpaired RNA + ATAC).

destVI / Stereoscope

用 VAE 做空間轉錄體的細胞型別解析 (deconvolution)——把 spot 視為 cell-type-specific latent 的混合。

VAE-based deconvolution for spatial transcriptomics — treats spots as mixtures of cell-type latents.

cell2location

階層貝氏 + VI，整合 spatial 與 scRNA-seq 參考估計每個 spot 的細胞型別豐度，包含技術因子建模。

Hierarchical Bayes + VI; integrates spatial + scRNA-seq reference to estimate per-spot cell-type abundances with technical modeling.

VAE for trajectory: VITAE, scTour

把 latent z 加上時間/軌跡先驗，做連續發育推論。scTour 用 Neural ODE 連續化潛變數。

Add temporal/trajectory priors to z; scTour uses Neural ODE for continuous latents.

生物應用

為什麼是單細胞的主流框架？

原生計數模型

直接用 NB / ZINB likelihood 模擬 raw counts，不需要前處理（log normalization）就能整合多批次。

Models raw counts directly via NB / ZINB likelihood; no preprocessing (log-norm) needed before batch integration.

批次效應

batch 當作 conditional input，decoder 學「在這個 batch 下」應該長什麼樣——天然移除技術變異。

Batch as conditional input; decoder learns "what x looks like in this batch" — natural batch correction.

不確定性量化

每個細胞都有 z 的後驗分布——可量化 imputation、DE 等下游分析的不確定性。

Posterior q(z|x) gives per-cell uncertainty — propagates to imputation, DE, etc.

可擴展

隨機梯度下降 + GPU → 數百萬細胞可在數小時訓練完畢。比 MCMC 快幾個數量級。

SGD + GPU → millions of cells in hours. Orders of magnitude faster than MCMC.

實務小提示：scVI 訓練時的 batch_key 一旦設定錯誤（例如把生物因子當 batch），會把真實生物差異「校正掉」。先想清楚什麼是「想保留的變異」、什麼是「想移除的變異」。
Practical tip: A wrong batch_key in scVI (treating a biological factor as batch) erases real biology. Always be clear what variation you want to keep vs remove.
深入解析

深入解析 VAE：ELBO、posterior collapse、likelihood 選擇

① ELBO 數學推導：為什麼最大化 ELBO ≈ 最小化 KL 後驗距離

對任意變分分布 qφ(z|x)：

log pθ(x) = Eq[log pθ(x,z) − log qφ(z|x)] + KL(qφ(z|x) ‖ pθ(z|x))
　　　　　= ELBO(x; θ, φ) + KL(q ‖ true posterior)

因為 KL ≥ 0，所以 ELBO ≤ log pθ(x)（這是「下界」evidence lower bound 的由來）。關鍵觀察：log p(x) 不依賴 φ，所以「對 φ 最大化 ELBO」等價於「對 φ 最小化 KL(q‖真實後驗)」——這就是 VAE 用 ELBO 訓練的根本原因。實務上把 ELBO 拆成 reconstruction term Eq[log p(x|z)] 減去 prior KL KL(q(z|x)‖p(z))，後者在高斯先驗下有閉合形式。

② 重參數技巧（reparameterization trick）的數學原因

訓練要計算 ∇φ Eqφ(z|x)[f(z)]。直接做的話 z ~ qφ 的取樣本身依賴 φ，梯度無法穿過取樣。Kingma & Welling (2014) 把 z = μφ(x) + σφ(x) · ε, ε ~ N(0, I)——把隨機性「外移」到 ε，z 變成 φ 的可微函數，於是 ∇φ Eq[f(z)] = Eε[∇φ f(μ + σε)] 可用 Monte Carlo 估計。等價於 pathwise / Jacobian-based gradient estimator，變異數比 score-function (REINFORCE) estimator 小好幾個數量級。離散 latent 不能直接重參數，要靠 Gumbel-Softmax 或 straight-through estimator（VQ-VAE 就是這條路）。

③ β-VAE 與 disentanglement (Higgins et al. 2017, ICLR)

把 ELBO 改寫為 Eq[log p(x|z)] − β · KL(q(z|x)‖p(z))。β = 1 是原本 VAE；β > 1 (常用 4–250) 強迫 latent 更接近先驗 N(0, I)，間接逼出「factor-disentangled」表徵（每個 latent 維度對應一個 generative factor）。代價：重建品質下降（trade-off）。在單細胞應用：β-VAE 變體可分離「細胞型別」與「batch / 技術因子」latent dimension。延伸：β-TCVAE (Chen 2018) 把 KL 拆成 mutual information / total correlation / dimension-wise KL，更精準地懲罰「跨維度耦合」而不傷重建。

④ Posterior collapse 問題與修補

當 decoder 太強（例如 PixelRNN 或大型 autoregressive）時，模型可以不靠 z 就重建 x，於是 q(z|x) → p(z)，KL → 0，latent 變無用。在 scRNA-seq 用 zero-inflated NB decoder 加深層時也會出現。修補策略：

  • KL annealing：訓練初期 β = 0 慢慢升到 1，給 encoder 機會先學會有意義的 z。
  • Free bits (Kingma 2016)：對每個 latent 維度的 KL 設下限 λ，KL < λ 不計入 loss——強迫每個維度至少傳 λ nats 的訊息。
  • Semi-amortized inference (Kim 2018)：用 encoder 給初值，再做幾步 SVI 細修——減少 amortization gap。
  • δ-VAE (Razavi 2019)：讓 prior 略偏離標準高斯，使 q→p 不再「免費」。
  • Weaken decoder：限制 decoder 容量或加 dropout——對 scVI 這類 Gaussian-decoder-of-modest-size 模型實務上很少出現 collapse。

⑤ Posterior expressiveness：normalizing flow 改進

標準 VAE 假設 q(z|x) = N(μ, diag(σ²))——對角高斯，不能表達 latent 維度間相關。改進：

  • Planar flow / Radial flow (Rezende & Mohamed 2015)：對 z0 ~ N 連續做 K 次可逆變換 zK = fK(...f1(z0))，log det Jacobian 可解析。
  • IAF (Inverse Autoregressive Flow, Kingma 2016)：用 autoregressive 結構，前向（取樣）快，反向（訓練）慢但可平行——適合 VAE encoder。
  • RealNVP / Glow 風格 coupling layer：對 scVI 加上 coupling-based flow encoder，能更好地表達細胞 latent 的多模態分布。

⑥ VQ-VAE (van den Oord et al. 2017, NeurIPS)：離散 latent codes

把連續 latent 換成 codebook 查表：encoder 輸出 ze(x)，找到 codebook 中最近的 ek 作為 zq(x)，decoder 用 zq。優點：(a) 自然對應「細胞型別 / 狀態 token」這種離散概念；(b) 可後接 transformer 做生成（OpenAI Jukebox, DALL-E 1 採用）；(c) 沒有 posterior collapse。在單細胞：可作為「細胞 vocabulary」基礎，啟發了部分 scLLM 設計。訓練的 straight-through estimator 與 commitment loss 是關鍵 trick。

⑦ MMD-VAE / InfoVAE (Zhao 2017)：避開 posterior collapse

把 KL(q(z|x)‖p(z)) 改成 MMD(q(z)‖p(z))——對 aggregated posterior 而非每樣本 posterior 配對先驗，數學上是 InfoVAE family 的特例。優點：不會 collapse（不直接懲罰 KL），latent 維度幾乎都活躍。代價：理論上沒有清楚的 likelihood lower bound，無法直接做 model comparison via ELBO。

⑧ Conditional VAE 與 scVI：把 batch 作為 conditional input

scVI (Lopez 2018) 把 batch label s 同時輸入 encoder f(x, s) 與 decoder g(z, s)。直觀理解：encoder 學到的 z 是「扣掉 batch 後的細胞 state」；decoder 「給定 state 加回 batch effect」生成 observed counts。這是 conditional VAE 思想在 scRNA-seq 的核心應用——讓 latent 自動 batch-corrected。延伸：scANVI 把 cell type 也作為 conditional 輸入，做 semi-supervised label transfer；totalVI 加上 protein modality 共同 condition。

⑨ NB 與 ZINB likelihood 數學：為什麼 scRNA-seq 用這個

scRNA-seq UMI counts 是非負整數，且有過度離散（variance ≫ mean）。Poisson 不夠（強制 var = mean），用 NB(μ, θ): variance = μ + μ²/θ，θ 為 dispersion。decoder 輸出 μg, θg for each gene g，log-likelihood 為：
log p(xg | μ, θ) = Γ(x+θ) − Γ(θ) − Γ(x+1) + θ log(θ/(θ+μ)) + x log(μ/(θ+μ))

ZINB 額外加 zero-inflation 機率 πg 處理 dropout（觀測 0 可能是 NB 採樣自然 0，也可能是 dropout 0）：p(x=0) = π + (1−π)·NB(0); p(x>0) = (1−π)·NB(x). 但 Svensson (2020) 指出 10x Chromium UMI 資料 dropout 沒原本想得那麼極端，NB 通常足夠，ZINB 是 over-modeling——scVI 預設用 ZINB 但允許 NB-only。

⑩ Hierarchical / multi-level VAE

multiVI (Ashuach 2023) 與 totalVI (Gayoso 2021) 都用 hierarchical：每個 modality 一個 modality-specific latent zRNA, zATAC，再經 shared latent zjoint 整合。totalVI 還做 mixture prior：背景蛋白訊號 + 真實表現雙成分混合。階層 VAE 的優點：可解釋（每層 latent 對應不同抽象等級）；可整合不完整資料（某些細胞只有 RNA）；理論上 tighter ELBO（NVAE, Vahdat 2020）。

⑪ Amortized vs non-amortized inference

Amortized：encoder 用單一 NN 對所有樣本攤平推論——快，但有「amortization gap」（單一 NN 不能精準擬合每個樣本的真實後驗）。Non-amortized：對每個樣本獨立優化 q（傳統 SVI）——準但極慢。Semi-amortized (Kim 2018, Marino 2018)：用 encoder 給初值再做幾步 per-sample SVI——在 scRNA-seq 大資料下實用性有限，但對「稀有細胞」可能有用（單純 amortized encoder 對少見樣本表現差）。

⑫ IWAE (Importance-Weighted VAE, Burda 2016)：tighter bound

用 K 個樣本估計：LK = E[log (1/K) Σk p(x, zk)/q(zk|x)]，K=1 時退化為 ELBO，K→∞ 時 → log p(x)。實務上 K = 5–50 給出顯著更緊的 bound 與更好的 log-likelihood，但 reparameterization 變異數隨 K 升高反而稀釋 encoder 梯度（Rainforth 2018）——所以 K 不是越大越好，且不能無腦套用。

① ELBO derivation: why maximizing ELBO ≈ minimizing posterior KL

For any variational qφ(z|x):

log pθ(x) = Eq[log pθ(x,z) − log qφ(z|x)] + KL(qφ(z|x) ‖ pθ(z|x))
　　　　　= ELBO(x; θ, φ) + KL(q ‖ true posterior)

Since KL ≥ 0, ELBO ≤ log p(x) — this is the "evidence lower bound." Key insight: log p(x) is independent of φ, so maximizing ELBO over φ is equivalent to minimizing KL(q ‖ true posterior). This is the foundation of VAE training. ELBO splits into a reconstruction term Eq[log p(x|z)] minus a prior KL KL(q(z|x)‖p(z)), the latter closed-form under Gaussian prior.

② Reparameterization trick — why it's needed

We need ∇φ Eqφ(z|x)[f(z)]. Direct sampling makes gradient untraceable through stochastic node. Kingma & Welling (2014): z = μφ(x) + σφ(x) · ε, ε ~ N(0, I) — externalize randomness to ε; z is now a differentiable function of φ, so ∇φ Eq[f(z)] = Eε[∇φ f(μ + σε)] via Monte Carlo. Equivalent to pathwise/Jacobian estimator; variance is orders of magnitude smaller than score-function (REINFORCE). Discrete latents can't be reparameterized — use Gumbel-Softmax or straight-through (VQ-VAE route).

③ β-VAE and disentanglement (Higgins et al. 2017, ICLR)

Modify ELBO: Eq[log p(x|z)] − β · KL(q(z|x)‖p(z)). β = 1 is vanilla VAE; β > 1 (often 4–250) pushes latent closer to prior, indirectly forcing factor-disentangled representation. Cost: reconstruction quality drops (trade-off). In single-cell: β-VAE variants can separate cell-type latents from batch/technical latents. β-TCVAE (Chen 2018) decomposes KL into mutual information / total correlation / dimension-wise KL — penalizes cross-dim coupling more surgically without harming reconstruction.

④ Posterior collapse and its fixes

When decoder is too powerful (PixelRNN, large autoregressive), the model reconstructs x without using z, so q(z|x) → p(z), KL → 0, latent becomes useless. Also happens in deep ZINB decoders for scRNA-seq. Fixes:

  • KL annealing: start β = 0, ramp to 1 — gives encoder time to learn meaningful z first.
  • Free bits (Kingma 2016): set per-dim KL floor λ; below λ not counted — each dim must carry ≥ λ nats.
  • Semi-amortized (Kim 2018): encoder gives init, then a few SVI steps — reduces amortization gap.
  • δ-VAE (Razavi 2019): slightly non-standard prior so q→p is no longer "free."
  • Weaken decoder: cap decoder capacity. In practice scVI-sized models rarely collapse.

⑤ Posterior expressiveness — normalizing flow improvements

Standard VAE uses q(z|x) = N(μ, diag(σ²)) — diagonal Gaussian, no inter-dim correlation. Improvements:

  • Planar / radial flow (Rezende & Mohamed 2015): K invertible transforms on z0 ~ N, log det Jacobian analytic.
  • IAF (Inverse Autoregressive Flow, Kingma 2016): autoregressive structure, fast forward (sampling), parallelizable backward (training) — fits VAE encoder well.
  • RealNVP / Glow coupling layers: adding coupling flow to scVI encoder enables multi-modal latent distributions for heterogeneous cells.

⑥ VQ-VAE (van den Oord et al. 2017, NeurIPS) — discrete latents

Replace continuous z with codebook lookup: encoder → ze(x); find nearest codebook entry ek as zq(x); decoder uses zq. Advantages: (a) naturally encodes "cell-state tokens"; (b) downstream transformer can model token sequence (used by OpenAI Jukebox, DALL-E 1); (c) no posterior collapse. In single-cell: foundation for "cell vocabulary," inspires some scLLM designs. Straight-through estimator + commitment loss are key training tricks.

⑦ MMD-VAE / InfoVAE (Zhao 2017) — avoid posterior collapse

Replace KL(q(z|x)‖p(z)) with MMD(q(z)‖p(z)) — match aggregated posterior to prior, not per-sample. InfoVAE family generalizes this. Advantage: no collapse (no direct KL penalty); virtually all latent dims stay active. Cost: no clean likelihood lower bound — model comparison via ELBO loses meaning.

⑧ Conditional VAE and scVI — batch as conditional input

scVI (Lopez 2018) feeds batch label s into both encoder f(x, s) and decoder g(z, s). Intuition: encoder learns z = "cell state after removing batch"; decoder adds batch effect back. This is the canonical conditional-VAE application in scRNA-seq — yields auto-batch-corrected latents. Extension: scANVI conditions on cell-type label for semi-supervised transfer; totalVI co-conditions on protein modality.

⑨ NB and ZINB likelihoods — why for scRNA-seq

scRNA-seq UMI counts are non-negative integers, overdispersed (variance ≫ mean). Poisson fails (variance = mean). Use NB(μ, θ): variance = μ + μ²/θ, θ = dispersion. Decoder outputs μg, θg per gene; log-likelihood:
log p(xg | μ, θ) = Γ(x+θ) − Γ(θ) − Γ(x+1) + θ log(θ/(θ+μ)) + x log(μ/(θ+μ))

ZINB adds zero-inflation πg for dropout: p(x=0) = π + (1−π)·NB(0); p(x>0) = (1−π)·NB(x). But Svensson (2020) showed 10x Chromium UMI data is less dropout-heavy than assumed — NB is usually enough, ZINB is over-modeling. scVI defaults to ZINB but supports NB.

⑩ Hierarchical / multi-level VAE

multiVI (Ashuach 2023) and totalVI (Gayoso 2021) use hierarchy: modality-specific latents zRNA, zATAC → shared latent zjoint. totalVI additionally uses mixture priors for background vs foreground protein signal. Hierarchical VAE benefits: interpretability per level; can integrate incomplete data (cells with only RNA); tighter ELBO theoretically (NVAE, Vahdat 2020).

⑪ Amortized vs non-amortized inference

Amortized: one shared encoder NN — fast but "amortization gap" (one NN can't fit every sample's true posterior). Non-amortized: per-sample q optimization (classical SVI) — accurate but very slow. Semi-amortized (Kim 2018, Marino 2018): encoder gives init, then a few per-sample SVI steps — limited practicality for large scRNA-seq, but may help rare-cell-type performance (pure amortized encoder underfits rare samples).

⑫ IWAE (Importance-Weighted VAE, Burda 2016) — tighter bound

LK = E[log (1/K) Σk p(x, zk)/q(zk|x)]; K=1 reduces to ELBO; K→∞ → log p(x). In practice K = 5–50 gives notably tighter bound and better log-likelihood, but reparameterization variance dilutes encoder gradient at high K (Rainforth 2018) — K is not unboundedly good.

詳細已知應用

VAE 在生物資訊的具體案例

🧬 案例 1：scVI (Lopez et al. 2018, Nature Methods) — 基準與里程碑

第一個將 VAE 完整套用到 scRNA-seq 的工作。架構：encoder 取 log-normalized counts + batch ID → 10–20 維 latent；decoder 還原 ZINB 參數 (μ, θ, π) for each gene。

  • 關鍵技術：batch ID 作為 encoder/decoder 共同 conditional input → 隱式 batch correction，不需要先做 Harmony/MNN。
  • 實證：在 PBMC、Tabula Muris 等資料上，與 Seurat-CCA、Harmony 整合品質相當或更好，且 latent 可直接用於下游 DE、分群、軌跡分析。
  • 影響：啟發後續整個 scvi-tools 生態系，至 2024 年累積引用超過 2000，是 Nature Methods 該年最熱門方法論論文。

🧬 案例 2：totalVI (Gayoso et al. 2021, Nature Methods) — CITE-seq 整合

CITE-seq 同步測 RNA + surface protein (Antibody-Derived Tags, ADT)。totalVI 用 hierarchical VAE：

  • RNA 用 NB likelihood；蛋白用 mixture (背景 + 前景) negative binomial。
  • 共享 latent z + modality-specific decoder head。
  • 能對 protein 做 batch correction、imputation、跨資料 transfer。
  • 實證：在 PBMC CITE-seq 與骨髓資料上 protein imputation MSE 比 Seurat WNN 降低 30–40%。

🧬 案例 3：multiVI (Ashuach et al. 2023, Nature Methods) — 整合 RNA + ATAC

對 10x Multiome 同步測 RNA + ATAC 配對細胞，建立統一 latent 空間。

  • 三模態 encoder：RNA-only / ATAC-only / paired，後兩者透過 KL alignment 拉到共同 latent。
  • 支援「半配對」資料：有些細胞只有 RNA、有些只有 ATAC，仍可整合。
  • 下游可預測 ATAC peak 從 RNA、或反之。
  • 實證：在 SHARE-seq, SNARE-seq 資料上比 Seurat v4 WNN、MOFA+ 表現好。

🧬 案例 4：PeakVI (Ashuach et al. 2022, Cell Reports Methods) — ATAC-only

專為 scATAC-seq 設計：peak × cell 矩陣 (Bernoulli/Binomial likelihood)，decoder 輸出每 peak 的開放機率。能做 differential accessibility、batch correction、imputation。比傳統 Signac/ArchR 在 noisy 資料上的細胞分群更穩定。

🧬 案例 5：destVI (Lopez et al. 2022, Nature Biotechnology) — 空間轉錄體去卷積

Visium 等空間轉錄體 spot 內含多種細胞型別，destVI 學 spot-level latent，分解出每種 cell type 的比例 + 該 type 在該 spot 的特異表現。雙層 VAE：CondSCVI 在參考 scRNA 上學 cell-type-specific generative model，再用 stLVM 推 spot composition。比 cell2location、RCTD 在低細胞型別解析時更準確。

🧬 案例 6：cell2location (Kleshchevnikov et al. 2022, Nature Biotechnology) — Hierarchical Bayesian

不是嚴格 VAE 但用 variational inference（Pyro 框架）做大規模 Bayesian 推論。為 Visium 設計，估計 spot 中每種 cell type 的「絕對細胞數」（不僅是比例）。hierarchical prior 結構：tissue-level → batch-level → spot-level。在 mouse brain Visium、人類淋巴結資料上找到罕見 cell type localization 訊號。

🧬 案例 7：scGen (Lotfollahi et al. 2019, Nature Methods) — 擾動預測

用 conditional VAE 學 "control" 與 "perturbed" 兩種狀態的 latent；在 latent space 做 vector arithmetic 預測「如果這個未見過的 cell type 也被擾動會如何」。在 LPS 刺激 PBMC 資料上預測 IFN-β 反應，能 generalize 到訓練未見過的 cell type。延伸 trVAE (2020) 加 MMD 對齊不同實驗條件。

🧬 案例 8：scArches (Lotfollahi et al. 2022, Nat Biotech) — Transfer learning 架構

核心想法：對預訓練 scVI/totalVI/scANVI 模型加 lightweight adapter（conditional weights），新資料只訓 adapter，原模型 frozen。在低資源情境（COVID-19 BAL、罕見疾病樣本）可只用 100s 細胞就 transfer 大型 atlas 的知識。

🧬 案例 9：LDVAE (Linear Decoder VAE, Svensson et al. 2020, Bioinformatics) — 可解釋性

把 scVI 的 decoder 改為單層線性（W·z + b），latent → gene weights 矩陣 W 可直接讀，類似 factor analysis 的 loading。雖然重建品質略遜，但「latent 維度 i 對應哪些基因」變得透明。在分析特定 disease atlas 時很有用，例如找「pro-inflammatory program」對應的 latent dim。

🧬 案例 10：SAUCIE (Amodio et al. 2019, Nature Methods)

多任務 autoencoder：同時做 batch correction、imputation、clustering、降維。用 Information Dimension 正則化把 latent 壓到離散細胞型別 token，與 VQ-VAE 思想接近但早 2 年。在 dengue PBMC 資料上找到傳統方法漏掉的細胞次群。

🧬 案例 11：scANVI (Xu et al. 2021, Mol Syst Biol) — Semi-supervised

scVI + cell-type label conditional + mixture-of-Gaussians prior。部分細胞有標註、部分沒，模型聯合學會「latent 內哪個 mixture component 對應哪個 type」與「未標註細胞屬於哪一群」。在 atlas-scale annotation transfer 上比 nearest neighbor mapping 更穩健於 batch shift。

🧬 案例 12：scVI-tools 生態系

scvi-tools 套件（Gayoso et al. 2022, Nat Biotech）統一了 scVI/totalVI/multiVI/PeakVI/destVI/scANVI/CellAssign 等十多個模型，PyTorch + Lightning 後端，到 2024 年成為 scRNA-seq 整合的 de facto 框架，與 Scanpy/AnnData 無縫整合。

🧬 Case 1: scVI (Lopez et al. 2018, Nature Methods) — the milestone baseline

First full VAE application to scRNA-seq. Architecture: encoder takes log-normalized counts + batch ID → 10–20 dim latent; decoder outputs ZINB parameters (μ, θ, π) per gene.

  • Key trick: batch ID as joint conditional input to both encoder & decoder → implicit batch correction, no need for Harmony/MNN pre-step.
  • Results: on PBMC, Tabula Muris, integration matches or beats Seurat-CCA / Harmony, and the latent is directly usable for DE, clustering, trajectory.
  • Impact: sparked the scvi-tools ecosystem; 2000+ citations by 2024.

🧬 Case 2: totalVI (Gayoso et al. 2021, Nature Methods) — CITE-seq integration

Hierarchical VAE for paired RNA + surface protein (ADT). NB likelihood for RNA, mixture NB (background + foreground) for protein. Shared latent z + modality-specific heads. Protein imputation MSE 30–40% lower than Seurat WNN on PBMC and bone marrow CITE-seq.

🧬 Case 3: multiVI (Ashuach et al. 2023, Nature Methods) — RNA + ATAC integration

Unified latent for 10x Multiome (paired RNA + ATAC). Three encoders (RNA-only, ATAC-only, paired) aligned via KL. Handles semi-paired data (cells with only one modality). Beats Seurat v4 WNN, MOFA+ on SHARE-seq, SNARE-seq.

🧬 Case 4: PeakVI (Ashuach et al. 2022, Cell Reports Methods) — ATAC-only

Designed for scATAC-seq: peak × cell with Bernoulli/Binomial likelihood. Outputs per-peak open probability. Enables differential accessibility, batch correction, imputation; more robust on noisy data than Signac / ArchR.

🧬 Case 5: destVI (Lopez et al. 2022, Nature Biotechnology) — spatial deconvolution

For Visium spots (which contain mixed cell types). Two-level VAE: CondSCVI learns cell-type-specific generative model from reference scRNA; stLVM infers per-spot composition + type-specific expression. Beats cell2location, RCTD when cell-type resolution is low.

🧬 Case 6: cell2location (Kleshchevnikov et al. 2022, Nature Biotechnology) — Hierarchical Bayesian

Not strictly VAE but uses variational inference (Pyro backend) for large-scale Bayesian inference. Estimates absolute cell-type counts per Visium spot (not just proportions). Hierarchical prior: tissue → batch → spot. Found rare cell-type localization signals in mouse brain Visium and human lymph node.

🧬 Case 7: scGen (Lotfollahi et al. 2019, Nature Methods) — perturbation prediction

Conditional VAE learns latents for control and perturbed states; vector arithmetic in latent predicts "what if this unseen cell type were perturbed." Predicts IFN-β response to LPS in unseen PBMC types. Extension trVAE (2020) adds MMD alignment across conditions.

🧬 Case 8: scArches (Lotfollahi et al. 2022, Nat Biotech) — transfer-learning architecture

Adds lightweight conditional adapters to pretrained scVI/totalVI/scANVI; frozen base model + adapter-only training. Hundreds of cells suffice to transfer atlas knowledge — used for COVID-19 BAL, rare disease cohorts.

🧬 Case 9: LDVAE (Svensson et al. 2020, Bioinformatics) — interpretability

Single-layer linear decoder: W·z + b. The latent→gene weight matrix W is directly readable, like factor analysis loadings. Trades slight reconstruction quality for clear "this latent dim ≈ pro-inflammatory program" interpretation.

🧬 Case 10: SAUCIE (Amodio et al. 2019, Nature Methods)

Multi-task autoencoder doing batch correction, imputation, clustering, embedding simultaneously. Information-dimension regularization compresses latent to discrete cell-type tokens — pre-dates VQ-VAE thinking in single-cell by 2 years. Found subpopulations missed by classical methods in dengue PBMC.

🧬 Case 11: scANVI (Xu et al. 2021, Mol Syst Biol) — semi-supervised

scVI + cell-type-label conditional + Gaussian-mixture prior. Some cells labeled, some not; learns which mixture component matches which type and assigns unlabeled cells. More batch-robust than nearest-neighbor mapping for atlas-scale annotation transfer.

🧬 Case 12: scvi-tools ecosystem

The scvi-tools package (Gayoso et al. 2022, Nat Biotech) unifies scVI/totalVI/multiVI/PeakVI/destVI/scANVI/CellAssign etc. on PyTorch + Lightning, integrated with Scanpy/AnnData — by 2024 the de facto framework for scRNA-seq integration.

潛在改良應用

VAE 還能改良 / 新解決哪些任務？

🎯 1. 三模態跨體學 VAE：RNA + ATAC + Protein 共同 latent

現有不足：multiVI 只整合 RNA + ATAC；totalVI 只整合 RNA + protein。沒有公開模型同時處理三模態（如 TEA-seq 資料）。新提案：tri-modal hierarchical VAE，每個 modality 有獨立 decoder（ZINB / Bernoulli / mixture-NB），共享 latent zshared + modality-specific zm，KL 對 zshared 做 modality-alignment。預期改進：對 TEA-seq、DOGMA-seq 等罕見三模態資料，可在 latent 同步看到「染色質→轉錄→蛋白」串聯訊號，比現有兩兩整合（先 multiVI 再 totalVI 拼接）更一致；可預測缺失模態的觀測值。

🎯 2. 因果 VAE：disentangle 「真實 biology」與「technical / batch」

現有不足：scVI 用 batch 作為 conditional 是「隱式」correction，無理論保證 latent 真的去除了 batch 訊息；可能仍殘留 batch-correlated 變異。新提案：用 causal disentanglement 框架（如 CausalVAE, Yang 2021）把 latent 拆成 zbio ⊥ ztech，用 batch 標籤作為 supervision 訓練 adversarial loss 強迫 zbio 對 batch 不可預測。預期改進：更乾淨的 cell state latent，可用於跨機構 atlas 真實整合；對下游 DE / GWAS 整合更可靠。

🎯 3. 發育過程的階層 VAE：時間 × 細胞型別兩層

現有不足：scVI 把細胞當作 i.i.d.，沒利用「同一胚胎在不同階段、不同 lineage」這種階層結構。新提案：三層 hierarchical VAE：embryo-level → lineage-level → cell-level latent。inspired by NVAE / very deep VAE。預期改進：對小鼠/斑馬魚胚胎發育 atlas（如 Pijuan-Sala 2019, Wagner 2018），能分離「個體差異」「lineage 分支」「瞬時細胞狀態」三尺度，找出真正的 lineage-determining gene programs，比 Monocle3 + scVI 串接清楚。

🎯 4. 病人特異性 fine-tuned VAE：精準醫療

現有不足：scArches 在 atlas-level fine-tune，但對「單一病人多時點縱貫資料」沒有專門設計；難以在 N=1 設計上提供個人化預測。新提案：把 scArches conditional adapter 擴充為「patient-conditional + time-conditional」，加上 Bayesian uncertainty propagation 量化 N=1 樣本下的 latent 不確定度。預期改進：對腫瘤跟蹤、發炎性疾病 longitudinal scRNA-seq，能預測「下次採樣時細胞分布」與「治療反應」，並給出可信區間。

🎯 5. scLLM × VAE 混合：兼具 token semantics 與 likelihood

現有不足：scGPT/Geneformer 用 transformer 學 gene token sequence 但沒有 generative likelihood；scVI 有 likelihood 但 latent 沒 token semantics。兩者各自有缺。新提案：VQ-VAE 風格——encoder 將細胞 → discrete token sequence，再用 scLLM transformer 對 token 學自回歸先驗 p(tokens)，decoder 從 tokens 還原 NB counts。預期改進：(a) 可像 LLM 一樣 prompt 一個細胞 prefix 自動生成後續 trajectory；(b) latent 可解釋為「細胞語彙」；(c) ELBO 仍可估 model evidence。

🎯 6. Bayesian uncertainty propagation 到下游分析

現有不足：scVI 輸出 posterior mean μz 作為「細胞座標」，下游 UMAP/clustering/DE 全用這個點估計，忽略了 σz。對「邊界細胞」、稀有 cell type 的不確定度從未傳遞。新提案：對下游每個分析步驟 sample K 次 z 重做（Monte Carlo over latent），給出每個細胞「分群信心」「DE p-value 不確定度」「軌跡分支機率」。預期改進：避免把高不確定細胞錯誤分到固定群；可量化「這群細胞是真的還是 noise」。

🎯 7. Counterfactual VAE：基因敲除模擬

現有不足：scGen 預測擾動效應但是 distribution-level；無法回答「這顆細胞如果敲除這個基因會變什麼樣？」instance-level counterfactual。新提案：把 latent 拆成 (zstate, zgene_program)，敲除模擬 = 把 zgene_program 中對應分量設為「該基因 = 0」conditional 下的 latent，再 decode。需要 paired Perturb-seq 資料訓練 disentanglement。預期改進：in silico screen 可大幅縮小 wet-lab Perturb-seq 規模；提供「對特定病人特定細胞」的個人化敲除模擬。

🎯 8. 異常細胞偵測：rare disease, doublet, dying cells

現有不足：doublet 偵測（Scrublet、DoubletFinder）獨立於整合 pipeline 做；rare cell detection 靠 clustering 後人工檢查；dying cells 靠 mitochondrial fraction 過濾。新提案：用 VAE 的 reconstruction error (per-cell log-likelihood) 作為統一 anomaly score——重建差的細胞可能是 doublet、rare type、apoptotic、或 contamination。可加 ensemble of K models 做 calibrated uncertainty。預期改進：一條 pipeline 統一三種 QC 任務，且能找出傳統規則錯失的細胞（例如表現「不像任何已知 type」的真實稀有細胞）。

🎯 9. Inverse design：給定目標表型反推細胞 state

現有不足：正向：給 cell state 預測表現容易；反向：「我想要一顆能對 X 藥反應的 CAR-T 細胞，gene expression 應該長什麼樣？」沒有系統方法。新提案：在 VAE latent 上做 gradient-based optimization：固定 decoder θ，對 z 做梯度下降最小化「實際表型 vs 目標表型」距離。類似 image inverse design (StyleGAN inversion)。預期改進：對細胞工程、再生醫學提供 in silico 候選 cell state 設計藍圖，再用 Perturb-seq 反向找對應的基因擾動組合。

🎯 10. 動態 VAE：時間動力學內嵌入 latent

現有不足：RNA velocity 把 spliced/unspliced 矩陣後處理出動力學；VAE latent 是 static 表徵。沒有單一模型同時學 generative + dynamic。新提案：把 latent z(t) 視為 stochastic differential equation 軌跡，encoder 同時輸出「當下 z」與「漂移 drift μ(z)、擴散 diffusion σ(z)」。類似 latent SDE / VeloVAE (Gu 2022)。預期改進：可同時給出細胞 embedding 與動力學方向，比兩步式 (scVI + scVelo) 更一致；對 lineage tracing + Perturb-seq 縱貫資料更適合。

🎯 11. 學習藥物效應的 perturbation latent space

現有不足：LINCS L1000、Connectivity Map 用 bulk profile 找藥物 signature；無 cell-state-level resolution。新提案：對 sci-Plex / Tahoe-100M 等 scRNA + drug 大規模資料訓 conditional VAE，drug ID + dose 為 conditional input，學「藥物擾動向量場」p(zperturbed | zcontrol, drug)。預期改進：給新藥的有限細胞資料即可預測「在未測試 cell type 上」的反應；可做 drug repurposing 透過「latent 向量相似度」找出機制相似的化合物。

🎯 1. Tri-modal VAE: RNA + ATAC + Protein

Gap: multiVI does RNA+ATAC; totalVI does RNA+protein. No public model handles all three (TEA-seq, DOGMA-seq). Proposal: hierarchical tri-modal VAE — independent decoder per modality (ZINB / Bernoulli / mixture-NB), shared latent zshared + modality-specific zm; modality-alignment KL on zshared. Expected gain: coherent "chromatin → transcription → protein" signal in one latent for TEA-seq / DOGMA-seq; can impute missing modalities.

🎯 2. Causal VAE — disentangle bio from technical

Gap: scVI's conditional batch correction is implicit, no guarantee z is batch-free. Proposal: causal disentanglement (CausalVAE, Yang 2021): split latent into zbio ⊥ ztech via adversarial loss forcing zbio unpredictable from batch. Expected gain: cleaner state latent for cross-institute atlas integration, more reliable downstream DE / GWAS overlay.

🎯 3. Developmental hierarchical VAE: time × lineage

Gap: scVI treats cells as IID, ignores embryo × stage × lineage hierarchy. Proposal: three-level hierarchical VAE: embryo → lineage → cell. Expected gain: on Pijuan-Sala / Wagner zebrafish atlases, separate individual variation / lineage branch / transient state — finds true lineage-determining programs, clearer than Monocle3 + scVI stacking.

🎯 4. Patient-specific fine-tuned VAE for precision medicine

Gap: scArches transfers at atlas level; no design for single-patient longitudinal data, no N=1 personalized prediction. Proposal: extend scArches adapters with patient + time conditional inputs + Bayesian uncertainty propagation. Expected gain: for tumor tracking, inflammatory disease longitudinal scRNA-seq — predict next time point's cell distribution and treatment response with credible intervals.

🎯 5. scLLM × VAE hybrid — token semantics + likelihood

Gap: scGPT/Geneformer model gene tokens but no generative likelihood; scVI has likelihood but no token semantics. Proposal: VQ-VAE encoder → discrete token sequence; transformer prior p(tokens); decoder → NB counts. Expected gain: LLM-style cell-prefix prompting to generate trajectories; interpretable "cell vocabulary"; ELBO still estimable.

🎯 6. Bayesian uncertainty propagation

Gap: scVI's posterior mean μz is used downstream as a point estimate; σz ignored. Border / rare cells' uncertainty is lost. Proposal: Monte Carlo over z for every downstream step — output clustering confidence, DE p-value uncertainty, branch probabilities. Expected gain: avoid forcing high-uncertainty cells into fixed groups; quantify whether a cluster is real or noise.

🎯 7. Counterfactual VAE — gene knockout simulation

Gap: scGen is distribution-level; can't answer "this exact cell, if KO X, becomes what?" instance counterfactual. Proposal: disentangle latent into (zstate, zprogram); KO simulation = condition zprogram on "gene = 0," decode. Train disentanglement on Perturb-seq. Expected gain: shrink wet-lab Perturb-seq scale; patient-specific in silico KO simulation.

🎯 8. Unified anomaly detection: doublets, rare cells, dying cells

Gap: Scrublet/DoubletFinder, rare-cell, mito-fraction filters are three independent steps. Proposal: per-cell VAE log-likelihood as unified anomaly score; ensemble of K models for calibrated uncertainty. Expected gain: one pipeline for three QC tasks; catches real rare cells missed by rule-based filters.

🎯 9. Inverse design — target phenotype → cell state

Gap: forward (state → expression) easy; reverse ("I want a CAR-T cell that responds to drug X, what expression?") not systematized. Proposal: gradient descent on z (decoder frozen) toward target phenotype, like StyleGAN inversion. Expected gain: in silico cell-engineering blueprints; pair with Perturb-seq to find which gene perturbations produce target z.

🎯 10. Dynamic VAE — embed dynamics into latent

Gap: RNA velocity post-processes spliced/unspliced; VAE latent is static. No single model that learns generative + dynamic together. Proposal: treat z(t) as latent SDE; encoder outputs z, drift μ(z), diffusion σ(z) (cf. VeloVAE, Gu 2022). Expected gain: embedding + dynamics consistent in one model; suits lineage-tracing + Perturb-seq longitudinal data.

🎯 11. Drug-effect perturbation latent space

Gap: LINCS L1000 / CMap use bulk profiles, no cell-state resolution. Proposal: conditional VAE on sci-Plex / Tahoe-100M with drug + dose as conditional; learn perturbation vector field p(zperturbed | zcontrol, drug). Expected gain: predict response on untested cell types with limited new-drug data; drug repurposing via latent vector similarity.

import torch, torch.nn as nn, torch.nn.functional as F

class VAE(nn.Module):
    def __init__(self, in_dim, h=128, z_dim=20):
        super().__init__()
        self.enc = nn.Sequential(nn.Linear(in_dim, h), nn.ReLU())
        self.mu = nn.Linear(h, z_dim); self.logvar = nn.Linear(h, z_dim)
        self.dec = nn.Sequential(nn.Linear(z_dim, h), nn.ReLU(), nn.Linear(h, in_dim))
    def forward(self, x):
        h = self.enc(x); mu, lv = self.mu(h), self.logvar(h)
        z = mu + torch.exp(0.5*lv) * torch.randn_like(mu)
        return self.dec(z), mu, lv

def elbo(x, x_hat, mu, lv, beta=1.0):
    rec = F.mse_loss(x_hat, x, reduction="sum")
    kl = -0.5 * torch.sum(1 + lv - mu**2 - lv.exp())
    return rec + beta * kl
