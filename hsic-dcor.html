HSIC 與 Distance Correlation

基於核方法 (kernel) 與能量距離 (energy distance) 的獨立性檢定——可推廣到向量、矩陣、甚至圖等任意結構的隨機變數。

Kernel-based and energy-distance independence tests — generalizable to vectors, matrices, and even graph-valued random variables.

核心概念

兩派思路：核 vs 距離

HSIC (Hilbert-Schmidt Independence Criterion, Gretton 2005)：把 X、Y 透過核函數映射到 RKHS（再生核 Hilbert 空間），計算映射後的協方差算子的 Hilbert-Schmidt 範數。獨立 ⇔ HSIC = 0（用「characteristic kernel」如 Gaussian/RBF 時）。

Distance Correlation, dCor (Székely 2007)：用 X 與 Y 的成對距離矩陣（中心化後）的相關來衡量依賴。獨立 ⇔ dCor = 0，這是一個比 Pearson 強得多的等價條件。

兩者數學上等價——HSIC 用核技巧描述、dCor 用距離描述，但本質都是「距離–距離相關」家族。

HSIC (Hilbert-Schmidt Independence Criterion, Gretton 2005): Map X, Y into a reproducing kernel Hilbert space (RKHS) via kernel functions; measure the Hilbert-Schmidt norm of the cross-covariance operator. Independence ⇔ HSIC = 0 (with a characteristic kernel like Gaussian/RBF).

Distance Correlation, dCor (Székely 2007): Use the correlation of (centered) pairwise distance matrices of X and Y. Independence ⇔ dCor = 0 — far stronger than Pearson.

The two are mathematically equivalent — HSIC speaks the kernel language, dCor speaks distances, but both belong to the "distance–distance correlation" family.

🔮

任意維度

X 與 Y 可以是不同維度的向量——例如 X 是 50D PCA 嵌入、Y 是細胞型別 one-hot。

X and Y can have different dimensions — e.g. X is 50D PCA embedding, Y is one-hot cell type.

🎯

獨立性檢定

虛無假設 H₀：X ⫫ Y 可透過 permutation 檢定取得 p 值。

Null H₀: X ⫫ Y can be tested via permutation to obtain p-values.

🧪

非線性敏感

和 Chatterjee 一樣對非單調關係敏感，且擴展到多變量。

Sensitive to non-monotonic relationships like Chatterjee, but generalizes to multivariate.

演算法

計算式

HSIC：給定核矩陣 K（X 的）、L（Y 的），令 H = I − (1/n)11ᵀ 為中心化矩陣：

HSICn(X,Y) = (1/(n−1)²) · tr(KHLH)

常用 Gaussian kernel：k(x, x') = exp(−‖x−x'‖² / 2σ²)，σ 用 median heuristic。

dCor：計算 X 與 Y 的歐式距離矩陣，雙重中心化得 A、B，定義 dCov² = (1/n²) Σᵢⱼ Aᵢⱼ Bᵢⱼ，再標準化：dCor = dCov / √(dVar(X) · dVar(Y))。

HSIC: Given kernel matrices K (on X), L (on Y), with centering matrix H = I − (1/n)11ᵀ:

HSICn(X,Y) = (1/(n−1)²) · tr(KHLH)

Gaussian kernel commonly used: k(x, x') = exp(−‖x−x'‖² / 2σ²) with σ from median heuristic.

dCor: Compute pairwise Euclidean distance matrices of X and Y; double-center to get A, B; define dCov² = (1/n²) Σᵢⱼ Aᵢⱼ Bᵢⱼ; normalize: dCor = dCov / √(dVar(X) · dVar(Y)).

核心直覺：如果 X 和 Y 獨立，那麼「X 兩兩之間有多遠」應與「Y 兩兩之間有多遠」毫無關係。HSIC/dCor 量化的正是這兩個距離模式的相關性。
Core intuition: If X ⫫ Y, then "how far apart X pairs are" should be unrelated to "how far apart Y pairs are." HSIC/dCor quantify exactly that correlation between distance patterns.

深入解析

實務細節

① Kernel 怎麼選？「characteristic kernel」的意義

HSIC 的「獨立性 ⇔ HSIC=0」這個等價條件，需要核函數是 characteristic kernel（嵌入到 RKHS 的映射是單射）。常用的 characteristic kernel：

  • Gaussian/RBF：k(x,y) = exp(−‖x−y‖²/2σ²)，最常用。對連續變數通用。
  • Laplacian：k(x,y) = exp(−‖x−y‖/σ)，比 Gaussian 對 outlier 更穩健。
  • Energy kernel：k(x,y) = ½(‖x‖+‖y‖−‖x−y‖)，與 distance correlation 等價。

非 characteristic 的 kernel（要避免）：純線性 kernel（HSIC 退化成 Pearson²）、多項式 kernel（看不出某些非線性關係）。

② σ 的選擇：median heuristic 與多核整合

Gaussian kernel 的頻寬 σ 通常用 median heuristic：σ = median{‖xᵢ − xⱼ‖ : i < j}。理論最佳但對極端數據敏感。實務上做 GRN 推論時建議用 multi-kernel HSIC：取多個 σ（如 0.5×, 1×, 2× median）取最大值，提升對不同尺度依賴的偵測力。

③ 計算複雜度與大規模擴展

naive HSIC 與 dCor 都是 O(n²) 記憶體 + O(n²) 計算——對 10⁴ 細胞已經吃緊。三個加速法：

  • Block HSIC：把資料切成 B 塊各算 HSIC、取平均。複雜度降至 O(n × block_size)。
  • Nyström approximation：用 m 個 anchor points 近似核矩陣，O(n × m²)。
  • RFF (Random Fourier Features)：把 Gaussian kernel 近似為線性內積，HSIC 退化成 O(n × D) 其中 D 為特徵數。在 scRNA-seq atlas-scale (10⁶ 細胞) 唯一可行的選擇。

④ HSIC 與 dCor 何時不等價？

用 energy kernel 的 HSIC 數學上 ≡ dCor 的平方。用 Gaussian kernel 的 HSIC 對「波長相符的非線性」更敏感（取決於 σ），但對極端的高頻變化（如鋸齒狀）可能不如 dCor。實務上：dCor 不需調參、HSIC 可透過 σ 適配資料尺度——選哪個看你願不願意花時間調 kernel。

⑤ HSIC Lasso：當特徵選擇遇上 HSIC

Yamada et al. (2014) 提出 HSIC Lasso：把目標變數 y 與每個候選特徵 x_k 的 HSIC 視為「相關性」分數，在凸最佳化框架下選出最相關且彼此非冗餘的特徵子集。比 mRMR、Boruta 等更善於處理非線性與高維。在 scRNA-seq 上特別適合篩選與細胞型別 label 強依賴的 HVG。

⑥ Conditional HSIC：控制 confounder 的獨立性檢定

cHSIC(X, Y | Z) 檢驗「給定 Z 後 X 與 Y 是否獨立」——這在生物學中至關重要（控制 cell cycle、batch、library size 等 confounder）。Fukumizu (2008) 與 Park (2020) 提供了一致估計子。實務應用：在做 GRN 推論時用 cHSIC 排除「兩基因都受第三基因驅動」的偽相關。

⑦ 變異數與 p-value 計算

HSIC 在虛無下的漸近分布是 χ² 加權和（樣本 HSIC 乘以 n 後）。實務上做檢定有三種選擇：

  1. Permutation test（最準但慢）：對 Y 隨機洗牌 B 次，計算 null 分布。
  2. Gamma approximation：用 null 的前兩階動差 fit gamma 分布，比 permutation 快數百倍且足夠準確。
  3. Block-wise HSIC + CLT：對 block HSIC 用 central limit theorem 直接給 z-test。

① Choosing a kernel: what "characteristic kernel" means

HSIC = 0 ⇔ independence requires a characteristic kernel (the RKHS embedding is injective). Common ones:

  • Gaussian/RBF: k(x,y) = exp(−‖x−y‖²/2σ²). Default for continuous data.
  • Laplacian: k(x,y) = exp(−‖x−y‖/σ). More outlier-robust than Gaussian.
  • Energy kernel: k(x,y) = ½(‖x‖+‖y‖−‖x−y‖). Equivalent to distance correlation.

Non-characteristic kernels to avoid: linear kernel (HSIC reduces to Pearson²), polynomial kernel (misses many non-linearities).

② Choosing σ: median heuristic + multi-kernel

For Gaussian kernel: σ = median{‖xᵢ − xⱼ‖} works well in theory but is sensitive to outliers. For GRN inference, use multi-kernel HSIC: try σ ∈ {0.5×, 1×, 2× median} and take the max — boosts detection across scales.

③ Compute and scaling

Naive HSIC and dCor are O(n²) memory and time — tight by 10⁴ cells. Three speed-ups:

  • Block HSIC: split into B blocks, compute HSIC per block, average. O(n × block_size).
  • Nyström approximation: use m anchor points, O(n × m²).
  • Random Fourier Features (RFF): approximate Gaussian kernel as linear inner product, HSIC → O(n × D). The only viable option at atlas-scale (10⁶ cells).

④ When do HSIC and dCor differ?

HSIC with energy kernel ≡ dCor². Gaussian-HSIC adapts to data scale via σ but may underperform dCor on extreme high-frequency variation. Practical pick: dCor for hands-off; HSIC if you can tune σ.

⑤ HSIC Lasso: feature selection meets HSIC

Yamada et al. (2014): treat HSIC(y, x_k) as a relevance score; pick non-redundant features in a convex framework. Beats mRMR and Boruta on non-linear high-dim data. Great for picking HVGs with strong dependence on cell-type labels.

⑥ Conditional HSIC: testing independence given confounders

cHSIC(X, Y | Z) tests whether X ⫫ Y | Z — vital for controlling cell cycle, batch, library size. Fukumizu (2008), Park (2020) provide consistent estimators. Use case: remove "two genes both driven by a third" spurious correlations in GRN inference.

⑦ Variance and p-values

Under H₀, HSIC has an asymptotic weighted-χ² distribution. Three test choices:

  1. Permutation: shuffle Y B times — accurate but slow.
  2. Gamma approximation: fit gamma to first two null moments — hundreds of times faster, near-equivalent accuracy.
  3. Block HSIC + CLT: z-test directly from block-averaged HSIC.
詳細已知應用

在生物資訊的具體應用

🧬 案例 1：MOFA / MOFA+ 中的因子顯著性檢定

MOFA+ (Argelaguet 2020, Genome Biology) 用 group-factor analysis 學跨模態 latent factor。每個 factor 對每個模態的「解釋變異」是 HSIC-like 量——測試「這個 factor 是否真的與這個 modality 有關」。在 CITE-seq、scNMT-seq 等多體學資料是標準分析步驟。

🧬 案例 2：cellAlign 與 SCOT 的對齊評估

cellAlign（Alpert 2018, Nature Methods）對齊不同條件下的 pseudotime 軌跡時，用 dCor 衡量「對齊後兩軌跡的相似度」——比 Pearson 對齊更穩健。SCOT 在無監督對齊不同模態時用 Gromov-HSIC 衡量「結構保持」。

🧬 案例 3：HSIC-based GRN 推論（GENIE3 變體）

GENIE3 是經典樹狀 GRN 方法，用 Random Forest variable importance。HSIC-Lasso GRN（Slawski et al. 2010 啟發）改用 HSIC Lasso 篩選每個目標基因的調控者——在 DREAM5 challenge benchmark 上比 GENIE3 在某些非線性 motif 上勝出。

🧬 案例 4：批次效應評估 — kBET, LISI, HSIC

kBET (Büttner 2019) 用 k-nearest-neighbor 統計檢定批次混合度；其變體用 HSIC(embedding, batch_label) 衡量「整合後是否還殘留 batch 訊號」。HSIC 對小規模殘留 batch effect 比 kBET 更敏感。

🧬 案例 5：CITE-seq 蛋白–RNA 對應的 dCor 篩選

給定 ~200 個蛋白抗體 panel，哪些 ADT 訊號真的與其對應的 mRNA 有關（很多抗體有 nonspecific binding）？dCor(ADT, mRNA) 比 Pearson 更敏感，能找到「在某 expression 區段才相關」的真實對應，幫助清理 panel。

🧪 案例 6：藥物機制研究

L1000 / Connectivity Map：給定 10⁵ 藥物擾動 × 1000 landmark genes，用 HSIC 衡量「藥物相似性」（兩藥物在 1000 維 expression 空間是否相關擾動）。比 Pearson 更能找到「機制相同但 magnitude 不同」的藥物。

🧪 案例 7：空間轉錄體的 spatial-feature dependence

Hotspot (DeTomaso & Yosef 2021) 用 local autocorrelation + HSIC 變體找空間變異基因，比 Moran's I 處理 zero-inflation 與 over-dispersion 更穩健。

🧬 Case 1: Factor significance in MOFA / MOFA+

MOFA+ (Argelaguet 2020, Genome Biology) uses group-factor analysis. Each factor's "variance explained" per modality is an HSIC-like quantity — testing whether the factor truly relates to the modality. Standard step in CITE-seq, scNMT-seq.

🧬 Case 2: cellAlign / SCOT alignment evaluation

cellAlign (Alpert 2018) uses dCor to compare aligned pseudotime trajectories across conditions — more robust than Pearson. SCOT uses Gromov-HSIC for unsupervised modality alignment, measuring "structure preservation."

🧬 Case 3: HSIC-based GRN inference

An HSIC-Lasso GRN variant of GENIE3 beats Random Forest baselines on non-linear motifs in DREAM5 benchmark.

🧬 Case 4: Batch effect evaluation — kBET, LISI, HSIC

kBET (Büttner 2019) and HSIC(embedding, batch) variants quantify residual batch signal post-integration. HSIC is more sensitive to small residuals than kBET.

🧬 Case 5: CITE-seq protein–RNA correspondence screening

Which ADT signals truly correspond to their mRNA (many antibodies have nonspecific binding)? dCor(ADT, mRNA) finds segment-specific real correspondences — helps clean panels.

🧪 Case 6: Drug mechanism studies

L1000 / Connectivity Map: HSIC between drug-perturbation profiles in 1000-D expression space identifies mechanism-similar drugs better than Pearson.

🧪 Case 7: Spatial feature dependence

Hotspot (DeTomaso & Yosef 2021) uses local autocorrelation + HSIC-like measures for SVGs, more robust to zero-inflation than Moran's I.

潛在改良應用

還能解決哪些問題？

🎯 1. 「多基因聯合 vs 表現型」依賴測試

傳統 GWAS 一次測一個 SNP；多基因聯合效應（epistasis）難測。HSIC(SNP_panel, phenotype) 把 SNP 組視為向量，直接測「整組是否依賴 phenotype」。提案：用於 polygenic risk score 開發前的「基因集是否真的有用」過濾。

🎯 2. scLLM embedding vs 細胞型別的全局評估

scGPT/Geneformer 的 cell embedding 怎麼評估「好不好」？HSIC(embedding, cell_type_one_hot) 給一個單一數字衡量「embedding 整體保留多少 cell type 訊息」——比 silhouette score 對非凸 cluster 更穩健。

🎯 3. 細胞–細胞通訊的「全模組」對應

CellChat 個別測 L–R 對。若一個訊號路徑涉及多 ligand + 多 receptor + 多 downstream effector，傳統法看不到全局。HSIC(ligand_set, receptor_set | spatial_proximity) 能測「ligand 整組與 receptor 整組是否依賴」。提案：「pathway-level」cell communication 分析。

🎯 4. 跨切片 spatial transcriptomics 對齊評估

連續切片之間的空間結構應該保留——但「保留多少」很難量化。dCor(slice1_gene_matrix, slice2_gene_matrix) 在對齊後給出單一相似度分數，可作為 PASTE/STAlign 等對齊方法的客觀評估。

🎯 5. 多體學 latent space 對齊驗證

MultiVI、totalVI 等模型把 RNA 與 ATAC/Protein 對應到共同 latent。HSIC(latent_RNA, latent_ATAC | cell_type) 檢驗「對齊後是否真的還保留模態特有結構」——若太低代表 modality collapse。

🎯 6. Variance-based DE（找變異度有差異的基因）

大部分 DE 找均值差異；某些疾病狀態下基因 mean 不變但 variance 增加（noisy gene expression）。HSIC(disease_label, gene_value) 同時敏感於均值與分布形狀變化。提案：autoimmune disease、senescence 等「expression noise」相關研究的標準工具。

🎯 7. CRISPR knockout screen 中的非線性 fitness

有些基因敲除單獨無顯著影響，但與其他基因組合敲除時有強效應（合成致死）。HSIC(perturbation_vector, fitness) 整體測試非線性 interaction。提案：取代/補充 MAGeCK 的線性測試。

🎯 8. 跨物種 atlas 整合的「真實對齊」驗證

跨物種 reference atlas 整合（人 ↔ 鼠 ↔ 斑馬魚）時，dCor 可衡量「同源細胞型別」在嵌入空間的真正對應。比 silhouette、ARI 更穩健於 cell-count imbalance。

🎯 9. 動態系統的「初始 vs 終態」依賴

在 fate decision 研究中：初期細胞狀態與最終 fate 的依賴有多強？HSIC(initial_state_vector, final_celltype) 可量化「fate 在多大程度由初期 state 決定」。提案：與 lineage tracing (CellTagging, LARRY) 整合。

🎯 10. scLLM fine-tune 的層級重要性分析

fine-tune 一個 scGPT 時，哪一層 attention 對特定下游 task 貢獻最大？HSIC(layer_i_activation, task_label) 給單一分數，比 gradient-based attribution 更穩健。

🎯 1. "Multi-gene joint vs phenotype" dependency test

GWAS tests one SNP at a time; epistasis is hard. HSIC(SNP_panel, phenotype) treats the SNP group as a vector. Proposal: filter "is this gene set useful at all" before building polygenic risk scores.

🎯 2. Global scLLM embedding vs cell-type evaluation

How to evaluate scGPT/Geneformer cell embeddings? HSIC(embedding, cell_type_one_hot) gives a single number — more robust than silhouette for non-convex clusters.

🎯 3. Pathway-level cell communication

CellChat tests individual L–R pairs. HSIC(ligand_set, receptor_set | spatial_proximity) tests whole-pathway dependence. Proposal: pathway-level cell communication analysis.

🎯 4. Spatial transcriptomics slice alignment evaluation

dCor(slice1, slice2) gives an objective similarity score for evaluating PASTE/STAlign-style spatial alignment.

🎯 5. Multi-omics latent alignment verification

HSIC(latent_RNA, latent_ATAC | cell_type) checks whether MultiVI/totalVI preserves modality-specific structure or has collapsed.

🎯 6. Variance-based DE

Most DE finds mean differences; some diseases change variance not mean. HSIC catches both. Proposal: standard tool for expression-noise diseases (autoimmunity, senescence).

🎯 7. Non-linear fitness in CRISPR screens

Synthetic lethality involves gene-gene combinations. HSIC(perturbation_vector, fitness) tests non-linear interactions holistically.

🎯 8. Cross-species atlas integration validation

dCor verifies whether homologous cell-types truly correspond in joint embeddings — more robust than silhouette / ARI to cell-count imbalance.

🎯 9. "Initial vs terminal state" dependence in dynamics

HSIC(initial_state_vector, final_celltype) quantifies how strongly fate is determined by initial state. Proposal: integrate with lineage tracing (CellTagging, LARRY).

🎯 10. Layer-wise importance in scLLM fine-tuning

Which attention layer contributes most to a downstream task? HSIC(layer_i_activation, task_label) — more robust than gradient attribution.

程式碼

實作範例

# install.packages(c("dHSIC","energy"))
library(dHSIC); library(energy)

# HSIC 獨立性檢定
res <- dhsic.test(x, y, method = "permutation", B = 200)
res$statistic; res$p.value

# Distance correlation
dcor(x, y)
dcor.test(x, y, R = 200)

# 多變量：X 為矩陣（cells × features），y 為向量
dcor(pca_embedding, batch_label)
import dcor
import numpy as np

dcor.distance_correlation(x, y)
dcor.independence.distance_covariance_test(x, y, num_resamples=200)

# HSIC via pyhsiclasso or custom
from sklearn.metrics.pairwise import rbf_kernel
def hsic(X, Y, sigma=None):
    n = X.shape[0]
    K = rbf_kernel(X.reshape(-1,1) if X.ndim==1 else X)
    L = rbf_kernel(Y.reshape(-1,1) if Y.ndim==1 else Y)
    H = np.eye(n) - 1/n
    return np.trace(K @ H @ L @ H) / (n-1)**2
