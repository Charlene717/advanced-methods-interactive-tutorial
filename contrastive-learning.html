自監督對比學習 (Contrastive Learning)

不需要 label 就能學表徵——同一物的不同 view 拉近、不同物推遠。SimCLR / MoCo / BYOL / CLIP 等方法在影像、文字、語音都已稱霸；今天它也是 scGPT、scCLIP、Concerto 等單細胞 foundation model 的核心模組。

Learn representations without labels — pull together different views of the same instance, push apart views of different instances. SimCLR / MoCo / BYOL / CLIP dominate vision, text, and audio; today they are also the core of scGPT, scCLIP, Concerto, and other single-cell foundation models.

核心概念

不用 label 怎麼學？

對比學習 (Contrastive Learning, CL) 的核心想法：對每筆資料 x 做兩種隨機 augmentation 得到 x⁺ 和 x'⁺——把它們視為「同一語意」的兩個 view，希望嵌入後彼此靠近；而 batch 中其他樣本 (negatives) 應該遠離。

InfoNCE loss (Oord et al. 2018) 形式化此目標：

LNCE = −E [ log ( exp(sim(z, z⁺)/τ) / Σk exp(sim(z, zk)/τ) ) ]

其中 sim 通常是 cosine、τ 是 temperature。可證明 LNCE 是 mutual information I(z; z⁺) 的下界——所以對比學習本質上是 MI maximization。

單細胞語境下「同一物的 view」可以是：同一細胞在 RNA + ATAC modality；同一細胞兩次 dropout augmentation；同一 patient 在不同時間點；同一基因在多個細胞——design 不同 view 就 design 了不同 self-supervised pretext task。

Contrastive Learning (CL) core idea: for each x, sample two random augmentations x⁺, x'⁺ — treat them as two views of the same semantic content. Embed them close together; push other samples (negatives) far apart.

InfoNCE loss (Oord et al. 2018) formalizes this:

LNCE = −E [ log ( exp(sim(z, z⁺)/τ) / Σk exp(sim(z, zk)/τ) ) ]

where sim is usually cosine and τ is temperature. LNCE is a lower bound on mutual information I(z; z⁺) — CL is essentially MI maximization.

In single-cell, "two views of one instance" can be: same cell in RNA + ATAC; same cell with two random dropouts; same patient at different times; same gene across multiple cells. Choosing views = choosing the self-supervised pretext task.

Positive pairs close, negatives far — simple but remarkably powerful.

τ controls hard-negative weight. Small τ → over-sensitive to nearest negatives; large τ → uniform weighting.

適合 scRNA-seq 這種「樣本多、label 少」場景——用 augmentation 取代手動標註。

Perfect for scRNA-seq's "many samples, few labels" regime — augmentation replaces manual annotation.

演算法家族

從 SimCLR 到 BYOL 到 VICReg

① SimCLR (Chen et al. 2020, ICML)

最直接的 InfoNCE 實作：每個 batch 對所有 sample 做兩次 augmentation，得到 2N 個 view。每個 view 把同 sample 的 view 視為 positive、其他 2N−2 個視為 negative。用 NT-Xent loss (normalized temperature-scaled cross-entropy)。需要大 batch（4096+）。

② MoCo (He et al. 2020, CVPR)

解決「大 batch 才有足夠 negative」的問題：用 momentum encoder 維護「dictionary queue」累積 65k+ 過去的 negative key。小 batch (256) 也能得到大量 negative。後續 MoCo-v3 (Chen 2021) 加 prediction head + ViT，效能再升。

③ BYOL (Grill et al. 2020, NeurIPS)

顛覆「需要 negative」的傳統信念：只用兩 view 的 prediction loss，搭配 momentum target network 與 prediction head——避免 collapse。理論上有些謎，但 empirically 強到爆。

④ VICReg (Bardes et al. 2022, ICLR) / Barlow Twins

用 variance + invariance + covariance regularization 直接約束 latent statistics——強制 latent dimensions 之間 decorrelated。不需 negative 也不需 momentum，理論最透明。Barlow Twins (Zbontar 2021) 是其 spiritual cousin。

⑤ CLIP (Radford et al. 2021, ICML)

Cross-modal contrastive：「image, text」配對為 positive、batch 中其他配對為 negative。學「跨模態對齊」的 joint embedding。scCLIP、scGPT-CLIP 都是這個架構在單細胞的版本——配對「同細胞的 RNA + image / RNA + ATAC」。

⑥ Supervised CL (Khosla 2020)

有 label 時，把同 class 都視為 positive（不只 augmentation pair）——把 CL 推廣到有部分監督的情境。對「半監督 cell type 整合」非常實用。

① SimCLR (Chen et al. 2020, ICML)

Most direct InfoNCE: augment each sample in a batch twice → 2N views. For each view, the same-sample view is positive; the other 2N−2 are negatives. Uses NT-Xent loss. Needs big batches (4096+).

② MoCo (He et al. 2020, CVPR)

Solves "need big batch for enough negatives" via a momentum encoder maintaining a dictionary queue of 65k+ past negatives. Small batches (256) work fine. MoCo-v3 (Chen 2021) adds a prediction head + ViT for further gains.

③ BYOL (Grill et al. 2020, NeurIPS)

Overturns the "negatives are necessary" dogma: only two views, prediction loss, momentum target network + prediction head avoid collapse. Theoretically puzzling but empirically very strong.

④ VICReg (Bardes et al. 2022, ICLR) / Barlow Twins

Variance + invariance + covariance regularization directly constrains latent statistics — decorrelates dimensions. No negatives, no momentum. Theoretically transparent. Barlow Twins (Zbontar 2021) is a spiritual cousin.

⑤ CLIP (Radford et al. 2021, ICML)

Cross-modal contrastive: "image, text" pairs are positive; other batch pairs are negative. Learns a cross-modal joint embedding. scCLIP, scGPT-CLIP apply this to single-cell — pairing "same-cell RNA + image" or "same-cell RNA + ATAC."

⑥ Supervised CL (Khosla 2020)

With labels, treat same-class samples as positive (not just augmentation pairs) — extends CL to semi-supervised regimes. Very useful for "semi-supervised cell-type integration."

關鍵：augmentation 設計。對 image 是 crop、color jitter 等；對 scRNA-seq 是 stochastic dropout、gene masking、count perturbation；對 scATAC 是 peak subsampling。augmentation 太弱 → trivial solution；太強 → 破壞語意。設計 augmentation 是 single-cell CL 的最大藝術。
Key: augmentation design. For images: crop, color jitter. For scRNA-seq: stochastic dropout, gene masking, count perturbation. For scATAC: peak subsampling. Too weak → trivial solutions; too strong → destroys semantics. Designing augmentations is the art of single-cell CL.

生物應用

對比學習在單細胞與多體學的應用

Concerto

Yang et al. (2022, Nature Communications)：第一個 large-scale 單細胞對比學習框架。用 stochastic gene masking 作 augmentation，學「不論 dropout 都不變」的細胞表徵。在 cell type annotation、batch correction、reference mapping 上都接近或勝過 scVI。

Yang et al. (2022, Nat Comm): the first large-scale single-cell CL framework. Stochastic gene masking as augmentation, learning representations invariant to dropouts. Matches or beats scVI on annotation, batch correction, reference mapping.

scCLIP / OmiCLIP

把 CLIP 跨模態對比學習用於配對的 (scRNA, scATAC) 或 (scRNA, image)。建構「跨模態語意空間」，能用 RNA query atlas、用 image 找對應 expression。也是 SpatialCLIP (2024) 對齊 spatial + histology 的基礎。

CLIP-style cross-modal CL for paired (scRNA, scATAC) or (scRNA, image). Builds a cross-modal semantic space — query atlas by RNA, find expression by image. Basis of SpatialCLIP (2024) aligning spatial + histology.

scBERT / scGPT 對比 head

Yang et al. (2022, NMI) 的 scBERT、Cui et al. (2024) 的 scGPT 在 mask-and-predict pretext 之外，都加入對比 head 強化「同細胞跨 view 一致性」。對下游 fine-tuning 提升明顯。

Yang et al. (2022, NMI) scBERT and Cui et al. (2024) scGPT add a contrastive head on top of mask-and-predict pretext — improving downstream fine-tuning.

SimCSE for proteins

借用 NLP 的 SimCSE (Gao 2021)：對 protein sequence 做兩次 random dropout，positive pair 是同蛋白質、negative 是 batch 其他蛋白。學「functional embedding」可用於 protein-protein interaction prediction、functional annotation。

Borrowed from NLP's SimCSE (Gao 2021): two random dropouts of a protein sequence form a positive pair; other proteins are negatives. Learns functional embeddings useful for PPI prediction and annotation.

ContrasCell

把同一病人的多 sample 視為 positive、跨病人為 negative，學「patient-level signature」——對 longitudinal cohort 跨時間追蹤特別有用。可作為「臨床預測 model」的輸入特徵。

Same-patient samples are positives, cross-patient pairs are negatives — learn patient-level signatures. Useful for longitudinal cohort tracking and as features for clinical prediction.

chromBERT (對比 chromatin)

把同一基因的 chromatin state vector 與 cell type label 配對為 positive，學「染色質 → 表現潛能」的可遷移表徵。可用於 enhancer-gene linking、TF binding prediction。

Pair a gene's chromatin state with its cell-type label as positive — learn transferable "chromatin → expression potential" representations. Useful for enhancer-gene linking and TF binding prediction.

深入解析

陷阱、原理與最新進展

① InfoNCE = MI 下界的數學

Poole et al. (2019) 證明：對 K 個 negative，InfoNCE 是 I(z; z⁺) 的下界，且 bound 為 log K − LNCE。所以「越多 negative」bound 越緊——這正是為何 SimCLR 需要大 batch、MoCo 維護 queue。實作 trick：對 K > 1000，bound 已非常接近真實 MI。

② Alignment + Uniformity 解釋

Wang & Isola (2020, ICML) 證明：InfoNCE 在 hypersphere 極限下分解為兩項：(1) alignment——positive pair 距離（要小）；(2) uniformity——所有 sample 在球面上的均勻度（要大）。這給對比學習「為何 work」一個幾何直觀。

③ Negative collapse / false negatives

單細胞 CL 的最大 pitfall：同 cell type 的「不同細胞」被當成 negative，但其實它們語意相同 → 強迫推遠它們會破壞 cell type 結構。解法：(a) Supervised CL 用 cell type label；(b) NN-CL (Dwibedi 2021) 把 nearest neighbor 也視為 positive；(c) Debiased CL (Chuang 2020) 修正 sampling bias。

④ Temperature τ 的精細調整

Wang & Liu (2021) 證明 τ 控制「hard negative mining」強度：τ→0 等價於只挑最近的 negative；τ→∞ 變成 uniform。實務上 τ ∈ [0.05, 0.5]，單細胞通常 0.1–0.2。Adaptive τ（隨訓練調整）是 2023 後的新方向。

⑤ BYOL 不用 negative 為何不 collapse？

Tian et al. (2021) 用 dynamical system 分析：predictor + target network 的 momentum update 等價於「對 representation 的 implicit ridge regularization」——防止 latent 萎縮到單點。沒有 negative 也能保持 diversity。

⑥ Multi-view 擴展與 augmentation 的選擇性

對單細胞，「好的」augmentation 是什麼？Tian et al. (2020) 的 InfoMin 原則：augmentation 應「最小化 view 間共享的 nuisance info、最大化 task-relevant info」。實務上：dropout augmentation 模擬 technical noise；gene masking 強迫學 global gene-gene 結構；count perturbation 模擬 capture efficiency。

⑦ Dimensional collapse

即使有 negative，CL 仍可能讓 latent 維度坍塌（部分 dim 失活）。Jing et al. (2022) 證明 BatchNorm + projection head 是緩解 key。實務檢查方法：latent 的 covariance matrix singular values 應均勻分布。

⑧ Spectral / EigenGame 理論

Balestriero & LeCun (2022) 證明：所有 CL 算法（SimCLR、BYOL、VICReg、Barlow Twins）在 limit 下都收斂到 spectral embedding ——某種 graph Laplacian eigenmaps。提供統一視角。

① InfoNCE = MI lower bound mathematics

Poole et al. (2019) proved: with K negatives, InfoNCE lower-bounds I(z; z⁺) by log K − LNCE. So "more negatives" tighten the bound — hence SimCLR's huge batches, MoCo's queue. For K > 1000 the bound is already close to true MI.

② Alignment + uniformity interpretation

Wang & Isola (2020, ICML): InfoNCE on the hypersphere decomposes into alignment (positive pair distance, small) + uniformity (spread of all samples on sphere, large) — a geometric intuition for why CL works.

③ Negative collapse / false negatives

The biggest pitfall in single-cell CL: cells of the same type are treated as negatives though semantically identical → forcing them apart destroys structure. Fixes: (a) Supervised CL with cell-type labels; (b) NN-CL (Dwibedi 2021) treats nearest neighbours as positives; (c) Debiased CL (Chuang 2020) corrects sampling bias.

④ Fine-tuning temperature τ

Wang & Liu (2021): τ controls hard-negative mining intensity. τ→0 = pick the closest negative; τ→∞ = uniform. Typical τ ∈ [0.05, 0.5]; single-cell uses 0.1–0.2. Adaptive τ is a post-2023 trend.

⑤ Why doesn't BYOL collapse without negatives?

Tian et al. (2021) used dynamical systems analysis: predictor + target network momentum = implicit ridge regularization on the representation — prevents latent collapse without explicit negatives.

⑥ Multi-view extension and augmentation selection

What's a "good" augmentation for single-cell? Tian et al. (2020) InfoMin: augmentation should minimize nuisance shared info between views and maximize task-relevant info. In practice: dropout mimics technical noise; gene masking forces global gene-gene structure learning; count perturbation mimics capture efficiency variation.

⑦ Dimensional collapse

Even with negatives, CL can have dimensional collapse (dead dimensions). Jing et al. (2022): BatchNorm + projection head are key mitigators. Diagnostic: latent covariance singular values should be roughly uniform.

⑧ Spectral / EigenGame theory

Balestriero & LeCun (2022): all CL methods (SimCLR, BYOL, VICReg, Barlow Twins) converge in some limit to spectral embeddings — a graph Laplacian eigenmap. Unifying view.

潛在改良應用

對比學習還能改良哪些生物資訊任務？

🎯 1. Cross-species cell contrastive

把「同 cell type 的人 vs 鼠細胞」設為 positive、跨 cell type 為 negative，學「物種無關的細胞型別表徵」。比經典 ortholog mapping 更語意化，預期能跨物種轉移細胞型別 label。

🎯 2. Patient-vs-patient atlas embedding

建構「patient-level」對比學習：把同病人 cells 視為 positive、跨病人為 negative → 學「個人化 disease signature」。可作為臨床 trial stratification 的特徵。

🎯 3. Perturbation contrastive (treated vs untreated)

把「同 perturbation 的細胞 pair」設為 positive、不同 perturbation 為 negative，學「擾動指紋」latent。預期可預測「未見過 perturbation 的反應」（zero-shot perturbation prediction）。

🎯 4. 多模態 spatial alignment

對 spatial transcriptomics + histology image，用 CLIP-style 把「同位置的 RNA + image patch」配對。預期得到「跨模態 niche representation」，比 PASTE 等 OT-only 方法更語意化。

🎯 5. 對比學習 + 因果結構先驗

InfoNCE 學 「invariant under augmentation」，但 augmentation 應反映真實 causal structure。提案：用 do-calculus 設計 augmentation（do(batch=k) 模擬 batch effect），讓 CL 學「對 batch 不變、對 phenotype 敏感」的因果表徵。

🎯 6. Time-aware CL

把「同細胞但不同 timepoint」設為 ordered positive（temporally weighted），學「沿軌跡平滑」的細胞表徵。可作為 trajectory inference 的新 backbone（取代 PCA + UMAP）。

🎯 7. Hierarchical CL for cell type taxonomy

對 cell type 分層 (T cell → CD4 T → Th1)：較細的「同 subtype」positive 強、粗的「同 lineage」positive 弱。學樹狀 hierarchical embedding。對 cell ontology 整合很有用。

🎯 8. CL on lineage tracing

有 barcode 的 lineage tracing 資料：同 clonal family 細胞為 positive、跨 clone 為 negative。學「lineage-aware embedding」，補強純基因表現的 trajectory inference。

🎯 9. Multi-task CL pretraining

同時定義多種 view-pair：(dropout)(mask)(modality)(time)，每對都有獨立 InfoNCE。學「universal cell embedding」，跨 atlas / 跨 task 都好用。預期是下一代 scLLM pretrain 範式。

🎯 10. CL for drug response prediction

把「同藥物作用的細胞」設為 positive，學「mechanism-of-action embedding」。可用於 drug repurposing（找相似 MoA 的新藥）與 combinatorial drug screening。

🎯 1. Cross-species cell contrastive

Same cell-type human/mouse cells = positive; different cell types = negative. Learns species-agnostic cell-type representations — more semantic than ortholog mapping, supports cross-species label transfer.

🎯 2. Patient-vs-patient atlas embedding

Patient-level CL: same-patient cells positive, cross-patient negative → learn personalised disease signatures. Useful as features for clinical trial stratification.

🎯 3. Perturbation contrastive (treated vs untreated)

Same-perturbation cells positive, different-perturbation negative — learns perturbation fingerprints. Could enable zero-shot perturbation prediction.

🎯 4. Multi-modal spatial alignment

CLIP-style pairing "same-location RNA + image patch" for spatial transcriptomics + histology. Learns cross-modal niche representations, more semantic than OT-only methods like PASTE.

🎯 5. CL + causal-structure priors

InfoNCE learns invariance under augmentation, but augmentation should reflect real causal structure. Use do-calculus to design augmentation (do(batch=k) simulating batch effects) → CL learns "batch-invariant, phenotype-sensitive" causal representations.

🎯 6. Time-aware CL

"Same cell, different timepoints" → temporally weighted positive. Learns smooth-along-trajectory representations. A new backbone for trajectory inference (replacing PCA + UMAP).

🎯 7. Hierarchical CL for cell type taxonomy

For nested cell types (T → CD4 → Th1): fine-subtype positive strong, coarse-lineage positive weak. Learns tree-structured hierarchical embeddings — useful for cell ontology integration.

🎯 8. CL on lineage tracing

For barcode lineage tracing: same clonal family = positive. Learn lineage-aware embeddings, complementing pure expression trajectory inference.

🎯 9. Multi-task CL pretraining

Define multiple view-pair tasks (dropout, mask, modality, time) with independent InfoNCEs. Learn a universal cell embedding usable across atlases and tasks. A likely next-gen scLLM pretraining paradigm.

🎯 10. CL for drug response prediction

Same-drug cells = positive → learn mechanism-of-action embeddings. Useful for drug repurposing (find similar MoA new drugs) and combinatorial screening.

# pip install lightly
import torch
from lightly.loss import NTXentLoss
from lightly.models.modules import SimCLRProjectionHead

class SCLR(torch.nn.Module):
    def __init__(self, d_in, d_hidden=256, d_z=128):
        super().__init__()
        self.backbone = torch.nn.Sequential(
            torch.nn.Linear(d_in, 512), torch.nn.ReLU(),
            torch.nn.Linear(512, d_hidden))
        self.proj = SimCLRProjectionHead(d_hidden, d_hidden, d_z)
    def forward(self, x):
        return self.proj(self.backbone(x))

criterion = NTXentLoss(temperature=0.1)

def aug(x, p=0.4):  # gene dropout augmentation
    mask = (torch.rand_like(x) > p).float()
    return x * mask
import pytorch_lightning as pl
import torch.nn.functional as F

class SCContrastive(pl.LightningModule):
    def __init__(self, model, tau=0.1):
        super().__init__(); self.m = model; self.tau = tau
    def training_step(self, batch, _):
        x = batch["expression"]
        z1 = F.normalize(self.m(aug(x)), dim=-1)
        z2 = F.normalize(self.m(aug(x)), dim=-1)
        N = x.size(0)
        z = torch.cat([z1, z2], dim=0)         # 2N
        sim = z @ z.T / self.tau                # 2N x 2N
        labels = torch.arange(N, device=z.device)
        labels = torch.cat([labels + N, labels])
        sim.fill_diagonal_(-1e9)
        loss = F.cross_entropy(sim, labels)
        return loss
import torch, torch.nn.functional as F

def info_nce(z1, z2, tau=0.1):
    """Symmetric InfoNCE for paired views z1, z2 (both [N, D])."""
    z1 = F.normalize(z1, dim=-1); z2 = F.normalize(z2, dim=-1)
    N = z1.size(0)
    logits = z1 @ z2.T / tau                  # [N, N]
    labels = torch.arange(N, device=z1.device)
    l_a = F.cross_entropy(logits, labels)
    l_b = F.cross_entropy(logits.T, labels)
    return (l_a + l_b) / 2

# Example: dropout-based augmentation
def dropout_aug(x, p=0.3):
    m = (torch.rand_like(x) > p).float()
    return x * m / (1 - p)
